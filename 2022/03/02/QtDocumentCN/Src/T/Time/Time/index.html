详细描述#

QTime 对象包含一个时钟时间，它可以表示为自午夜以来的小时数、分钟数、秒数和毫秒数。它提供了比较时间和通过添加毫秒数来操纵时间的功能。 QTime 对象应该通过值而不是通过引用来传递 const；他们只是打包int。

QTime 使用 24 小时时钟格式；它没有 AM/PM 的概念。与 QDateTime 不同，QTime 对时区或夏令时 (DST) 一无所知。

QTime 对象通常通过明确给出小时、分钟、秒和毫秒数或使用静态函数 currentTime() 来创建，该函数创建一个表示系统本地时间的 QTime 对象。

hour()、minute()、second() 和 msec() 函数提供对小时数的访问，时间的分钟、秒和毫秒。 toString() 函数以文本格式提供相同的信息。

addSecs() 和 addMSecs() 函数提供比给定时间晚给定秒数或毫秒数的时间。相应地，可以使用 secsTo() 或 msecsTo() 找到两次之间的秒数或毫秒数。

QTime 提供了一套完整的运算符来比较两个 QTime 对象；较早的时间被认为小于较晚的时间；如果 A.msecsTo(B) 为正，则 A < B。

QTime 对象也可以使用 fromString() 从文本表示创建，并使用 toString() 转换为字符串表示。所有与字符串格式之间的转换都是使用 C 语言环境完成的。有关本地化转换，请参阅 QLocale。

See also QDate and QDateTime.

QTime::toString( format) const#

QTime::toString(const &format) const#

以字符串形式返回时间。format参数确定结果字符串的格式。

可以使用这些表达式：

表达 输出
h 没有前导零的小时（如果显示 AM/PM，则为 0 到 23 或 1 到 12）
hh 带前导零的小时（如果显示 AM/PM，则为 00 到 23 或 01 到 12）
h 没有前导零的小时（0 到 23，即使是 AM/PM 显示）
HH 带前导零的小时（00 到 23，即使是 AM/PM 显示）
m 没有前导零的分钟（0 到 59）
mm 带前导零的分钟（00 到 59）
s 整秒，没有任何前导零（0 到 59）
ss 整秒，在适用的情况下带有前导零（00 到 59）
z 秒的小数部分，在小数点之后，没有尾随零（0 到 999）。因此，“ s.z”将秒报告为完全可用（毫秒）精度，而没有尾随零。
zzz 秒的小数部分，精确到毫秒，包括适用的尾随零（000 到 999）。
AP or A 使用 AM/PM 显示。A/AP将替换为“AM”或“PM”
ap or a 使用上午/下午显示。a/ap将被 ‘am’ 或 ‘pm’ 取代
t 时区（例如“CEST”）

任何用单引号括起来的非空字符序列都将逐字包含在输出字符串中（去掉引号），即使它包含格式化字符。在输出中，两个连续的单引号 (“‘’”) 被替换为单引号。格式字符串中的所有其他字符都逐字包含在输出字符串中。

支持不带分隔符的格式（例如“ddMM”），但必须小心使用，因为结果字符串并不总是可靠地可读（例如，如果“dM”产生“212”，它可能意味着 12 月 2 日或 21 日二月）。

示例格式字符串（假设QTime是 14:13:09.042 并且系统区域设置是en_US）

格式 结果
hh:mm:ss.zzz 14:13:09.042
hⓂs ap 下午 2:13:9
HⓂS A 下午 14:13:9

如果时间无效，将返回一个空字符串。

注意：如果需要本地化形式的 am 或 pm（AP、ap、A 或 a 格式），请使用QLocale::system ()。字符串（）。

QTime::QTime()#

构造一个空时间对象。对于 null 时间，isNull () 返回true并且isValid () 返回false。如果您需要零时间，请使用 QTime(0, 0)。对于一天的开始，请参见QDate::startOfDay ()。

QTime QTime::addMSecs(int ms) const#

返回一个QTime对象，该对象包含比该对象的时间晚的时间ms毫秒（如果ms为负数，则更早）。

请注意，如果过了午夜，时间将会结束。有关示例，请参见addSecs ()。

如果此时间无效，则返回空时间。

另请参见addSecs ()、msecsTo () 和QDateTime::addMSecs ()。

QTime QTime::addSecs(int s) const#

返回一个QTime对象，该对象包含比该对象的时间晚s秒的时间（如果s为负数，则更早）。

请注意，如果过了午夜，时间将会结束。

如果此时间无效，则返回空时间。

例子

QTime n(14, 0, 0);                // n == 14:00:00
QTime t;
t = n.addSecs(70);                // t == 14:01:10
t = n.addSecs(-70);               // t == 13:58:50
t = n.addSecs(10 * 60 * 60 + 5);  // t == 00:00:05
t = n.addSecs(-15 * 60 * 60);     // t == 23:00:00

See also addMSecs(), secsTo(), and QDateTime::addSecs().

[static]QTime QTime::currentTime()#

返回系统时钟报告的当前时间。

注意，准确度取决于底层操作系统的准确度；并非所有系统都提供 1 毫秒的精度。

此外， currentTime() 只会在每天增加；每次午夜过去，它会减少 24 小时；并且，除此之外，如果夏令时过渡干预，它的变化可能与经过的时间不对应。

另见QDateTime::currentDateTime () 和QDateTime::currentDateTimeUtc ()。

[static]QTime QTime::fromMSecsSinceStartOfDay(int msecs)#

返回一个新的QTime实例，其时间设置为从一天开始的毫秒数，即从 00:00:00 开始。

如果毫秒超出有效范围，将返回无效的QTime 。

另见mssecsSinceStartOfDay ()。

[static]QTime QTime::fromString(const &string, const &format)#

使用给定的format返回由string表示的QTime，如果无法解析字符串，则返回无效时间。

这些表达式可用于以下格式：

表达 输出
h 没有前导零的小时（如果显示 AM/PM，则为 0 到 23 或 1 到 12）
hh 带前导零的小时（如果显示 AM/PM，则为 00 到 23 或 01 到 12）
H 没有前导零的小时（0 到 23，即使是 AM/PM 显示）
HH 带前导零的小时（00 到 23，即使是 AM/PM 显示）
m 没有前导零的分钟（0 到 59）
mm 带前导零的分钟（00 到 59）
s 整秒，没有任何前导零（0 到 59）
ss 整秒，在适用的情况下带有前导零（00 到 59）
z 秒的小数部分，在小数点之后，没有尾随零（0 到 999）。因此，“ s.z”将秒报告为完全可用（毫秒）精度，而没有尾随零。
zzz 秒的小数部分，精确到毫秒，包括适用的尾随零（000 到 999）。
AP or A 解释为 AM/PM 时间。A/AP将匹配“AM”或“PM”。
ap or a 解释为上午/下午时间。a/ap将匹配 ‘am’ 或 ‘pm’。

所有其他输入字符将被视为文本。任何包含在单引号中的非空字符序列也将被视为（去掉引号）文本而不被解释为表达式。

QTime time = QTime::fromString("1mm12car00", "m'mm'hcarss");
// time is 12:01.00

如果不满足格式，则返回无效的QTime。不期望给出前导零的表达式（h、m、s 和 z）是贪婪的。这意味着他们将使用两位数（或三位，对于 z），即使这使他们超出了可接受值的范围并且为其他部分留下了太少的位数。例如，以下字符串可能表示 00:07:10，但 m 会抓取两位数，导致时间无效： 

QTime time = QTime::fromString("00:710", "hh:ms"); // invalid

格式中未表示的任何字段都将设置为零。例如： 

QTime time = QTime::fromString("1.30", "m.s");
// time is 00:01:30.000

注意：如果使用本地化形式的 am 或 pm（AP、ap、A 或 a 格式），请使用QLocale::system ().toTime()。

另见toString ()、QDateTime::fromString ()、QDate::fromString () 和QLocale::toTime ()。

int QTime::hour() const#

返回时间的小时部分（0 到 23）。

如果时间无效，则返回 -1。

See also minute(), second(), and msec().

bool QTime::isNull() const#

如果时间为空，则返回true（即QTime对象是使用默认构造函数构造的）；否则返回假。空时间也是无效时间。

另请参见isValid ()。

bool QTime::isValid() const#

如果时间有效则返回true；否则返回false。例如，时间 23:30:55.746 有效，但 24:12:30 无效。

另请参见isNull ()。

[static]bool QTime::isValid(int h, int m, int s, int ms = 0)#

这是一个重载功能。

如果指定的时间有效，则返回true；否则返回假。

如果h在 0 到 23 范围内，m和s在 0 到 59 范围内，ms在 0 到 999 范围内，则时间有效。

例子：

QTime::isValid(21, 10, 30); // returns true
QTime::isValid(22, 5,  62); // returns false

int QTime::msecsTo(QTime t) const#

返回从该时间到t的毫秒数。如果t早于这个时间，则返回的毫秒数为负数。

因为QTime测量一天内的时间，一天有 86400 秒，所以结果总是在 -86400000 到 86400000 毫秒之间。

如果任一时间无效，则返回 0。

另请参见secsTo ()、addMSecs () 和QDateTime::msecsTo ()。

int QTime::second() const#

返回时间的第二部分（0 到 59）。

如果时间无效，则返回 -1。

See also hour(), minute(), and msec().

int QTime::secsTo(QTime t) const#

返回从该时间到t的秒数。如果t早于这个时间，则返回的秒数为负数。

因为QTime测量一天内的时间，一天有 86400 秒，所以结果总是在 -86400 和 86400 之间。

secsTo() 不考虑任何毫秒。

如果任一时间无效，则返回 0。

另见addSecs () 和QDateTime::secsTo ()。

QTime::toString( format = Qt::TextDate) const#

这是一个重载功能。

以字符串形式返回时间。format参数确定字符串的格式。

如果format 为Qt::TextDate，则字符串格式为 HH:mm:ss；例如，午夜前 1 秒将是“23:59:59”。

如果format是Qt::ISODate，则字符串格式对应于日期表示的 ISO 8601 扩展规范，由 HH:mm:ss 表示。要在 ISO 8601 日期中包含毫秒，请使用格式 Qt::ISODateWithMs，它对应于 HH:mm:ss.zzz。

如果格式为Qt::RFC2822Date，则字符串以RFC 2822兼容的方式格式化。这种格式的一个例子是“23:59:20”。

如果时间无效，将返回一个空字符串。

另见fromString ()、QDate::toString () 、QDateTime:: toString ()和QLocale::toString ()。
