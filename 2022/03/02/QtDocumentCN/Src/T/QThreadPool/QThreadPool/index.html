详细描述#

QThreadPool 管理和回收各个QThread对象，以帮助减少使用线程的程序中的线程创建成本。每个 Qt 应用程序都有一个全局 QThreadPool 对象，可以通过调用globalInstance () 来访问该对象。

要使用 QThreadPool 线程之一，子类QRunnable并实现 run() 虚函数。然后创建该类的对象并将其传递给QThreadPool::start ()。

class HelloWorldTask : public QRunnable
{
    void run() override
    {
        qDebug() << "Hello world from thread" << QThread::currentThread();
    }
};

HelloWorldTask *hello = new HelloWorldTask();
// QThreadPool takes ownership and deletes 'hello' automatically
QThreadPool::globalInstance()->start(hello);

QThreadPool 默认会自动删除QRunnable。使用QRunnable::setAutoDelete () 更改自动删除标志。

QThreadPool 支持通过从QRunnable::run () 中调用tryStart (this) 多次执行相同的QRunnable。如果启用了 autoDelete，QRunnable将在最后一个线程退出 run 函数时被删除。启用 autoDelete 时，使用相同的QRunnable多次调用start ()会产生竞争条件，因此不建议这样做。

在一定时间内未使用的线程将过期。默认到期超时为 30000 毫秒（30 秒）。这可以使用setExpiryTimeout () 进行更改。设置负到期超时会禁用到期机制。

调用maxThreadCount () 查询最大使用线程数。如果需要，您可以使用setMaxThreadCount () 更改限制。默认的maxThreadCount () 是QThread::idealThreadCount ()。activeThreadCount () 函数返回当前正在工作的线程数。

reserveThread ( ) 函数保留一个线程供外部使用。使用完线程后使用releaseThread ()，以便它可以被重用。本质上，这些函数会暂时增加或减少活动线程数，并且在实现对 QThreadPool 不可见的耗时操作时很有用。

请注意，QThreadPool 是用于管理线程的低级类，请参阅 Qt Concurrent 模块以获取更高级别的替代方案。

另请参阅QRunnable。

属性文档#

[read-only]activeThreadCount : const int#

此属性保存线程池中的活动线程数。

注意：此函数可能返回一个大于maxThreadCount () 的值。有关详细信息，请参阅reserveThread ()。

访问功能：

int activeThreadCount() const

See also reserveThread() and releaseThread().

expiryTimeout : int#

此属性以毫秒为单位保存线程到期超时值。

expiryTimeout毫秒内未使用的线程被认为已过期并将退出。此类线程将根据需要重新启动。默认的expiryTimeout为 30000 毫秒（30 秒）。如果expiryTimeout为负数，则新创建的线程不会过期，例如，在线程池被销毁之前它们不会退出。

请注意，设置expiryTimeout对已经运行的线程没有影响。只有新创建的线程才会使用新的expiryTimeout。我们建议在创建线程池之后，但在调用start ()之前立即设置expiryTimeout 。

访问功能：

int expiryTimeout() const
void setExpiryTimeout(int expiryTimeout)

maxThreadCount : int#

此属性保存线程池使用的最大线程数。在创建QThreadPool对象时，此属性将默认为QThread::idealThreadCount ()的值。

注意：线程池将始终使用至少 1 个线程，即使maxThreadCount限制为零或负数。

默认的maxThreadCount是QThread::idealThreadCount ()。

访问功能：

int maxThreadCount() const
void setMaxThreadCount(int maxThreadCount)

[since 5.10]stackSize : #

此属性保存线程池工作线程的堆栈大小。

该属性的值仅在线程池创建新线程时使用。更改它对已创建或正在运行的线程没有影响。

默认值为 0，这使得QThread使用操作系统默认的堆栈大小。

这个属性是在 Qt 5.10 中引入的。

访问功能：

uint stackSize() const
void setStackSize(uint stackSize)

[since 6.2]threadPriority : #

此属性保存新工作线程的线程优先级。

该属性的值仅在线程池启动新线程时使用。更改它对已经运行的线程没有影响。

默认值为QThread::InheritPriority，这使得QThread使用与QThreadPool对象所在的优先级相同的优先级。

这个属性是在 Qt 6.2 中引入的。

访问功能：

QThread::Priority threadPriority() const
void setThreadPriority(QThread::Priority priority)

See also QThread::Priority.

[static] *QThreadPool::globalInstance()#

返回全局QThreadPool实例。

void QThreadPool::releaseThread()#

释放先前通过调用reserveThread () 保留的线程。

注意：在没有预先保留线程的情况下调用此函数会暂时增加maxThreadCount ()。当一个线程进入睡眠等待更多工作时，这很有用，允许其他线程继续。完成等待时一定要调用reserveThread ()，这样线程池才能正确维护activeThreadCount ()。

See also reserveThread().

void QThreadPool::reserveThread()#

保留一个线程，忽略activeThreadCount ()和maxThreadCount ()。

一旦你完成了线程，调用releaseThread () 允许它被重用。

注意：即使保留maxThreadCount () 线程或更多，线程池仍将允许最少一个线程。

注意：此函数将增加报告的活动线程数。这意味着通过使用这个函数，activeThreadCount () 可以返回一个大于maxThreadCount () 的值。

See also releaseThread().

void QThreadPool::start( *runnable, int priority = 0)#

预留一个线程并用它来运行runnable，除非这个线程会使当前线程数超过maxThreadCount ()。在这种情况下，runnable被添加到运行队列中。priority参数可用于控制运行队列的执行顺序。

注意，如果runnable->autoDelete ()返回，线程池将拥有runnable的所有权， runnable- >run ()返回后，线程池会自动删除runnable。如果runnable->autoDelete () 返回，runnable的所有权仍然属于调用者。请注意，在调用此函数后更改runnable上的自动删除会导致未定义的行为。true``false

[since 5.15]void QThreadPool::start(std::function<void ()> functionToRun, int priority = 0)#

这是一个重载功能。

保留一个线程并使用它来运行functionToRun，除非该线程会使当前线程数超过maxThreadCount ()。在这种情况下，functionToRun被添加到运行队列中。priority参数可用于控制运行队列的执行顺序。

这个函数是在 Qt 5.15 中引入的。

bool QThreadPool::tryStart( *runnable)#

尝试保留一个线程来运行runnable。

如果在调用时没有可用的线程，则此函数不执行任何操作并返回false。否则，runnable会立即使用一个可用线程运行，并且此函数返回true.

请注意，成功时，如果runnable->autoDelete () 返回，线程池将拥有可运行对象的所有权，并且在runnable- >run () 返回后，线程池将自动删除可运行对象。如果runnable->autoDelete () 返回，runnable的所有权仍然属于调用者。请注意，在调用此函数后更改runnable上的自动删除会导致未定义的行为。true``false

[since 5.15]bool QThreadPool::tryStart(std::function<void ()> functionToRun)#

这是一个重载功能。

尝试保留一个线程来运行functionToRun。

如果在调用时没有可用的线程，则此函数不执行任何操作并返回false。否则，functionToRun立即使用一个可用线程运行，并且此函数返回true.

这个函数是在 Qt 5.15 中引入的。

[since 5.9]bool QThreadPool::tryTake( *runnable)#

如果尚未启动，则尝试从队列中删除指定的可运行文件。如果 runnable 尚未启动，则返回true，并将runnable的所有权转移给调用者（即使是runnable->autoDelete() == true）。否则返回false。

注意：如果runnable->autoDelete() == true，此函数可能会删除错误的可运行文件。这被称为ABA 问题：原来的可运行对象可能已经执行并且已经被删除。内存被重新用于另一个可运行的，然后被删除而不是预期的。因此，我们建议仅对非自动删除的可运行文件调用此函数。

这个函数是在 Qt 5.9 中引入的。

See also start() and QRunnable::autoDelete().

bool QThreadPool::waitForDone(int msecs = -1)#

等待所有线程退出并从线程池中删除所有线程最多毫秒。true如果所有线程都被删除，则返回；否则返回false。如果msecs为 -1（默认值），则忽略超时（等待最后一个线程退出）。
