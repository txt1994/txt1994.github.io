详细描述#

文本光标是用于通过模仿文本编辑器中光标行为的编程接口访问和修改文本文档的内容和底层结构的对象。 QTextCursor 包含有关光标在 QTextDocument 中的位置以及它所做的任何选择的信息。

QTextCursor 以文本光标在文本编辑器中的行为方式为模型，提供通过用户界面执行标准操作的编程方式。可以将文档视为单个字符串。光标的当前 位置() 总是在字符串中的两个连续字符之间，或者在第一个字符之前或在字符串中的最后一个字符之后。除了文本之外，文档还可以包含表格、列表、图像和其他对象，但从开发人员的角度来看，文档可以被视为一个长字符串。该字符串的某些部分可以被认为位于特定的块（例如段落）内，或者在表格的单元格内，或者在列表的项目内，或其他结构元素内。当我们提到“当前字符”时，我们指的是文档中光标 位置() 之前的字符。类似地，“当前块”是包含光标 position() 的块。

QTextCursor 也有一个 anchor() 位置。 anchor() 和 position() 之间的文本是选择。如果 anchor() == position() 则没有选择。

可以使用 setPosition() 和 movePosition() 以编程方式更改光标位置；后者也可用于选择文本。有关选择，请参见 selectionStart()、selectionEnd()、hasSelection()、clearSelection() 和 removeSelectedText()。

如果 position() 位于块的开头，则 atBlockStart() 返回 true；如果它位于块的末尾，则 atBlockEnd() 返回 true。当前字符的格式由charFormat()返回，当前块的格式由blockFormat()返回。

可以使用 setCharFormat()、mergeCharFormat()、setBlockFormat() 和 mergeBlockFormat() 将格式应用于当前文本文档职能。 ‘set’ 函数将替换光标的当前字符或块格式，而 ‘merge’ 函数将给定的格式属性添加到光标的当前格式。如果光标有选择，则给定格式将应用于当前选择。请注意，当仅选择块的一部分时，块格式将应用于整个块。可以使用 createList() 将当前字符位置的文本转换为列表。

可以使用 deleteChar()、deletePreviousChar() 和 removeSelectedText() 来实现删除。

文本字符串可以使用 insertText() 函数插入到文档中，块（表示新段落）可以使用 insertBlock() 插入。

可以使用 insertFragment() 插入现有的文本片段，但是，如果要插入各种格式的文本片段，通常更容易使用 insertText() 并提供字符格式。

还可以使用光标将各种类型的高级结构插入到文档中：

  • 列表是用项目符号或符号装饰的块元素的有序序列。这些以使用 insertList() 的指定格式插入。
  • 使用 insertTable() 函数插入表格，并且可以提供可选格式。这些包含可以使用光标遍历的单元格数组。
  • 使用 insertImage() 插入内联图像。要使用的图像可以以图像格式或名称指定。
  • 通过调用具有指定格式的 insertFrame() 来插入帧。

可以使用 beginEditBlock() 和 endEditBlock() 对操作进行分组（即，将其视为撤消/重做的单个操作）。

光标移动仅限于有效的光标位置。在拉丁文写作中，这是在文本中任意两个连续字符之间、第一个字符之前或最后一个字符之后。在其他一些书写系统中，光标移动仅限于“簇”（例如梵文中的音节，或基本字母加变音符号）。 movePosition() 和 deleteChar() 等函数将光标移动限制在这些有效位置。

See also Rich Text Processing.

无效QTextCursor:: beginEditBlock ()#

指示文档上的一组编辑操作的开始，从撤消/重做的角度来看，该操作应显示为单个操作。

例如：

QTextCursor cursor(textDocument);
cursor.beginEditBlock();
cursor.insertText("Hello");
cursor.insertText("World");
cursor.endEditBlock();

textDocument->undo();

对 undo() 的调用将导致两个插入都被撤消，从而导致“World”和“Hello”都被删除。

可以嵌套调用 beginEditBlock 和 endEditBlock。最上面的对将确定撤消/重做操作的范围。

另见 endEditBlock()。

bool QTextCursor::hasComplexSelection() const#

如果光标包含的选择不仅仅是从 selectionStart() 到 selectionEnd() 的范围，则返回 true；否则返回“假”。

复杂选择是跨越表格中至少两个单元格的选择；它们的范围由 selectedTableCells() 指定。

*QTextCursor::insertTable(int rows, int columns, const &format)#

这是一个重载函数。

创建一个具有给定数量的 rows 和 columns 的新表，将其插入到文档中的当前光标 position() 处，并返回表对象。光标移动到第一个单元格的开头。

表格中必须至少有一行和一列。

另见 currentTable()。

void QTextCursor::insertText(const &text)#

使用当前字符格式在当前位置插入 text。

如果有选择，则删除选择并替换为 text，例如：

cursor.clearSelection();
cursor.movePosition(QTextCursor::NextWord, QTextCursor::KeepAnchor);
cursor.insertText("Hello World");

这将清除所有现有选择，选择光标处的单词（即从 position() 向前），并将选择替换为短语“Hello World”。

插入文本中的任何 ASCII 换行符 (\n) 都将转换为 unicode 块分隔符，对应于 insertBlock() 调用。

See also charFormat() and hasSelection().

void QTextCursor::insertText(const &text, const &format)#

这是一个重载函数。

使用给定的 format 在当前位置插入 text。

bool QTextCursor::isCopyOf(const &other) const#

如果此光标和 other 是彼此的副本，则返回 true，即其中一个被创建为另一个的副本并且此后两者都没有移动。这比平等要严格得多。

另见 operator=() 和 operator==()。

bool QTextCursor::isNull() const#

如果光标为空，则返回 true；否则返回“假”。空游标由默认构造函数创建。

void QTextCursor::joinPreviousEditBlock()#

像 beginEditBlock() 表示应该作为撤消/重做的单个操作出现的编辑操作块的开始。然而，与 beginEditBlock() 不同的是，它不会开始一个新块，而是反转对 endEditBlock() 的先前调用，因此使以下操作成为先前创建的编辑块的一部分。

例如：

QTextCursor cursor(textDocument);
cursor.beginEditBlock();
cursor.insertText("Hello");
cursor.insertText("World");
cursor.endEditBlock();

// ...

cursor.joinPreviousEditBlock();
cursor.insertText("Hey");
cursor.endEditBlock();

textDocument->undo();

对 undo() 的调用将导致所有三个插入都被撤消。

另见 beginEditBlock() 和 endEditBlock()。

void QTextCursor::mergeCharFormat(const &modifier)#

将光标的当前字符格式与格式 modifier 描述的属性合并。如果光标有选择，此函数将 modifier 中设置的所有属性应用于作为选择的一部分的所有字符格式。

另请参阅 hasSelection() 和 setCharFormat()。

bool QTextCursor::movePosition( operation, mode = MoveAnchor, int n = 1)#

通过执行给定的操作 n次，使用指定的模式移动光标，如果所有操作都成功完成，则返回true；否则返回“假”。

例如，如果重复使用此函数查找下一个单词的末尾，则最终会在到达文档末尾时失败。

默认情况下，移动操作执行一次 (n = 1)。

如果 mode 是 KeepAnchor，则光标选择它移动的文本。这与用户在按住 Shift 键并使用光标键移动光标时达到的效果相同。

另见 setVisualNavigation()。

int QTextCursor::position() const#

返回光标在文档中的绝对位置。光标位于字符之间。

注意： 本例中的“字符”指的是 QChar 对象的字符串，即 16 位 Unicode 字符，位置被视为该字符串的索引。这不一定对应于书写系统中的单个字素，因为单个字素可能由多个 Unicode 字符表示，例如在代理对、语言连字或变音符号的情况下。

另请参阅 setPosition()、movePosition()、anchor() 和 positionInBlock()。

int QTextCursor::positionInBlock() const#

返回光标在块内的相对位置。光标位于字符之间。

这相当于position() - block().position()。

注意： 本例中的“字符”指的是 QChar 对象的字符串，即 16 位 Unicode 字符，位置被视为该字符串的索引。这不一定对应于书写系统中的单个字素，因为单个字素可能由多个 Unicode 字符表示，例如在代理对、语言连字或变音符号的情况下。

See also position().

void QTextCursor::selectedTableCells(int *firstRow, int *numRows, int *firstColumn, int *numColumns) const#

如果选择跨越表格单元格，firstRow 填充选择中第一行的编号，firstColumn 填充选择中第一列的编号，numRows 和 numColumns 填充选择中的数量选择中的行和列。如果选择不跨越任何表格单元格，则结果是无害的，但未定义。

QTextCursor::selectedText() const#

返回当前选择的文本（可能为空）。这仅返回文本，没有富文本格式信息。如果您想要一个文档片段（即格式化的富文本），请改用 selection()。

注意： 如果从编辑器获得的选择跨越换行符，则文本将包含 Unicode U+2029 段落分隔符，而不是换行符 \n 字符。使用 QString::replace() 将这些字符替换为换行符。

QTextCursor::selection() const#

注意： 与 QTextDocumentFragment::toPlainText() 不同，selectedText() 可能包含特殊的 unicode 字符，例如 [QChar::ParagraphSeparator](qchar.html#特殊字符枚举）。

另见 QTextDocumentFragment::toPlainText()。

void QTextCursor::setKeepPositionOnInsert(bool b)#

定义在光标当前位置插入文本时光标是否应保持当前位置。

如果 b 为真，则在光标所在位置插入文本时，光标将保持其当前位置。如果 b 为 false，则光标会随插入的文本一起移动。

默认值为假。

请注意，在光标当前位置之前插入文本时，光标始终移动，而在光标当前位置之后插入文本时，光标始终保持其位置。

另见 keepPositionOnInsert()。

void QTextCursor::setPosition(int pos, m = MoveAnchor)#

使用 m 指定的 MoveMode 将光标移动到由 pos 指定的文档中的绝对位置。光标位于字符之间。

注意： 本例中的“字符”指的是 QChar 对象的字符串，即 16 位 Unicode 字符，pos 被认为是该字符串的索引。这不一定对应于书写系统中的单个字素，因为单个字素可能由多个 Unicode 字符表示，例如在代理对、语言连字或变音符号的情况下。对于更通用的文档导航方法，请使用 movePosition()，这将尊重文本中的实际字形边界。

另请参阅 位置()、movePosition() 和 anchor()。

void QTextCursor::setVerticalMovementX(int x)#

将垂直光标移动的视觉 x 位置设置为 x。

垂直移动x位置在光标水平移动时自动清除，在光标垂直移动时保持不变。该机制允许光标在具有比例字体的视觉直线上上下移动，并轻轻“跳过”短线。

值 -1 表示没有预定义的 x 位置。下次光标向上或向下移动时，它将自动设置。

另见 verticalMovementX()。

void QTextCursor::setVisualNavigation(bool b)#

将视觉导航设置为 b。

视觉导航意味着跳过隐藏的文本段落。默认值为假。

See also visualNavigation() and movePosition().
