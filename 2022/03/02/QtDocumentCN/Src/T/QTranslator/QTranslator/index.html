详细描述#

此类的对象包含一组从源语言到目标语言的翻译。 QTranslator 提供了在翻译文件中查找翻译的功能。 翻译文件是使用 Qt Linguist 创建的。

QTranslator 最常见的用途是：加载翻译文件，并使用 QCoreApplication::installTranslator() 安装它。

这是一个使用 QTranslator 的示例 main() 函数：

int main(int argc, char *argv[])
{
    QApplication app(argc, argv);

    QTranslator translator;
    // look up e.g. :/i18n/myapp_de.qm
    if (translator.load(QLocale(), QLatin1String("myapp"), QLatin1String("_"), QLatin1String(":/i18n")))
        QCoreApplication::installTranslator(&translator);

    QPushButton hello(QCoreApplication::translate("main", "Hello world!"));
    hello.resize(100, 30);

    hello.show();
    return app.exec();
}

请注意，必须在 之前 应用程序的小部件创建转换器。

大多数应用程序永远不需要对这个类做任何其他事情。 此类提供的其他功能对于处理翻译文件的应用程序很有用。

查找翻译#

可以使用 translate() 查找翻译（如 tr() 和 QCoreApplication::translate() 做） . translate() 函数最多需要三个参数：

  • context - 通常是 tr() 调用者的类名。
  • 源文本 - 通常是 tr() 的参数。
  • disambiguation - 一个可选字符串，有助于消除相同上下文中相同文本的不同用途的歧义。

例如，当程序以波兰语运行时，对话框中的“取消”可能有“Anuluj”（在这种情况下，源文本将是“取消”）。上下文（通常）是对话框的类名；通常没有评论，翻译后的文本将是“Anuluj”。

但这并不总是那么简单。带有双面打印和装订设置的西班牙语版本的打印机对话框可能需要“Activado”和“Activada”作为“已启用”的翻译。在这种情况下，源文本在这两种情况下都将是“启用”，上下文将是对话框的类名，但是这两个项目会有歧义，例如一个为“双面打印”，另一个为“绑定”。消歧使翻译者能够为西班牙语版本选择适当的性别，并使 Qt 能够区分翻译。

使用多个翻译#

可以在一个应用程序中安装多个翻译文件。翻译按安装时的相反顺序搜索，因此首先搜索最近安装的翻译文件，最后搜索最早的翻译文件。一旦找到包含匹配字符串的翻译，搜索就会停止。

这种机制使得特定翻译可以被“选择”或优先于其他翻译；只需将翻译器从应用程序中卸载，将其传递给 QCoreApplication::removeTranslator() 函数，然后使用 QCoreApplication::installTranslator() 重新安装它。然后它将是第一个搜索匹配字符串的翻译。

See also QCoreApplication::installTranslator(), QCoreApplication::removeTranslator(), QObject::tr(), QCoreApplication::translate(), I18N Example, Hello tr() Example, Arrow Pad Example, and Troll Print Example.

返回加载的翻译文件的路径。

如果此转换器为空，则返回 true，否则返回 false。 此功能适用于剥离和未剥离的翻译文件。

bool QTranslator::load(const &filename, const &directory = QString(), const &search_delimiters = QString(), const &suffix = QString())#

加载 filename + suffix（如果没有指定 suffix，则为“.qm”），它可以是绝对文件名或相对于 directory。如果翻译成功加载，则返回 true；否则返回“假”。

如果未指定 directory，则使用当前目录（即，作为 currentPath()）。

此翻译器对象的先前内容将被丢弃。

如果文件名不存在，则按以下顺序尝试其他文件名：

  1. 不带后缀的文件名。
  2. 去掉 search_delimiters 中的一个字符后带有文本的文件名（“_.”是 search_delimiters 的默认值，如果它是一个空字符串）和 suffix。
  3. 删除文件名，不附加后缀。

4.文件名进一步剥离等。

例如，在 fr_CA 语言环境（加拿大法语区）中运行的应用程序可能会调用 load(“foo.fr_ca”, “/opt/foolib”)。 load() 然后会尝试从这个列表中打开第一个现有的可读文件：

1./opt/foolib/foo.fr_ca.qm
2./opt/foolib/foo.fr_ca
3./opt/foolib/foo.fr.qm
4./opt/foolib/foo.fr
5./opt/foolib/foo.qm
6./opt/foolib/foo

通常，最好使用 QTranslator::load(const QLocale &, const QString &, const QString &, const [QString] (qstring.html) &, const QString &) 函数，因为它使用 QLocale::uiLanguages() 而不仅仅是语言环境名称，它指的是日期和数字的格式，不一定是 UI 语言。

bool QTranslator::load(const &locale, const &filename, const &prefix = QString(), const &directory = QString(), const &suffix = QString())#

加载 filename + prefix + ui 语言名称 + suffix（如果没有指定 suffix 则为“.qm”），它可以是绝对文件名或相对于目录。如果翻译成功加载，则返回 true；否则返回“假”。

此翻译器对象的先前内容将被丢弃。

如果文件名不存在，则按以下顺序尝试其他文件名：

  1. 不带后缀的文件名。
  2. 删除“_”字符和后缀后带有ui语言部分的文件名。
  3. 去掉了ui语言部分的文件名，不附加后缀。
  4. 进一步剥离 ui 语言部分的文件名等。

例如，在 locale 中运行的应用程序具有以下 ui 语言 - “es”, “fr-CA”, “de” 可能会调用 load(QLocale(), “foo” 、“.”、“/opt/foolib”、“.qm”）。 load() 会在 ui 语言中将 ‘-‘（破折号）替换为 ‘_’（下划线），然后尝试从该列表中打开第一个现有的可读文件：

1./opt/foolib/foo.es.qm
2./opt/foolib/foo.es
3./opt/foolib/foo.fr_CA.qm
4./opt/foolib/foo.fr_CA
5./opt/foolib/foo.fr.qm
6./opt/foolib/foo.fr
7./opt/foolib/foo.de.qm
8./opt/foolib/foo.de
9./opt/foolib/foo.qm
10./opt/foolib/foo。
11./opt/foolib/foo

在文件系统区分大小写的操作系统上，QTranslator 还尝试加载区域设置名称的小写版本。

bool QTranslator::load(const *data, int len, const &directory = QString())#

此函数重载 load()。

将长度为 len 的 QM 文件数据 data 加载到转换器中。

数据不会被复制。 调用者必须能够保证 data 不会被删除或修改。

directory 仅用于在加载 QM 文件的依赖项时指定基本目录。 如果文件没有依赖关系，则忽略此参数。

[virtual] QTranslator::translate(const char *context, const char *sourceText, const char *disambiguation = nullptr, int n = -1) const#

返回键的翻译 (context, sourceText, disambiguation)。 如果没有找到，也尝试 (context, sourceText, “”)。 如果仍然失败，则返回一个空字符串。

注意： 不完整的翻译可能会导致意外行为：如果未提供 (context, sourceText, “”) 的翻译，则在这种情况下，该方法实际上可能会返回不同 disambiguation 的翻译。

如果 n 不是 -1，它用于为翻译选择适当的形式（例如，“找到%n 个文件”与“找到%n 个文件”）。

如果您需要以编程方式将翻译插入 QTranslator，则可以重新实现此函数。

See also load().
