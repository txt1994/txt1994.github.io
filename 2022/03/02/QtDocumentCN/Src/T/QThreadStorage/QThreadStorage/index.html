详细描述#

QThreadStorage 是一个提供每个线程数据存储的模板类。

setLocalData() 函数为调用线程存储单个线程特定的值。 稍后可以使用 localData() 访问数据。

hasLocalData() 函数允许程序员确定数据是否先前已使用 setLocalData() 函数设置。 这对于延迟初始化也很有用。

如果 T 是指针类型，QThreadStorage 会取得数据的所有权（必须在堆上用 new 创建）并在线程正常退出或通过终止退出时将其删除。

例如，以下代码使用 QThreadStorage 为调用 cacheObject() 和 removeFromCache() 函数的每个线程存储单个缓存。 调用线程退出时，缓存会自动删除。

QThreadStorage<QCache<QString, SomeClass> > caches;

void cacheObject(const QString &key, SomeClass *object)
{
    caches.localData().insert(key, object);
}

void removeFromCache(const QString &key)
{
    if (!caches.hasLocalData())
        return;

    caches.localData().remove(key);
}

注意事项#

  • QThreadStorage 析构函数不会删除每个线程的数据。 QThreadStorage 仅在线程退出或多次调用 setLocalData() 时删除每个线程的数据。
  • QThreadStorage 可用于为 main() 线程存储数据。 当 QApplication 被销毁时，QThreadStorage 会删除 main() 线程的所有数据集，无论 main() 线程是否实际完成。

See also QThread.

成员函数文档#

QThreadStorage::QThreadStorage()#

构造一个新的每线程数据存储对象。

QThreadStorage::~QThreadStorage()#

销毁每线程数据存储对象。

注意：存储的每个线程数据不会被删除。 QThreadStorage 中留下的任何数据都会泄露。 在删除 QThreadStorage 之前，请确保所有使用 QThreadStorage 的线程都已退出。

See also hasLocalData().

bool QThreadStorage::hasLocalData() const#

如果 T 是指针类型，如果调用线程有可用的非零数据，则返回 true。

如果 T 是值类型，则返回是否已经通过调用 setLocalData 或 localData 构造了数据。

See also localData().

T &QThreadStorage::localData()#

返回对调用线程设置的数据的引用。

如果没有设置数据，这将创建一个类型为 T 的默认构造实例。

See also setLocalData() and hasLocalData().

T QThreadStorage::localData() const#

这是一个重载函数。

返回由调用线程设置的数据的副本。

See also hasLocalData().

void QThreadStorage::setLocalData(T data)#

将调用线程的本地数据设置为 data。 稍后可以使用 localData() 函数访问它。

如果 T 是指针类型，则 QThreadStorage 获取数据的所有权并在线程退出（正常或通过终止）或再次调用 setLocalData() 时自动删除它。

See also localData() and hasLocalData().
