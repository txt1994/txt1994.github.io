详细描述#

QFileDialog类使用户能够遍历文件系统，以选择一个或多个文件或一个目录。

创建一个QFileDialog的最简单的方法是使用静态函数。

fileName = QFileDialog::getOpenFileName(this,
    tr("Open Image"), "/home/jana", tr("Image Files (*.png *.jpg *.bmp)"));

在上面的例子中，一个模式化的QFileDialog是用一个静态函数创建的。该对话框最初显示”/home/jana “目录下的内容，并显示与字符串 “Image Files (*.png .jpg .bmp) “中给出的模式匹配的文件。文件对话框的父级被设置为this，窗口标题被设置为 “打开图像”。

如果你想使用多个过滤器，请用两个分号分隔每个过滤器。比如说:

"Images (*.png *.xpm *.jpg);;Text files (*.txt);;XML files (*.xml)"

你可以创建你自己的 QFileDialog 而不使用静态函数。通过调用 setFileMode()，你可以指定用户必须在对话框中选择什么:

QFileDialog dialog(this);
dialog.setFileMode(QFileDialog::AnyFile);

在上面的例子中，文件对话框的模式被设置为AnyFile，意味着用户可以选择任何文件，甚至可以指定一个不存在的文件。这种模式对于创建 “另存为 “文件对话框非常有用。如果用户必须选择一个现有的文件，则使用 ExistingFile，如果只能选择一个目录，则使用 Directory。请参阅 QFileDialog::FileMode 枚举，以获得完整的模式列表。

fileMode 属性包含了对话框的操作模式；这表明用户应该选择什么类型的对象。使用 setNameFilter() 来设置对话框的文件过滤器。例如:

dialog.setNameFilter(tr("Images (*.png *.xpm *.jpg)"));

在上面的例子中，过滤器被设置为"图像(*.png *.xpm *.jpg)"，这意味着只有扩展名为png、xpm或jpg的文件会被显示在QFileDialog中。你可以通过使用setNameFilters()来应用多个过滤器。使用 selectNameFilter() 来选择你所给的过滤器之一作为文件对话框的默认过滤器。

文件对话框有两种视图模式。列表和细节。List以文件和目录名称的列表形式显示当前目录的内容。Detail也显示文件和目录名称的列表，但在每个名称旁边提供额外的信息，如文件大小和修改日期。用setViewMode()设置模式:

dialog.setViewMode(QFileDialog::Detail);

在创建你自己的文件对话框时，你需要使用的最后一个重要函数是selectedFiles()。

QStringList fileNames;
if (dialog.exec())
    fileNames = dialog.selectedFiles();

在上面的例子中，一个模式化的文件对话框被创建和显示。如果用户点击了确定，他们选择的文件就被放在fileName中。

对话框的工作目录可以用setDirectory()来设置。当前目录中的每个文件都可以用selectFile()函数来选择。

Standard Dialogs 示例展示了如何使用 QFileDialog 以及其他内置的 Qt 对话框。

默认情况下，如果平台上有一个平台原生的文件对话框，它将被使用。在这种情况下，原本用于构建对话框的部件将不会被实例化，所以相关的访问器，如 layout() 和 itemDelegate() 将返回 null。你可以设置DontUseNativeDialog选项以确保使用基于部件的实现而不是本地对话框。

See also QDir, QFileInfo, QFile, QColorDialog, QFontDialog, Standard Dialogs Example, and Qt Widgets - Application Example.

enum QFileDialog::FileMode#

这个枚举用于指示用户在文件对话框中可以选择什么；也就是说，如果用户点击确定，对话框将返回什么。

Constant Value Description
QFileDialog::AnyFile 0 一个文件的名称，无论它是否存在。
QFileDialog::ExistingFile 1 单个现有文件的名称。
QFileDialog::Directory 2 一个目录的名称。文件和目录都可以显示。然而，本地Windows文件对话框不支持在目录选择器中显示文件。
QFileDialog::ExistingFiles 3 零个或多个现有文件的名称。

See also setFileMode().

enum QFileDialog::Option flags QFileDialog::Options#

Constant Value Description
QFileDialog::ShowDirsOnly 0x00000001 在文件对话框中只显示目录。默认情况下，文件和目录都会显示。(仅在目录文件模式下有效。)
QFileDialog::DontResolveSymlinks 0x00000002 不在文件对话框中解析符号链接。默认情况下，符号链接会被解析。
QFileDialog::DontConfirmOverwrite 0x00000004 如果选择了一个现有的文件，就不要要求确认。默认情况下会要求确认。

注意：在macOS上使用本地文件对话框时，不支持这个选项。

Constant Value Description
QFileDialog::DontUseNativeDialog 0x00000008 不要使用本地文件对话框。默认情况下，除非你使用包含 Q_OBJECT 宏的 QFileDialog 子类，或者平台没有你所需要的本地对话框类型，否则会使用本地文件对话框。

注意：此选项必须在改变对话框属性或显示对话框之前设置。

Constant Value Description
QFileDialog::ReadOnly 0x00000010 表示该模型是只读的。
QFileDialog::HideNameFilterDetails 0x00000020 表示是否隐藏文件名过滤器的详细信息。
QFileDialog::DontUseCustomDirectoryIcons 0x00000040 始终使用默认的目录图标。一些平台允许用户设置一个不同的图标。自定义图标查询会对网络或可移动驱动器的性能产生很大影响。设置这个值将启用图标提供者中的QFileIconProvider::DontUseCustomDirectoryIcons选项。这个枚举值是在Qt 5.2中添加的。

Options类型是QFlags

Constant Value Description

特性文档#

acceptMode : #

此属性持有对话框的接受模式

行动模式定义了该对话框是用于打开还是保存文件。

默认情况下，此属性被设置为AcceptOpen。

访问函数:

QFileDialog::AcceptMode acceptMode() const
void setAcceptMode(QFileDialog::AcceptMode mode)

See also AcceptMode.

defaultSuffix : #

如果没有指定其他后缀，添加到文件名中的后缀

此属性指定了一个字符串，如果文件名没有后缀，则会被添加到文件名中。后缀通常用于指示文件类型（例如，”txt “表示文本文件）。

如果第一个字符是一个点（’.’），它将被删除。

访问函数:

QString defaultSuffix() const
void setDefaultSuffix(const QString &suffix)

fileMode : #

此属性持有对话框的文件模式

文件模式定义了希望用户在对话框中选择的项目的数量和类型。

默认情况下，此属性被设置为任意文件。

此函数将设置文件名和接受DialogLabel的标签。可以在调用setFileMode()后设置自定义文本。

访问函数:

QFileDialog::FileMode fileMode() const
void setFileMode(QFileDialog::FileMode mode)

See also FileMode.

options : #

此属性持有影响对话框外观和感觉的各种选项

默认情况下，所有选项都被禁用。

应该在改变对话框属性或显示对话框之前设置选项（特别是DontUseNativeDialogs选项）。

在对话框可见时设置选项，不能保证对对话框立即产生影响（取决于选项和平台）。

在改变其他属性后设置选项可能导致这些值没有效果。

访问函数:

QFileDialog::Options options() const
void setOptions(QFileDialog::Options options)

See also setOption() and testOption().

[since 5.6]supportedSchemes : #

此属性持有文件对话框应允许导航到的URL方案。

设置此属性可以限制用户能够选择的URL的类型。它是应用程序声明其将支持获取文件内容的协议的一种方式。一个空列表意味着不应用任何限制（默认）。对本地文件的支持（”file “方案）是隐含的，并且总是启用的；没有必要在限制中包括它。

这个属性是在Qt 5.6中引入的。

访问函数:

QStringList supportedSchemes() const
void setSupportedSchemes(const QStringList &schemes)

viewMode : #

此属性持有文件和目录在对话框中的显示方式

默认情况下，Detail模式用于显示文件和目录的信息。

访问函数:

QFileDialog::ViewMode viewMode() const
void setViewMode(QFileDialog::ViewMode mode)

See also ViewMode.

See also setMimeTypeFilters() and setNameFilters().

void QFileDialog::setNameFilters(const &filters)#

设置文件对话框中使用的filters过滤器。

注意过滤器***.是不可移植的，因为文件扩展名决定文件类型的历史假设在每个操作系统上都不一致。有可能出现一个名称中没有点的文件（例如，Makefile）。在本地Windows文件对话框中，.*将匹配这样的文件，而在其他类型的文件对话框中，它可能不匹配。因此，如果你想选择任何文件，最好使用***。

const QStringList filters({"Image files (*.png *.xpm *.jpg)",
                           "Text files (*.txt)",
                           "Any files (*)"
                          });
QFileDialog dialog(this);
dialog.setNameFilters(filters);
dialog.exec();

setMimeTypeFilters()的优点是为每个文件类型提供所有可能的名称过滤器。例如，JPEG图像有三种可能的扩展名；如果你的应用程序可以打开这样的文件，选择image/jpegmime类型作为过滤器将允许你打开所有的文件。

See also nameFilters().

void QFileDialog::setOption( option, bool on = true)#

如果on为真，则设置给定的option为启用；否则，清除给定的选项。

选项（特别是DontUseNativeDialogs选项）应该在改变对话框属性或显示对话框之前设置。

在对话框可见时设置选项并不能保证对对话框产生即时的影响（这取决于选项和平台）。

在改变其他属性后设置选项可能导致这些值没有效果。

See also options and testOption().

void QFileDialog::setProxyModel( *proxyModel)#

将视图的模型设置为给定的proxyModel。如果你想修改底层模型，这是非常有用的；例如，添加列、过滤数据或添加驱动器。

任何现有的代理模型将被移除，但不会被删除。文件对话框将取得proxyModel的所有权。

See also proxyModel().
