# QFileOpenEvent Class

QFileOpenEvent类提供了一个事件，当有请求打开一个文件或一个URL时，该事件将被发送。. More…

详细描述#

当操作系统要求打开一个文件或URL时，文件打开事件将被发送至QApplication::instance()。这是一个高层次的事件，根据用户的桌面环境，可以由不同的用户操作引起；例如，在macOS上双击Finder中的文件图标。

该事件仅用于通知应用程序的请求。它可以被安全地忽略。

注意：该类目前只支持macOS。

macOS Example#

为了在macOS上触发事件，应用程序必须被配置成让操作系统知道它应该对哪种文件做出反应。

例如，下面的Info.plist文件声明该应用程序可以作为PNG扩展名的文件的查看器：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleDocumentTypes</key>
    <array>
        <dict>
            <key>CFBundleTypeExtensions</key>
            <array>
                <string>png</string>
            </array>
            <key>CFBundleTypeRole</key>
            <string>Viewer</string>
        </dict>
    </array>
</dict>
</plist>

下面是一个QApplication子类的实现，它打印了文件的路径，例如，被丢在应用程序的Dock图标上。

#include <QApplication>
#include <QDebug>
#include <QFileOpenEvent>
#include <QPushButton>

class MyApplication : public QApplication
{
public:
    MyApplication(int &argc, char **argv)
        : QApplication(argc, argv)
    {
    }

    bool event(QEvent *event) override
    {
        if (event->type() == QEvent::FileOpen) {
            QFileOpenEvent *openEvent = static_cast<QFileOpenEvent *>(event);
            qDebug() << "Open file" << openEvent->file();
        }

        return QApplication::event(event);
    }
};

成员函数文档#

QFileOpenEvent::file() const#

返回正在打开的文件。

bool QFileOpenEvent::openFile( &file, flags) const#

以flags指定的模式，在该事件所引用的文件上打开一个QFile。如果成功返回 “true”；否则返回 “false”。

这是必要的，因为有些文件不能通过名字打开，而是需要存储在此事件中的特定信息。

QFileOpenEvent::url() const#

返回正在打开的URL。
