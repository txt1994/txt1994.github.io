详细描述#

QFileInfo 可以获取任一文件的名字、路径、访问权限、是否是文件夹及是否是符号链接等信息，也可以获取文件的大小、上次修改时间、上次读取时间，还可以获取 Qt 资源文件 的信息。

QFileInfo 可以表示相对路径也可以表示绝对路径。绝对路径以 “/” （windows 上以驱动符号）开头（例如 “/tmp/quartz”），相对路径以目录名或文件名开头，相对于当前工作目录（例如 “src/fatlib”）。可以用 isRelative() 检查一个 QFileInfo 表示的是相对路径还是绝对路径。可以用 makeAbsolute() 将相对路径转为绝对路径。

QFileInfo 可以在构造函数中或稍后在 setFile() 设置文件，可以用 exists() 查看文件是否存在，可以用 size() 获取文件大小。

文件类型可以用 isFile()、isDir() 和 isSymLink() 获取。symLinkTarget() 函数提供了符号链接指向的文件名。

在 Unix 系统（包括 macOS 和 iOS），此类的属性的 getter 函数（例如获取时间或大小）返回的都是目标文件的值，而不是符号链接的（因为Unix透明地处理符号链接）。使用 QFile 打开符号链接实际上是打开链接的目标。举例如下：

#ifdef Q_OS_UNIX

QFileInfo info1("/home/bob/bin/untabify");
info1.isSymLink();          // returns true
info1.absoluteFilePath();   // returns "/home/bob/bin/untabify"
info1.size();               // returns 56201
info1.symLinkTarget();      // returns "/opt/pretty++/bin/untabify"

QFileInfo info2(info1.symLinkTarget());
info2.isSymLink();          // returns false
info2.absoluteFilePath();   // returns "/opt/pretty++/bin/untabify"
info2.size();               // returns 56201

#endif

在Windows上，快捷方式（“.lnk”文件）当前被视为符号链接。 和Unix系统一样，属性的 getter 返回目标文件的大小，而不是.lnk文件本身。 此行为已被弃用，并且可能会在Qt的未来版本中删除，此后，.lnk 文件将被视为常规文件。

#ifdef Q_OS_WIN

QFileInfo info1("C:\\Documents and Settings\\Bob\\untabify.lnk");
info1.isSymLink();          // returns true
info1.absoluteFilePath();   // returns "C:/Documents and Settings/Bob/untabify.lnk"
info1.size();               // returns 743
info1.symLinkTarget();      // returns "C:/Pretty++/untabify"

QFileInfo info2(info1.symLinkTarget());
info2.isSymLink();          // returns false
info2.absoluteFilePath();   // returns "C:/Pretty++/untabify"
info2.size();               // returns 63942

#endif

可以使用 path() 和 fileName() 提取文件名的元素。 fileName() 的部分可以用 baseName()、suffix() 或 completeSuffix() 提取。 由 Qt 类创建的目录的 QFileInfo 对象将没有尾随文件分隔符。 如果您希望在自己的文件信息对象中使用尾随分隔符，只需将一个附加到给定构造函数或 setFile() 的文件名。

extern Q_CORE_EXPORT int qt_ntfs_permission_lookup;

然后，通过将qt_ntfs_permission_lookup递增和递减 1 来打开和关闭权限检查。

qt_ntfs_permission_lookup++; // 打开检查
qt_ntfs_permission_lookup--; // 再次关闭

性能问题#

QFileInfo 的一些函数查询文件系统，但出于性能原因，一些函数只对文件名本身进行操作。 例如：要返回相对文件名的绝对路径，absolutePath() 必须查询文件系统。 但是，path() 函数可以直接处理文件名，因此速度更快。

注意: 为了提高性能，QFileInfo 缓存有关文件的信息。

因为文件可以被其他用户或程序更改，甚至可以被同一程序的其他部分更改，所以有一个刷新文件信息的函数：refresh()。 如果您想关闭 QFileInfo 的缓存并在每次向它请求信息时强制它访问文件系统，请调用 setCaching(false)。

另请参阅 QDir and QFile.

QFileInfo::absoluteFilePath() const#

返回包含文件名的绝对路径。

绝对路径名由完整路径和文件名组成。 在Unix上，它将始终以根目录“ /”开头。 在Windows上，它将始终以“ D：/”开头，其中D是驱动器号，但未映射到驱动器号的网络共享除外，在这种情况下，路径将以“ // sharename /”开头。 QFileInfo 将大写驱动器号。

返回包含文件名的绝对路径。

绝对路径名由完整路径和文件名组成。 在 Unix 上，这将始终以根目录“/”开头。 在 Windows 上，这将始终以“D:/”开头，其中 D 是驱动器号，但未映射到驱动器号的网络共享除外，在这种情况下，路径将以“//sharename/”开头。 QFileInfo 将大写驱动器号。 请注意，QDir 不会这样做。 下面的代码片段显示了这一点。

QFileInfo fi("c:/temp/foo"); => fi.absoluteFilePath() => "C:/temp/foo"

此函数返回与 filePath() 相同的结果，除非 isRelative() 为真。 与 canonicalFilePath() 相比，符号链接或多余的“.” 或“..”元素不一定被删除。

警告❤️如果 filePath() 为空，则此函数的行为未定义。

另请参阅 filePath(), canonicalFilePath(), and isRelative().

QFileInfo::absolutePath() const#

返回文件的路径绝对路径。 这不包括文件名。

在 Unix 上，绝对路径总是以根目录“/”开头。 在 Windows 上，这将始终以“D:/”开头，其中 D 是驱动器号，但未映射到驱动器号的网络共享除外，在这种情况下，路径将以“//sharename/”开头。

与 canonicalPath() 符号链接或多余的“.”相反 或“..”元素不一定被删除。

警告❤️如果 filePath() 为空，则此函数的行为未定义。

另请参阅 absoluteFilePath(), path(), canonicalPath(), fileName(), and isRelative().

QFileInfo::baseName() const#

返回不带路径的文件的基本名称。

基本名称由文件中的所有字符组成，直到（但不包括）第一个 ‘.’ 特点。

例子：

QFileInfo fi("/tmp/archive.tar.gz");
QString base = fi.baseName();  // base = "archive"

文件的基本名称在所有平台上均等计算，与文件命名约定无关（例如，Unix 上的“.bashrc”有一个空的基本名称，后缀是“bashrc”）。

另请参阅 fileName(), suffix(), completeSuffix(), and completeBaseName()

QFileInfo::birthTime() const#

返回文件创建/诞生的日期和时间。

如果文件出生时间不可用，该函数返回无效的QDateTime

如果文件是符号链接，则返回目标文件的时间（不是符号链接）。

这个函数是在 Qt 5.10 中引入的。

另请参阅 lastModified(), lastRead(), and metadataChangeTime()

QFileInfo::bundleName() const#

返回捆绑包的名称。

在 macOS 和 iOS 上，如果路径为 isBundle()，则会返回包的正确本地化名称。 在所有其他平台上，返回一个空的 QString。

例子：

QFileInfo fi("/Applications/Safari.app");
QString bundle = fi.bundleName();                // name = "Safari"

这个函数是在 Qt 4.3 中引入的。

另请参阅 isBundle(), filePath(), baseName(), and suffix().

QFileInfo::canonicalFilePath() const#

返回包括文件名在内的规范路径，即一个没有符号链接或多余的”. “或”. “元素的绝对路径。

如果文件不存在，canonicalFilePath()返回一个空字符串。

另请参阅 filePath(), absoluteFilePath(), and dir().

QFileInfo::canonicalPath() const#

返回文件的典型路径（不包括文件名），即一个没有符号链接或多余的”. “或”. “元素的绝对路径。

如果文件不存在，canonicalPath()返回一个空字符串。

另请参阅 path() and absolutePath().

QFileInfo::completeBaseName() const#

返回文件的完整基本名称，不包括路径。

完整的基本名称包括文件中的所有字符，直到（但不包括）最后一个’.’字符。

例如:

QFileInfo fi("/tmp/archive.tar.gz");
QString base = fi.completeBaseName();  // base = "archive.tar"

另请参阅 fileName(), suffix(), completeSuffix(), and baseName().

QFileInfo::completeSuffix() const#

返回文件的完整后缀（扩展名）。

完整的后缀包括文件中第一个’.’之后（但不包括）的所有字符。

例如:

QFileInfo fi("/tmp/archive.tar.gz");
QString ext = fi.completeSuffix();  // ext = "tar.gz"

另请参阅 fileName(), suffix(), baseName(), and completeBaseName().

QFileInfo::dir() const#

返回对象的父目录的路径为QDir对象。

注意 👉返回的QDir总是对应于对象的父目录，即使QFileInfo代表一个目录。

对于下面的每一个，dir()返回 QDir "~/examples/191697"。

QFileInfo fileInfo2("~/examples/191697/..");

对于以下各项，dir()返回QDir ".".

QFileInfo fileInfo4(".");
QFileInfo fileInfo5("..");
QFileInfo fileInfo6("main.cpp");

另请参阅 absolutePath(), filePath(), fileName(), isRelative(), and absoluteDir().

bool QFileInfo::exists() const#

如果文件存在，返回 “true”；否则返回 “false”。

注意 👉如果文件是一个指向不存在的文件的符号链接，则返回false。

[static]bool QFileInfo::exists(const &file)#

如果file存在，返回 “true”；否则返回 “false”。

注意 👉如果file是一个指向不存在的文件的符号链接，则返回false。

注意 👉使用这个函数比使用QFileInfo(file).exists()来访问文件系统要快。

QFileInfo::fileName() const#

返回文件的名称，不包括路径。

例子:

QFileInfo fi("/tmp/archive.tar.gz");
QString name = fi.fileName();                // name = "archive.tar.gz"

注意，如果这个QFileInfo对象被赋予一个以斜线结尾的路径，那么文件的名字就会被认为是空的。

另请参阅 isRelative(), filePath(), baseName(), and suffix().

QFileInfo::filePath() const#

返回文件名，包括路径（可以是绝对路径或相对路径）。

另请参阅 absoluteFilePath(), canonicalFilePath(), and isRelative().

QFileInfo::fileTime( time) const#

返回由时间指定的文件时间。如果不能确定时间，将返回无效的日期时间。

如果文件是一个符号链接，将返回目标文件的时间（而不是符号链接）。

这个函数是在 Qt 5.10 中引入的。

另请参阅 QFile::FileTime and QDateTime::isValid().

QFileInfo::group() const#

返回该文件的组。在Windows下，在文件没有分组的系统中，或者如果发生错误，将返回一个空字符串。

这个函数在Unix下可能很耗时（以毫秒为单位）。

如果文件是一个符号链接，这个函数返回目标文件的组（而不是符号链接）。

另请参阅 groupId(), owner(), and ownerId().

QFileInfo::groupId() const#

返回文件所属组的 id。

在 Windows 和文件没有组的系统上，此函数始终返回 (uint) -2。

如果文件是符号链接，则此函数返回拥有目标（而不是符号链接）的组的 ID。

另请参阅 group(), owner(), and ownerId().

bool QFileInfo::isAbsolute() const#

如果文件路径名是绝对的，返回 “true”；如果是相对的，则返回 “false”。

另请参阅 isRelative().

bool QFileInfo::isBundle() const#

如果此对象指向包或指向 macOS 和 iOS 上包的符号链接，则返回 true； 否则返回false。

如果文件是符号链接，如果目标是包（不是符号链接），则此函数返回 true。

这个函数是在 Qt 4.3 中引入的。

另请参阅 isDir(), isSymLink(), and isFile().

bool QFileInfo::isDir() const#

如果此对象指向目录或指向目录的符号链接，则返回 true； 否则返回false。

如果文件是符号链接，如果目标是目录（不是符号链接），则此函数返回 true。

另请参阅 isFile(), isSymLink(), and isBundle().

bool QFileInfo::isExecutable() const#

如果文件是可执行的，返回 “true”；否则返回 “false”。

如果文件是一个符号链接，如果目标是可执行的（不是符号链接），该函数返回true。

另请参阅 isReadable(), isWritable(), and permission().

bool QFileInfo::isFile() const#

如果这个对象指向一个文件或一个文件的符号链接，返回true。如果这个对象指向的不是一个文件，例如一个目录，则返回false。

如果文件是一个符号链接，如果目标是一个普通文件（不是符号链接），该函数返回true。

另请参阅 isDir(), isSymLink(), and isBundle().

bool QFileInfo::isHidden() const#

如果这是一个 “隐藏 “的文件，返回 “true”；否则返回 “false”。

注意 👉这个函数对Unix上的特殊条目”. “和”. “返回true，尽管QDir::entryList威胁它们，如图。请注意，由于这个函数检查的是文件名，在Unix上它将检查符号链接的名称，如果这个文件是一个符号链接，而不是目标的名称。

在Windows上，如果目标文件是隐藏的（而不是符号链接），这个函数返回 “true”。

bool QFileInfo::isJunction() const#

如果对象指向一个联结点，则返回 true； 否则返回false。

连接只存在于 Windows 的 NTFS 文件系统上，通常由 mklink 命令创建。 它们可以被认为是目录的符号链接，并且只能为本地卷上的绝对路径创建。

这个函数是在 Qt 5.15 中引入的。

bool QFileInfo::isNativePath() const#

如果文件路径可以直接与本机 API 一起使用，则返回 true。 如果 Qt 中的虚拟文件系统（例如 Qt 资源系统）支持该文件，则返回 false。

注意 👉本机路径可能仍需要转换路径分隔符和字符编码，具体取决于本机 API 的平台和输入要求。

这个函数是在 Qt 5.0 中引入的。

另请参阅 QDir::toNativeSeparators(), QFile::encodeName(), filePath(), absoluteFilePath(), and canonicalFilePath().

bool QFileInfo::isReadable() const#

如果用户可以读取文件，则返回 true； 否则返回false。

如果文件是符号链接，如果目标是可读的（不是符号链接），则此函数返回 true。

注意 👉如果未启用 NTFS 权限 检查，Windows 上的结果将仅反映文件是否存在。

另请参阅 isWritable(), isExecutable(), and permission().

bool QFileInfo::isRelative() const#

如果文件路径名是相对的，则返回 true，否则，如果路径是绝对的，则返回 false（例如，在 Unix 下，如果路径以“/”开头，则该路径是绝对的）。

另请参阅 isAbsolute().

bool QFileInfo::isRoot() const#

如果该对象指向一个目录或一个目录的符号链接，并且该目录是根目录，则返回 “true”；否则返回 “false”。

bool QFileInfo::isShortcut() const#

如果这个对象指向一个快捷方式，返回 “true”；否则返回 “false”。

快捷方式只存在于Windows上，通常是.lnk文件。例如，Windows上的快捷方式（*.lnk文件）将返回true，但Unix（包括macOS和iOS）上将返回false。

快捷方式（.lnk）文件被视为普通文件。打开这些文件将打开.lnk文件本身。为了打开一个快捷方式引用的文件，它必须在一个快捷方式上使用symLinkTarget()。

注意 👉👉即使一个快捷方式（破损的快捷方式）指向一个不存在的文件，isShortcut()也会返回true。

另请参阅 isFile(), isDir(), isSymbolicLink(), and symLinkTarget().

bool QFileInfo::isSymLink() const#

如果这个对象指向一个符号链接或快捷方式，返回 “true”；否则返回 “false”。

符号链接存在于Unix（包括macOS和iOS）和Windows，通常分别由ln -s或mklink命令创建。打开一个符号链接可以有效地打开[链接的目标]（#symLinkTarget）。

此外，Windows上的快捷方式（*.lnk文件）将返回true。这种行为已经过时了，可能会在未来的Qt版本中改变。打开这些文件将打开.lnk文件本身。

例子:

QFileInfo info(fileName);
if (info.isSymLink())
    fileName = info.symLinkTarget();

注意 👉If the symlink points to a non existing file, exists() returns false.

另请参阅 isFile(), isDir(), and symLinkTarget().

bool QFileInfo::isSymbolicLink() const#

如果此对象指向符号链接，则返回 true； 否则返回false。

符号链接存在于 Unix（包括 macOS 和 iOS）和 Windows（NTFS 符号链接）上，通常分别由 ln -s 或 mklink 命令创建。

Unix 透明地处理符号链接。 打开符号链接有效地打开了链接的目标。

与 isSymLink() 相比，Windows 上的快捷方式（*.lnk 文件）将返回 false。 请改用 QFileInfo::isShortcut()。

注意 👉如果符号链接指向一个不存在的文件，exists() 返回 false。

另请参阅 isFile(), isDir(), isShortcut(), and symLinkTarget().

bool QFileInfo::isWritable() const#

如果用户可以写入文件，则返回 true； 否则返回false。

如果文件是符号链接，如果目标是可写的（不是符号链接），则此函数返回 true。

注意 👉如果未启用 NTFS 权限 检查，Windows 上的结果将仅反映文件是否标记为只读。

另请参阅 isReadable(), isExecutable(), and permission().

QFileInfo::lastModified() const#

返回上次修改文件的日期和本地时间。

如果文件是符号链接，则返回目标文件的时间（不是符号链接）。

另请参阅 birthTime(), lastRead(), metadataChangeTime(), and fileTime().

QFileInfo::lastRead() const#

返回上次读取（访问）文件的日期和本地时间。

在此信息不可用的平台上，返回与 lastModified() 相同的值。

如果文件是符号链接，则返回目标文件的时间（不是符号链接）。

另请参阅 birthTime(), lastModified(), metadataChangeTime(), and fileTime().

bool QFileInfo::makeAbsolute()#

如果文件的路径还不是绝对路径，则将其转换为绝对路径。返回 “true “表示路径被转换；否则返回 “false “表示路径已经是绝对路径。

另请参阅 filePath() and isRelative().

QFileInfo::metadataChangeTime() const#

返回文件元数据更改的日期和时间。 创建文件时会发生元数据更改，但每当用户写入或设置 inode 信息时也会发生元数据更改（例如，更改文件权限）。

如果文件是符号链接，则返回目标文件的时间（不是符号链接）。

这个函数是在 Qt 5.10 中引入的。

另请参阅 lastModified() and lastRead().

QFileInfo::owner() const#

返回文件的所有者。在文件没有所有者的系统中，或者如果发生错误，将返回一个空字符串。

这个函数在Unix下可能很耗时（以毫秒为单位）。在Windows下，它将返回一个空字符串，除非NTFS权限检查已被启用。

如果文件是一个符号链接，这个函数将返回目标文件的所有者（而不是符号链接）。

另请参阅 ownerId(), group(), and groupId().

QFileInfo::ownerId() const#

返回文件所有者的ID。

在Windows和文件没有所有者的系统中，该函数返回（（uint）-2）。

如果文件是一个符号链接，这个函数返回目标文件（而不是符号链接）的所有者的ID。

另请参阅 owner(), group(), and groupId().

QFileInfo::path() const#

返回文件的路径。这并不包括文件名。

注意，如果这个QFileInfo对象被赋予一个以斜线结尾的路径，那么文件的名称被认为是空的，这个函数将返回整个路径。

另请参阅 filePath(), absolutePath(), canonicalPath(), dir(), fileName(), and isRelative().

bool QFileInfo::permission( permissions) const#

测试文件的权限。权限参数可以是几个QFile::Permissions类型的标志，它们被OR在一起以检查权限组合。

在文件没有权限的系统中，这个函数总是返回 “true”。

注意 👉如果NTFS权限检查没有被启用，那么在Windows上的结果可能是不准确的。

例子:

QFileInfo fi("/tmp/archive.tar.gz");
if (fi.permission(QFile::WriteUser | QFile::ReadGroup))
    qWarning("I can change the file; my group can read the file");
if (fi.permission(QFile::WriteGroup | QFile::WriteOther))
    qWarning("The group or others can change the file");

如果文件是一个符号链接，这个函数检查目标文件（而不是符号链接）的权限。

另请参阅 isReadable(), isWritable(), and isExecutable().

QFileInfo::permissions() const#

返回文件的 QFile::Permissions 的完整OR-ed together组合。

注意 👉如果NTFS权限检查没有被启用，那么在Windows上的结果可能是不准确的。

如果文件是一个符号链接，该函数返回目标（而不是符号链接）的权限。

void QFileInfo::refresh()#

刷新关于文件的信息，即在下次获取缓存属性时从文件系统中读入信息。

void QFileInfo::setCaching(bool enable)#

如果enable为true，启用文件信息的缓存。如果使能为假，则禁用缓存。

当启用缓存时，QFileInfo会在第一次需要的时候从文件系统中读取文件信息，但通常不会在以后读取。

缓存在默认情况下是启用的。

另请参阅 refresh() and caching().

void QFileInfo::setFile(const &file)#

设置 QFileInfo 提供有关文件的信息的文件。

该文件还可以包括绝对或相对文件路径。 绝对路径以目录分隔符（例如 Unix 下的“/”）或驱动器规范（Windows 下）开头。 相对文件名以目录名或文件名开头，并指定相对于当前目录的路径。

例子：

QString absolute = "/local/bin";
QString relative = "local/bin";
QFileInfo absFile(absolute);
QFileInfo relFile(relative);

QDir::setCurrent(QDir::rootPath());
// absFile and relFile now point to the same file

QDir::setCurrent("/tmp");
// absFile now points to "/local/bin",
// while relFile points to "/tmp/local/bin"

另请参阅 isFile(), isRelative(), QDir::setCurrent(), and QDir::isRelativePath().

void QFileInfo::setFile(const &file)#

这是一个重载函数。

设置 QFileInfo 提供有关文件的信息的文件。

如果 file 包含相对路径，则 QFileInfo 也将具有相对路径。

另请参阅 isRelative().

void QFileInfo::setFile(const &dir, const &file)#

这是一个重载函数。

将 QFileInfo 提供的信息设置到目录 dir 中的文件。

如果文件包含相对路径，则 QFileInfo 也将具有相对路径。

另请参阅 isRelative().

QFileInfo::size() const#

返回文件大小（以字节为单位）。 如果文件不存在或无法获取，则返回 0。

如果文件是符号链接，则返回目标文件的大小（不是符号链接）。

另请参阅 exists().

QFileInfo::suffix() const#

返回文件的后缀（扩展名）。

后缀由文件中最后一个“.”之后（但不包括）的所有字符组成。

例子：

QFileInfo fi("/tmp/archive.tar.gz");
QString ext = fi.suffix();  // ext = "gz"

文件的后缀在所有平台上均等计算，与文件命名约定无关（例如，Unix 上的“.bashrc”有一个空的基本名称，后缀是“bashrc”）。

另请参阅 fileName(), completeSuffix(), baseName(), and completeBaseName().

void QFileInfo::swap(QFileInfo &other)#

与其他交换此文件信息。 此功能非常快且永不失败。

这个函数是在 Qt 5.0 中引入的。

QFileInfo::symLinkTarget() const#

返回符号链接指向的文件或目录的绝对路径，如果对象不是符号链接，则返回空字符串。

此名称可能不代表现有文件； 它只是一个字符串。 QFileInfo::exists() 如果符号链接指向现有文件，则返回 true。

这个函数是在 Qt 4.2 中引入的。

另请参阅 exists(), isSymLink(), isDir(), and isFile().

bool QFileInfo::operator!=(const QFileInfo &fileinfo) const#

如果此 QFileInfo 对象引用的文件与 fileinfo 指定的文件不同，则返回 true； 否则返回false。

另请参阅 operator==().

bool QFileInfo::operator==(const QFileInfo &fileinfo) const#

如果此 QFileInfo 对象引用与 fileinfo 位于同一位置的文件，则返回 true； 否则返回false。

请注意，比较两个不包含文件引用（不存在或为空的文件路径）的空 QFileInfo 对象的结果是未定义的。

警告❤️这不会比较指向同一文件的两个不同的符号链接。

警告❤️在 Windows 上引用同一文件的长文件名和短文件名被视为引用不同的文件。

另请参阅 operator!=()
