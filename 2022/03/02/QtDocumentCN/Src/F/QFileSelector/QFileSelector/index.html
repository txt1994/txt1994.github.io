详细描述#

QFileSelector 是一种基于平台或设备特性来选择文件变体的便利工具。这允许你在某些情况下更容易开发和部署一个包含所有不同变体的代码库，例如在部署步骤中无法确定正确的变体。

Using QFileSelector#

如果你总是使用同一个文件，你就不需要使用QFileSelector。

考虑一下下面的用法示例，你想在不同的语言环境中使用不同的设置文件。你可能会像这样在不同的locales之间选择代码:

QString defaultsBasePath = "data/";
QString defaultsPath = defaultsBasePath + "defaults.conf";
QString localizedPath = defaultsBasePath
    + QString("%1/defaults.conf").arg(QLocale().name());
if (QFile::exists(localizedPath))
    defaultsPath = localizedPath;
QFile defaults(defaultsPath);

同样，如果你想根据目标平台挑选一个不同的数据文件，你的代码可能看起来像这样:

QString defaultsPath = "data/defaults.conf";
#if defined(Q_OS_ANDROID)
defaultsPath = "data/android/defaults.conf";
#elif defined(Q_OS_IOS)
defaultsPath = "data/ios/defaults.conf";
#endif
QFile defaults(defaultsPath);

QFileSelector 提供了一个方便的替代方案来编写这样的模板代码，在后一种情况下，它允许你开始使用特定平台的配置，而无需重新编译。QFileSelector 还允许以一种方便的方式将多个选择器串联起来，例如，只在特定的平台和地域组合中选择不同的文件。例如，要根据平台和/或语言环境进行选择，代码如下:

QFileSelector selector;
QFile defaultsFile(selector.select("data/defaults.conf"));

要选择的文件被放在以’+’和选择器名称命名的目录中。在上面的例子中，你可以通过将平台配置放在以下位置来选择它们:

data/defaults.conf
    data/+android/defaults.conf
    data/+ios/+en_GB/defaults.conf

为了找到被选中的文件，QFileSelector会在与基础文件相同的目录中寻找。如果有任何形式的+的目录有一个激活的选择器，QFileSelector将优先选择该目录中具有相同文件名的文件而不是基础文件。这些目录可以被嵌套以检查多个选择器，例如:

images/background.png
    images/+android/+en_GB/background.png

有了这些文件，你会在android平台上选择一个不同的文件，但前提是locale是en_GB。

为了在没有有效选择器的情况下进行错误处理，建议在基本文件位置上有一个默认的或错误处理的文件，即使你期望选择器在所有部署中都存在。

在未来的版本中，一些选择器可能会被标记为部署时静态的，并在部署步骤中作为一种优化被移动。由于选择器是有性能成本的，建议在涉及性能关键代码的情况下避免使用它们。

Adding Selectors#

通常可用的选择器有

  • 平台，与应用程序所运行的平台相匹配的以下任何一个字符串（列表不详尽）：android, ios, osx, darwin, mac, macos, linux, qnx, unix, windows。在Linux上，如果可以确定的话，也可以确定发行版的名称，如debian、fedora或opensuse。
  • locale，与QLocale().name()相同。

进一步的选择器将从QT_FILE_SELECTORS环境变量中添加，当设置时，它应该是一组逗号分隔的选择器。注意，这个变量只会被读取一次；如果该变量在应用程序运行时发生变化，选择器可能不会更新。初始的选择器集只被评估一次，在第一次使用时。

你也可以在运行时为自定义行为添加额外的选择器。这些将被用于今后对选择()的任何调用。如果额外的选择器列表被改变了，对选择()的调用将使用新的列表，并可能返回不同的结果。

Conflict Resolution when Multiple Selectors Apply#

当多个选择器可以应用于同一个文件时，将选择第一个匹配的选择器。选择器被检查的顺序是。

  1. 通过setExtraSelectors()设置的选择器，按其在列表中的顺序排列
  2. QT_FILE_SELECTORS环境变量中的选择器，从左到右
  3. 地区设置
  4. 平台

下面是一个涉及多个选择器同时匹配的例子。它使用了平台选择器，另外还有一个名为 “admin “的选择器是由应用程序根据用户证书设置的。这个例子的排序是这样的：如果所有的选择器都存在，将选择最低的匹配文件:

images/background.png
    images/+linux/background.png
    images/+windows/background.png
    images/+admin/background.png
    images/+admin/+linux/background.png

因为额外的选择器会在平台之前被检查，当管理员选择器被设置时，+admin/background.png会在Windows上被选择，而当管理员选择器未被设置时，+windows/background.png会在Windows上被选择。在Linux上，如果设置了管理员，将选择+admin/+linux/background.png，如果没有设置，则选择+linux/background.png。

Member Function Documentation#

QFileSelector::QFileSelector( *parent = nullptr)#

创建一个QFileSelector实例。这个实例将拥有与其他QFileSelector实例相同的静态选择器，但有自己的一组额外的选择器。

如果提供的话，它将拥有给定的QObject parent。

[virtual]QFileSelector::~QFileSelector()#

销毁这个选择器实例。

QFileSelector::allSelectors() const#

返回这个实例所使用的完整的、有序的选择器列表。

QFileSelector::extraSelectors() const#

返回以编程方式添加到该实例的额外选择器的列表。

See also setExtraSelectors().

QFileSelector::select(const &filePath) const#

这个函数根据运行时的条件，返回路径的选择版本。如果没有可选择的文件，返回原始的filePath。

如果原始文件不存在，则返回原始filePath。这意味着你必须有一个基础文件作为后盾，你不能只有可选择的子目录中的文件。

关于选择算法，见类概述。

QFileSelector::select(const &filePath) const#

这是在QUrl对象上操作select的一个方便版本。如果方案不是file或qrc，filePath将立即返回。否则，选择被应用于filePath的路径，并返回一个QUrl，其选择的路径和其他QUrl部分与filePath相同。

关于选择算法，见类概述。

void QFileSelector::setExtraSelectors(const &list)#

设置以编程方式添加到该实例的额外选择器的list。

这些选择器的优先级高于任何已被自动拾取的选择器。

See also extraSelectors().
