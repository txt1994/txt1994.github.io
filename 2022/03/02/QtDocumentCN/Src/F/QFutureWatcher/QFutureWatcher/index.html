详细描述#

QFutureWatcher提供关于一个QFuture的信息和通知。使用setFuture()函数来开始观察一个特定的QFuture。future()函数用setFuture()返回所设定的未来。

为了方便起见，QFuture的几个函数在QFutureWatcher中也可用。progressValue(), progressMinimum(), progressMaximum(), progressText(), isStarted(), isFinished(), [isRunning](https://doc. qt.io/qt-6/qfuturewatcher.html#isRunning)(), isCanceled(), isSuspending(), isSuspended(), waitForFinished(), result (), 以及 resultAt () 。cancel(), setSuspended(), suspend(), resume(), 和 toggleSuspended() 函数是QFutureWatcher的槽。

状态变化通过started()、finished()、canceled()、suspending()、suspended()、resumed()、resultReadyAt ()和resultsReadyAt() 信号报告。进度信息由progressRangeChanged()、void progressValueChanged()和progressTextChanged() 信号提供。

节流控制是由setPendingResultsLimit()函数提供的。当等待的resultReadyAt()或resultsReadyAt()信号的数量超过限制，未来所代表的计算将被自动节流。一旦待定信号的数量降到极限以下，计算就会恢复。

例子。启动一个计算，完成后得到一个槽的回调:

// Instantiate the objects and connect to the finished signal.
MyClass myObject;
QFutureWatcher<int> watcher;
connect(&watcher, &QFutureWatcher<int>::finished, &myObject, &MyClass::handleFinished);

// Start the computation.
QFuture<int> future = QtConcurrent::run(...);
watcher.setFuture(future);

请注意，并非所有正在运行的异步计算都可以被取消或暂停。例如，由QtConcurrent::run()返回的fulture不能被取消；但由QtConcurrent::mappedReduced()返回的fulture可以。

QFutureWatcher被专门化，不包含任何获取结果的函数。任何QFuture也可以被一个QFutureWatcher所监视。如果只需要状态或进度信息，而不需要实际的结果数据，这就很有用。

See also QFuture and Qt Concurrent.

成员函数文档#

QFutureWatcher::QFutureWatcher( *parent = nullptr)#

构建一个新的QFutureWatcher，并给定parent。在用setFuture()设置fulture之前，函数isStarted()、isCanceled()和isFinished()返回true。

[slot]void QFutureWatcher::cancel()#

取消由future()代表的异步计算。注意，该取消是异步的。当你需要同步取消时，在调用cancel()后使用waitForFinished()。

目前可用的结果仍然可以在被取消的QFuture上被访问，但是新的结果在调用这个函数后将不会*变得可用。此外，这个QFutureWatcher一旦被取消，就不会传递进度和结果的准备信号。这包括progressValueChanged()、progressRangeChanged()、progressTextChanged()、resultReadyAt()和resultsReadyAt ()信号。

请注意，并非所有正在运行的异步计算都可以被取消。例如，由QtConcurrent::run()返回的QFuture不能被取消；但由QtConcurrent::mappedReduced()返回的QFuture可以。

[signal]void QFutureWatcher::canceled()#

如果被监视的fulture被取消，就会发出这个信号。

[signal]void QFutureWatcher::finished()#

这个信号在被监视的fulture完成时发出。

[signal]void QFutureWatcher::progressRangeChanged(int minimum, int maximum)#

被观察的fulture的进度范围 已改为minimum和maximum。

[signal]void QFutureWatcher::progressTextChanged(const &progressText)#

当被观察的未来报告文本进度信息progressText时，该信号被发射出来。

[signal]void QFutureWatcher::progressValueChanged(int progressValue)#

当被监视的未来报告进展时，这个信号被发射出来，progressValue给出了当前的进展。为了避免GUI事件循环的重载，QFutureWatcher限制了进度信号的发射率。这意味着连接到这个槽的听众可能不会得到未来的所有进度报告。最后的进度更新（其中progressValue等于最大值）将始终被传递。

[signal]void QFutureWatcher::resultReadyAt(int index)#

当被监视的future在index处报告一个准备好的结果时，这个信号会被发出。如果future报告了多个结果，索引将表明是哪一个。结果可以不按顺序报告。要获得结果，请调用 resultAt(index)。

[signal]void QFutureWatcher::resultsReadyAt(int beginIndex, int endIndex)#

当被观察的fulture报告准备好的结果时，这个信号被发射出来。这些结果的索引是从beginIndex到endIndex。

[slot]void QFutureWatcher::resume()#

恢复由future()代表的异步计算。这是一个方便的方法，简单地调用setSuspended(false)。

See also suspend().

[signal]void QFutureWatcher::resumed()#

当被监视的fulture被恢复时，这个信号被发射出来。

[slot, since 6.0]void QFutureWatcher::setSuspended(bool suspend)#

如果suspend为真，这个函数将暂停由future()代表的异步计算。如果计算已经暂停，这个函数不做任何事情。QFutureWatcher在暂停未来时不会立即停止传递进度和结果就绪信号。在暂停的那一刻，可能仍然有正在进行的计算，并且不能被停止。这些计算的信号仍将被传递。

如果suspend为false，这个函数将恢复异步计算。如果计算之前没有被暂停，这个函数什么也不做。

请注意，不是所有的计算都可以被暂停。例如，由QtConcurrent::run()返回的QFuture不能被暂停；但由QtConcurrent::mappedReduced()返回的QFuture可以。

这个函数是在Qt 6.0中引入的。

See also suspended(), suspend(), resume(), and toggleSuspended().

[signal]void QFutureWatcher::started()#

当这个QFutureWatcher开始观察用setFuture()设置的fulture时，这个信号就会发出来。

[slot, since 6.0]void QFutureWatcher::suspend()#

暂停这个future所代表的异步计算。这是一个方便的方法，简单地调用setSuspended(true)。

这个函数是在Qt 6.0中引入的。

See also resume().

[signal, since 6.0]void QFutureWatcher::suspended()#

这个信号是在suspend()生效时发出的，意味着不再有运行中的计算。在收到这个信号后，就不会再有结果准备或进度报告信号了。

这个函数是在Qt 6.0中引入的。

See also setSuspended(), suspend(), and suspended().

[signal, since 6.0]void QFutureWatcher::suspending()#

当被监视的fulture的状态被设置为暂停时，这个信号就会被发射出来。

注意：该信号仅通知已请求暂停。它并不表示所有的后台操作都停止了。在暂停时正在进行的计算的信号仍将被传递。要想获知暂停实际生效的时间，请使用susabled()信号。

这个函数是在Qt 6.0中引入的。

See also setSuspended(), suspend(), and suspended().

[slot, since 6.0]void QFutureWatcher::toggleSuspended()#

切换异步计算的暂停状态。换句话说，如果计算当前正在暂停或中止，调用这个函数就可以恢复计算；如果计算正在运行，则是暂停。这是调用setSuspended(!(isSuspending() || isSuspended())的一个方便方法。）

这个函数是在Qt 6.0中引入的。

See also setSuspended(), suspend(), and resume().

[virtual]QFutureWatcher::~QFutureWatcher()#

销毁这个 QFutureWatcher.

QFutureWatcher::future() const#

返回被监视的fulture。

See also setFuture().

bool QFutureWatcher::isCanceled() const#

如果异步计算已经被cancel()函数取消，或者没有设置fulture，则返回true；否则返回false。

请注意，即使这个函数返回 “true”，计算可能仍然在运行。更多细节见cancel()。

bool QFutureWatcher::isFinished() const#

如果future()所代表的异步计算已经完成，或者没有设置future，则返回true；否则返回false。

bool QFutureWatcher::isRunning() const#

如果future()所代表的异步计算目前正在运行，则返回 “true”；否则返回 “false”。

bool QFutureWatcher::isStarted() const#

如果future()所代表的异步计算已经开始，或者没有设置future，则返回true；否则返回false。

[since 6.0]bool QFutureWatcher::isSuspended() const#

如果请求暂停异步计算，则返回 “true”，并且它是有效的，这意味着没有更多的结果或进度变化被期待。

这个函数是在Qt 6.0中引入的。

See also suspended(), setSuspended(), and isSuspending().

[since 6.0]bool QFutureWatcher::isSuspending() const#

如果异步计算已经用suspend()函数中止，但工作尚未中止，计算仍在运行，则返回true。否则返回 “false”。

要检查暂停是否真的生效，请使用isSuspended()代替。

这个函数是在Qt 6.0中引入的。

See also setSuspended(), toggleSuspended(), and isSuspended().

int QFutureWatcher::progressMaximum() const#

返回最大progressValue()。

See also progressValue() and progressMinimum().

int QFutureWatcher::progressMinimum() const#

返回最大 progressValue().

See also progressValue() and progressMaximum().

QFutureWatcher::progressText() const#

返回由异步计算报告的进度的（可选）文本表示。

请注意，并不是所有的计算都提供进度的文本表示，因此，这个函数可能返回一个空字符串。

int QFutureWatcher::progressValue() const#

返回当前的进度值，它介于progressMinimum()和progressMaximum()之间。

See also progressMinimum() and progressMaximum().

template <typename U, typename> T QFutureWatcher::result() const#

返回future()中的第一个结果。如果结果不是立即可用的，这个函数将阻塞并等待结果变得可用。这是一个调用resultAt(0)的方便方法。

See also resultAt().

template <typename U, typename> T QFutureWatcher::resultAt(int index) const#

返回future()中index处的结果。如果结果不是立即可用的，这个函数将阻塞并等待结果变得可用。

See also result().

void QFutureWatcher::setFuture(const &future)#

开始监视给定的future。

其中一个信号可能会针对future的当前状态而被发射出来。例如，如果fulture已经停止，将发出完成信号。

为了避免竞赛条件，重要的是在*做完连接后调用这个函数。

See also future().

void QFutureWatcher::setPendingResultsLimit(int limit)#

setPendingResultsLimit()提供节流控制。当等待的resultReadyAt()或resultsReadyAt()信号的数量超过limit时，未来所代表的计算将被自动节流。一旦等待信号的数量下降到限制以下，计算将恢复。

void QFutureWatcher::waitForFinished()#

等待异步计算完成（包括cancel()ed计算），即直到isFinished()返回true。
