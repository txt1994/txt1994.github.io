详细描述#

这个类不应该被直接使用。请使用QSqlDatabase代替。

如果你想创建你自己的SQL驱动，你可以子类化这个类并重新实现它的纯虚拟函数和那些你需要的虚拟函数。参见如何编写你自己的数据库驱动程序以了解更多信息。

See also QSqlDatabase and QSqlResult.

成员类型文档#

enum QSqlDriver::DriverFeature#

此枚举包含驱动程序可能支持的功能列表。 使用 hasFeature() 查询是否支持某个功能。

Constant Value Description
QSqlDriver::Transactions 0 驱动程序是否支持 SQL 事务。
QSqlDriver::QuerySize 1 数据库是否能够报告一个查询的大小。注意，有些数据库不支持返回查询的大小（即返回的行数），在这种情况下，QSqlQuery::size()将返回-1。
QSqlDriver::BLOB 2 驱动程序是否支持二进制大对象字段。
QSqlDriver::Unicode 3 如果数据库服务器支持Unicode字符串，驱动程序是否支持Unicode字符串。
QSqlDriver::PreparedQueries 4 驱动程序是否支持预准备查询执行。
QSqlDriver::NamedPlaceholders 5 驱动程序是否支持使用命名占位符。
QSqlDriver::PositionalPlaceholders 6 驱动程序是否支持使用位置占位符。
QSqlDriver::LastInsertId 7 驱动程序是否支持返回最后触摸行的 Id。
QSqlDriver::BatchOperations 8 驱动是否支持批量操作，见QSqlQuery::execBatch()
QSqlDriver::SimpleLocking 9 驱动程序是否不允许对表进行写锁定，而其他查询对其具有读锁定。
QSqlDriver::LowPrecisionNumbers 10 驱动程序是否允许以低精度获取数值。
QSqlDriver::EventNotifications 11 驱动程序是否允许以低精度获取数值。
QSqlDriver::FinishQuery 12 调用 QSqlQuery::finish() 时，驱动程序是否可以进行任何低级资源清理。
QSqlDriver::MultipleResultSets 13 驱动程序是否可以访问从批处理语句或存储过程返回的多个结果集。
QSqlDriver::CancelQuery 14 驱动程序是否允许取消正在运行的查询。

有关支持的功能的更多信息，请参阅 Qt SQL 驱动程序 文档。

See also hasFeature().

Constant Value Description

QSqlDriver::SelfSource 1 通知源是这个连接
QSqlDriver::OtherSource 2 通知源是另一个连接

enum QSqlDriver::StatementType#

此枚举包含驱动程序可以创建的 SQL 语句（或子句）类型列表。

Constant Value Description
QSqlDriver::WhereStatement 0 一个SQL WHERE 语句（例如，WHERE f = 5）。
QSqlDriver::SelectStatement 1 SQL SELECT 语句（例如，SELECT f FROM t）。
QSqlDriver::UpdateStatement 2 SQL UPDATE 语句（例如，UPDATE TABLE t set f = 1）。
QSqlDriver::InsertStatement 3 SQL INSERT 语句（例如，INSERT INTO t (f) values (1)）。
QSqlDriver::DeleteStatement 4 SQL DELETE 语句（例如，DELETE FROM t）。

See also sqlStatement().

[signal, since 5.0]void QSqlDriver::notification(const &name, source, const &payload)#

当数据库发布驱动程序订阅的事件通知时，会发出此信号。 name 标识事件通知，source 表示信号源，payload 保存可选随通知传递的额外数据。

该函数在 Qt 5.0 引入。

See also subscribeToNotification().

[virtual]bool QSqlDriver::beginTransaction()#

调用此函数以开始事务。 如果成功，则返回 true，否则返回 false。 默认实现什么都不做并返回false。

See also commitTransaction() and rollbackTransaction().

[pure virtual]void QSqlDriver::close()#

派生类必须重新实现这个纯虚函数才能关闭数据库连接。 成功返回真，失败返回假。

See also open() and setOpen().

[virtual]bool QSqlDriver::commitTransaction()#

调用此函数来提交事务。 如果成功，则返回 true，否则返回 false。 默认实现什么都不做并返回false。

See also beginTransaction() and rollbackTransaction().

[pure virtual] *QSqlDriver::createResult() const#

在数据库上创建一个空的 SQL 结果。 派生类必须重新实现此函数并将适合其数据库的 QSqlResult 对象返回给调用者。

[virtual] QSqlDriver::escapeIdentifier(const &identifier, type) const#

返回根据数据库规则转义的 identifier。 identifier 可以是表名或字段名，取决于 type。

默认实现什么也不做。

See also isIdentifierEscaped().

[virtual] QSqlDriver::formatValue(const &field, bool trimStrings = false) const#

返回数据库的 field 值的字符串表示形式。 例如，在构造 INSERT 和 UPDATE 语句时使用它。

默认实现根据以下规则返回格式化为字符串的值：

  • 如果 field 是字符数据，则返回值用单引号括起来，这适用于许多 SQL 数据库。 任何嵌入的单引号字符都会被转义（替换为两个单引号字符）。 如果 trimStrings 为真（默认为假），则从字段中删除所有尾随空格。
  • 如果 field 是日期/时间数据，则该值采用 ISO 格式格式化并用单引号括起来。 如果日期/时间数据无效，则返回“NULL”。
  • 如果 field 是 bytearray 数据，并且驱动程序可以编辑二进制字段，则该值被格式化为十六进制字符串。
  • 对于任何其他字段类型，对其值调用 toString() 并返回其结果。

See also QVariant::toString().

[virtual] QSqlDriver::handle() const#

返回包装在 QVariant 中的低级数据库句柄，如果没有句柄，则返回无效变体。

警告： 请谨慎使用此功能，并且仅在您知道自己在做什么的情况下使用。

警告： 如果连接被修改（例如，如果您关闭连接），此处返回的句柄可能会变成一个过时的指针。

警告： 如果连接尚未打开，则句柄可以为 NULL。

这里返回的句柄是依赖于数据库的，你应该在访问它之前查询变量的类型名称。

此示例检索与 sqlite 的连接的句柄：

QSqlDatabase db = QSqlDatabase::database();
QVariant v = db.driver()->handle();
if (v.isValid() && (qstrcmp(v.typeName(), "sqlite3*") == 0)) {
    // v.data() returns a pointer to the handle
    sqlite3 *handle = *static_cast<sqlite3 **>(v.data());
    if (handle) {
        // ...
    }
}

此代码段返回 PostgreSQL 或 MySQL 的句柄:

if (qstrcmp(v.typeName(), "PGconn*") == 0) {
    PGconn *handle = *static_cast<PGconn **>(v.data());
    if (handle) {
        // ...
    }
}

if (qstrcmp(v.typeName(), "MYSQL*") == 0) {
    MYSQL *handle = *static_cast<MYSQL **>(v.data());
    if (handle) {
        // ...
    }
}

See also QSqlResult::handle().

[pure virtual]bool QSqlDriver::hasFeature( feature) const#

如果驱动程序支持功能 feature，则返回 true； 否则返回false。

请注意，某些数据库需要 open() 才能确定。

See also DriverFeature.

[virtual] QSqlDriver::stripDelimiters(const &identifier, type) const#

返回删除前导和尾随分隔符的 identifier，identifier 可以是表名或字段名，取决于 type。如果 identifier 没有前导和尾随分隔符，则返回 identifier 而不进行修改。

如果您想在 QSqlDriver 子类中提供自己的实现，请重新实现此函数，

另请参阅 isIdentifierEscaped()。

[virtual]bool QSqlDriver::subscribeToNotification(const &name)#

调用此函数以订阅来自数据库的事件通知。 name 标识事件通知。

如果成功，则返回 true，否则返回 false。

调用此函数时必须打开数据库。当通过调用 close() 关闭数据库时，所有订阅的事件通知都会自动取消订阅。请注意，在已打开的数据库上调用 open() 可能会隐式导致调用 close()，这将导致驱动程序取消订阅所有事件通知。

当数据库发布由 name 标识的事件通知时，会发出 notification() 信号。

如果您想在自己的 QSqlDriver 子类中提供事件通知支持，请重新实现此函数，

另请参阅 unsubscribeFromNotification()、subscribedToNotifications() 和 QSqlDriver::hasFeature()。

[virtual] QSqlDriver::subscribedToNotifications() const#

返回当前订阅的事件通知的名称列表。

如果您想在自己的 QSqlDriver 子类中提供事件通知支持，请重新实现此函数，

另请参阅 subscribeToNotification() 和 unsubscribeFromNotification()。

[virtual] QSqlDriver::tables( tableType) const#

返回数据库中表的名称列表。默认实现返回一个空列表。

tableType 参数描述了应该返回什么类型的表。由于二进制兼容性，字符串包含枚举 QSql::TableTypes 的值作为文本。为了向后兼容，应将空字符串视为 QSql::Tables。

[virtual]bool QSqlDriver::unsubscribeFromNotification(const &name)#

调用此函数以取消订阅来自数据库的事件通知。 name 标识事件通知。

如果成功，则返回 true，否则返回 false。

调用此函数时必须打开数据库。当调用 close() 函数时，所有订阅的事件通知都会自动取消订阅。

调用 this 函数后，当数据库发布由 name 标识的事件通知时，将不再发出 notification() 信号。

如果您想在自己的 QSqlDriver 子类中提供事件通知支持，请重新实现此函数，

See also subscribeToNotification() and subscribedToNotifications().
