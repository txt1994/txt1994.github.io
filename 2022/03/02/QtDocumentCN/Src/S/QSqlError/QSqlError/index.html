详细描述#

QSqlError 对象可以提供特定于数据库的错误数据，包括 driverText() 和 databaseText() 消息（或两者连接在一起作为 text()）， 和 nativeErrorCode() 和 type()。

See also QSqlDatabase::lastError() and QSqlQuery::lastError().

成员类型文档#

enum QSqlError::ErrorType#

此枚举类型描述了发生错误的上下文，例如，连接错误、语句错误等。

Constant Value Description
QSqlError::NoError 0 没有发生错误。
QSqlError::ConnectionError 1 连接错误
QSqlError::StatementError 2 SQL 语句语法错误。
QSqlError::TransactionError 3 事务失败的错误。
QSqlError::UnknownError 4 未知错误。

成员函数文档#

[since 5.10]QSqlError::QSqlError( &&other)#

移动构造一个 QSqlError 实例，使其指向 other 指向的同一个对象。

注意： 移出的对象 other 处于部分形成状态，其中唯一有效的操作是销毁和分配新值。

这个函数是在 Qt 5.10 中引入的。

QSqlError::QSqlError(const &other)#

创建 other 的副本。

QSqlError::QSqlError(const &driverText = QString(), const &databaseText = QString(), type = NoError, const &code = QString())#

构造一个包含驱动程序错误文本 driverText、数据库特定错误文本 databaseText、类型 type 和错误代码 code 的错误。

注意： DB2：DB2 可能会报告多个错误代码。 发生这种情况时，; 用作错误代码之间的分隔符。

[since 5.10] &QSqlError::operator=( &&other)#

将 other 移动分配给此 QSqlError 实例。

注意： 移出的对象 other 处于部分形成状态，其中唯一有效的操作是销毁和分配新值。

这个函数是在 Qt 5.10 中引入的。

&QSqlError::operator=(const &other)#

将 other 错误的值分配给此错误。

QSqlError::~QSqlError()#

销毁对象并释放所有分配的资源。

QSqlError::databaseText() const#

返回数据库报告的错误文本。 这可能包含特定于数据库的描述； 它可能是空的。

See also driverText() and text().

QSqlError::driverText() const#

返回驱动程序报告的错误文本。 这可能包含特定于数据库的描述。 它也可能是空的。

See also databaseText() and text().

bool QSqlError::isValid() const#

如果设置了错误，则返回 true，否则返回 false。

例子：

QSqlQueryModel model;
model.setQuery("select * from myTable");
if (model.lastError().isValid())
    qDebug() << model.lastError();

See also type().

QSqlError::nativeErrorCode() const#

返回特定于数据库的错误代码，如果无法确定，则返回空字符串。

[since 5.10]void QSqlError::swap( &other)#

用此错误交换错误 other。 此操作非常快且永不失败。

这个函数是在 Qt 5.10 中引入的。

QSqlError::text() const#

这是一个方便的函数，它返回连接成单个字符串的 databaseText() 和 driverText()。

See also driverText() and databaseText().

QSqlError::type() const#

返回错误类型，如果无法确定类型，则返回 -1。

bool QSqlError::operator!=(const &other) const#

将 other 错误的值与此错误进行比较，如果不相等则返回 true。

bool QSqlError::operator==(const &other) const#

将 other 错误的值与此错误进行比较，如果相等则返回 true。
