详细描述#

QSharedMemory提供了多个线程和进程对一个共享内存段的访问。它还提供了一种方法，使单个线程或进程能够锁定内存，以便独占访问。

在使用这个类时，要注意以下平台的差异。

  • Windows。QSharedMemory并不 “拥有 “共享内存段。当所有拥有QSharedMemory实例的线程或进程连接到一个特定的共享内存段时，要么销毁其QSharedMemory实例，要么退出，Windows内核会自动释放共享内存段。

  • Unix。QSharedMemory “拥有 “该共享内存段。当最后一个拥有QSharedMemory实例的线程或进程连接到一个特定的共享内存段时，通过销毁其QSharedMemory实例从该内存段中分离，Unix内核会释放该共享内存段。但如果最后一个线程或进程在没有运行QSharedMemory析构器的情况下崩溃了，共享内存段就会在崩溃后存活下来。

  • HP-UX：每个进程只允许附加到一个共享内存段。这意味着在HP-UX中，QSharedMemory不应该在同一进程的多个线程中使用。

  • 苹果平台。沙盒应用程序（包括通过苹果应用商店发货的应用程序）需要使用POSIX共享内存（而不是System V共享内存），这增加了一些限制，包括。

    • 密钥必须是/的形式，如这里和这里所记载的。

    • 密钥的长度被限制在30个字符。

    • 在进程退出时，命名的共享内存条目没有被清理，所以重新启动应用程序并以相同的名称重新创建共享内存将会失败。为了解决这个问题，可以退回到附加到现有的共享内存条目。

      QSharedMemory shm("DEVTEAMID.app-group/shared")。
    if (!shm.create(42) && shm.error() == QSharedMemory::AlreadyExists)
        shm.attach()。

在读取或写入共享内存之前，记得用lock()锁定共享内存，并在完成后记得用unlock()释放该锁。

当QSharedMemory的最后一个实例从共享内存段中分离出来时，QSharedMemory会自动销毁该共享内存段，并且没有对该段的引用。

警告： QSharedMemory以Qt特有的方式改变键，除非另有规定。通过首先用QSharedMemory()创建一个默认的共享内存，然后用setNativeKey()设置一个本地键来实现与非Qt应用程序的互操作。当使用本地密钥时，共享内存不会被保护，以防止对其进行多重访问（例如，无法lock()），应使用用户定义的机制来实现这种保护。

成员类型文档#

enum QSharedMemory::AccessMode#

Constant Value Description
QSharedMemory::ReadOnly 0 共享内存段是只读的。不允许对共享内存段进行写入。试图向以ReadOnly创建的共享内存段写入会导致程序中止。
QSharedMemory::ReadWrite 1 读取和写入共享内存段都是允许的。

enum QSharedMemory::SharedMemoryError#

Constant Value Description
QSharedMemory::NoError 0 没有发生错误。
QSharedMemory::PermissionDenied 1 操作失败，因为调用者没有必要的权限。
QSharedMemory::InvalidSize 2 创建操作失败，因为请求的大小无效。
QSharedMemory::KeyError 3 操作失败是因为一个无效的密钥。
QSharedMemory::AlreadyExists 4 create()操作失败，因为一个具有指定键的共享内存段已经存在。
QSharedMemory::NotFound 5 一个attach()失败了，因为找不到一个具有指定键的共享内存段。
QSharedMemory::LockError 6 由于create()或attach()失败并返回错误，或者由于QSystemSemaphore::acquisition()中发生系统错误，因此lock()共享内存段的尝试失败。
QSharedMemory::OutOfResources 7 一个create()操作失败了，因为没有足够的内存来满足请求。
QSharedMemory::UnknownError 8 还发生了一些事情，而且很糟糕。

成员函数文档#

QSharedMemory::QSharedMemory(const &key, *parent = nullptr)#

用给定的parent构建一个共享内存对象，并将其键值设置为key。因为它的键被设置了，它的create()和attach()函数可以被调用。

See also setKey(), create(), and attach().

QSharedMemory::QSharedMemory( *parent = nullptr)#

这个函数重载了QSharedMemory()。

用给定的parent构造一个共享内存对象。该共享内存对象的键没有被构造函数设置，所以该共享内存对象没有附加一个底层的共享内存段。在使用create()或attach()之前，必须用setKey()来设置密钥。

See also setKey().

[virtual]QSharedMemory::~QSharedMemory()#

解构器清除了键，这迫使共享内存对象从它的底层共享内存段detach。如果这个共享内存对象是连接到共享内存段的最后一个对象，那么detach()操作将销毁共享内存段。

See also detach() and isAttached().

bool QSharedMemory::attach( mode = ReadWrite)#

试图将进程附加到由传递给构造函数或调用setKey()或setNativeKey()的键标识的共享内存段。访问mode默认为ReadWrite。它也可以是ReadOnly。如果附加操作成功，返回 “true”。如果返回false，则调用error()来确定发生了哪种错误。附加共享内存段后，可以通过调用data()获得共享内存的指针。

参见 isAttached(), detach(), 和 create()。

const void *QSharedMemory::constData() const#

如果有共享内存段，返回一个指向共享内存段内容的常量指针。否则，它返回null。在从共享内存读取或写入共享内存之前，记得用lock()锁定共享内存，并在完成后记得用unlock()释放锁。

参见 attach() 和 create()。

bool QSharedMemory::create(qsizetype size, mode = ReadWrite)#

创建一个size字节的共享内存段，密钥传递给构造函数，用setKey()设置或用setNativeKey()设置，然后用给定的访问mode附加到新的共享内存段，并返回true。如果键识别的共享内存段已经存在，则不执行附加操作，返回false。当返回值为 “false “时，调用error()来确定发生了哪种错误。

也见 error()。

void *QSharedMemory::data()#

如果有共享内存段的话，返回一个指向共享内存段内容的指针。否则，它返回null。在从共享内存读取或写入共享内存之前，记得用lock()锁定共享内存，并在完成后记得用unlock()释放锁。

参见 attach()。

const void *QSharedMemory::data() const#

这个函数重载了data()。

bool QSharedMemory::detach()#

将进程从共享内存段中分离出来。如果这是连接到共享内存段的最后一个进程，那么共享内存段被系统释放，也就是说，内容被销毁。如果该函数分离了共享内存段，则返回 “true”。如果它返回 “false”，这通常意味着该段没有被连接，或者被其他进程锁定。

参见 attach() 和 isAttached() 。

QSharedMemory::error() const#

返回一个值，表明是否发生了错误，如果是的话，是哪个错误。

也见 errorString()。

QSharedMemory::errorString() const#

返回最后发生的错误的文本描述。如果error()返回一个error值，调用这个函数来得到一个描述错误的文本字符串。

参见 error()。

bool QSharedMemory::isAttached() const#

如果该进程连接到共享内存段，则返回 “true”。

参见 attach() 和 detach()。

QSharedMemory::key() const#

返回用setKey()分配给这个共享内存的键，如果没有分配键，或者段使用的是nativeKey()，则为空键。键是Qt应用程序用来识别共享内存段的标识符。

你可以通过调用nativeKey()找到操作系统所使用的本地的、特定平台的键。

参见 setKey() 和 setNativeKey()。

bool QSharedMemory::lock()#

这是一个信号，锁定共享内存段，供本进程访问，并返回 “true”。如果另一个进程锁定了该段，该函数就会阻塞，直到该锁被释放。然后它获得锁并返回true。如果该函数返回 “false”，意味着你忽略了create()或attach()的错误返回，你用setNativeKey()设置了密钥，或者由于未知的系统错误，QSystemSemaphore::acquire ()失败。

See also unlock(), data(), and QSystemSemaphore::acquire().

QSharedMemory::nativeKey() const#

返回这个共享内存对象的本地、平台特定的键。本机键是操作系统用来识别共享内存段的标识符。

你可以使用本地键来访问未被Qt创建的共享内存段，或授予非Qt应用程序共享内存访问权。

参见 setKey() 和 setNativeKey()。

void QSharedMemory::setKey(const &key)#

为这个共享内存对象设置独立于平台的key。如果key与当前的key相同，该函数不做任何事情就返回。

你可以调用key()来找回独立于平台的键。在内部，QSharedMemory会将这个键转换为一个平台特定的键。如果你调用nativeKey()，你将得到平台特定的、转换后的密钥。

如果共享内存对象被附加到一个底层的共享内存段，在设置新的键之前，它将detach与之分离。这个函数不做attach()。

参见 key(), nativeKey(), 以及 isAttached()。

void QSharedMemory::setNativeKey(const &key)#

为这个共享内存对象设置本地的、平台特定的key。如果key与当前的本地键相同，该函数将返回而不做任何事情。如果你只是想给一个段分配一个键，你应该调用setKey()来代替。

你可以调用nativeKey()来检索本地键。如果已经分配了一个本地密钥，调用key()将返回一个空字符串。

如果共享内存对象被附加到一个底层共享内存段，在设置新的键之前，它将detach从它那里。这个函数不做attach()。

如果你设置了一个本地密钥，应用程序将不能被移植。

参见 nativeKey(), key(), 以及 isAttached()。

qsizetype QSharedMemory::size() const#

返回附加的共享内存段的大小。如果没有附加共享内存段，则返回0。

注意：段的大小可能大于传递给create()的请求大小。

参见 create() 和 attach()。

bool QSharedMemory::unlock()#

释放共享内存段上的锁，如果该锁目前由该进程持有，则返回true。如果该段没有被锁定，或者该锁被其他进程持有，则不会发生任何事情，返回false。

See also lock().
