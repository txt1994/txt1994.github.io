详细描述#

QSqlRelationalTableModel 的行为类似于 QSqlTableModel，但允许将列设置为其他数据库表的外键。

左侧的屏幕截图显示了 [QTableView] (qtableview.html) 中的普通 QSqlTableModel。 外键（city 和 country）无法解析为人类可读的值。 右侧的屏幕截图显示了一个 QSqlRelationalTableModel，外键解析为人类可读的文本字符串。

以下代码片段显示了如何设置 QSqlRelationalTableModel:

model->setTable("employee");

model->setRelation(2, QSqlRelation("city", "id", "name"));
model->setRelation(3, QSqlRelation("country", "id", "name"));

setRelation() 函数调用在两个表之间建立关系。 第一次调用指定表employee中的第2列是与表city的字段id映射的外键，并且视图应该向用户显示city的name字段。 第二个调用与第 3 列类似。

如果您使用读写 QSqlRelationalTableModel，您可能希望在视图上使用 QSqlRelationalDelegate。 与默认委托不同，QSqlRelationalDelegate 为其他表的外键字段提供了一个组合框。 要使用该类，只需在具有 [QSqlRelationalDelegate](https:// /doc.qt.io/qt-6/qsqlrelationaldelegate.html)：

std::unique_ptr<QTableView> view{new QTableView};
view->setModel(model);
view->setItemDelegate(new QSqlRelationalDelegate(view.get()));

relationaltablemodel 示例说明了如何将 QSqlRelationalTableModel 与 QSqlRelationalDelegate 为表提供外键支持。

笔记：

  • 表必须声明一个主键。
  • 表的主键可能不包含与另一个表的关系。
  • 如果关系表包含引用引用表中不存在的行的键，则包含无效键的行将不会通过模型公开。用户或数据库负责保持参照完整性。
  • 如果一个关系的显示列名也用作关系表中的列名，或者如果它在多个关系中用作显示列名，它将被别名。别名是关系的表名、显示列名和由下划线连接的唯一 id（例如 tablename_columnname_id）。 QSqlRecord::fieldName() 将返回别名列名。检测到重复时，所有出现的重复显示列名都会被别名，但不会对主表中的列名进行别名。别名不会影响 QSqlRelation，因此 QSqlRelation::displayColumn() 将返回原始显示列名称。
  • 引用表名称是别名。别名是单词“relTblAl”和由下划线连接的相关列索引（例如 relTblAl_2）。别名可用于过滤表（例如，setFilter(“relTblAl_2=’Oslo’ OR relTblAl_3=’USA’”)）。
  • 使用 setData() 时，角色应始终为 Qt::EditRole，使用 data() 时，角色应始终为Qt::DisplayRole。

See also QSqlRelation, QSqlRelationalDelegate, and Relational Table Model Example.

enum QSqlRelationalTableModel::JoinMode#

Constant Value Description
QSqlRelationalTableModel::InnerJoin 0 - 内连接模式，当两个表中至少有一个匹配时返回行。
QSqlRelationalTableModel::LeftJoin 1 `- 左连接模式，返回左表 (table_name1) 中的所有行，即使右表 (table_name2) 中没有匹配项。

See also QSqlRelationalTableModel::setJoinMode().

[override virtual]bool QSqlRelationalTableModel::setData(const &index, const &value, int role = Qt::EditRole)#

重新实现: QSqlTableModel::setData(const QModelIndex &index, const QVariant &value, int role).

将具有指定 index 的项目中 role 的数据设置为给定的 value。 根据编辑策略，该值可能会立即应用于数据库，也可能会缓存在模型中。

如果可以设置值，则返回 true，否则返回 false（例如，如果 index 超出范围）。

对于关系列，value 必须是索引，而不是显示值。 索引也必须存在于引用的表中，否则函数返回“false”。

See also editStrategy(), data(), submit(), and revertRow().

void QSqlRelationalTableModel::setJoinMode( joinMode)#

设置 SQL joinMode 以显示或隐藏具有 NULL 外键的行。 在 InnerJoin 模式（默认）下，这些行将不会显示：如果要显示它们，请使用 LeftJoin 模式。

See also QSqlRelationalTableModel::JoinMode.

[virtual]void QSqlRelationalTableModel::setRelation(int column, const &relation)#

让指定的 column 成为 relation 指定的外部索引。

例子：

model->setTable("employee");

model->setRelation(2, QSqlRelation("city", "id", "name"));

setRelation() 调用指定表 employee 中的第 2 列是与表 city 的字段 id 映射的外键，并且视图应该向用户显示 city 的 name 字段 .

注意：表的主键可能不包含与另一个表的关系。

See also relation().
