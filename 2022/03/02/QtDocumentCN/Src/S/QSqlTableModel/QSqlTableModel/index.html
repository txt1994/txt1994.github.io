详细描述#

QSqlTableModel 是用于从单个表中读取和写入数据库记录的高级接口。 它建立在较低级别的 QSqlQuery 之上，可用于为视图类提供数据，例如 QTableView。 例如：

QSqlTableModel *model = new QSqlTableModel;
model->setTable("employee");
model->setEditStrategy(QSqlTableModel::OnManualSubmit);
model->select();
model->setHeaderData(0, Qt::Horizontal, tr("Name"));
model->setHeaderData(1, Qt::Horizontal, tr("Salary"));

QTableView *view = new QTableView;
view->setModel(model);
view->hideColumn(0); // don't show the ID
view->show();

我们设置了 SQL 表的名称和编辑策略，然后我们设置了在视图标题中显示的标签。 编辑策略指示用户在视图中所做的更改何时实际应用于数据库。 可能的值为 OnFieldChange、OnRowChange 和 OnManualSubmit。

QSqlTableModel 也可用于以编程方式访问数据库，而无需将其绑定到视图：

QSqlTableModel model;
model.setTable("employee");
model.select();
int salary = model.record(4).value("salary").toInt();

上面的代码片段从查询SELECT * from employee的结果集中的记录 4 中提取salary字段。

可以使用 setFilter() 设置过滤器，或使用 setSort() 修改排序顺序。 最后，您必须调用 select() 以使用数据填充模型。

tablemodel 示例说明了如何使用 QSqlTableModel 作为 QTableView 的数据源。

QSqlTableModel 不直接支持外键。 如果要解析外键，请使用 QSqlRelationalTableModel 和 QSqlRelationalDelegate。

See also QSqlRelationalTableModel, QSqlQuery, Model/View Programming, Table Model Example, and Cached Table Example.

成员类型文档#

enum QSqlTableModel::EditStrategy#

此枚举类型描述了在编辑数据库中的值时选择哪种策略。

Constant Value Description
QSqlTableModel::OnFieldChange 0 对模型的所有更改都将立即应用于数据库。
QSqlTableModel::OnRowChange 1 当用户选择不同的行时，将应用对行的更改。
QSqlTableModel::OnManualSubmit 2 在调用 submitAll() 或 revertAll() 之前，所有更改都将缓存在模型中。

注意：为了防止仅将部分初始化的行插入到数据库中，OnFieldChange 对于新插入的行的行为类似于OnRowChange。

See also setEditStrategy().

成员函数文档#

QSqlTableModel::QSqlTableModel( *parent = nullptr, const &db = QSqlDatabase())#

创建一个空的 QSqlTableModel 并将父级设置为 parent 并将数据库连接设置为 db。 如果 db 无效，将使用默认数据库连接。

默认编辑策略是 OnRowChange。

[signal]void QSqlTableModel::beforeDelete(int row)#

该信号由 deleteRowFromTable() 在从当前活动的数据库表中删除 row 之前发出。

[signal]void QSqlTableModel::beforeInsert( &record)#

此信号由 insertRowIntoTable() 在将新行插入当前活动的数据库表之前发出。 即将插入的值存储在 record 中，可以在插入之前进行修改。

[signal]void QSqlTableModel::beforeUpdate(int row, &record)#

该信号由 updateRowInTable() 在当前活动数据库表中的 row 使用 record 中的值更新之前发出。

请注意，只有标记为已生成的值才会被更新。 生成的标志可以使用 QSqlRecord::setGenerated() 设置并使用 QSqlRecord::isGenerated() 检查。

See also QSqlRecord::isGenerated().

[signal]void QSqlTableModel::primeInsert(int row, &record)#

此信号由 insertRows() 发出，当在当前活动数据库表的给定 row 中启动插入时。 record 参数可以被写入（因为它是一个引用），例如用默认值填充一些字段并设置字段的生成标志。 处理此信号时，请勿尝试通过 setData() 或 setRecord() 等其他方式编辑记录。

[override virtual slot]void QSqlTableModel::revert()#

重新实现: QAbstractItemModel::revert().

当用户取消编辑当前行时，项目委托将调用此重新实现的插槽。

如果模型的策略设置为 OnRowChange 或 OnFieldChange，则还原更改。 对 OnManualSubmit 策略没有任何作用。

使用 revertAll() 还原 OnManualSubmit 策略的所有待处理更改，或使用 revertRow() 还原特定行。

See also submit(), submitAll(), revertRow(), and revertAll().

[slot]void QSqlTableModel::revertAll()#

还原所有挂起的更改。

See also revert(), revertRow(), and submitAll().

[virtual slot]bool QSqlTableModel::select()#

使用指定的过滤器和排序条件，使用通过 setTable() 设置的表中的数据填充模型，如果成功则返回 true； 否则返回“假”。

注意： 调用 select() 将还原所有未提交的更改并删除所有插入的列。

See also setTable(), setFilter(), and selectStatement().

[virtual slot, since 5.0]bool QSqlTableModel::selectRow(int row)#

使用与主键值匹配的数据库表行中的值刷新模型中的 row。 如果没有主键，所有列值都必须匹配。 如果没有找到匹配的行，模型将显示一个空行。

如果成功，则返回 true； 否则返回false。

此功能是在 Qt 5.0 中引入的。

See also select().

[override virtual slot]bool QSqlTableModel::submit()#

重新实现: QAbstractItemModel::submit().

当用户停止编辑当前行时，项目委托会调用此重新实现的插槽。

如果模型的策略设置为 OnRowChange 或 OnFieldChange，则提交当前编辑的行。 对 OnManualSubmit 策略没有任何作用。

使用 submitAll() 提交 OnManualSubmit 策略的所有未决更改。

成功时返回 true； 否则返回false。 使用 lastError() 查询详细的错误信息。

不会自动重新填充模型。 提交的行会在成功时从数据库中刷新。

See also revert(), revertRow(), submitAll(), revertAll(), and lastError().

[slot]bool QSqlTableModel::submitAll()#

提交所有挂起的更改并在成功时返回 true。 错误返回false，详细的错误信息可以通过lastError()获取。

在 OnManualSubmit 中，成功时将重新填充模型。 任何呈现它的视图都将失去其选择。

注意：在 OnManualSubmit 模式下，submitAll() 失败时不会从缓存中清除已提交的更改。 这允许在不丢失数据的情况下回滚和重新提交事务。

See also revertAll() and lastError().

bool QSqlTableModel::insertRecord(int row, const &record)#

在 row 位置插入 record。 如果 row 为负数，则记录将附加到末尾。 在内部调用 insertRows() 和 setRecord()。

如果可以插入记录，则返回 true，否则返回 false。

立即提交 OnFieldChange 和 OnRowChange 的更改。 失败不会在模型中留下新行。

See also insertRows(), removeRows(), and setRecord().

[virtual protected]bool QSqlTableModel::insertRowIntoTable(const &values)#

将值 values 插入到当前活动的数据库表中。

这是一种直接对数据库进行操作的低级方法，不应直接调用。 使用 insertRow() 和 setData() 插入值。 模型将根据其编辑策略决定何时修改数据库。

如果可以插入值，则返回 true，否则返回 false。 可以使用 lastError() 检索错误信息。

See also lastError(), insertRow(), and insertRows().

[override virtual]bool QSqlTableModel::insertRows(int row, int count, const &parent = QModelIndex())#

重新实现: QAbstractItemModel::insertRows(int row, int count, const QModelIndex &parent).

在 row 位置插入 count 个空行。 注意 parent 必须是无效的，因为这个模型不支持父子关系。

对于编辑策略 OnFieldChange 和 OnRowChange，一次只能插入一行，并且模型可能不包含其他缓存的更改。

将为每个新行发出 primeInsert() 信号。 如果要使用默认值初始化新行，请连接到它。

无论编辑策略如何，都不提交行。

如果参数超出范围或无法插入行，则返回 false； 否则返回 true。

See also primeInsert() and insertRecord().

bool QSqlTableModel::isDirty(const &index) const#

如果索引 index 处的值是脏的，则返回 true，否则返回 false。 脏值是在模型中修改但尚未写入数据库的值。

如果 index 无效或指向不存在的行，则返回 false。

[override virtual]bool QSqlTableModel::removeRows(int row, int count, const &parent = QModelIndex())#

重新实现: QAbstractItemModel::removeRows(int row, int count, const QModelIndex &parent).

从 row 开始删除 count 行。由于此模型不支持分层结构，parent 必须是无效的模型索引。

当编辑策略为 OnManualSubmit 时，从数据库中删除行会延迟，直到调用 submitAll()。

对于 OnFieldChange 和 OnRowChange，一次只能删除一行，并且只有在其他行没有缓存更改的情况下才能删除。删除将立即提交到数据库。模型为成功删除的行保留一个空白行，直到使用 select() 刷新。

删除失败后，模型中的操作不会恢复。申请可以重新提交或恢复。

在要删除的范围内插入但尚未成功提交的行会立即从模型中删除。

在从数据库中删除一行之前，会发出 beforeDelete() 信号。

如果 row < 0 或 row + count > rowCount()，则不执行任何操作并返回 false。如果可以删除所有行，则返回 true；否则返回false。可以使用 lastError() 检索详细的数据库错误信息。

See also removeColumns() and insertRows().

[override virtual]bool QSqlTableModel::setData(const &index, const &value, int role = Qt::EditRole)#

重新实现: QAbstractItemModel::setData(const QModelIndex &index, const QVariant &value, int role).

将角色 role 的项目 index 的数据设置为 value。

对于编辑策略 OnFieldChange，只有在没有其他索引缓存更改的情况下，索引才可能收到更改。 更改会立即提交。 但是，尚未插入数据库的行可以随意更改，不会自动提交。 提交的更改不会在失败时恢复。

对于 OnRowChange，仅当没有其他行具有缓存更改时，索引才可能接收更改。 更改不会自动提交。

如果 value 等于当前值，则返回 true。 但是，该值不会提交到数据库。

如果可以设置该值，则返回 true 或错误时返回 false，例如如果 index 超出范围。

如果角色不是 Qt::EditRole，则返回 false。 要为 EditRole 以外的角色设置数据，请使用自定义代理模型或子类 QSqlTableModel。

See also editStrategy(), data(), submit(), submitAll(), and revertRow().

[virtual]void QSqlTableModel::setEditStrategy( strategy)#

[virtual]void QSqlTableModel::setFilter(const &filter)#

将当前过滤器设置为 filter。

过滤器是一个没有关键字 WHERE 的 SQL WHERE 子句（例如，name='Josephine'）。

如果模型已经填充了数据库中的数据，则模型会使用新过滤器重新选择它。 否则，过滤器将在下次调用 select() 时应用。

See also filter(), select(), selectStatement(), and orderByClause().

See also primaryKey() and QSqlDatabase::primaryIndex().

[protected]void QSqlTableModel::setQuery(const &query)#

这个函数简单地调用 QSqlQueryModel::setQuery(query)。 通常不应在 QSqlTableModel 上调用它。 相反，使用 setTable()、setSort()、setFilter() 等来设置查询。

See also selectStatement().

bool QSqlTableModel::setRecord(int row, const &values)#

将 values 应用于模型中的 row。 源字段和目标字段按字段名称映射，而不是按记录中的位置映射。

请注意，values 中生成的标志被保留，以确定在将更改提交到数据库时是否使用相应的字段。 默认情况下，QSqlRecord 中的所有字段都设置为 true。 对于 values 中的任何值，您必须使用 setGenerated(false) 将标志设置为 false，以将更改保存回数据库。

对于编辑策略 OnFieldChange 和 OnRowChange，只有在没有其他行有缓存更改时，一行才可能收到更改。 更改会立即提交。 提交的更改不会在失败时恢复。

如果可以设置所有值，则返回 true； 否则返回假。

See also record() and editStrategy().

[virtual]void QSqlTableModel::setSort(int column, order)#

将 column 的排序顺序设置为 order。 这不会影响当前数据，要使用新的排序顺序刷新数据，请调用 select()。

See also sort(), select(), and orderByClause().

[virtual]void QSqlTableModel::setTable(const &tableName)#

将模型操作的数据库表设置为 tableName。 不从表中选择数据，但获取其字段信息。

要使用表的数据填充模型，请调用 select()。

可以使用 lastError() 检索错误信息。

See also select(), setFilter(), and lastError().

[override virtual]void QSqlTableModel::sort(int column, order)#

重新实现: QAbstractItemModel::sort(int column, Qt::SortOrder order).

使用排序顺序 order 按 column 对数据进行排序。 这将立即选择数据，使用 setSort() 设置排序顺序，而不用数据填充模型。

See also setSort(), select(), and orderByClause().

QSqlTableModel::tableName() const#

返回当前选定表的名称。

[virtual protected]bool QSqlTableModel::updateRowInTable(int row, const &values)#

用指定的 values 更新当前活动数据库表中的给定 row。 如果成功，则返回 true； 否则返回false。

这是一种直接对数据库进行操作的低级方法，不应直接调用。 使用 setData() 更新值。 模型将根据其编辑策略决定何时修改数据库。

请注意，只有设置了生成标志的值才会更新。 生成标志可以使用 QSqlRecord::setGenerated() 设置并使用 QSqlRecord::isGenerated() 进行测试。

See also QSqlRecord::isGenerated() and setData().
