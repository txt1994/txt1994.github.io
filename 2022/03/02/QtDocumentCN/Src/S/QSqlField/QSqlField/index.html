详细描述#

QSqlField 表示数据库表或视图中单个列的特征，例如数据类型和列名。字段还包含数据库列的值，可以查看或更改。

字段数据值存储为 QVariants。不允许使用不兼容的类型。例如：

QSqlField field("age", QMetaType::fromType<int>());
field.setValue(QPixmap()); // 错误的

但是，该字段将在可能的情况下尝试将某些数据类型转换为该字段数据类型：

QSqlField field("age", QMetaType::fromType<int>());
field.setValue(QString("123")); // 将 QString 转换为 int

QSqlField 对象很少在应用程序代码中显式创建。它们通常通过已经包含字段列表的 QSqlRecords 间接访问。例如：

QSqlQuery query;
...
    QSqlRecord record = query.record();
QSqlField field = record.field("country");

QSqlField 对象可以提供一些关于字段的元数据，例如，它的 name()、variant type()、length()、precision( )、defaultValue()、typeID() 及其 requiredStatus()、isGenerated() 和 isReadOnly()。可以检查该字段的数据以查看它是否 isNull() 以及它的 value() 是否检索到。编辑数据时，可以使用 setValue() 设置或使用 clear() 设置为 NULL。

See also QSqlRecord.

成员类型文档#

enum QSqlField::RequiredStatus#

指定该字段是必需的还是可选的。

Constant Value Description
QSqlField::Required 1 插入记录时必须指定该字段。
QSqlField::Optional 0 插入记录时不必指定字段。
QSqlField::Unknown -1 数据库驱动程序无法确定该字段是必需的还是可选的。

See also requiredStatus().

void QSqlField::clear()#

清除字段的值并将其设置为 NULL。 如果该字段是只读的，则不会发生任何事情。

See also setValue(), isReadOnly(), and requiredStatus().

QSqlField::defaultValue() const#

返回字段的默认值(可能为 NULL)。

See also setDefaultValue(), metaType(), requiredStatus(), length(), precision(), and isGenerated().

bool QSqlField::isAutoValue() const#

如果值是由数据库自动生成的，则返回 true，例如自动递增主键值。

注意： 使用 ODBC 驱动程序时，由于 ODBC API 中的限制，”isAutoValue()”字段仅填充在通过执行 SELECT 查询获得的 QSqlRecord Q.html SqlField 中。在 QSqlField 中，它是flase，由 QSqlRecord 返回，返回自 QSqlDatabase：：record() 或 QSqlDatabase：:p rimaryIndex()。

See also setAutoValue().

bool QSqlField::isGenerated() const#

如果生成了字段，则返回true;否则返回 false。

See also setGenerated(), metaType(), requiredStatus(), length(), precision(), and defaultValue().

bool QSqlField::isNull() const#

如果字段的值为 NULL，则返回true;否则返回 false。

See also value().

bool QSqlField::isReadOnly() const#

如果字段的值是只读的，则返回true;否则返回 false。

See also setReadOnly(), metaType(), requiredStatus(), length(), precision(), defaultValue(), and isGenerated().

bool QSqlField::isValid() const#

如果字段的变体类型有效，则返回true;否则返回”false”。

int QSqlField::length() const#

返回字段的长度。

如果返回的值为负数，则表示无法从数据库中获取该信息。

See also setLength(), metaType(), requiredStatus(), precision(), defaultValue(), and isGenerated().

QSqlField::metaType() const#

返回存储在数据库中的字段类型。请注意，实际值可能具有不同的类型，由于太大而无法存储在长整数或双精度中，数值通常存储为字符串以防止精度损失。

See also setMetaType().

QSqlField::name() const#

返回字段的名称。

See also setName().

int QSqlField::precision() const#

返回字段的精度;这仅对数值类型有意义。

如果返回的值为负数，则表示无法从数据库中获取该信息。

See also setPrecision(), metaType(), requiredStatus(), length(), defaultValue(), and isGenerated().

QSqlField::requiredStatus() const#

如果这是必填字段，则返回true;否则返回”false”。如果必填字段没有值，则”INSERT”将失败。

See also setRequiredStatus(), metaType(), length(), precision(), defaultValue(), and isGenerated().

void QSqlField::setAutoValue(bool autoVal)#

如果 autoVal 为 true，则将字段标记为自动生成的值。

See also isAutoValue().

void QSqlField::setDefaultValue(const &value)#

将此字段使用的默认值设置为 value。

See also defaultValue(), value(), setMetaType(), setRequiredStatus(), setLength(), setPrecision(), setGenerated(), and setReadOnly().

void QSqlField::setGenerated(bool gen)#

设置生成的状态。如果 gen 为假，则不会为此字段生成任何 SQL;否则，Qt 类(如 QSqlQueryModel和 QSqlTableModel 将为此字段生成 SQL。

See also isGenerated(), setMetaType(), setRequiredStatus(), setLength(), setPrecision(), setDefaultValue(), and setReadOnly().

void QSqlField::setLength(int fieldLength)#

将字段的长度设置为fieldLength。对于字符串，这是字符串可以容纳的最大字符数;对于其他类型，含义各不相同。

See also length(), setMetaType(), setRequiredStatus(), setPrecision(), setDefaultValue(), setGenerated(), and setReadOnly().

void QSqlField::setMetaType( type)#

将字段的变体类型设置为 type。

See also metaType(), setRequiredStatus(), setLength(), setPrecision(), setDefaultValue(), setGenerated(), and setReadOnly().

void QSqlField::setName(const &name)#

将字段的名称设置为 name。

See also name().

void QSqlField::setPrecision(int precision)#

设置字段的 precision。这只会影响数值字段。

See also precision(), setMetaType(), setRequiredStatus(), setLength(), setDefaultValue(), setGenerated(), and setReadOnly().

void QSqlField::setReadOnly(bool readOnly)#

将字段值的只读标志设置为 readOnly。只读字段不能使用 setValue() 设置其值，并且不能使用 clear() 清除为 NULL。

See also isReadOnly().

void QSqlField::setRequired(bool required)#

如果 required为真，则将此字段的必需状态设置为 required;否则，将其设置为 Optional。

See also setRequiredStatus() and requiredStatus().

void QSqlField::setRequiredStatus( required)#

将此字段的必填状态设置为 必填。

See also requiredStatus(), setMetaType(), setLength(), setPrecision(), setDefaultValue(), setGenerated(), and setReadOnly().

void QSqlField::setTableName(const &table)#

将字段的 tableName 设置为 table。

See also tableName().

void QSqlField::setValue(const &value)#

将字段的值设置为 value。如果该字段是只读的(isReadOnly() 返回true)，则不会发生任何反应。

如果 value 的数据类型与字段的当前数据类型不同，则尝试将其转换为正确的类型。这将在赋值的情况下保留字段的数据类型，例如 QString 为整数数据类型。

要将值设置为 NULL，请使用 clear()。

See also value(), isReadOnly(), and defaultValue().

QSqlField::tableName() const#

返回字段的表名称。

注意： 使用 QPSQL 驱动程序时，由于 libpq 库中的限制，”tableName()”字段不会填充在由只进查询的 QSqlQuery：：record() 获得的 QSqlRecord() 生成的 QSqlField 中。

See also setTableName().

QSqlField::value() const#

以 QVariant 的形式返回字段的值。

使用 isNull() 检查字段的值是否为 NULL。

See also setValue().

bool QSqlField::operator!=(const &other) const#

如果字段不等于 other，则返回true;否则返回 false。

bool QSqlField::operator==(const &other) const#

如果字段等于 other，则返回 true;否则返回 false。
