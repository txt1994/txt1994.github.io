void QSslSocket::addCaCertificate(const &certificate)#

这个函数已经过时了。它是为了保持旧的源代码工作而提供的。我们强烈建议不要在新代码中使用它。

使用QSslConfiguration::addCaCertificate()代替。

将证书添加到这个套接字的CA证书数据库中。CA证书数据库在握手阶段被套接字使用，以验证对等人的证书。

要添加多个证书，请使用addCaCertificates()。

参见 QSslConfiguration::caCertificates()和 QSslConfiguration::setCaCertificates()。

bool QSslSocket::addCaCertificates(const &path, format = QSsl::Pem, [QRegExp::PatternSyntax] (qregexp.html#PatternSyntax-enum) syntax = QRegExp::FixedString)#

这个函数已经过时了。它的提供是为了保持旧的源代码工作。我们强烈建议不要在新代码中使用它。

使用 QSslConfiguration::addCaCertificates()代替。

在path中的所有文件中搜索以指定的format编码的证书，并将它们添加到这个套接字的CA证书数据库。path必须是一个文件或匹配一个或多个文件的模式，如syntax所指定。如果一个或多个证书被添加到该套接字的CA证书数据库中，则返回 “true”；否则返回 “false”。

CA证书数据库在握手阶段被套接字使用，以验证对方的证书。

为了更精确地控制，请使用addCaCertificate()。

参见 addCaCertificate()和QSslCertificate::fromPath()。

[static] bool QSslSocket::addDefaultCaCertificates(const &path, encoding = QSsl::Pem, syntax = QRegExp::FixedString)#

这个函数已经过时了。它是为了保持旧的源代码工作而提供的。我们强烈建议不要在新代码中使用它。

在默认的QSslConfiguration上使用QSslConfiguration::addCaCertificates()来代替。

在路径的所有文件中搜索具有指定编码的证书，并将它们添加到默认的CA证书数据库中。path可以是一个明确的文件，也可以包含syntax所指定格式的通配符。如果有任何CA证书被添加到默认数据库中，则返回`true’。

每个SSL套接字的CA证书数据库被初始化为默认CA证书数据库。

参见 QSslConfiguration::caCertificates(), QSslConfiguration::addCaCertificates(), 和 QSslConfiguration::addCaCertificate ()。

使用默认的QSslConfiguration上的QSslConfiguration::addCaCertificates()代替。

参见 QSslConfiguration::caCertificates() 和 QSslConfiguration::addCaCertificates() 。

<> QSslSocket::caCertificates() const#

这个函数已经过时了。它的提供是为了保持旧的源代码工作。我们强烈建议不要在新代码中使用它。

使用QSslConfiguration::caCertificates()代替。

返回这个套接字的CA证书数据库。CA证书数据库在握手阶段被套接字使用，以验证对方的证书。它可以在握手之前用addCaCertificate()、addCaCertificates()和setCaCertificates()进行调节。

注意：在Unix上，如果根证书是按需加载的，这个方法可能会返回一个空列表。

参见 addCaCertificate(), addCaCertificates(), and setCaCertificates()。

<> QSslSocket::ciphers() const#

这个函数已经过时了。它的出现是为了让旧的源代码继续工作。我们强烈建议不要在新代码中使用它。

使用QSslConfiguration::ciphers()代替。

返回此套接字的当前加密密码套件。这个列表在套接字的握手阶段用于选择一个会话密码。返回的密码列表是按优先级降序排列的。(即，列表中的第一个密码是最受欢迎的密码）。会话密码将是列表中第一个也被对方支持的密码。

默认情况下，握手阶段可以选择本系统的SSL库所支持的任何密码，这可能因系统而异。本系统的SSL库支持的密码列表由supportedCiphers()返回。你可以通过调用setCiphers()来限制用于为该套接字选择会话密码的密码列表，其中包括一个支持的密码子集。你可以通过调用setCiphers()和supportedCiphers()返回的列表恢复到使用整个密码集。

你可以通过调用setDefaultCiphers()来限制用于为所有套接字选择会话密码的密码列表。你可以通过调用setCiphers()和supportedCiphers()返回的列表恢复到使用整个密码集。

See also setCiphers(), defaultCiphers(), setDefaultCiphers(), and supportedCiphers().

[static] <> QSslSocket::defaultCaCertificates()#

这个函数已经过时了。它是为了保持旧的源代码工作而提供的。我们强烈建议不要在新代码中使用它。

使用默认的 QSslConfiguration::caCertificates() 来代替。

返回当前默认的CA证书数据库。这个数据库最初被设置为你的系统默认CA证书数据库。如果没有找到系统默认的数据库，将返回一个空数据库。你可以使用setDefaultCaCertificates()用自己的CA证书数据库覆盖默认CA证书数据库。

每个SSL套接字的CA证书数据库被初始化为默认的CA证书数据库。

注意：在Unix上，如果根证书是按需加载的，该方法可能返回一个空列表。

参见 setDefaultCaCertificates() 和 caCertificates()。

[static] <> QSslSocket::defaultCiphers()#

这个函数已经过时了。它的存在是为了保持旧的源代码的工作。我们强烈建议不要在新代码中使用它。

使用QSslConfiguration::ciphers()来代替默认的QSslConfiguration。

返回此应用程序中所有套接字的默认密码套件。这个列表是在套接字的握手阶段与对等体协商选择会话密码时使用的。该列表是按优先级排序的（即，列表中的第一个密码是最优先的密码）。

默认情况下，握手阶段可以选择本系统的SSL库所支持的任何密码，这些密码可能因系统而异。本系统的SSL库所支持的密码列表由supportedCiphers()返回。

参见 setDefaultCiphers()和supportedCiphers()。

void QSslSocket::setCaCertificates(const <> &certificates)#

这个函数已经过时了。它的存在是为了保持旧的源代码的工作。我们强烈建议不要在新代码中使用它。

请使用QSslConfiguration::setCaCertificates()代替。

设置此套接字的CA证书数据库为certificates。证书数据库必须在SSL握手之前设置。在握手阶段，CA证书数据库被套接字使用，以验证对等体的证书。

CA证书数据库可以通过调用此函数和defaultCaCertificates()返回的CA证书列表重置为当前默认的CA证书数据库。

参见 caCertificates()和defaultCaCertificates()。

void QSslSocket::setCiphers(const < > &ciphers)#

这个函数已经过时了。它的存在是为了保持旧的源代码的工作。我们强烈建议不要在新代码中使用它。

请使用 QSslConfiguration::setCiphers()代替。

将此套接字的密码套件设置为ciphers，其中必须包含由supportedCiphers()返回的列表中的密码子集。

限制密码套件必须在选择会话密码的握手阶段之前完成。

See also ciphers(), setDefaultCiphers(), and supportedCiphers().

void QSslSocket::setCiphers(const &ciphers)#

这个函数已经过时了。它是为了保持旧的源代码工作而提供的。我们强烈建议不要在新代码中使用它。

请使用QSslConfiguration::setCiphers()代替。

将此套接字的密码套件设置为ciphers，这是一个用冒号分隔的密码套件名称的列表。这些密码按优先顺序列出，从最优先的密码开始。例如。

QSslSocket socket;
socket.setCiphers("DHE-RSA-AES256-SHA：DHE-DSS-AES256-SHA：AES256-SHA")。

ciphers中的每个密码名称必须是supportedCiphers()返回的列表中的一个密码的名称。限制密码套件必须在选择会话密码的握手阶段之前完成。

参见 ciphers(), setDefaultCiphers(), 和 supportedCiphers ()。

[static] void QSslSocket::setDefaultCaCertificates(const <> &certificates)#

这个函数已经过时了。它的存在是为了保持旧的源代码的工作。我们强烈建议不要在新代码中使用它。

使用默认的 QSslConfiguration::setCaCertificates() 来代替。

将默认的CA证书数据库设置为certificates。默认的CA证书数据库最初被设置为你系统的默认CA证书数据库。你可以使用这个函数用你自己的CA证书数据库来覆盖默认的CA证书数据库。

每个SSL插口的CA证书数据库都被初始化为默认的CA证书数据库。

参见 defaultCaCertificates()和addDefaultCaCertificate()。

[static] void QSslSocket::setDefaultCiphers(const <> &ciphers)#

这个函数已经过时了。它的存在是为了保持旧的源代码的工作。我们强烈建议不要在新代码中使用它。

请使用默认的QSslConfiguration上的QSslConfiguration::setCiphers代替。

将此应用程序中所有套接字的默认密码套件设置为ciphers，其中必须包含由supportedCiphers()返回的列表中的密码子集。

限制默认密码套件只影响到在默认密码套件被改变后执行握手阶段的SSL套接字。

See also setCiphers(), defaultCiphers(), and supportedCiphers().

<> QSslSocket::sslErrors() const#

这个函数已经过时了。它是为了保持旧的源代码工作而提供的。我们强烈建议不要在新代码中使用它。

使用sslHandshakeErrors()代替。

返回最后发生的SSL错误的列表。这与QSslSocket通过sslErrors()信号传递的列表相同。如果连接已经被加密且没有错误，这个函数将返回一个空的列表。

参见 connectToHostEncrypted()和sslHandshakeErrors() 。

[static] <> QSslSocket::supportedCiphers()#

这个函数已经过时了。它的存在是为了保持旧的源代码的工作。我们强烈建议不要在新代码中使用它。

使用 QSslConfiguration::supportedCiphers()代替。

返回本系统支持的加密密码的列表。这个列表是由系统的SSL库设置的，可能因系统而异。

参见 defaultCiphers(), ciphers(), 和 setCiphers()。

[static] <> **QSslSocket **::systemCaCertificates()#

这个函数已经过时了。它的提供是为了保持旧的源代码工作。我们强烈建议不要在新代码中使用它。

请使用QSslConfiguration::systemDefaultCaCertificates代替。

这个函数提供了由操作系统提供的CA证书数据库。这个函数返回的CA证书数据库被用来初始化由defaultCaCertificates()返回的数据库。你可以用setDefaultCaCertificates()用你自己的数据库替换该数据库。

注意： 。在OS X上，只有在钥匙串中为所有目的信任或为SSL目的信任的证书才会被返回。

See also caCertificates(), defaultCaCertificates(), and setDefaultCaCertificates().
