详细描述#

通常，您将使用 QSqlQuery 而不是 QSqlResult，因为 QSqlQuery 为特定于数据库的 QSqlResult 实现提供了一个通用包装器。

如果您正在实现自己的 SQL 驱动程序（通过子类化 QSqlDriver），您将需要提供自己的 QSqlResult 子类来实现您需要的所有纯虚函数和其他虚函数。

See also QSqlDriver.

成员类型文档#

enum QSqlResult::BindingSyntax#

此枚举类型指定用于在准备好的查询中指定占位符的不同语法。

Constant Value Description
QSqlResult::PositionalBinding 0 使用 ODBC 样式的位置语法，带有”?” 作为占位符。
QSqlResult::NamedBinding 1 使用带有命名占位符的 Oracle 样式语法（例如，”:id”）

See also bindingSyntax().

使用数据库驱动程序 db 创建一个 QSqlResult。 对象被初始化为非活动状态。

[protected]int QSqlResult::at() const#

返回结果的当前（从零开始）行位置。可能返回特殊值 QSql::BeforeFirstRow 或 QSql::AfterLastRow。

另请参阅 setAt() 和 isValid()。

[virtual protected]void QSqlResult::bindValue(int index, const &val, paramType)#

将参数类型 paramType 的值 val 绑定到当前记录（行）中的位置 index。

另请参阅 addBindValue()。

[virtual protected]void QSqlResult::bindValue(const &placeholder, const &val, paramType)#

这是一个过载功能。

将参数类型 paramType 的值 val 绑定到当前记录（行）中的 placeholder 名称。

注意： 绑定未定义的占位符将导致未定义的行为。

另见 QSqlQuery::bindValue()。

返回当前记录（行）中位置 index 处绑定的值。

See also bindValue() and boundValues().

[protected] QSqlResult::boundValue(const &placeholder) const#

这是一个过载功能。

返回当前记录（行）中给定 placeholder 名称所绑定的值。

另见 boundValues()。

[protected] QSqlResult::boundValueName(int index) const#

返回当前记录（行）中位置 index 处的绑定值的名称。

另请参阅 boundValue()。

[protected]<> &QSqlResult::boundValues() const#

返回当前记录（行）的结果边界值的向量。

另请参阅 boundValueCount()。

[protected]void QSqlResult::clear()#

清除整个结果集并释放任何关联的资源。

[pure virtual protected] QSqlResult::data(int index)#

以 QVariant 形式返回当前行中字段 index 的数据。仅当结果处于活动状态并位于有效记录上且 index 为非负数时才调用此函数。派生类必须重新实现此函数并返回字段 index 的值，如果无法确定则返回 QVariant()。

[protected]const *QSqlResult::driver() const#

返回与结果关联的驱动程序。这是传递给构造函数的对象。

[virtual protected]bool QSqlResult::exec()#

执行查询，成功则返回true；否则返回假。

另见 prepare()。

[protected] QSqlResult::executedQuery() const#

返回实际执行的查询。这可能与传递的查询不同，例如，如果绑定值与准备好的查询一起使用并且基础数据库不支持准备好的查询。

See also exec() and setQuery().

[pure virtual protected]bool QSqlResult::fetch(int index)#

将结果定位到任意（从零开始）行index。

仅当结果处于活动状态时才调用此函数。派生类必须重新实现此函数并将结果定位到行index，并使用适当的值调用setAt()。返回 true 表示成功，返回 false 表示失败。

另请参阅 isActive()、fetchFirst()、fetchLast()、fetchNext() 和 [fetchPrevious] (#fetchPrevious)()。

[pure virtual protected]bool QSqlResult::fetchFirst()#

将结果定位到结果中的第一条记录（第 0 行）。

仅当结果处于活动状态时才调用此函数。派生类必须重新实现此函数并将结果定位到第一条记录，并使用适当的值调用 setAt()。返回 true 表示成功，返回 false 表示失败。

另见 fetch() 和 fetchLast()。

[pure virtual protected]bool QSqlResult::fetchLast()#

将结果定位到结果中的最后一条记录（最后一行）。

仅当结果处于活动状态时才调用此函数。派生类必须重新实现此函数并将结果定位到最后一条记录，并使用适当的值调用 setAt()。返回 true 表示成功，返回 false 表示失败。

另请参阅 fetch() 和 fetchFirst()。

[virtual protected]bool QSqlResult::fetchNext()#

将结果定位到结果中的下一个可用记录（行）。

仅当结果处于活动状态时才调用此函数。默认实现使用下一个索引调用 fetch()。派生类可以重新实现此函数并以其他方式将结果定位到下一条记录，并使用适当的值调用 setAt()。返回 true 表示成功，返回 false 表示失败。

See also fetch() and fetchPrevious().

[virtual protected]bool QSqlResult::fetchPrevious()#

将结果定位到结果中的前一条记录（行）。

仅当结果处于活动状态时才调用此函数。 默认实现使用前一个索引调用 fetch()。 派生类可以重新实现此函数并以其他方式将结果定位到下一条记录，并使用适当的值调用 setAt()。 返回 true 表示成功，返回 false 表示失败。

[virtual] QSqlResult::handle() const#

返回包装在 QVariant 中的此结果集的低级数据库句柄，如果没有句柄，则返回无效的 QVariant。

警告： 请谨慎使用此功能，并且仅在您知道自己在做什么的情况下使用。

警告： 如果结果被修改（例如，如果你清除它），这里返回的句柄可能会变成一个过时的指针。

警告： 如果结果尚未执行，则句柄可以为 NULL。

警告： PostgreSQL：在只进模式下，QSqlResult 的句柄在调用 fetch(), fetchFirst(), fetchLast)()、fetchNext()、fetchPrevious()、nextResult()。

这里返回的句柄是依赖于数据库的，你应该在访问它之前查询变量的类型名称。

此示例检索 sqlite 结果的句柄：

QSqlDatabase db = QSqlDatabase::database("sales");
QSqlQuery query("SELECT NAME, DOB FROM EMPLOYEES", db);

QVariant v = query.result()->handle();
if (v.isValid() && qstrcmp(v.typeName(), "sqlite3_stmt*") == 0) {
    // v.data() returns a pointer to the handle
    sqlite3_stmt *handle = *static_cast<sqlite3_stmt **>(v.data());
    if (handle) {
        // ...
    }
}

此代码段返回 PostgreSQL 或 MySQL 的句柄：

if (qstrcmp(v.typeName(), "PGresult*") == 0) {
    PGresult *handle = *static_cast<PGresult **>(v.data());
    if (handle) {
        // ...
    }
}

if (qstrcmp(v.typeName(), "MYSQL_STMT*") == 0) {
    MYSQL_STMT *handle = *static_cast<MYSQL_STMT **>(v.data());
    if (handle) {
        // ...
    }
}

See also QSqlDriver::handle().

[protected]bool QSqlResult::hasOutValues() const#

如果查询的至少一个绑定值是 QSql::Out 或 QSql::InOut，则返回 true；否则返回“假”。

另请参阅 bindValueType()。

[protected]bool QSqlResult::isActive() const#

如果结果有要检索的记录，则返回 true；否则返回“假”。

[protected]bool QSqlResult::isForwardOnly() const#

如果只能向前滚动结果集，则返回 true；否则返回“假”。

另请参阅 setForwardOnly()。

[pure virtual protected]bool QSqlResult::isNull(int index)#

如果当前行中 index 位置的字段为 null，则返回 true；否则返回“假”。

[protected]bool QSqlResult::isSelect() const#

如果当前结果来自“SELECT”语句，则返回“true”；否则返回“假”。

另见 setSelect()。

[protected]bool QSqlResult::isValid() const#

如果结果定位在有效记录上（即结果不定位在第一条记录之前或最后一条记录之后），则返回 true；否则返回“假”。

See also at().

[protected] QSqlResult::lastError() const#

返回与结果关联的最后一个错误。

See also setLastError().

[virtual protected] QSqlResult::lastInsertId() const#

如果数据库支持，则返回最近插入行的对象 ID。 如果查询未插入任何值或数据库未报告 id，则将返回无效的 QVariant。 如果插入操作多于一行，则行为未定义。

请注意，对于 Oracle 数据库，将返回行的 ROWID，而对于 MySQL 数据库，将返回行的自动增量字段。

See also QSqlDriver::hasFeature().

[protected] QSqlResult::lastQuery() const#

返回当前 SQL 查询文本，如果没有则返回空字符串。

See also setQuery().

[pure virtual protected]int QSqlResult::numRowsAffected()#

返回受执行的最后一个查询影响的行数，如果无法确定或查询是 SELECT 语句，则返回 -1。

See also size().

[virtual protected]bool QSqlResult::prepare(const &query)#

准备给定的 query 以供执行； 查询通常会使用占位符，以便可以重复执行。 如果查询准备成功，则返回 true； 否则返回“假”。

See also exec().

[virtual protected] QSqlResult::record() const#

如果查询处于活动状态，则返回当前记录； 否则返回一个空的 QSqlRecord。

默认实现总是返回一个空的 QSqlRecord。

See also isActive().

[pure virtual protected]bool QSqlResult::reset(const &query)#

将结果设置为使用 SQL 语句 query 进行后续数据检索。

派生类必须重新实现此函数并将 query 应用于数据库。 此函数仅在结果设置为非活动状态并位于新结果的第一条记录之前调用。 如果查询成功并准备好使用，派生类应该返回 true，否则返回 false。

See also setQuery().

[protected]void QSqlResult::resetBindCount()#

重置绑定参数的数量。

[virtual protected]bool QSqlResult::savePrepare(const &query)#

准备给定的query，尽可能使用底层数据库功能。 如果查询准备成功，则返回 true； 否则返回“假”。

注意：这个方法应该被称为“safePrepare()”。

See also prepare().

[virtual protected]void QSqlResult::setActive(bool active)#

该函数是为派生类提供的，用于将内部活动状态设置为active。

See also isActive().

[virtual protected]void QSqlResult::setAt(int index)#

此函数是为派生类提供的，用于将内部（从零开始）行位置设置为 index。

See also at().

[virtual protected]void QSqlResult::setForwardOnly(bool forward)#

将只转发模式设置为 forward。如果 forward 为真，则只允许 fetchNext() 用于导航结果。仅转发模式需要更少的内存，因为不必缓存结果。默认情况下，此功能被禁用。

将 forward only 设置为 false 是对数据库引擎的建议，它对结果集是仅向前还是可滚动有最终决定权。 isForwardOnly() 将始终返回结果集的正确状态。

注意： 在执行查询后调用 setForwardOnly 充其量会导致意外结果，最坏情况下会导致崩溃。

注意： 为确保只进查询成功完成，应用程序不仅应在执行查询后检查 lastError() 错误，而且应在导航查询结果后检查错误。

警告： PostgreSQL：在只进模式下浏览查询结果时，不要在同一数据库连接上执行任何其他 SQL 命令。这将导致查询结果丢失。

See also isForwardOnly(), fetchNext(), and QSqlQuery::setForwardOnly().

[virtual protected]void QSqlResult::setLastError(const &error)#

此函数是为派生类提供的，用于将最后一个错误设置为 error。

另见 lastError()。

[virtual protected]void QSqlResult::setQuery(const &query)#

将结果的当前查询设置为 query。 您必须调用 reset() 来对数据库执行查询。

另请参阅 reset() 和 lastQuery()。

[virtual protected]void QSqlResult::setSelect(bool select)#

该函数是为派生类提供的，用于指示当前语句是否为 SQL SELECT 语句。 如果语句是 SELECT 语句，select 参数应该为真； 否则它应该是错误的。

另请参阅 isSelect()。

[pure virtual protected]int QSqlResult::size()#

返回 SELECT 结果的大小，如果无法确定或查询不是 SELECT 语句，则返回 -1。

See also numRowsAffected().
