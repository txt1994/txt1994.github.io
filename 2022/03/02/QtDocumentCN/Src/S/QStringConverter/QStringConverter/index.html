详细描述#

Qt 使用 UTF-16 来存储、绘制和操作字符串。 在许多情况下，您可能希望处理使用不同编码的数据。 大多数通过文件和网络连接传输的文本数据都以 UTF-8 编码。

QStringConverter 类是 QStringEncoder 和 QStringDecoder 类的基类，有助于在不同的文本编码之间进行转换。 QStringDecoder 可以将字符串从编码表示解码为 UTF-16，这是 Qt 内部使用的格式。 QStringEncoder 执行相反的操作，将 UTF-16 编码数据（通常以 QString 的形式）编码为请求的编码。

支持的编码是：

  • UTF-8
  • UTF-16
  • UTF-16BE
  • UTF-16LE
  • UTF-32
  • UTF-32BE
  • UTF-32LE
  • ISO-8859-1 (Latin-1)
  • 系统编码

QStringConverters 可用于将某些编码字符串与 UTF-16 进行转换。

假设您有一些以 UTF-8 编码的字符串，并希望将其转换为 QString。 简单的方法是使用 QStringDecoder 像这样：

QByteArray encodedString = "...";
auto toUtf16 = QStringDecoder(QStringDecoder::Utf8);
QString string = toUtf16(encodedString);

在此之后，string 以解码的形式保存文本。 使用 QStringEncoder 类将字符串从 Unicode 转换为本地编码同样简单：

QString string = "...";
auto fromUtf16 = QStringEncoder(QStringEncoder::Utf8);
QByteArray encodedString = fromUtf16(string);

要以各种编码读取或写入文本文件，请使用 QTextStream 及其 setEncoding() 函数。

尝试以块的形式转换数据时必须小心，例如，通过网络接收数据时。 在这种情况下，一个多字节字符可能会被分成两个块。 充其量这可能会导致角色丢失，最坏的情况是导致整个转换失败。

QStringEncoder 和 QStringDecoder 都通过在内部状态中跟踪它来简化此操作。 因此，只需使用下一个数据块再次调用编码器或解码器，将自动继续正确编码或解码数据：

auto toUtf16 = QStringDecoder(QStringDecoder::Utf8);

QString string;
while (new_data_available()) {
    QByteArray chunk = get_new_data();
    string += toUtf16(chunk);
}

QStringDecoder 对象维护块之间的状态，因此即使在块之间拆分多字节字符也能正常工作。

QStringConverter 对象由于其内部状态而无法复制，但可以移动。

See also QTextStream, QStringDecoder, and QStringEncoder.

enum QStringConverter::Encoding#

Constant Value Description
QStringConverter::Utf8 0 创建与 UTF-8 之间的转换器
QStringConverter::Utf16 1 创建与 UTF-16 之间的转换器。 解码时，字节顺序将由前导字节顺序标记自动检测。 如果不存在或在编码时，将假定系统字节顺序。
QStringConverter::Utf16BE 3 创建与大端 UTF-16 之间的转换器。
QStringConverter::Utf16LE 2 创建与 little-endian UTF-16 之间的转换器。
QStringConverter::Utf32 4 `创建与 UTF-32 之间的转换器。 解码时，字节顺序将由前导字节顺序标记自动检测。 如果不存在或在编码时，将假定系统字节顺序。
QStringConverter::Utf32BE 6 创建与大端 UTF-32 之间的转换器。
QStringConverter::Utf32LE 5 创建与 little-endian UTF-32 之间的转换器。
QStringConverter::Latin1 7 创建与 ISO-8859-1 (Latin1) 之间的转换器。
QStringConverter::System 8 创建与操作系统区域设置的底层编码之间的转换器。 对于基于 Unix 的系统，始终假定为 UTF-8。 在 Windows 上，这将与语言环境代码页相互转换。

enum class QStringConverter::Flag flags QStringConverter::Flags#

Constant Value Description
QStringConverter::Flag::Default 0 默认转换规则适用。
QStringConverter::Flag::ConvertInvalidToNull 0x2 如果设置了此标志，则每个无效输入字符都将作为空字符输出。 如果未设置，如果输出编码可以表示该字符，则无效输入字符表示为 QChar::ReplacementCharacter，否则表示为问号。
QStringConverter::Flag::WriteBom 0x4 从 QString 转换为输出编码时，如果输出编码支持，则将 QChar::ByteOrderMark 作为第一个字符。 UTF-8、UTF-16 和 UTF-32 编码就是这种情况。
QStringConverter::Flag::ConvertInitialBom 0x8 当从输入编码转换为 QString 时，QStringDecoder 通常会跳过前导 QChar::ByteOrderMark。 设置此标志时，字节顺序标记将不会被跳过，而是转换为 utf-16 并插入到创建的 QString 的开头。
QStringConverter::Flag::Stateless 0x1 忽略不同函数调用之间可能的转换器状态以编码或解码字符串。 如果遇到不完整的数据序列，这也会导致 QStringConverter 引发错误。

Flags 类型是 QFlags 的 typedef。 它存储标志值的 OR 组合。

成员函数文档#

[static]std::optional<> QStringConverter::encodingForData( data, char16_t expectedFirstCharacter = 0)#

如果可以确定，则返回 data 内容的编码。 expectedFirstCharacter 可以作为附加提示传递，以帮助确定编码。

如果编码不清楚，则返回的可选项为空。

[static]std::optional<> QStringConverter::encodingForHtml( data)#

尝试通过查看 HTML 元标记中的前导字节顺序标记或字符集说明符来确定 data 中 HTML 的编码。 如果可选项为空，则 QStringConverter 不支持指定的编码。 如果未检测到编码，则该方法返回 Utf8。

[static]std::optional<> QStringConverter::encodingForName(const char *name)#

返回 name 的可选编码。 如果名称无法转换为有效编码，则该选项为空。

bool QStringConverter::hasError() const#

如果转换无法正确转换字符，则返回 true。 例如，这可能由无效的 UTF-8 序列触发，或者由于目标编码的限制而无法转换字符。

bool QStringConverter::isValid() const#

如果这是可用于编码或解码文本的有效字符串转换器，则返回 true。

默认构造的字符串转换器或使用不受支持的名称构造的转换器无效。

const char *QStringConverter::name() const#

返回此 QStringConverter 可以编码或解码的编码的规范名称。 如果转换器无效，则返回 nullptr。

See also isValid().

[static]const char *QStringConverter::nameForEncoding( e)#

返回编码 e 的规范名称。

void QStringConverter::resetState()#

重置转换器的内部状态，清除潜在错误或部分转换。
