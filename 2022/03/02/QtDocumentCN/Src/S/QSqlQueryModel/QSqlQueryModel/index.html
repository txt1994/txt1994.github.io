详细描述#

QSqlQueryModel 是用于执行 SQL 语句和遍历结果集的高级接口。 它建立在较低级别的 QSqlQuery 之上，可用于为视图类提供数据，例如 QTableView。 例如：

QSqlQueryModel *model = new QSqlQueryModel;
model->setQuery("SELECT name, salary FROM employee");
model->setHeaderData(0, Qt::Horizontal, tr("Name"));
model->setHeaderData(1, Qt::Horizontal, tr("Salary"));
QTableView *view = new QTableView;
view->setModel(model);
view->show();

我们设置模型的查询，然后设置视图标题中显示的标签。

QSqlQueryModel 也可用于以编程方式访问数据库，而无需将其绑定到视图：

QSqlQueryModel model;
model.setQuery("SELECT name, salary FROM employee");
int salary = model.record(4).value("salary").toInt();

上面的代码片段从SELECT查询的结果集中的记录 4 中提取salary字段。 由于 salary 是第 2 列（或列索引 1），我们可以将最后一行重写如下：

int salary = model.data(model.index(4, 1)).toInt();

该模型默认为只读。 要使其可读写，您必须对其进行子类化并重新实现 setData() 和 flags()。 另一种选择是使用QSqlTableModel，它提供基于单个数据库表的读写模型。

querymodel 示例说明了如何使用 QSqlQueryModel 显示查询结果。 它还展示了如何继承 QSqlQueryModel 以在向用户显示数据之前自定义数据的内容，以及如何创建基于 QSqlQueryModel 的读写模型。

如果数据库未返回查询中选定行的数量，则模型将增量获取行。 有关详细信息，请参阅 fetchMore()。

See also QSqlTableModel, QSqlRelationalTableModel, QSqlQuery, Model/View Programming, and Query Model Example.

[override virtual]bool QSqlQueryModel::canFetchMore(const &parent = QModelIndex()) const#

重新实现: QAbstractItemModel::canFetchMore(const QModelIndex &parent) const.

如果可以从数据库中读取更多行，则返回 true。 这只会影响不报告查询大小的数据库（请参阅 QSqlDriver::hasFeature()）。

parent 应该始终是无效的 QModelIndex。

See also fetchMore().

[override virtual]void QSqlQueryModel::fetchMore(const &parent = QModelIndex())#

重新实现：QAbstractItemModel::fetchMore(const QModelIndex &parent)。

从数据库中获取更多行。 这只会影响不报告查询大小的数据库（请参阅 QSqlDriver::hasFeature()）。

要强制获取整个结果集，您可以使用以下命令：

while (myModel->canFetchMore())
    myModel->fetchMore();

parent 应该始终是无效的 QModelIndex。

See also canFetchMore().

[override virtual] QSqlQueryModel::headerData(int section, orientation, int role = Qt::DisplayRole) const#

重新实现: QAbstractItemModel::headerData(int section, Qt::Orientation orientation, int role) const.

See also setHeaderData().

[virtual protected] QSqlQueryModel::indexInQuery(const &item) const#

返回模型中给定 item 的数据库结果集中值的索引。

如果没有插入、删除或移动任何列或行，则返回值与 item 相同。

如果 item 超出范围或 item 未指向结果集中的值，则返回无效的模型索引。

See also QSqlTableModel::indexInQuery(), insertColumns(), and removeColumns().

[override virtual]bool QSqlQueryModel::insertColumns(int column, int count, const &parent = QModelIndex())#

重新实现：QAbstractItemModel::insertColumns(int column, int count, const QModelIndex &parent)。

将 count 列插入模型的位置 column。 parent 参数必须始终是无效的 QModelIndex，因为模型不支持父子关系。

如果 column 在范围内，则返回 true； 否则返回“假”。

默认情况下，插入的列是空的。 要用数据填充它们，请重新实现 data() 并分别处理任何插入的列：

QVariant MyModel::data(const QModelIndex &item, int role) const
{
    if (item.column() == m_specialColumnNo) {
        // handle column separately
    }
    return QSqlQueryModel::data(item, role);
}

See also removeColumns().

[virtual protected]void QSqlQueryModel::queryChange()#

每当查询更改时，都会调用此虚函数。 默认实现什么也不做。

query() 返回新的查询。

See also query() and setQuery().

QSqlQueryModel::record(int row) const#

返回包含有关当前查询的字段信息的记录。 如果 row 是有效行的索引，则记录将填充该行的值。

如果模型未初始化，则返回空记录。

See also QSqlRecord::isEmpty().

QSqlQueryModel::record() const#

这是一个重载函数。

返回一个空记录，其中包含有关当前查询的字段的信息。

如果模型未初始化，则返回空记录。

See also QSqlRecord::isEmpty().

[override virtual]bool QSqlQueryModel::removeColumns(int column, int count, const &parent = QModelIndex())#

重新实现：QAbstractItemModel::removeColumns(int column, int count, const QModelIndex &parent)。

从位置 column 开始从模型中删除 count 列。 parent 参数必须始终是无效的 QModelIndex，因为模型不支持父子关系。

删除列可以有效地隐藏它们。 它不影响底层的 QSqlQuery。

如果列被删除，则返回 true； 否则返回“假”。

[override virtual]int QSqlQueryModel::rowCount(const &parent = QModelIndex()) const#

重新实现：QAbstractItemModel::rowCount(const QModelIndex &parent) const。

如果数据库支持返回查询的大小（参见QSqlDriver::hasFeature()），则返回当前查询的行数。否则，返回客户端当前缓存的行数。

parent 应该始终是无效的 QModelIndex。

另请参阅 canFetchMore() 和 QSqlDriver::hasFeature()。

[override virtual]bool QSqlQueryModel::setHeaderData(int section, orientation, const &* value*, int role = Qt::EditRole)#

重新实现：QAbstractItemModel::setHeaderData(int section, Qt::Orientationorientation, const QVariant &value, int role)。

将指定 role 的水平标题的标题设置为 value。如果模型用于在视图中显示数据（例如，QTableView），这很有用。

如果 orientation 是 Qt::Horizontal 并且 section 指的是有效部分，则返回 true；否则返回假。

请注意，此函数不能用于修改数据库中的值，因为模型是只读的。

另请参阅 headerData() 和 data()。

[protected]void QSqlQueryModel::setLastError(const &error)#

受保护的函数，它允许派生类将数据库上发生的最后一个错误的值设置为 error。

See also lastError().

[since 6.2]void QSqlQueryModel::setQuery( &&query)#

重置模型并将数据提供者设置为给定的query。 请注意，查询必须是活动的，并且不能是 isForwardOnly()。

如果设置查询时出错，lastError() 可用于检索详细信息。

注意： 调用 setQuery() 将删除所有插入的列。

这个函数是在 Qt 6.2 中引入的。

See also query(), QSqlQuery::isActive(), QSqlQuery::setForwardOnly(), and lastError().

void QSqlQueryModel::setQuery(const &query, const &db = QSqlDatabase())#

这是一个重载函数。

对给定的数据库连接 db 执行查询 query。 如果未指定数据库（或无效数据库），则使用默认连接。

如果设置查询时出错，lastError() 可用于检索详细信息。

例子：

QSqlQueryModel model;
model.setQuery("select * from MyTable");
if (model.lastError().isValid())
    qDebug() << model.lastError();

See also query(), queryChange(), and lastError().
