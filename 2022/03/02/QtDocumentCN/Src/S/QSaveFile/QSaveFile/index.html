详细描述#

QSaveFile 是一种用于写入文本和二进制文件的 I/O 设备，如果写入操作失败，则不会丢失现有数据。

写入时，内容将被写入一个临时文件，如果没有发生错误，commit() 会将其移动到最终文件中。这样可以确保在写入时发生错误时不会丢失最终文件中的数据，并且最终位置不会出现部分写入的文件。将整个文档保存到磁盘时始终使用 QSaveFile。

QSaveFile 在写入时自动检测错误，例如分区满的情况，其中 write() 无法写入所有字节。它会记住发生了错误，并会丢弃 commit() 中的临时文件。

很像 QFile，文件是用 open() 打开的。数据通常使用 QDataStream 或 QTextStream 读取和写入，但您也可以调用 QIODevice 继承的函数 read()、readLine()、readAll()、write()。

与 QFile 不同，不允许调用 close()。 commit() 替换它。如果未调用 commit() 并且 QSaveFile 实例被销毁，则临时文件将被丢弃。

要因应用程序错误而中止保存，请调用 cancelWriting()，这样即使稍后调用 commit() 也不会保存。

See also QTextStream, QDataStream, QFileInfo, QDir, QFile, and QTemporaryFile.

void QSaveFile::cancelWriting()#

取消写入新文件。

如果应用程序在保存时改变主意，它可以调用 cancelWriting()，它会设置一个错误代码，以便 commit() 将丢弃临时文件。

或者，它可以简单地确保不调用 commit()。

调用该方法后可以进行进一步的写操作，但是没有任何作用，写的文件会被丢弃。

使用直接写入回退时，此方法无效。 在只读目录中保存现有文件时会出现这种情况：无法创建临时文件，因此无论如何都会覆盖现有文件，而 cancelWriting() 对此无能为力，现有文件的内容将是 丢失。

See also commit().

bool QSaveFile::commit()#

如果所有先前的写入都成功，则将更改提交到磁盘。

必须在保存操作结束时调用它，否则文件将被丢弃。

如果写入过程中发生错误，则删除临时文件并返回 false。 否则，将其重命名为最终的 fileName 并在成功时返回 true。 最后，关闭设备。

See also cancelWriting().

bool QSaveFile::directWriteFallback() const#

如果启用了将文件保存在只读目录中的后备解决方案，则返回 true。

See also setDirectWriteFallback().

[override virtual] QSaveFile::fileName() const#

重新实现：QFileDevice::fileName() const。

返回由 setFileName() 或 QSaveFile 构造函数设置的名称。

See also setFileName().

[override virtual]bool QSaveFile::open( mode)#

重新实现: QIODevice::open(QIODeviceBase::OpenMode mode).

使用 OpenMode mode 打开文件，如果成功则返回 true； 否则为假。

重要提示：mode 必须包含 QIODevice::WriteOnly。 它还可能有其他标志，例如 QIODevice::Text 和 QIODevice::Unbuffered。

目前不支持 QIODevice::ReadWrite、QIODevice::Append、QIODevice::NewOnly 和 QIODevice::ExistingOnly。

See also QIODevice::OpenMode and setFileName().

void QSaveFile::setDirectWriteFallback(bool enabled)#

如有必要，允许覆盖现有文件。

QSaveFile 在与最终文件相同的目录中创建一个临时文件并自动重命名它。但是，如果目录权限不允许创建新文件，则这是不可能的。为了保持原子性保证，open() 在无法创建临时文件时会失败。

为了允许用户在权限受限的目录中编辑具有写入权限的文件，调用 setDirectWriteFallback() 并将 enabled 设置为 true，然后对 open() 的以下调用将回退到打开现有的直接写入文件并写入其中，无需使用临时文件。这没有原子性保证，即应用程序崩溃或例如电源故障可能导致磁盘上的部分写入文件。这也意味着 cancelWriting() 在这种情况下无效。

通常，要保存用户编辑的文档，调用 setDirectWriteFallback(true)，以及保存应用程序内部文件（配置文件、数据文件等），保持默认设置以确保原子性。

See also directWriteFallback().

void QSaveFile::setFileName(const &name)#

设置文件的name。 该名称可以没有路径、相对路径或绝对路径。

See also QFile::setFileName() and fileName().
