详细描述#

一个QItemSelectionModel记录了一个视图中的选定项目，或者在同一个模型上的几个视图中的选定项目。它也跟踪视图中当前选择的项目。

QItemSelectionModel类是模型/视图类之一，是Qt的模型/视图框架的一部分。

选定的项目是用范围来存储的。每当你想修改选中的项目时，使用select()并提供一个QItemSelection，或者一个QModelIndex和一个QItemSelectionModel::SelectionFlag 。

QItemSelectionModel对选择管理采取了两层方法，既处理已经提交的选择项，又处理属于当前选择的项。当前选择的项目是当前交互式选择的一部分（例如，橡皮筋选择或键盘移动选择）。

要更新当前选择的项目，请使用QItemSelectionModel::Current和其他任何SelectionFlag的位法OR。如果你省略了QItemSelectionModel::Current命令，一个新的当前选择将被创建，而之前的选择将被添加到整个选择中。所有的函数都在两个层上操作；例如，selecteditems()将返回两个层的项目。

注意：从5.5开始，model、hasSelection和currentIndex是元对象属性。

See also Model/View Programming, QAbstractItemModel, and Chart Example.

成员类型文档#

enum QItemSelectionModel::SelectionFlag flags QItemSelectionModel::SelectionFlags#

这个枚举描述了选择模型将被更新的方式。

Constant Value Description
QItemSelectionModel::NoUpdate 0x0000 将不进行选择。
QItemSelectionModel::Clear 0x0001 完整的选择将被清除。
QItemSelectionModel::Select 0x0002 所有指定的索引将被选中。
QItemSelectionModel::Deselect 0x0004 所有指定的索引将被取消选择。
QItemSelectionModel::Toggle 0x0008 所有指定的索引将被选择或取消选择，这取决于它们当前的状态。
QItemSelectionModel::Current 0x0010 当前的选择将被更新。
QItemSelectionModel::Rows 0x0020 所有的索引将被扩展到跨行。
QItemSelectionModel::Columns 0x0040 所有的索引将被扩展到跨列。
QItemSelectionModel::SelectCurrent `Select Current`
QItemSelectionModel::ToggleCurrent `Toggle Current`
QItemSelectionModel::ClearAndSelect `Clear Select`

SelectionFlags类型是QFlags的一个类型定义。它存储了SelectionFlag值的一个OR组合。

[signal]void QItemSelectionModel::currentChanged(const &current, const &previous)#

每当当前项目发生变化时，这个信号就会被发射出来。previous模型的项目索引被current索引所取代，作为选择的当前项目。

注意，当项目模型被重置时，这个信号将不会被发出。

See also currentIndex(), setCurrentIndex(), and selectionChanged().

[signal]void QItemSelectionModel::currentColumnChanged(const &current, const &previous)#

如果current项目发生变化，并且其列与previous当前项目的列不同，就会发出这个信号。

请注意，当项目模型被重置时，这个信号将不会被发出。

See also currentChanged(), currentRowChanged(), currentIndex(), and setCurrentIndex().

[signal]void QItemSelectionModel::currentRowChanged(const &current, const &previous)#

如果current项目发生变化，并且它的行与previous当前项目的行不同，则发出这个信号。

请注意，当项目模型被重置时，这个信号将不会被发出。

See also currentChanged(), currentColumnChanged(), currentIndex(), and setCurrentIndex().

[signal]void QItemSelectionModel::selectionChanged(const &selected, const &deselected)#

每当选择发生变化时，这个信号就会被发射出来。选择的变化被表示为deselected项的项目选择和selected项的项目选择。

请注意，当前索引的变化是独立于选择的。还要注意，当项目模型被重置时，这个信号将不会被发出。

保持选择但改变索引的项目不包括在selected和deselected中。因此，如果只有被选中的项目的指数发生变化，这个信号可能会在selected和deselected都为空的情况下发出。

注意：属性selectedIndexes的通知器信号。

See also select() and currentChanged().

[virtual slot]void QItemSelectionModel::setCurrentIndex(const &index, command)#

设置模型项目index为当前项目，并发出currentChanged()。当前项目用于键盘导航和焦点指示；它与任何选定的项目无关，尽管一个选定的项目也可以是当前项目。

根据指定的命令，索引也可以成为当前选择的一部分。

See also currentIndex() and select().

[virtual]QItemSelectionModel::~QItemSelectionModel()#

破坏选择模式。

[invokable]bool QItemSelectionModel::columnIntersectsSelection(int column, const &parent = QModelIndex()) const#

如果有任何项目在column中被选中，并有给定的parent，则返回true。

注意：从Qt 5.15开始，parent的默认参数是一个空的模型索引。

**注意:**这个函数可以通过元对象系统和QML来调用。参见Q_INVOKABLE。

QItemSelectionModel::currentIndex() const#

返回当前项目的模型项目索引，如果没有当前项目，则返回一个无效的索引。

See also setCurrentIndex().

[protected]void QItemSelectionModel::emitSelectionChanged(const &newSelection, const &oldSelection)#

比较两个选择newSelection和oldSelection，并发出selectionChanged()，说明取消选择和选择的项目。

bool QItemSelectionModel::hasSelection() const#

如果选择模型包含任何选定的项目，返回 “true”，否则返回 “false”。

[invokable]bool QItemSelectionModel::isColumnSelected(int column, const &parent = QModelIndex()) const#

如果在给定的父列中的所有项目都被选中，则返回true。

请注意，这个函数通常比在同一列的所有项目上调用isSelected()要快，而且不可选择的项目会被忽略。

注意：从Qt 5.15开始，parent的默认参数是一个空的模型索引。

**注意:**这个函数可以通过元对象系统和QML来调用。参见Q_INVOKABLE。

[invokable]bool QItemSelectionModel::isRowSelected(int row, const &parent = QModelIndex()) const#

如果在给定的parent 的row中，所有项目都被选中，则返回true。

请注意，这个函数通常比在同一行的所有项目上调用isSelected()要快，而且不可选择的项目会被忽略掉。

注意：从Qt 5.15开始，parent的默认参数是一个空的模型索引。

**注意:**这个函数可以通过元对象系统和QML来调用。参见Q_INVOKABLE。

[invokable]bool QItemSelectionModel::isSelected(const &index) const#
