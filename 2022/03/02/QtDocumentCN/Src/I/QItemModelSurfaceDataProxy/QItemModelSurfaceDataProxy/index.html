详细描述#

QItemModelSurfaceDataProxy允许你使用QAbstractItemModel衍生的模型作为Q3DSurface的数据源。它使用定义的映射将数据从模型映射到 Q3DSurface 图的行、列和表面点。

只要映射或模型发生变化，数据就会被异步解析。当数据被解析后，QSurfaceDataProxy::arrayReset()被发射出来。然而，当useModelCategories属性被设置为true时，单项变化被同步解析，除非同一帧也包含一个导致整个模型被解析的变化。

映射可以通过以下方式使用。

  • 如果useModelCategories属性被设置为true，这个代理将把QAbstractItemModel的行和列映射为Q3DSurface的行和列，并且默认使用Qt::DisplayRole返回的值作为Y-position。如果行和列的标题可以转换为浮点数，则默认使用Z-position和X-position。否则就使用行和列的索引。如果Qt::DisplayRole不适合，可以重新定义要使用的Y-position角色。如果标题或索引不合适，要使用的Z-位置和X-位置角色可以被重新定义。
  • 对于没有数据已经被整齐地分类为行和列的模型，比如基于 QAbstractListModel的模型，你可以从模型中定义一个角色来映射行、列和 Y-位置中的每一个。
  • 如果你不想包括模型中包含的所有数据，或者自动生成的行和列的顺序不符合你的要求，你可以通过为行和列中的任何一个或两个定义一个明确的类别列表来指定哪些行和列应该被包括，以及以何种顺序。

例如，假设你有一个自定义的QAbstractItemModel来存储表面地形数据。模型中的每个项目都有 “经度”、”纬度 “和 “高度 “等角色。项目模型已经包含了正确排序的数据，所以经度和纬度是以正确的顺序首先遇到的，这使得我们能够利用行和列的类别自动生成。你可以做以下工作，在一个表面图中显示数据：

QItemModelSurfaceDataProxy *proxy = new QItemModelSurfaceDataProxy(customModel,
                                                                   QStringLiteral("longitude"), // Row role
                                                                   QStringLiteral("latitude"), // Column role
                                                                   QStringLiteral("height")); // Y-position role

如果模型的字段不包含你需要的确切格式的数据，你可以为每个角色指定一个搜索模式正则表达式和替换规则，以获得你需要的格式的值。关于使用正则表达式进行替换的更多信息，请参阅 QString::replace(const QRegularExpression&rx, const QString&after) 函数文档。请注意，使用正则表达式对性能有影响，所以利用项目模式更有效率，在这种情况下，做搜索和替换不一定能得到所需的值。

关于结合角色使用搜索模式的例子，见ItemModelBarDataProxy在Qt Quick 2 Bars Example中的用法。

另请参阅 Qt Data Visualization 数据处理 .

enum QItemModelSurfaceDataProxy:: MultiMatchBehavior#

QItemModelSurfaceDataProxy::multiMatchBehavior的行为类型。

常量 值 描述
QItemModelSurfaceDataProxy::MMBFirst 0 位置值取自项目模型中与每个行/列组合相匹配的第一个项目。
QItemModelSurfaceDataProxy::MMBLast 1 位置值取自项目模型中与每个行/列组合相匹配的最后一个项目。
QItemModelSurfaceDataProxy::MMBAverage 2 匹配每行/每列组合的所有项目的位置值被平均到一起，平均数被用作表面点位置。
QItemModelSurfaceDataProxy::MMBCumulativeY 3 对于X和Z值，其作用与MMBAverage相同，但Y值被加在一起，而不是平均，总和被用作表面点的Y位置。

multiMatchBehavior : #

如何处理每个行/列组合的多重匹配。
