详细描述#

QIcon 可以从给定像素图集，生成更小、更大、活动及禁用的像素图。Qt 小部件使用这种像素图来展示表示特定动作的图标。

QIcon 的最简单用法是创建一个从 QPixmap 文件或资源，然后使用它，允许 Qt 算出所有要求图标的样式和大小。例如：

QToolButton *button = new QToolButton;
button->setIcon(QIcon("open.xpm"));

要撤销一个QIcon，只需在其位置上设置一个空图标。

button->setIcon(QIcon());

使用 QImageReader::supportedImageFormats () 和 QImageWriter::supportedImageFormats () 函数来检索支持文件格式的完整列表。

当你使用pixmap(QSize, Mode, State)检索一个像素图，并且没有用addFile() 或 addPixmap() 添加这个尺寸、模式和状态的像素图，那么QIcon将在运行中生成一个。这种像素图的生成发生在QIconEngine中。如果需要的话，默认的引擎会将像素图缩小，但绝不会放大，而且它使用当前的风格来计算一个禁用的外观。通过使用自定义图标引擎，你可以自定义生成的图标的每一个方面。通过QIconEnginePlugin，可以为不同的文件后缀注册不同的图标引擎，使第三方有可能在Qt包含的图标引擎之外提供额外的图标引擎。

注意：从Qt 4.2开始，一个支持SVG的图标引擎被包含在内。

制作使用 QIcon 的类#

如果你自己编写的小部件有一个设置小像素图的选项，可以考虑允许为该像素图设置一个QIcon。Qt类QToolButton就是这样一个小部件的例子。

提供一个方法来设置一个QIcon，当你绘制图标时，选择适合你的widget的当前状态的任何像素图。比如说：

void MyWidget::drawIcon(QPainter *painter, QPoint pos)
{
    QPixmap pixmap = icon.pixmap(QSize(22, 22),
                                   isEnabled() ? QIcon::Normal
                                               : QIcon::Disabled,
                                   isChecked() ? QIcon::On
                                               : QIcon::Off);
    painter->drawPixmap(pos, pixmap);
}

你也可以利用 “激活 “模式，也许当鼠标在小部件上时使你的小部件 “激活”（见 QWidget::enterEvent ()），当鼠标被按下等待释放将激活该功能时，或者当它是当前选择的项目时。如果该小组件可以被切换，”开启 “模式可能被用来绘制不同的图标。

注意： QIcon 需要 QGuiApplication 实例在创建图标之前。

高 DPI 图标#

QIcon有两种方式支持高DPI图标：凭借addFile() 和 fromTheme()。

如果你有自己的自定义目录结构，并且不需要使用freedesktop.org图标主题规范，那么addFile()就很有用。通过这种方法创建的图标使用Qt的“@nx “高DPI语法。

如果你打算遵循Icon主题规范，使用fromTheme()是必要的。要让QIcon使用高DPI版本的图片，请在相应的 “index.theme “文件中添加一个额外的条目：

[Icon Theme]
Name=Test
Comment=Test Theme
Directories=32x32/actions,32x32@2/actions
[32x32/actions]
Size=32
Context=Actions
Type=Fixed
# High DPI version of the entry above.
[32x32@2/actions]
Size=32
Scale=2
Type=Fixed

然后你的图标主题目录将看起来像这样：

├── 32x32
│   └── actions
│       └── appointment-new.png
├── 32x32@2
│   └── actions
│       └── appointment-new.png
└── index.theme

另请参阅 GUI 设计手册：标志性标签 and 图标范例 .

成员类型文档#

enum QIcon:: Mode#

此枚举类型描述旨在使用的像素图模式。目前定义的模式：

常量 值 描述
QIcon::Normal 0 显示像素图当用户未与图标交互时，但由图标表示的功能是可用的。
QIcon::Disabled 1 当图标所代表的功能不可用时，显示该像素图。
QIcon::Active 2 当图标所代表的功能可用，并且用户正在与图标互动时，例如，将鼠标移到它上面或点击它，就会显示该像素图。
QIcon::Selected 3 显示像素图当由图标表示的项被选中时。

enum QIcon:: State#

此枚举描述旨在使用的像素图状态。 state 可以是：

常量 值 描述
QIcon::Off 1 当小组件处于 “关闭 “状态时显示像素图。
QIcon::On 0 当小组件处于 “开启 “状态时显示像素图。

成员函数文档#

QIcon:: QIcon ( * engine )#

创建图标采用特定图标 engine 。图标拥有引擎的所有权。

QIcon:: QIcon (const & fileName )#

构造图标从文件采用给定 fileName 。将按需加载文件。

若 fileName 包含相对路径(如：仅文件名)，必须相对运行时工作目录查找相关文件。

文件名可以引用实际磁盘文件或应用程序嵌入资源之一。见 资源系统 概述，了解如何在应用程序可执行文件中嵌入图像及其它资源文件的细节。

使用 QImageReader::supportedImageFormats () 和 QImageWriter::supportedImageFormats () 函数来检索支持文件格式的完整列表。

移动构造 QIcon 实例，使它指向相同对象的 other 所指向的。

QIcon:: QIcon (const & other )#

构造副本为 other 。这非常快。

QIcon:: QIcon (const & pixmap )#

构造图标从 pixmap .

QIcon:: QIcon ()#

构造 null 图标。

QIcon:: actualSize (const & size , mode = Normal, state = Off) const#

返回请求的size ，mode ，和state的图标的实际大小。结果可能比要求的要小，但绝不会大。返回的尺寸是与设备无关的像素（这与高像素地图有关）。

另请参阅 pixmap () 和 paint ().

void QIcon:: addFile (const & fileName , const & size = QSize(), mode = Normal, state = Off)#

从给定的fileName到图标，作为特殊化为size、mode和state的文件中添加一个图像。该文件将在需要时被加载。注意：自定义图标引擎可以自由地忽略额外添加的像素图。

若 fileName 包含相对路径(如：仅文件名)，必须相对运行时工作目录查找相关文件。

文件名可以引用实际磁盘文件或应用程序嵌入资源之一。见 资源系统 概述，了解如何在应用程序可执行文件中嵌入图像及其它资源文件的细节。

使用 QImageReader::supportedImageFormats () 和 QImageWriter::supportedImageFormats () 函数来检索支持文件格式的完整列表。

如果图像的高分辨率版本存在（由基本名称的后缀@2x标识），它将被自动加载并添加，设备像素比率设置为2。这可以通过设置环境变量QT_HIGHDPI_DISABLE_2X_IMAGE_LOADING（见QImageReader）来禁用。

注意：当你给QIcon添加一个非空的文件名时，即使该文件不存在或指向一个损坏的文件，该图标也会变成非空的。

另请参阅 addPixmap () 和 QPixmap::devicePixelRatio ().

void QIcon:: addPixmap (const & pixmap , mode = Normal, state = Off)#

添加 pixmap 到图标，作为特殊化为 mode and state .

自定义图标引擎可自由忽略额外添加的像素图。

另请参阅 addFile ().

< > QIcon:: availableSizes ( mode = Normal, state = Off) const#

返回可用图标大小的列表为指定 mode and state .

QIcon:: cacheKey () const#

返回一个数字，用于识别这个QIcon对象的内容。如果不同的QIcon对象指的是相同的内容，它们可以有相同的键。

当通过addPixmap()或addFile()改变图标时，cacheKey()将会改变。

缓存键大多与缓存结合使用。

另请参阅 QPixmap::cacheKey ().

[static, since 5.11] QIcon:: fallbackSearchPaths ()#

返回图标的后备搜索路径。

默认值将取决于平台。

该函数在 Qt 5.11 引入。

另请参阅 setFallbackSearchPaths () 和 themeSearchPaths ().

[static, since 5.12] QIcon:: fallbackThemeName ()#

返回后备图标主题的名称。

在X11上，如果没有设置，后备图标主题取决于你的桌面设置。在其他平台上，默认情况下是不设置的。

该函数在 Qt 5.12 引入。

另请参阅 setFallbackThemeName () 和 themeName ().

[static] QIcon:: fromTheme (const & name )#

返回 QIcon 对应于当前图标主题中的name。

最新版本的freedesktop图标规范和命名规范可以在这里获得:

  • http://standards.freedesktop.org/icon-theme-spec/icon-theme-spec-latest.html
  • http://standards.freedesktop.org/icon-naming-spec/icon-naming-spec-latest.html

从当前的图标主题中获取一个图标:

QIcon undoicon = QIcon::fromTheme("edit-undo");

注意：默认情况下，只有X11会支持主题图标。为了在Mac和Windows上使用主题图标，你必须在你的themeSearchPaths()中捆绑一个符合要求的主题，并设置适当的themeName()。

注意： Qt将利用GTK的icon-theme.cache（如果存在的话）来加快查找速度。这些缓存可以用gtk-update-icon-cache生成。https://developer.gnome.org/gtk3/stable/gtk-update-icon-cache.html 。

注意：如果在当前主题中找不到一个图标，那么它将在fallbackSearchPaths()中作为一个无主题的图标被搜索到。

另请参阅 themeName (), setThemeName (), themeSearchPaths ()，和 fallbackSearchPaths ().

[static] QIcon:: fromTheme (const & name , const & fallback )#

这是重载函数。

返回 QIcon ，与当前图标主题中的name对应。如果在当前主题中没有找到这样的图标fallback被返回取而代之。

如果你想为不支持主题图标的平台提供一个保证的回退，你可以使用第二个参数:

QIcon undoicon = QIcon::fromTheme("edit-undo", QIcon(":/undo.png"));

[static] bool QIcon:: hasThemeIcon (const & name )#

返回 true 如果在当前的图标主题中，name有一个可用的图标，否则返回 false 。

另请参阅 themeSearchPaths (), fromTheme ()，和 setThemeName ().

[since 5.6] bool QIcon:: isMask () const#

返回 true 如果这个图标被标记为遮罩图像。某些平台对遮罩图标的渲染方式不同（例如，macOS上的菜单图标）。

该函数在 Qt 5.6 引入。

另请参阅 setIsMask ().

bool QIcon:: isNull () const#

返回 true 若图标为空；否则返回 false .

如果一个图标既没有像素图也没有文件名，它就是空的。

注意：即使是一个非空的图标也可能无法创建有效的像素图，例如，如果文件不存在或不能被读取。

QIcon:: name () const#

返回用于创建图标的名称，若可用。

根据图标的创建方式，它可能有一个相关的名字。对于使用fromTheme()创建的图标或使用支持QIconEngine::IconNameHook的QIconEngine的图标来说，就是这种情况。

另请参阅 fromTheme () 和 QIconEngine .

QIcon:: pixmap (const & size , mode = Normal, state = Off) const#

返回一个具有请求的size ，mode ，和state的像素图，必要时生成一个。该像素图可能比要求的要小，但绝不会更大，除非返回的像素图的设备-像素比大于1。

另请参阅 actualSize () 和 paint ().

QIcon:: pixmap ( int w , int h , mode = Normal, state = Off) const#

QIcon:: pixmap ( int extent , mode = Normal, state = Off) const#

[static] QIcon:: theeName ()#

返回当前图标主题的名称。

在X11上，当前的图标主题取决于你的桌面设置。在其他平台上，它不是默认设置的。

另请参阅 setThemeName (), themeSearchPaths (), fromTheme ()，和 hasThemeIcon ().

[static] QIcon:: themeSearchPaths ()#

返回图标主题的搜索路径。

默认值将取决于平台。

在X11上，搜索路径将使用XDG_DATA_DIRS环境变量（如果有）。

默认情况下，所有平台都会有资源目录:\icons作为备用。你可以使用 “rcc -project “从你的图标主题生成一个资源文件。

另请参阅 setThemeSearchPaths (), fromTheme ()，和 setThemeName ().
