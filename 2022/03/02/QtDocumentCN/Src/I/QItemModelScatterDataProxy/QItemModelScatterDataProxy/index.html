详细描述#

QItemModelScatterDataProxy允许你使用QAbstractItemModel衍生模型作为Q3DScatter的数据源。它将QAbstractItemModel的角色映射到Q3DScatter点的XYZ值。

每当映射或模型发生变化时，数据会被异步解析。当数据被解析后，QScatterDataProxy::arrayReset()会被发射出来。然而，模型初始化后的插入、移除和单个数据项的变化都是同步解析的，除非同一帧也包含一个导致整个模型被解析的变化。

映射忽略了 [QAbstractItemModel] (qabstractitemmodel.html)的行和列，并平等地对待所有项目。它要求模型为数据项提供角色，这些角色可以被映射到散点的X、Y和Z值。

例如，假设你有一个自定义的QAbstractItemModel，用于存储对材料样本进行的各种测量，提供诸如 “密度”、”硬度 “和 “电导率 “等角色数据。你可以用这个代理在散点图上将这些属性可视化:

// Map "density" value to X-axis, "hardness" to Y-axis and "conductivity" to Z-axis.
QItemModelScatterDataProxy *proxy = new QItemModelScatterDataProxy(customModel,
                                                                   QStringLiteral("density"),
                                                                   QStringLiteral("hardness"),
                                                                   QStringLiteral("conductivity"));

如果模型的字段不包含你所需要的确切格式的数据，你可以为每个角色指定一个搜索模式正则表达式和一个替换规则，以获得你需要的格式的值。关于使用正则表达式进行替换的更多信息，请参见 QString::replace (const QRegularExpression &rx, const QString &after) 函数文档。请注意，使用正则表达式对性能有影响，所以利用项目模式更有效率，在这种情况下，做搜索和替换不一定能得到所需的值。

关于结合角色使用搜索模式的例子，请看ItemModelBarDataProxy在Qt Quick 2 Bars Example的用法。

另请参阅 Qt Data Visualization 数据处理 .

rotationRole : #

这个属性持有项目模型的角色，以映射到项目的旋转。

模型可以提供旋转的值，可以是直接转换为QQuaternion的变体，也可以是字符串的代表之一。"scalar,x,y,z" 或"@angle,x,y,z" 。前者将用给定的值直接构造四元数，后者将使用QQuaternion::fromAxisAndAngle() 方法构造四元数。
