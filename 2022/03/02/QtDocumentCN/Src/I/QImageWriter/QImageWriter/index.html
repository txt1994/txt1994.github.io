# QImageWriter Class

QImageWriter类提供了一个独立于格式的接口，用于将图像写入文件或其他设备。. 更 多…

详细描述#

QImageWriter支持在存储图像之前设置特定格式的选项，如压缩级别和质量。如果你不需要这些选项，你可以使用 QImage::save () 或 QPixmap::save () 代替。

要存储一个图像，你首先要构造一个QImageWriter对象。将文件名或设备指针以及图像格式传给QImageWriter的构造函数。然后你可以设置几个选项，比如质量（通过调用setQuality()）。canWrite () 返回 true 如果QImageWriter可以写入图像（即，图像格式被支持，并且设备为写入而开放）。调用write()将图像写到设备上。

如果在写入图像时发生任何错误，write ()将返回false。然后你可以调用error ()来查找发生的错误类型，或者调用errorString ()来获得对出错原因的可读描述。

调用 supportedImageFormats () 获取QImageWriter可以写入的格式列表。QImageWriter支持所有内置的图像格式，此外还有任何支持写入的图像格式插件。

注意： QImageWriter假定对被分配的文件或设备具有排他性的控制。在QImageWriter对象的生命周期内，任何试图修改分配的文件或设备的行为都将产生未定义的结果。如果需要立即访问一个资源，建议使用一个范围。

例如：

QString imagePath(QStringLiteral("path/image.jpeg"));
QImage image(64, 64, QImage::Format_RGB32);
image.fill(Qt::red);
{
    QImageWriter writer(imagePath);
    writer.write(image);
}
QFile::rename(imagePath,
              QStringLiteral("path/other_image.jpeg"));

另请参阅 QImageReader , QImageIOHandler , QImageIOPlugin ，和 QColorSpace .

enum QImageWriter:: ImageWriterError#

这个枚举描述了用QImageWriter写入图像时可能发生的错误。

常量 值 描述
QImageWriter::DeviceError 1 QImageWriter在写入图像数据时遇到了一个设备错误。请咨询你的设备，了解更多关于出错的细节。
QImageWriter::UnsupportedFormatError 2 Qt 不支持请求的图像格式。
QImageWriter::InvalidImageError 3 有人试图写入一个无效的QImage。无效图像的一个例子是一个空的QImage。
QImageWriter::UnknownError 0 发生了一个未知的错误。如果你在调用write()后得到这个值，很可能是由QImageWriter的一个错误引起的。

QImageWriter:: QImageWriter (const & fileName , const & format = QByteArray())#

构建一个QImageWriter对象，该对象将写入一个名称为fileName的文件，使用图像格式format。如果没有提供format，QImageWriter将通过检查fileName的扩展名检测图像格式。

QImageWriter:: QImageWriter ( * device , const & format )#

使用设备 device 和图像格式 format 构造一个 QImageWriter 对象。

QImageWriter:: QImageWriter ()#

构建一个空的QImageWriter对象。在写之前，你必须调用setFormat()来设置图像格式，然后再调用setDevice() 或 setFileName()。

QImageWriter:: ~QImageWriter ()#

销毁 QImageWriter 对象。

bool QImageWriter:: canWrite () const#

返回 true 如果 QImageWriter 可以写入图像；也就是说，图像格式被支持，并且指定的设备被打开供读取。

另请参阅 write (), setDevice ()，和 setFormat ().

void QImageWriter:: setCompression ( int compression )#

这是一个针对图像格式的函数，用于设置图像的压缩。对于不支持设置压缩的图像格式，这个值会被忽略。

compression 的值范围取决于图像格式。例如，”tiff “格式支持两个值，0（无压缩）和1（LZW-压缩）。

另请参阅 压缩 ().

void QImageWriter:: setDevice ( * device )#

设置QImageWriter的设备为device。如果设备已被设置，旧的设备将从QImageWriter中移除，否则将保持不变。

如果设备尚未打开，QImageWriter将试图通过调用open()在QIODevice::WriteOnly模式下打开该设备。请注意，这对某些设备不起作用，比如QProcess、QTcpSocket和QUdpSocket，这些设备需要更多逻辑来打开。

另请参阅 device () 和 setFileName ().

void QImageWriter:: setFileName (const & fileName )#

设置QImageWriter的文件名为fileName。在内部，QImageWriter将创建一个QFile并在QIODevice::WriteOnly模式下打开它，并在写入图像时使用该文件。

另请参阅 fileName () 和 setDevice ().

void QImageWriter:: setFormat (const & format )#

设置格式 QImageWriter 将在写图像时使用，以format。format是一个不区分大小写的文本字符串。

例子:

QImageWriter writer;
writer.setFormat("png"); // same as writer.setFormat("PNG");

可以调用 supportedImageFormats() 获得完整的格式列表 QImageWriter支持。

另请参阅 format ().

[since 5.5] void QImageWriter:: setOptimizedWrite ( bool optimize )#

这是一个针对图像格式的函数，在写图像时设置optimize 标志。对于不支持设置optimize 标志的图像格式，此值被忽略。

默认为 false。

该函数在 Qt 5.5 引入。

另请参阅 optimizedWrite ().

[since 5.5] void QImageWriter:: setProgressiveScanWrite ( bool progressive )#

这是一个特定于图像格式的函数，在写入图像时开启progressive 扫描。对于不支持设置progressive 扫描标志的图像格式，此值被忽略。

默认为 false。

该函数在 Qt 5.5 引入。

另请参阅 progressiveScanWrite ().

void QImageWriter:: setQuality ( int quality )#

将图像格式的品质设置设为 quality .

一些图像格式，特别是有损的图像格式，需要在a）生成的图像的视觉质量和b）编码执行时间和压缩水平之间进行权衡。这个函数为支持它的图像格式设置该权衡的水平。对于其他格式，这个值被忽略。

quality 的值范围取决于图像格式。例如，”jpeg “格式支持的质量范围从0（低视觉质量，高压缩）到100（高视觉质量，低压缩）。

另请参阅 quality ().

[since 5.4] void QImageWriter:: setSubType (const & type )#

这是一个针对图像格式的函数，将图像的子类型设置为type。子类型可以被处理程序用来确定它在保存图像时应该使用哪种格式。

例如，用A8R8G8R8子类型保存一个DDS格式的图像:

QImageWriter writer("some/image.dds");
if (writer.supportsOption(QImageIOHandler::SubType))
    writer.setSubType("A8R8G8B8");
writer.write(image);

该函数在 Qt 5.4 引入。

另请参阅 subType ().

void QImageWriter:: setText (const & key , const & text )#

将与键key相关的图像文本设置为text。这对于存储版权信息或其他有关图像的信息很有用。

例子:

QImage image("some/image.jpeg");
QImageWriter writer("images/outimage.png", "png");
writer.setText("Author", "John Smith");
writer.write(image);

如果你想存储一个单一的数据块（例如，一个评论），你可以传递一个空的键，或者使用一个通用的键，如 “描述”。

在调用write()后，键和文本将被嵌入图像数据中。

支持此选项的实现是透过 QImageIOHandler::Description .

另请参阅 QImage::setText () 和 QImageReader::text ().

该函数在 Qt 5.5 引入。

另请参阅 transformation () 和 write ().

[static] < > QImageWriter:: supportedImageFormats ()#

返回 QImageWriter 支持的图像格式列表。

默认情况下，Qt可以写入以下格式:

格式 MIME 类型 描述
BMP image/bmp Windows 位图
JPG image/jpeg JPEG (联合摄影专家组)
PNG image/png PNG (便携式网络图形)
PBM image/x-portable-bitmap PBM (便携式位图)
PGM image/x-portable-graymap PGM (便携式灰度图)
PPM image/x-portable-pixmap Portable Pixmap (便携式像素图)
XBM image/x-xbitmap X11 Bitmap (X11 位图)
XPM image/x-xpixmap X11 Pixmap (X11 像素图)

通过Qt SVG模块支持读写SVG文件。Qt Image Formats模块提供对其他图像格式的支持。

注意， QApplication 实例必须被创建在调用此函数之前。

另请参阅 setFormat (), QImageReader::supportedImageFormats ()，和 QImageIOPlugin .

bool QImageWriter:: supportsOption ( option ) const#

返回 true 如果作者支持option ；否则返回 false。

不同的图像格式支持不同的选项。调用此函数以确定当前格式是否支持某个选项。例如，PNG格式允许你将文本嵌入到图像的元数据中（见text()）。

QImageWriter writer(fileName);
if (writer.supportsOption(QImageIOHandler::Description))
    writer.setText("Author", "John Smith");

选项可以在写作者与格式关联后进行测试。

另请参阅 QImageReader::supportsOption () 和 setFormat ().

[since 5.5] QImageWriter:: transformation () const#

返回图像被设置为写入的变换和方向。

该函数在 Qt 5.5 引入。

另请参阅 setTransformation ().

bool QImageWriter:: write (const & image )#

写入图像 image到指定的设备或文件名。返回 true 当成功时；否则返回 false 。如果操作失败，你可以调用error()来查找发生的错误类型，或者调用errorString()来获得对错误的可读描述。

另请参阅 canWrite (), error ()，和 errorString ().
