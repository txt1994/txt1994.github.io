详细描述#

每个QInputEvent包含一个QInputDevice指针，允许访问设备的特定属性，如类型、能力和座位。平台或通用插件有责任通过QWindowSystemInterface::registerInputDevice()来发现、创建和注册这个类的一个实例，以对应每个可用的输入设备，然后再生成指向该设备的任何输入事件。

应用程序不需要实例化这个类，但可以读取由QInputEvent::device()和QInputDevice::devices()指向的实例。

成员类型文档编制#

[since 6.2] enum class QInputDevice:: Capability flags QInputDevice:: Capabilities#

表示输入设备或其驱动程序能提供什么样的信息。

常量 值 描述
QInputDevice::Capability::None 0 没有关于输入设备能力的信息可用。
QInputDevice::Capability::Position 0x0001 表示位置信息是可用的，也就是说，触摸点中的position()系列函数返回有效的点。
QInputDevice::Capability::Area 0x0002 表示触摸区域信息是可用的，意味着触摸点中的QEventPoint::ellipseDiameters ()返回有效值。
QInputDevice::Capability::Pressure 0x0004 表示压力信息是可用的，意味着QEventPoint::pressure ()返回一个有效值。
QInputDevice::Capability::Velocity 0x0008 表示速度信息是可用的，意味着QEventPoint::velocity ()返回一个有效的向量。
QInputDevice::Capability::NormalizedPosition 0x0020 表示归一化的位置是可用的，意味着QEventPoint::globalPosition ()返回一个有效值。
QInputDevice::Capability::MouseEmulation 0x0040 表示设备合成了鼠标事件。
QInputDevice::Capability::Scroll 0x0100 表示该设备具有滚动功能。
QInputDevice::Capability::PixelScroll 0x0080 表示该设备(通常是触摸板)以像素精度滚动。
QInputDevice::Capability::Hover 0x0200 表示该设备有悬停功能。
QInputDevice::Capability::Rotation 0x0400 指示rotation信息是可用的。
QInputDevice::Capability::XTilt 0x0800 指示tiltX轴的信息是可用的。
QInputDevice::Capability::YTilt 0x1000 指示 tilt 为Y轴提供信息。
QInputDevice::Capability::TangentialPressure 0x2000 指示切向压力信息是可用的。
QInputDevice::Capability::ZPosition 0x4000 表示Z轴的位置信息可用。
QInputDevice::Capability::All 0x7FFFFFFF

这个枚举是在Qt 6.2中引入或修改的。

Capabilities 类型是 typedef 对于 QFlags 。它存储 Capability 值的 OR 组合。

enum class QInputDevice:: DeviceType flags QInputDevice:: DeviceTypes#

这个枚举代表了产生QPointerEvent的设备类型。

常量 值 描述
QInputDevice::DeviceType::Unknown 0x0000 该设备无法被识别。
QInputDevice::DeviceType::Mouse 0x0001 鼠标。
QInputDevice::DeviceType::TouchScreen 0x0002 在这种类型的设备中，触摸表面和显示器是一体化的。这意味着表面和显示器通常具有相同的尺寸，这样，触摸点的物理位置和QEventPoint报告的坐标之间就有了直接的关系。因此，Qt允许用户同时与多个QWidgets、QGraphicsItems或Qt Quick Items直接交互。
QInputDevice::DeviceType::TouchPad 0x0004 在这种类型的设备中，触摸表面与显示器是分开的。物理触摸位置和屏幕上的坐标之间没有直接关系。相反，它们是相对于当前的鼠标位置计算的，而用户必须使用触摸板来移动这个参考点。与触摸屏不同，Qt允许用户每次只与一个QWidget或QGraphicsItem互动。
QInputDevice::DeviceType::Stylus 0x0010 用于绘图板（如Wacom平板电脑）或触摸屏上的一种类似笔的装置，提供单独的手写笔感应功能。
QInputDevice::DeviceType::Airbrush 0x0020 带指轮的测针可以调节tangentialPressure。
QInputDevice::DeviceType::Puck 0x0008 一种类似于平面鼠标的设备，有一个带十字线的透明圆圈。
QInputDevice::DeviceType::Keyboard 0x1000 一个键盘。
QInputDevice::DeviceType::AllDevices 0x7FFFFFFF 以上任何一项（用作默认的过滤值）。

DeviceTypes类型是QFlags的一个类型定义。它存储了DeviceType值的一个OR组合。

成员函数文档编制#

QInputDevice:: QInputDevice (const & name , id , type , const & seatName = QString(), * parent = nullptr)#

创建一个新的输入设备实例。给定的name通常是制造商指定的型号名称（如果有的话），或者其他可识别的东西；id是一个平台特定的数字，每个设备都是唯一的（例如X11上的xinput ID）；type识别什么样的设备。在能够同时处理来自多个用户或一组输入设备的输入的窗口系统上（如Wayland或X11），seatName标识了将被一起使用的一组设备的名称。如果该设备是一个子设备或从属设备（例如，可以轮流移动 “核心指针 “的几个小鼠之一），则应将主设备作为parent。

平台插件创建、注册并继续拥有每个设备实例；即使某个设备没有主设备，为了内存管理的目的，通常也应该给出parent。

默认情况下， capabilities () are None .

QInputDevice:: QInputDevice ( * parent = nullptr)#

创建一个新的无效的输入设备实例，作为parent的子设备。

QInputDevice:: availableVirtualGeometry () const#

返回该设备可以访问的虚拟桌面中的区域。

例如，一个TouchScreen输入设备被固定在一个物理屏幕上，并且通常被校准为这个区域与QScreen::geometry()相同；而一个Mouse可能能够访问虚拟桌面上的所有屏幕。Wacom绘图板的配置方式可能是映射到所有屏幕，或者只映射到用户喜欢创建绘图的屏幕，或者映射到绘图的窗口。与触摸屏集成的手写笔设备可能被物理限制在该屏幕上。

如果返回的矩形是null，这意味着这个设备可以访问整个虚拟桌面。

注意：属性availableVirtualGeometry的getter函数。

QInputDevice:: capabilities () const#

返回设备的能力。

注意：属性capabilities的getter函数。

[static] <const *> QInputDevice:: devices ()#

返回所有注册的输入设备（键盘和指向性设备）的列表。

注意：返回的列表不能用于添加新设备。要为自动测试添加一个模拟的触摸屏，可以使用QTest::createTouchDevice()。平台插件应该调用QWindowSystemInterface::registerInputDevice()来添加被发现的设备。

bool QInputDevice:: hasCapability ( capability ) const#

返回设备能力是否包括给定的capability。

QInputDevice:: name () const#

返回设备名称。

这个字符串可能是空的。然而，它在有多个输入设备的系统中很有用：它可以用来区分一个QPointerEvent发源地是哪个设备。

注意：属性名称的getter函数。

[static] const *QInputDevice:: primaryKeyboard (const & seatName = QString())#

返回给定座位上的核心或主键盘 seatName .

QInputDevice:: seatName () const#

如果知道的话，返回与该设备相关的座位；否则为空。

那些打算由一个用户共同使用的设备可以被配置为具有相同的座位名称。到目前为止，这只在Wayland和X11平台上是可行的。

注意：属性seatName的getter函数。

QInputDevice:: systemId () const#

返回平台特定的系统ID（例如X11平台的xinput ID）。

所有的平台都应该为每个设备提供一个唯一的系统ID。

注意：属性systemId的getter函数。

QInputDevice:: type () const#

返回设备类型。

注意：属性type的getter函数。
