详细描述#

一个QIterator只能由一个QIterable实例创建，并且可以以类似于其他stl风格的迭代器的方式使用。一般来说，QIterator不应该被直接使用，而是通过它的派生类QSequentialIterable和QAssociativeIterable提供。

另请参阅 QIterable .

QIterator:: QIterator ( < Container > * iterable , void * iterator )#

从一个iterable和一个指向本地iterator的指针创建一个迭代器。

bool QIterator:: operator!= (const < Container > & other ) const#

返回 true if other 指向与此迭代器不同的项；否则返回 false .

另请参阅 operator== ().

< Container > QIterator:: operator+ ( qsizetype j ) const#

返回一个从该迭代器向前j位置的项目的迭代器。

另请参阅 operator- () 和 operator+= ().

< Container > &QIterator:: operator++ ()#

前缀++操作符（ ++it ）将迭代器推进到容器中的下一个项目，并返回到当前新项目的迭代器。

在QSequentialIterable::end()上调用这个函数会导致未定义的结果。

另请参阅 operator– ().

< Container > QIterator:: operator++ ( int )#

这是重载函数。

后缀++操作符（ it++ ）将迭代器推进到容器中的下一个项目，并返回一个迭代器到之前的当前项目。

< Container > &QIterator:: operator+= ( qsizetype j )#

将迭代器按j项推进。

另请参阅 operator-= () 和 operator+ ().

< Container > QIterator:: operator- ( qsizetype j ) const#

返回一个从这个迭代器向后j位置的项目的迭代器。

如果QVariant中的容器不支持双向迭代，调用此函数会导致未定义的结果。

另请参阅 operator+ (), operator-= ()，和 QIterable::canReverseIterate ().

qsizetype QIterator:: operator- (const < Container > & j ) const#

返回两个迭代器之间的距离。

另请参阅 operator+ (), operator-= ()，和 QIterable::canReverseIterate ().

< Container > &QIterator:: operator– ()#

前缀–操作符（ --it ）使前面的项成为当前项，并返回一个指向新的当前项的迭代器。

在QSequentialIterable::begin()上调用这个函数会导致未定义的结果。

如果QVariant中的容器不支持双向迭代，调用这个函数会导致未定义的结果。

另请参阅 operator++ () 和 QIterable::canReverseIterate ().

< Container > QIterator:: operator– ( int )#

这是重载函数。

后缀–操作符（ it-- ）使前面的项成为当前项，并返回到之前的当前项的一个迭代器。

如果QVariant中的容器不支持双向迭代，调用这个函数会导致未定义的结果。

另请参阅 QIterable::canReverseIterate ().

< Container > &QIterator:: operator-= ( qsizetype j )#

使得迭代器往回走j项。

如果QVariant中的容器不支持双向迭代，调用这个函数会导致未定义的结果。

另请参阅 operator+= (), operator- ()，和 QIterable::canReverseIterate ().

bool QIterator:: operator== (const < Container > & other ) const#

返回 true 如果 other 指向与此迭代器相同的项目；否则返回 false 。

另请参阅 operator!= ().
