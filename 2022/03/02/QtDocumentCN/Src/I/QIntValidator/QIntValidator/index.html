# QIntValidator 类

详细描述#

用法范例：

QValidator *validator = new QIntValidator(100, 999, this);
QLineEdit *edit = new QLineEdit(this);
// the edit lineedit will only accept integers between 100 and 999
edit->setValidator(validator);

下面我们介绍一些验证器的例子。在实践中，它们通常会像上面的例子一样，与一个小部件相关联。

QString str;
int pos = 0;
QIntValidator v(100, 900, this);
str = "1";
v.validate(str, pos);     // returns Intermediate
str = "012";
v.validate(str, pos);     // returns Intermediate
str = "123";
v.validate(str, pos);     // returns Acceptable
str = "678";
v.validate(str, pos);     // returns Acceptable
str = "999";
v.validate(str, pos);    // returns Intermediate
str = "1234";
v.validate(str, pos);     // returns Invalid
str = "-123";
v.validate(str, pos);     // returns Invalid
str = "abc";
v.validate(str, pos);     // returns Invalid
str = "12cm";
v.validate(str, pos);     // returns Invalid

预告，值 999 返回中间值。由等于或小于最大值的数字组成的值被认为是中间值。这是因为阻止一个数字进入范围的数字不一定是最后输入的数字。这也意味着，一个中间数可以有前导零。

最小值和最大值可以用setRange()一次设置，或者用setBottom()和setTop()单独设置。

QIntValidator使用其locale()来解释该数字。例如，在阿拉伯地区，QIntValidator将接受阿拉伯数字。

注意： QLocale::NumberOptions在locale()上的设置也会影响数字被解释的方式。例如，由于QLocale::RejectGroupSeparator默认没有设置，验证器将接受分组分隔符。因此建议使用QLocale::toInt()来获得数值。

另请参阅 QDoubleValidator , QRegularExpressionValidator , QLocale::toInt ()，和 行编辑范例 .

特性文档编制#

bottom : int#

此属性持有验证器的最低可接受值

默认情况下，这个属性的值来自于最低的有符号整数（通常是-2147483647）。

访问函数：

int bottom () const
void setBottom (int)

通知程序信号：

void bottomChanged (int bottom )

另请参阅 setRange ().

top : int#

此属性持有验证器的最高可接受值

默认情况下，此特性值派生自最高可用有符号整数 (通常为 2147483647)。

访问函数：

int top () const
void setTop (int)

通知程序信号：

[override virtual] QIntValidator:: validate ( & input , int & pos ) const#

重实现： QValidator::validate (QString &input, int &pos) const.

返回 Acceptable若input是有效范围内的一个整数。如果input的位数最多与范围的顶部一样多，或者是有效范围内整数的前缀，则返回 中间体。否则，返回 无效。

如果有效范围只是由正整数组成（例如，32到100），而input是一个负整数，那么返回Invalid。(另一方面，如果范围由负整数组成（例如，-100到-32），并且input是一个正整数，那么返回Intermediate，因为用户可能正要输入减号（特别是对于从右到左的语言）。

同样，如果有效范围在46和53之间，那么41和59将被评估为中间体，因为否则用户就无法将一个值从49改为51。

int pos = 0;
s = "abc";
v.validate(s, pos);    // returns Invalid
s = "5";
v.validate(s, pos);    // returns Intermediate
s = "50";
v.validate(s, pos);    // returns Acceptable

默认情况下，pos参数不被该验证器使用。
