详细描述#

QItemModelBarDataProxy允许你使用QAbstractItemModel衍生模型作为Q3DBars的数据源。它使用定义的映射将数据从模型映射到 Q3DBars 图的行、列和值。

每当映射或模型发生变化时，数据会被异步解析。当数据被解析后，QBarDataProxy::arrayReset()被发射出来。然而，当useModelCategories属性被设置为 “true “时，单项变化被同步解析，除非同一帧也包含一个导致整个模型被解析的变化。

映射可以通过以下方式使用。

  • 若useModelCategories属性设置为true，该代理将直接把QAbstractItemModel的行和列映射到Q3DBars的行和列，并且默认使用Qt::DisplayRole返回的值作为bar值。如果 Qt::DisplayRole 不适合，可以重新定义要使用的值角色。
  • 对于那些没有将数据整齐地分类为行和列的模型，例如基于 QAbstractListModel的模型，你可以从模型中定义一个角色来映射行、列和值。
  • 如果你不想包括模型中包含的所有数据，或者自动生成的行和列没有按照你的意愿排序，你可以通过为行和列中的任何一个或两个定义一个明确的类别列表来指定哪些行和列应该被包括以及以何种顺序。

例如，假设你有一个自定义的QAbstractItemModel用于存储与企业相关的各种月度值。模型中的每个项目都有 “年”、”月”、”收入 “和 “支出 “等角色。你可以做以下工作，以条形图的形式显示这些数据:

// By defining row and column categories, you tell the mapping which row and column each item
// belongs to. The categories must match the data stored in the model in the roles you define
// for row and column mapping. In this example we expect "year" role to return four digit year
// and "month" to return three letter designation for the month.
//
// An example of an item in model would be:
// Requested role -> Returned data
// "year" -> "2006" // Matches the first row category, so this item is added to the first row.
// "month" -> "jan" // Matches the first column category, so this item is added as first item in the row.
// "income" -> "12.1"
// "expenses" -> "9.2"
QStringList years;
QStringList months;
years << "2006" << "2007" << "2008" << "2009" << "2010" << "2011" << "2012";
months << "jan" << "feb" << "mar" << "apr" << "may" << "jun" << "jul" << "aug" << "sep" << "oct" << "nov" << "dec";
QItemModelBarDataProxy *proxy = new QItemModelBarDataProxy(customModel,
                                                           QStringLiteral("year"), // Row role
                                                           QStringLiteral("month"), // Column role
                                                           QStringLiteral("income"), // Value role
                                                           years, // Row categories
                                                           months); // Column categories
//...
// To display different data later, you can simply change the mapping.
proxy->setValueRole(QStringLiteral("expenses"));

如果模型的字段不包含你需要的确切格式的数据，你可以为每个角色指定一个搜索模式正则表达式和替换规则，以获得你需要的格式的值。关于使用正则表达式进行替换的更多信息，请参见 QString::replace (const QRegularExpression &rx, const QString &after) 函数文档。请注意，使用正则表达式对性能有影响，所以利用项目模式更有效率，在这种情况下，做搜索和替换不一定能得到所需的值。

关于结合角色使用搜索模式的例子，请看Qt Quick 2 Bars Example。

另请参阅 Qt Data Visualization 数据处理 .

enum QItemModelBarDataProxy:: MultiMatchBehavior#

QItemModelBarDataProxy::multiMatchBehavior特性的行为类型。

常量 值 描述
QItemModelBarDataProxy::MMBFirst 0 该值取自项目模型中与每行/列组合相匹配的第一个项目。
QItemModelBarDataProxy::MMBLast 1 该值取自项目模型中与每个行/列组合相匹配的最后一个项目。
QItemModelBarDataProxy::MMBAverage 2 匹配每一行/每一列组合的所有项目的值被平均到一起，平均数被用作条形值。
QItemModelBarDataProxy::MMBCumulative 3 匹配每一行/每一列组合的所有项目的值被加在一起，总和被用作条形值。

autoColumnCategories : bool#

这个属性决定了是否自动生成列的类别。

当设置为 “true “时，映射会忽略任何明确设置的列类别，并在模型的数据被解析时用自动生成的列类别覆盖它们。默认为true。

multiMatchBehavior : #

如何处理每个行/列组合的多重匹配。

默认为 QItemModelBarDataProxy::MMBLast . 选择的行为会影响条形图的值和旋转。

例如，你可能有一个带有时间戳的数据的项目模型，你想用条形图来显示每一天的数据项目的总价值。这可以通过指定行和列的类别，使每个条形图代表一天，并将multiMatchBehavior设置为QItemModelBarDataProxy::MMBCumulative来实现。

用itemModel和可选parent构造QItemModelBarDataProxy。代理不接受itemModel的所有权，因为通常项目模型是由其他控件拥有的。角色映射是用rowRole, columnRole, valueRole ，和rotationRole来设置。行和列的类别用rowCategories和columnCategories来设置。这个构造函数还设置autoRowCategories和autoColumnCategories为假。
