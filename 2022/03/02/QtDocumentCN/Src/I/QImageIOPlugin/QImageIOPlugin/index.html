详细描述#

QImageIOPlugin是一个创建QImageIOHandler对象的工厂，这些对象被QImageReader和QImageWriter内部使用，以增加对Qt的不同图像格式支持。

编写图像I/O插件是通过子类化这个基类，重新实现纯虚拟函数capabilities()和create()，和导出类采用Q_PLUGIN_METADATA() 宏。参见 如何创建Qt 插件了解细节。

一个图像格式插件可以支持三种能力：读（CanRead），写（CanWrite）和增量读（CanReadIncremental）。在你的子类中重新实现capabilities()以暴露你的图像格式的能力。

create()应该创建一个你的QImageIOHandler子类的实例，并正确设置提供的设备和格式，并返回这个处理程序。

插件的json元数据文件需要包含关于插件支持的图像格式的信息，以及相应的MIME类型（每种格式一个）。对于一个jpeg插件来说，例如，这可以看起来如下。

{
  "Keys": [ "jpg", "jpeg" ],
  "MimeTypes": [ "image/jpeg", "image/jpeg" ]
}

不同的插件可以支持不同的功能。例如，你可能有一个支持读取GIF格式的插件，而另一个则支持写入。Qt将根据capabilities()的返回值，为这项工作选择正确的插件。如果有几个插件支持同样的能力，Qt将任意选择一个。

另请参阅 QImageIOHandler and 如何创建 Qt 插件 .

成员类型文档编制#

enum QImageIOPlugin:: Capability flags QImageIOPlugin:: Capabilities#

这个枚举描述了一个QImageIOPlugin的能力。

常量 值 描述
QImageIOPlugin::CanRead 0x1 该插件可以读取图像。
QImageIOPlugin::CanWrite 0x2 该插件可以写入图像。
QImageIOPlugin::CanReadIncremental 0x4 该插件可以递增地读取图片。

Capabilities 类型是 typedef 对于 QFlags 。它存储 Capability 值的 OR 组合。

[virtual] QImageIOPlugin:: ~QImageIOPlugin ()#

销毁图片格式插件。

从不需要明确调用这。Qt 自动销毁插件当不再使用时。

[pure virtual] QImageIOPlugin:: capabilities ( * device , const & format ) const#

返回该插件的能力，基于device中的数据和格式format。如果device是0，它应该简单地报告该格式是否可以被读取或写入。否则，它应该试图确定是否可以从device中读出或写入给定的格式（或者如果format为空，则为插件支持的任何格式）。它应该在不改变device的状态的情况下做到这一点（通常通过使用QIODevice::peek()）。

例如，如果QImageIOPlugin支持BMP格式，format为空或"bmp"，并且设备中的数据以"BM"字符开始。这个函数应该返回CanRead。如果format是"bmp"，device是0，并且处理程序支持读和写，这个函数应该返回CanRead| CanWrite。

格式名称总是以小写字母给出。

[pure virtual] QImageIOPlugin:: create ( QIODevice device , const & format = QByteArray()) const#

创建和返回 QImageIOHandler子类，并设置device和format。format必须来自插件元数据中的 “Keys “条目中列出的值，或者是空的。如果是空的，device中的数据必须被capabilities()方法所识别（同样是空格式）。

格式名称总是以小写字母给出。
