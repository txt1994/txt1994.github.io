详细描述#

读取图像的最常见方式是透过 QImage and QPixmap 的构造函数，或通过调用 QImage::load () 和 QPixmap::load (). QImageReader是一个专门的类，在读取图像时给你更多的控制。例如，你可以通过调用setScaledSize()将图像读成一个特定的尺寸，你还可以通过调用setClipRect()选择一个剪辑矩形，有效地只加载图像的一部分。根据图像格式的底层支持，这可以节省内存并加快图像的加载速度。

要读取一个图像，你首先要构造一个QImageReader对象。将文件名或设备指针以及图像格式传给QImageReader的构造函数。然后，你可以设置几个选项，比如剪辑矩形（通过调用setClipRect ()）和缩放大小（通过调用setScaledSize ()）。canRead ()如果QImageReader可以读取图像（即，图像格式被支持，并且设备被打开用于读取），则返回该图像。调用read()来读取图像。

若读取图像时发生任何错误， read () 将返回 null QImage . 然后你可以调用error()来查找发生的错误类型，或者调用errorString()来获得对出错原因的可读描述。

注意： QImageReader假定对被分配的文件或设备有独占控制权。在QImageReader对象的生命周期内，任何试图修改分配的文件或设备的行为都会产生未定义的结果。

格式#

调用 supportedImageFormats() 获取QImageReader可以读取的格式列表。QImageReader支持所有内置的图像格式，此外还有任何支持阅读的图像格式插件。调用supportedMimeTypes()来获得一个支持的MIME类型的列表，例如，它可以传递给QFileDialog::setMimeTypeFilters() 。

QImageReader默认会自动检测图像格式，通过查看提供的（可选）格式字符串、文件名后缀和数据流内容。你可以通过调用setAutoDetectImageFormat()来启用或禁用这个功能。

图像的高分辨率版本#

如果设备像素和设备独立像素之间的缩放生效，可以提供图像的高分辨率版本。

高分辨率版本在基本名称上用后缀@2x标记。读取的图像的设备像素比率将设置为2的值。

这可以通过设置环境变量QT_HIGHDPI_DISABLE_2X_IMAGE_LOADING来禁用。

另请参阅 QImageWriter , QImageIOHandler , QImageIOPlugin , QMimeDatabase , QColorSpace , QImage::devicePixelRatio (), QPixmap::devicePixelRatio (), QIcon , QPainter::drawPixmap ()，和 QPainter::drawImage ().

enum QImageReader:: ImageReaderError#

这个枚举描述了用QImageReader读取图像时可能发生的不同类型的错误。

常量 值 描述
QImageReader::FileNotFoundError 1 QImageReader 被用于一个文件名，但没有找到这个名字的文件。如果文件名不包含扩展名，而具有正确扩展名的文件不被Qt支持，也会发生这种情况。
QImageReader::DeviceError 2 QImageReader在读取图像时遇到了一个设备错误。你可以咨询你的特定设备，了解更多出错的细节。
QImageReader::UnsupportedFormatError 3 Qt 不支持请求的图像格式。
QImageReader::InvalidDataError 4 图像数据是无效的，QImageReader无法从中读取图像。如果图像文件被损坏，就会发生这种情况。
QImageReader::UnknownError 0 发生了一个未知的错误。如果你在调用read()后得到这个值，很可能是由QImageReader的一个错误引起的。

bool QImageReader:: canRead () const#

返回 true 如果可以为该设备读取图像（即图像格式被支持，并且该设备似乎包含有效的数据）；否则返回 false 。

canRead()是一个轻量级的函数，只做一个快速测试，看看图像数据是否有效。read()在canRead()返回`true后仍可能返回false，如果图像数据被破坏了。

注意： QMimeDatabase 查找通常是比这个函数更好的方法来识别潜在的非图像文件或数据。

对于支持动画的图像，canRead()在所有帧都被读取时返回 false。

另请参阅 read (), supportedImageFormats ()，和 QMimeDatabase .

QImageReader:: read ()#

从设备上读取一个图像。成功时，将返回被读取的图像；否则，将返回一个空的QImage。然后你可以调用error()来查找发生的错误类型，或者调用errorString()来获得对错误的可读描述。

bool QImageReader:: read ( * image )#

这是重载函数。

从设备中读取一个图像到image中，它必须指向一个QImage。

如果 image 的格式和大小与将要读取的图像数据相同，该函数可能不需要在读取前分配一个新的图像。正因为如此，它比其他read()重载要快，后者总是构建一个新的图像；特别是在读取几个具有相同格式和大小的图像时.

QImage icon(64, 64, QImage::Format_RGB32);
QImageReader reader("icon_64x64.bmp");
if (reader.read(&icon)) {
    // Display icon
}

对于支持动画的图像格式，重复调用read()将返回下一帧。当所有帧都被读取后，将返回一个空图像。

另请参阅 canRead (), supportedImageFormats (), supportsAnimation ()，和 QMovie .

[static, since 6.0] void QImageReader:: setAllocationLimit ( int mbLimit )#

设置分配限制为mbLimit兆字节。需要QImage内存分配超过这个限制的图像将被拒绝。

另请参阅 allocationLimit ().

void QImageReader:: setAutoDetectImageFormat ( bool enabled )#

若enabled为真，则启用图像格式自动检测；否则，将禁用。默认情况下，自动检测是启用的。

QImageReader使用一种广泛的方法来检测图像格式；首先，如果你向QImageReader传递一个文件名，如果给定的文件名没有指向一个现有的文件，它将尝试检测文件扩展名，方法是将支持的默认扩展名附加到给定的文件名上，一次一个。然后，它使用以下方法来检测图像格式。

  • 首先查询图像插件，基于可选的格式字符串或文件名后缀（如果源设备是一个文件）。在这个阶段没有进行内容检测。QImageReader将选择第一个支持该格式的读取的插件。
  • 如果没有插件支持该图像格式，Qt的内置处理程序会根据可选的格式字符串或文件名后缀进行检查。
  • 如果没有找到有能力的插件或内置处理程序，则通过检查数据流的内容对每个插件进行测试。
  • 如果没有插件可以根据数据内容检测图像格式，则通过检测内容来测试每个内置图像处理程序。
  • 最后，如果上述方法都失败了，QImageReader在试图读取图像时将报告失败。

通过禁用图像格式自动检测，QImageReader将只根据格式字符串查询插件和内置处理程序（即不测试文件名扩展名）。

另请参阅 autoDetectImageFormat (), QImageIOHandler::canRead ()，和 QImageIOPlugin::capabilities ().

void QImageReader:: setDecideFormatFromContent ( bool ignored )#

如果ignored被设置为true，那么图像阅读器将忽略指定的格式或文件扩展名，只根据数据流中的内容来决定使用哪个插件。

设置这个标志意味着所有图像插件都会被加载。每个插件将读取图像数据中的第一个字节，并决定该插件是否兼容。

这还禁用自动检测图像格式。

另请参阅 decideFormatFromContent ().

void QImageReader:: setDevice ( * device )#

设置QImageReader的设备为device。如果设备已被设置，旧的设备将从QImageReader中移除，否则将保持不变。

若设备尚未打开，QImageReader将试图通过调用open()在ReadOnly模式下打开设备。请注意，这对某些设备不起作用，例如QProcess、QTcpSocket和QUdpSocket，这些设备需要更多逻辑来打开设备。

另请参阅 device () 和 setFileName ().

void QImageReader:: setFileName (const & fileName )#

设置QImageReader的文件名为fileName。在内部，QImageReader将创建一个QFile对象，并以ReadOnly模式打开，并在读取图像时使用它。

如果fileName不包括文件扩展名（例如，.png或.bmp），QImageReader将循环浏览所有支持的扩展名，直到找到一个匹配的文件。

另请参阅 fileName (), setDevice ()，和 supportedImageFormats ().

void QImageReader:: setFormat (const & format )#

设置格式 QImageReader在读取图片时，将使用format。format是一个不区分大小写的文本字符串。

例子：

QImageReader reader;
reader.setFormat("png"); // same as reader.setFormat("PNG");

可以调用 supportedImageFormats () for the full list of formats QImageReader 支持。

另请参阅 format ().

void QImageReader:: setQuality ( int quality )#

将图像格式的品质设置设为 quality .

一些图像格式，特别是有损的图像格式，需要在a）产生的图像的视觉质量和b）解码执行时间之间进行权衡。这个函数为支持它的图像格式设置这种权衡的水平。

在读取缩放图像的情况下，质量设置也可能影响视觉质量和缩放算法的执行速度之间的权衡水平。

quality 的取值范围取决于图像格式。例如，”jpeg “格式支持的质量范围从0（低视觉质量）到100（高视觉质量）。

另请参阅 quality () 和 setScaledSize ().

void QImageReader:: setScaledClipRect (const & rect )#

将缩放后的剪辑矩形设置为rect。缩放后的剪辑矩形是在图像被缩放后应用的剪辑矩形（也被称为ROI，或兴趣区域）。

另请参阅 scaledClipRect () 和 setScaledSize ().

void QImageReader:: setScaledSize (const & size )#

将图像的比例缩放尺寸设为 size 。缩放是在初始剪辑矩形之后进行的，但在应用缩放的剪辑矩形之前。用于缩放的算法取决于图像格式。默认情况下（即，如果图像格式不支持缩放），QImageReader将使用QImage::scale()与Qt::SmoothScaling。

另请参阅 scaledSize (), setClipRect ()，和 setScaledClipRect ().

QImageReader:: size () const#

返回图像的大小，而不实际读取图像内容。

如果图像格式不支持这个功能，这个函数会返回一个无效的尺寸。Qt的内置图像处理程序都支持这个功能，但自定义图像格式插件不需要这样做。

另请参阅 QImageIOHandler::ImageOption , QImageIOHandler::option ()，和 QImageIOHandler::supportsOption ().

[since 5.4] QImageReader:: subType () const#

返回图像的子类型。

该函数在 Qt 5.4 引入。

bool QImageReader:: supportsOption ( option ) const#

返回 true 若读取器支持 option ；否则返回 false。

不同的图像格式支持不同的选项。调用这个函数来确定当前格式是否支持某个选项。例如，PNG格式允许你将文本嵌入到图像的元数据中（见text ()），BMP格式允许你确定图像的大小，而无需将整个图像加载到内存中（见size ()）。

QImageReader reader(":/image.png");
if (reader.supportsOption(QImageIOHandler::Size))
    qDebug() << "Size:" << reader.size();

另请参阅 QImageWriter::supportsOption ().
