详细描述#

一个图标引擎为一个QIcon提供了渲染功能。每个图标都有一个相应的图标引擎，负责以要求的尺寸、模式和状态来绘制图标。

图标是由paint ()函数渲染的，另外还可以通过pixmap ()函数获得一个像素图（默认实现只是使用paint ()来实现）。addPixmap ()函数可以用来向图标引擎添加新的像素图，并被QIcon 用来添加专门的自定义像素图。

paint (), pixmap (), 和 addPixmap ()函数都是虚拟的，因此可以在QIconEngine的子类中被重新实现。

另请参阅 QIconEnginePlugin .

成员类型文档编制#

enum QIconEngine:: IconEngineHook#

这些枚举值用于virtual_hook()，以允许对图标引擎进行额外的查询而不破坏二进制兼容性。

常量 值 描述
QIconEngine::IsNullHook 3 允许查询这个引擎是否代表一个空图标。virtual_hook](#virtual_hook)()的data参数是一个指向bool的指针，如果图标为空，可以设置为true。这个枚举值是在Qt 5.7中添加的。
QIconEngine::ScaledPixmapHook 4 提供了一种方法来获得一个按照给定比例（通常等于设备像素比率）缩放的像素图。virtual_hook](#virtual_hook)()函数的data参数是一个ScaledPixmapArgument指针，包含输入和输出参数。这个枚举值是在Qt 5.9中添加的。

另请参阅 virtual_hook ().

成员函数文档编制#

[since 5.6] QIconEngine:: QIconEngine ()#

构造图标引擎。

该函数在 Qt 5.6 引入。

[virtual] QIconEngine:: ~QIconEngine ()#

销毁图标引擎。

[virtual] QIconEngine:: actualSize (const & size , mode , state )#

返回引擎为请求的size、mode和state提供的图标的实际尺寸。默认实现返回给定的size。

[virtual] void QIconEngine:: addFile (const & fileName , const & size , mode , state )#

由QIcon::addFile()调用。从具有给定的fileName、size、mode和state的文件中添加一个专门的像素图。默认的基于像素图的引擎会存储任何提供的文件名，如果像素图的大小与请求的图标大小相匹配，它会按需加载像素图，而不是使用缩放的像素图。实现可缩放矢量格式的自定义图标引擎可以自由地忽略任何额外的文件。

[virtual] void QIconEngine:: addPixmap (const & pixmap , mode , state )#

由QIcon::addPixmap()调用。添加一个专门的pixmap为给定mode和state。默认的基于像素图的引擎会存储任何提供的像素图，如果像素图的大小与请求的图标大小相匹配，它会使用它们而不是按比例的像素图。实现可伸缩矢量格式的自定义图标引擎可以自由地忽略任何额外的像素图。

[virtual] < > QIconEngine:: availableSizes ( mode = QIcon::Normal, state = QIcon::Off)#

返回特定mode和state 的引擎中包含的所有图像的尺寸。

[pure virtual] *QIconEngine:: clone () const#

重新实现此方法以返回此图标引擎的克隆。

[virtual] QIconEngine:: iconName ()#

返回用于创建引擎的名称，如果有的话。

[virtual, since 5.7] bool QIconEngine:: isNull ()#

如果这个图标引擎代表一个空的QIcon，返回true。

注意：这是一个辅助方法，实际工作由virtual_hook()方法完成，因此这个方法取决于对图标引擎的支持，可能不会在所有的图标引擎上工作。

该函数在 Qt 5.7 引入。

[virtual] QIconEngine:: key () const#

返回一个标识这个图标引擎的键。

[pure virtual] void QIconEngine:: paint ( * painter , const & rect , mode , state )#

使用给定的painter将具有所需的mode和state的图标绘制到矩形rect中。

[virtual] QIconEngine:: pixmap (const & size , mode , state )#

以像素图的形式返回所需的size ，mode ，和state的图标。默认实现是创建一个新的pixmap并调用paint()来填充它。

[virtual] bool QIconEngine:: read ( & in )#

从QDataStreamin读取图标引擎内容。如果内容被读取，返回true；否则返回false。

QIconEngine的默认实现总是返回false。

[virtual, since 5.9] QIconEngine:: scaledPixmap (const & size , mode , state , scale )#

返回一个给定的size、mode、state和scale的像素图。

scale参数通常等于显示器的设备像素比率。

注意：这是一个辅助方法，实际工作由virtual_hook()方法完成，因此这个方法取决于图标引擎的支持，可能不适合所有的图标引擎。

注意：有些引擎可能会将scale铸成一个整数。

该函数在 Qt 5.9 引入。

另请参阅 ScaledPixmapArgument .

[virtual] void QIconEngine:: virtual_hook ( int id , void * data )#

额外的方法允许在不增加新的虚拟方法的情况下扩展QIconEngine（并且不破坏二进制兼容性）。实际的动作和数据格式取决于id参数，实际上它是IconEngineHook枚举中的一个常数

另请参阅 IconEngineHook .

[virtual] bool QIconEngine:: write ( & out ) const#

将此引擎的内容写入QDataStreamout。返回 true 如果内容被写入；否则返回false 。

QIconEngine的默认实现总是返回false。
