详细描述#

QItemDelegate可以用来为基于QAbstractItemView子类的项目视图提供自定义显示功能和编辑部件。为此，使用一个代理允许显示和编辑机制被定制并独立于模型和视图而开发。

QItemDelegate类是模型/视图类且属于Qt的模型/视图框架。注意，QStyledItemDelegate已经接管了绘制Qt项视图的工作。我们推荐在创建新的委托时使用QStyledItemDelegate。

当在标准视图中显示来自自定义模型的项目时，通常只需确保模型为每个角色返回适当的数据即可，这些角色决定了项目在视图中的外观。Qt的标准视图所使用的默认委托使用这些角色信息来显示用户所期望的大多数常见形式的项目。然而，有时需要对项目的外观进行更多的控制，而不是默认委托所能提供的。

这个类提供了在视图中绘制项目数据和编辑项目模型数据的函数的默认实现。在QAbstractItemDelegate中定义的paint()和sizeHint()虚拟函数的默认实现被提供，以确保该委托实现了视图所期望的正确基本行为。你可以在子类中重新实现这些函数来定制项的外观。

当在项目视图中编辑数据时，QItemDelegate提供了一个编辑器部件，这是一个在编辑时被放置在视图顶部的部件。编辑器是用QItemEditorFactory创建的；QItemEditorFactory提供的默认静态实例被安装在所有项目委托上。你可以使用setItemEditorFactory()设置一个自定义工厂，或者使用QItemEditorFactory::setDefaultFactory()设置一个新的默认工厂。被编辑的是存储在具有Qt::EditRole的项目模型中的数据。

这里只重新实现了基于widget的委托的标准编辑功能。

  • createEditor ()返回用于改变模型中数据的widget，可以重新实现以定制编辑行为。
  • setEditorData()为小组件提供可操作的数据。
  • updateEditorGeometry()确保编辑器相对于项目视图的正确显示。
  • setModelData()返回更新的数据到模型中。

closeEditor()信号表明，用户已经完成了数据的编辑，编辑器部件可以被销毁。

子类化#

当子类化QItemDelegate以创建一个使用自定义渲染器显示项目的委托时，重要的是要确保该委托能够为所有需要的状态适当地渲染项目；例如，选中、禁用、选中。paint()函数的文档包含了一些提示，说明如何实现这一点。

你可以通过使用QItemEditorFactory来提供自定义编辑器。色彩编辑器工厂范例展示了如何用默认的项目编辑器工厂为委托人提供自定义编辑器。这样一来，就不需要对QItemDelegate进行子类化。另一种方法是重新实现createEditor(), setEditorData(), setModelData(), 和 updateEditorGeometry()。这个过程在自旋框委托范例中描述。

QStyledItemDelegate vs. QItemDelegate#

从Qt 4.4开始，有两个委托类。QItemDelegate和[QStyledItemDelegate]（qstyleditemdelegate.html）。然而，默认的委托是QStyledItemDelegate。这两个类是为视图中的项目绘制和提供编辑器的独立替代品。它们之间的区别是，QStyledItemDelegate使用当前的样式来绘制其项目。因此我们推荐在实现自定义委托或使用Qt样式表时使用QStyledItemDelegate作为基类。这两个类所需的代码应该是一样的，除非自定义委托需要使用样式来绘制。

另请参阅 委托类 , QStyledItemDelegate , QAbstractItemDelegate , 自旋框委托范例 , 设置编辑器范例 ，和 图标范例 .

[override virtual protected] bool QItemDelegate:: eventFilter ( * editor , * event )#

重实现： QObject::eventFilter (QObject *watched, QEvent *event).

返回 true 若给定 editor是一个有效的QWidget和给定 event被处理；否则返回false 。以下按键事件默认被处理。

  • Tab
  • Backtab
  • Enter
  • 返回
  • Esc

在Tab*、Backtab、**Enter和返回键事件的情况下，editor的数据被提交到模型中，编辑器被关闭。如果事件是Tab键，视图将在视图的下一个项目上打开一个编辑器。同样地，如果事件是Backtab*键，视图将在视图中的上一*项上打开一个编辑器。

如果该事件是一个Esc键的事件，那么editor就会被关闭，而不会提交其数据。

另请参阅 commitData () 和 closeEditor ().

[override virtual] void QItemDelegate:: paint ( * painter , const & option , const & index ) const#

重实现： QAbstractItemDelegate::paint (QPainter *painter, const QStyleOptionViewItem &option, const QModelIndex &index) const.

使用给定的painter和样式option为index指定的项目渲染委托。

当在子类中重新实现这个函数时，你应该更新选项的rect变量所持有的区域，使用选项的state变量来确定要显示的项目的状态，并相应地调整其绘制的方式。

例如，一个被选中的项目可能需要与未被选中的项目以不同的方式显示，如以下代码所示:

if (option.state & QStyle::State_Selected)
    painter->fillRect(option.rect, option.palette.highlight());
const int size = qMin(option.rect.width(), option.rect.height());
const int brightness = index.model()->data(index, Qt::DisplayRole).toInt();
const double radius = (size / 2.0) - (brightness / 255.0 * size / 2.0);
if (qFuzzyIsNull(radius))
    return;
painter->save();
painter->setRenderHint(QPainter::Antialiasing, true);
painter->setPen(Qt::NoPen);
if (option.state & QStyle::State_Selected)
    painter->setBrush(option.palette.highlightedText());
else
    ...

在绘画之后，你应该确保绘画器被返回到它在调用这个函数时的状态。例如，在绘画前调用 QPainter::save ()，在绘画后调用 QPainter::restore ()可能是有用的。

另请参阅 QStyle::State .
