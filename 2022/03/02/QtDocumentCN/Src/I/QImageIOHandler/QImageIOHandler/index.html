详细描述#

Qt使用QImageIOHandler来通过QImageReader和QImageWriter读写图像。你也可以使用Qt的插件机制，从这个类派生出你自己的图像格式处理器。

调用setDevice()来给处理程序分配一个设备，以及setFormat()来给它分配一个格式。一个QImageIOHandler可以支持一个以上的图像格式。canRead () 返回 true 如果可以从设备上读取图像，read () 和 write () 返回 true 如果读取或写入图像已经成功完成。

QImageIOHandler也支持动画格式，通过函数loopCount (), imageCount (), nextImageDelay () 和 currentImageNumber ()。

为了确定一个图像处理程序支持哪些选项，Qt将调用 supportedOption () 和 setOption ()。如果你能为ImageOption中的任何选项提供支持，请确保重新实现这些函数。

要编写你自己的图像处理程序，你至少要重新实现canRead() 和 read()。然后创建一个QImageIOPlugin，它可以创建处理程序。最后，安装你的插件，QImageReader和QImageWriter就会自动加载该插件，并开始使用它。

另请参阅 QImageIOPlugin , QImageReader ，和 QImageWriter .

成员类型文档编制#

enum QImageIOHandler:: ImageOption#

这个枚举描述了QImageIOHandler所支持的不同选项。一些选项用于查询图像的属性，而另一些选项则用于切换图像的写入方式。

常量 值 描述
QImageIOHandler::Size 0 一个图像的原始尺寸。支持这个选项的处理程序应该从图像元数据中读取图像的大小，并从option()中以QSize的形式返回这一大小。
QImageIOHandler::ClipRect 1 剪辑矩形，或ROI(Region Of Interest)。一个支持这个选项的处理程序被期望只在read()中从原始图像中读取所提供的QRect区域，然后再应用任何其他转换。
QImageIOHandler::ScaledSize 4 图像的比例大小。一个支持这个选项的处理程序在应用任何剪辑矩形变换（ClipRect）之后，会将图像缩放到所提供的尺寸（一个QSize）。如果处理程序不支持这个选项，QImageReader将在图像被读取后执行缩放。
QImageIOHandler::ScaledClipRect 3 缩放后的图像的剪辑矩形（或ROI，感兴趣的区域）。一个支持这个选项的处理程序被期望在应用任何缩放（ScaleSize）或常规剪裁（ClipRect）之后，应用所提供的剪辑矩形（一个QRect）。如果处理程序不支持这个选项，QImageReader将在图像被读取后应用缩放的剪辑矩形。
QImageIOHandler::Description 2 图像描述。一些图像格式，如GIF和PNG，允许在图像数据中嵌入文本或注释（例如，用于存储版权信息）。常见的是文本以键值对的形式存储，但有些格式将所有文本存储在一个连续的块中。QImageIOHandler将文本作为一个QString返回，其中键和值用’:’隔开，键值对用两个换行符(\n\n)分开。例如，”标题。Sunset\nAuthor: Jim Smith\nSarah Jonesn\n”。在单个区块中存储文本的格式可以使用 “描述 “作为键。
QImageIOHandler::CompressionRatio 5 图像数据的压缩率。支持该选项的处理程序在写入时应根据该选项的值（一个int）来设置其压缩率。
QImageIOHandler::Gamma 6 图像的伽马等级。支持该选项的处理程序在写入时应根据该选项的值（一个浮点数）设置图像的伽玛等级。
QImageIOHandler::Quality 7 图像的质量水平。支持该选项的处理程序在写入时应根据该选项的值（一个int）来设置图像质量等级。
QImageIOHandler::Name 8 图像的名称。支持这个选项的处理程序应该从图像元数据中读取名称，并以QString的形式返回，或者在写入图像时，它应该将名称存储在图像元数据中。
QImageIOHandler::SubType 9 图像的子类型。支持这个选项的处理程序可以在读写图像时使用子类型值来帮助。例如，一个PPM处理程序可以有一个 “ppm “或 “ppmraw “的子类型值。
QImageIOHandler::IncrementalReading 10 一个支持这个选项的处理程序被期望分几次读取图像，就像它是一个动画一样。QImageReader将把图像当作一个动画。
QImageIOHandler::Endianness 11 图像的字节数。某些图像格式可以被存储为BigEndian或LittleEndian。支持Endianness的处理程序使用这个选项的值来决定图像应该如何存储。
QImageIOHandler::Animation 12 支持动画的图像格式在supportsOption()中对该值返回真，否则返回假。
QImageIOHandler::BackgroundColor 13 某些图像格式允许指定背景颜色。支持BackgroundColor的处理程序在读取图像时将背景颜色初始化为这个选项（一个[QColor]（qcolor.html））。
QImageIOHandler::ImageFormat 14 处理程序返回的图像的数据格式。这可以是QImage::Format中所列的任何格式。
QImageIOHandler::SupportedSubTypes 15 支持不同保存变体的图像格式应该在这个选项中返回一个支持的变体名称列表（QList< QByteArray>）。
QImageIOHandler::OptimizedWrite 16 支持该选项的处理程序在写入时应打开优化标志。
QImageIOHandler::ProgressiveScanWrite 17 支持该选项的处理程序应将图像写成逐行扫描图像。
QImageIOHandler::ImageTransformation 18 支持该选项的处理程序可以读取图像的转换元数据。支持这个选项的处理程序不应该自己应用转换。

[since 5.5] enum QImageIOHandler:: Transformation flags QImageIOHandler:: 变换#

这个枚举描述了一些图像格式所支持的不同变换或方向，通常是通过EXIF。

常量 值 描述
QImageIOHandler::TransformationNone 0 不应进行转换。
QImageIOHandler::TransformationMirror 1 水平镜像图像。
QImageIOHandler::TransformationFlip 2 镜像图像的垂直方向。
QImageIOHandler::TransformationRotate180 `TransformationMirror TransformationFlip`
QImageIOHandler::TransformationRotate90 4 将图像旋转90度。
QImageIOHandler::TransformationMirrorAndRotate90 `TransformationMirror TransformationRotate90`
QImageIOHandler::TransformationFlipAndRotate90 `TransformationFlip TransformationRotate90`
QImageIOHandler::TransformationRotate270 `TransformationRotate180 TransformationRotate90`

该枚举在 Qt 5.5 引入或被修改。

Transformations类型是QFlags的一个类型定义。它存储了Transformation值的OR组合。

另请参阅 QImageReader::transformation (), QImageReader::setAutoTransform ()，和 QImageWriter::setTransformation ().

构造 QImageIOHandler 对象。

销毁 QImageIOHandler 对象。

[static, since 6.0] bool QImageIOHandler:: allocateImage ( size , format , * image )#

这是一个方便的方法，用于子类中的读取函数。如果需要的分配超出了当前的分配限制，图像格式处理程序必须拒绝加载图像。这个函数检查参数和限制，并在有效和需要的情况下进行分配。成功返回后，image将是一个有效的、分离的QImage为给定size和format。

该函数在 Qt 6.0 引入。

另请参阅 QImageReader::allocationLimit ().

[pure virtual] bool QImageIOHandler:: canRead () const#

返回 true 如果可以从设备上读取图像（即支持图像格式，可以从设备上读取，并且初始头信息表明可以读取图像）；否则返回 false 。

当重新实现canRead()时，确保I/O设备(device())处于原始状态(例如，通过使用peek()而不是read())。

另请参阅 read () 和 QIODevice::peek ().

[virtual] int QImageIOHandler:: currentImageNumber () const#

对于支持动画的图像格式，该函数返回动画中当前图像的序列号。如果在任何图像被读()之前调用此函数，将返回-1。序列中第一个图像的编号为0。

如果图像格式不支持动画，则返回0。

另请参阅 read ().

[virtual] QImageIOHandler:: currentImageRect () const#

返回当前图像的矩形。如果没有为图像定义矩形，将返回空的QRect()。

这个函数对动画很有用，因为在动画中，每次只有部分帧可以被更新。

*QImageIOHandler:: device () const#

返回当前分配给[QImageIOHandler]（）的设备。如果没有设备被分配，nullptr 被返回。

另请参阅 setDevice ().

QImageIOHandler:: format () const#

返回当前分配给QImageIOHandler的格式。如果没有分配格式，将返回一个空字符串。

另请参阅 setFormat ().

[virtual] int QImageIOHandler:: imageCount () const#

对于支持动画的图像格式，此函数返回动画中的图像数量。如果图像格式不支持动画，或者无法确定图像的数量，则返回0。

如果canRead()返回 true ；否则返回0.默认实现返回1。

[virtual] bool QImageIOHandler:: jumpToImage ( int imageNumber )#

对于支持动画的图像格式，该函数跳转到序列号为imageNumber的图像。下一次调用read()将试图读取这个图像。

默认实现不做任何事情，并返回 “false”。

[virtual] bool QImageIOHandler:: jumpToNextImage ()#

对于支持动画的图像格式，该函数会跳转到下一张图像。

默认实现不做任何事情，并返回 “false”。

[virtual] int QImageIOHandler:: loopCount () const#

对于支持动画的图像格式，该函数返回动画应该循环的次数。如果图像格式不支持动画，则返回0。

[virtual] int QImageIOHandler:: nextImageDelay () const#

对于支持动画的图像格式，该函数返回在读取下一幅图像之前要等待的毫秒数。如果图像格式不支持动画，则返回0。

[virtual] QImageIOHandler:: option ( option ) const#

返回分配给option的值作为[QVariant]（qvariant.html）。该值的类型取决于选项。例如，option(Size)返回一个QSize变体。

另请参阅 setOption () 和 supportsOption ().

[pure virtual] bool QImageIOHandler:: read ( * image )#

从设备中读取图像，并将其存储在image中。返回 true 如果图像被成功读取；否则返回false。

对于支持增量加载的图像格式，以及动画格式，图像处理程序可以假定image指向前一帧。

另请参阅 canRead ().

void QImageIOHandler:: setDevice ( * device )#

设置QImageIOHandler的设备为device。在读写图像时，图像处理程序将使用这个设备。

设备只能被设置一次，而且必须在调用canRead()、read()、write()等之前设置。如果你需要读取多个文件，请构建适当的QImageIOHandler子类的多个实例。

另请参阅 device ().

void QImageIOHandler:: setFormat (const & format )#

将QImageIOHandler的格式设置为format。该格式对支持多种图像格式的处理程序最有用。

另请参阅 format ().

void QImageIOHandler:: setFormat (const & format ) const#

将QImageIOHandler的格式设置为format。该格式对支持多种图像格式的处理程序最有用。

这个函数被声明为常数，所以它可以从canRead()调用。

另请参阅 format ().

[virtual] void QImageIOHandler:: setOption ( option , const & value )#

设置选项 option 采用值 value .

另请参阅 option () 和 ImageOption .

[virtual] bool QImageIOHandler:: supportsOption ( option ) const#

返回 true 若 QImageIOHandler 支持选项 option ；否则返回 false 。例如，若 QImageIOHandler 支持 Size 选项，supportsOption(Size) 必须返回 true。

另请参阅 setOption () 和 option ().

[virtual] bool QImageIOHandler:: write (const & image )#

写入图像 image 到赋值设备。返回 true 当成功时；否则返回 false .

默认实现什么都不做，且仅返回 false .
