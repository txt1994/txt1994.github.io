详细描述#

QHttpPart类持有一个在HTTP多部分MIME消息（由QHttpMultiPart类表示）中使用的主体部分。一个QHttpPart由一个头块和一个数据块组成，它们之间由两个连续的新行分开。一个部分的例子是：

Content-Type: text/plain
Content-Disposition: form-data; name="text"

here goes the body

对于设置标题，使用setHeader()和setRawHeader()，它们的行为与QNetworkRequest::setHeader()和QNetworkRequest::setRawHeader()完全一样。

对于读取小块的数据，使用setBody()；对于较大的数据块，如图像，使用setBodyDevice()。后一种方法不在内部复制数据，而是直接从设备上读取，从而节省了内存。这意味着在通过QNetworkAccessManager::post()在网络上发送包含身体部分的多部分消息时，该设备必须被打开并可读。

为了构造一个具有小体的QHttpPart，请考虑下面的片段（这产生了上面例子中所示的数据）：

QHttpPart textPart;
textPart.setHeader(QNetworkRequest::ContentTypeHeader, QVariant("text/plain"));
textPart.setHeader(QNetworkRequest::ContentDispositionHeader, QVariant("form-data; name=\"text\""));
textPart.setBody("here goes the body");

为了构造一个从设备（例如一个文件）读取的QHttpPart，可以应用以下内容：

QHttpPart imagePart;
imagePart.setHeader(QNetworkRequest::ContentTypeHeader, QVariant("image/jpeg"));
imagePart.setHeader(QNetworkRequest::ContentDispositionHeader, QVariant("form-data; name=\"image\""));
imagePart.setRawHeader("Content-ID", "my@content.id"); // add any headers you like via setRawHeader()
QFile *file = new QFile("image.jpg");
file->open(QIODevice::ReadOnly);
imagePart.setBodyDevice(file);

请注意，QHttpPart在被设置时并没有取得设备的所有权，所以当它不再被需要时，销毁它是开发者的责任。一个好的主意可能是将多部件消息设置为设备的父对象，正如QHttpMultiPart的文档所记载的那样。

See also QHttpMultiPart and QNetworkAccessManager.

void QHttpPart::setBody(const &body)#

将这个MIME部分的主体设置为body。除非通过setBodyDevice()设置设备，否则将使用用此方法设置的body。对于大量的数据（如图片），请使用setBodyDevice()，它将不会在内部复制数据。

See also setBodyDevice().

void QHttpPart::setBodyDevice( *device)#

把要读取内容的设备设置为device。对于大量的数据，这个方法应该比setBody()更受欢迎，因为使用这个方法时，内容不会被复制，而是直接从设备读取。device必须是开放的和可读的。QHttpPart并不占有device的所有权，也就是说，如果有必要的话，必须关闭并销毁该设备。如果device是连续的（例如，套接字，但不是文件），QNetworkAccessManager::post()应该在device发出 finished()之后被呼叫。对于通过setBody()取消设置设备和使用数据集，使用 “setBodyDevice(0)”。

See also setBody() and QNetworkAccessManager::post().

void QHttpPart::setHeader( header, const &value)#

将已知的头信息header的值设置为value，覆盖之前设置的任何头信息。

See also QNetworkRequest::KnownHeaders, setRawHeader(), and QNetworkRequest::setHeader().

void QHttpPart::setRawHeader(const &headerName, const &headerValue)#

设置头信息headerName的值为headerValue。如果headerName对应于一个已知的头（见QNetworkRequest::KnownHeaders），原始格式将被解析，相应的 “熟 “头也将被设置。

注意：两次设置同一个头会覆盖之前的设置。为了完成多个同名HTTP头的行为，你应该把两个值连接起来，用逗号（”,”）分开，然后设置一个单一的原始头。

See also QNetworkRequest::KnownHeaders, setHeader(), and QNetworkRequest::setRawHeader().

[since 5.0]void QHttpPart::swap( &other)#

将这个HTTP部分与other的部分交换。这个功能非常快，而且从未失败。

此函数被加入于 5.0.

bool QHttpPart::operator!=(const &other) const#

如果这个对象与other对象不一样，返回true。

See also operator==().

bool QHttpPart::operator==(const &other) const#

如果该对象与other对象相同，则返回 “true”（即，如果它们有相同的标题和正文）。

See also operator!=().
