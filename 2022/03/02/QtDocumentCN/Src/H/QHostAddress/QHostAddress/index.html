详细描述#

这个类以一种与平台和协议无关的方式持有一个IPv4或IPv6地址。

QHostAddress通常与QTcpSocket、QTcpServer和QUdpSocket一起使用，以连接到一个主机或建立一个服务器。

用setAddress()设置主机地址，用toIPv4Address()、toIPv6Address()或toString()检索。你可以用protocol()来检查该类型。

注意：请注意，QHostAddress不做DNS查询。为此需要QHostInfo。

该类也支持常见的预定义地址。Null, LocalHost, LocalHostIPv6, Broadcast, 以及 Any。

See also QHostInfo, QTcpSocket, QTcpServer, and QUdpSocket.

成员类型文件#

[since 5.8]enum QHostAddress::ConversionModeFlag flags QHostAddress::ConversionMode#

Constant Value Description
QHostAddress::StrictConversion 0 在比较两个不同协议的QHostAddress对象时，不要将IPv6地址转换为IPv4，因此它们将永远被认为是不同的。
QHostAddress::ConvertV4MappedToIPv4 1 在比较时转换IPv4映射的IPv6地址（RFC 4291第2.5.5.2节）。因此，QHostAddress(“::fff:192.168.1.1”)将与QHostAddress(“192.168.1.1”)比较，两者相等。
QHostAddress::ConvertV4CompatToIPv4 2 在比较时，转换与IPv4兼容的IPv6地址（RFC 4291第2.5.5.1节）。因此，QHostAddress(“:192.168.1.1”)将与QHostAddress(“192.168.1.1”)比较，两者相等。
QHostAddress::ConvertLocalHost 8 在比较时，将IPv6环回地址转换为其IPv4等价物。因此，例如，QHostAddress(“::1”)将与QHostAddress(“127.0.0.1”)进行比较。
QHostAddress::ConvertUnspecifiedAddress 4 所有未指定的地址将比较相等，即AnyIPv4、AnyIPv6 和Any。
QHostAddress::TolerantConversion 0xff 设置前面所有三个标志。

这个枚举是在Qt 5.8中引入或修改的。

ConversionMode类型是QFlags的一个类型定义。它存储了ConversionModeFlag值的OR组合。

See also isEqual().

enum QHostAddress::SpecialAddress#

Constant Value Description
QHostAddress::Null 0 0空的地址对象。相当于QHostAddress()。参见QHostAddress::isNull()。
QHostAddress::LocalHost 2 IPv4本地主机地址。相当于QHostAddress(“127.0.0.1”)。
QHostAddress::LocalHostIPv6 3 IPv6的本地主机地址。相当于QHostAddress(“::1”)。
QHostAddress::Broadcast 1 IPv4广播地址。相当于QHostAddress(“255.255.255.255”)。
QHostAddress::AnyIPv4 6 IPv4任意地址。相当于QHostAddress(“0.0.0.0”)。用这个地址绑定的套接字将只在IPv4接口上监听。
QHostAddress::AnyIPv6 5 IPv6的任意地址。相当于QHostAddress(“:“)。与此地址绑定的套接字将只在IPv6接口上监听。
QHostAddress::Any 4 双栈任意地址。用这个地址绑定的套接字将同时监听IPv4和IPv6接口。

[since 5.11]bool QHostAddress::isGlobal() const#

如果地址是IPv4或IPv6全局地址，返回 true，否则返回 false。全局地址是指没有为特殊目的（如环回或组播）或未来目的保留的地址。

请注意，IPv6唯一的本地单播地址被认为是全局地址（参见isUniqueLocalUnicast()），正如RFC 1918为本地网络保留的IPv4地址。

还要注意的是，IPv6站点-本地地址已被废弃，在新的应用中应被视为全局地址。这个函数对站点-本地地址也返回true。

这个函数是在Qt 5.11中引入的。

See also isLoopback(), isSiteLocal(), and isUniqueLocalUnicast().

bool QHostAddress::isInSubnet(const &subnet, int netmask) const#

如果这个IP在网络前缀subnet和网络掩码netmask所描述的子网中，返回true。

如果一个IP包含在子网中最低和最高的地址之间，那么它就被认为属于一个子网。在IP版本4的情况下，最低地址是网络地址，而最高地址是广播地址。

子网参数不一定是实际的网络地址（子网中的最低地址）。它可以是属于该子网的任何有效IP。特别是，如果它等于这个对象所持有的IP地址，这个函数将总是返回true（前提是netmask是一个有效的值）。

See also parseSubnet().

bool QHostAddress::isInSubnet(const QPair<, int> &subnet) const#

这是一个重载函数。

[since 5.11]bool QHostAddress::isSiteLocal() const#

如果地址是IPv6站点本地地址，返回 “true”，否则返回 “false”。

[since 5.11]bool QHostAddress::isUniqueLocalUnicast() const#

如果地址是一个IPv6唯一的本地单播地址，则返回 “true”，否则返回 “false”。

一个IPv6唯一本地单播地址是网络fc00::/7中的一个。更多信息请参见IANA IPv6地址空间注册表。

请注意，唯一本地单播地址也算作全球地址。RFC 4193说，在实践中，”应用程序可以把这些地址当作全局范围的地址”。只有路由器需要关心这种区别。

这个函数是在Qt 5.11中引入的。

See also isLoopback(), isGlobal(), isMulticast(), isLinkLocal(), and isUniqueLocalUnicast().

[static]QPair<, int> QHostAddress::parseSubnet(const &subnet)#

解析subnet中包含的IP和子网信息，并返回该网络的网络前缀和其前缀长度。

IP地址和网络掩码必须用斜线（/）分开。

这个函数支持以下形式的参数。

  • 123.123.123.123/n 其中n是0到32之间的任何数值
  • 123.123.123.123/255.255.255.255
  • /n 其中n是0到128之间的任何数值

对于IP版本4，该函数也接受缺失的尾部组件（即少于4个八位数，如 “192.168.1”），后面是否有一个点。如果在这种情况下，netmask也丢失了，它将被设置为实际传递的八位数（在上面的例子中，它将是24，为3个八位数）。

See also isInSubnet().

QHostAddress::scopeId() const#

返回一个IPv6地址的范围ID。对于IPv4地址，或者如果地址不包含范围ID，则返回一个空的QString。

IPv6范围ID指定了非全球IPv6地址的*可达范围，限制了该地址可使用的区域。所有的IPv6地址都与这样一个可达范围相关联。作用域ID被用来区分那些不能保证是全球唯一的地址。

IPv6规定了以下四个级别的可达性。

  • Node-local。仅用于与同一接口上的服务进行通信的地址（例如，回环接口”::1”）。
  • 链路本地。属于网络接口（链接）的本地地址。主机上的每个IPv6接口总是有一个链路本地地址。链路本地地址（”fe80…”）是由本地网络适配器的MAC地址生成的，并不保证是唯一的。
  • 全局。用于全球可路由的地址，如互联网上的公共服务器。

当使用链接本地或站点本地地址进行IPv6连接时，你必须指定范围ID。链路本地地址的范围ID通常与接口名称（如 “eth0”、”en1”）或数字（如 “1”、”2”）相同。

See also setScopeId(), QNetworkInterface, and QNetworkInterface::interfaceFromName.

QHostAddress::toIPv4Address(bool *ok = nullptr) const#

返回IPv4地址为一个数字。

例如，如果地址是127.0.0.1，返回值是2130706433（即0x7f000001）。

如果协议()是IPv4Protocol，或者协议是IPv6Protocol，并且IPv6地址是IPv4映射的地址(RFC4291)，这个值就有效。在这些情况下，ok将被设置为true。否则，它将被设置为false。

See also toString().

Q_IPV6ADDR QHostAddress::toIPv6Address() const#

返回IPv6地址为一个Q_IPV6ADDR结构。该结构由16个无符号字符组成。

Q_IPV6ADDR addr = hostAddr.toIPv6Address();
// addr contains 16 unsigned characters

for (int i = 0; i < 16; ++i) {
    // process addr[i]
}

如果协议()是IPv6Protocol，这个值就有效。如果协议是IPv4Protocol，那么返回的地址是一个IPv4映射的IPv6地址。(RFC4291)

See also toString().

QHostAddress::toString() const#

返回地址为一个字符串。

See also toIPv4Address().
