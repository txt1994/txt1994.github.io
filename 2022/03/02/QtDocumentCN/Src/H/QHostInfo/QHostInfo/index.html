详细描述#

QHostInfo查找与主机名相关的IP地址，或者与IP地址相关的主机名。该类提供了两个静态的便利函数：一个是异步工作的，一旦找到主机就会发出一个信号；另一个是阻塞并返回一个QHostInfo对象。

要异步查询主机的IP地址，可以调用lookupHost()，该函数将主机名或IP地址、接收器对象和插槽签名作为参数并返回一个ID。你可以通过调用abortHostLookup()和查找的ID来中止查找工作。

例子:

// To find the IP address of qt-project.org
QHostInfo::lookupHost("qt-project.org",
                      this, SLOT(printResults(QHostInfo)));

// To find the host name for 4.2.2.1
QHostInfo::lookupHost("4.2.2.1",
                      this, SLOT(printResults(QHostInfo)));

当结果准备好时，该槽被调用。结果存储在一个QHostInfo对象中。调用addresses()来获得主机的IP地址列表，调用hostName()来获得被查询的主机名称。

如果查询失败，error()返回发生错误的类型。errorString()给出查找错误的人可读描述。

如果你想要一个阻塞的查询，使用QHostInfo::fromName()函数:

QHostInfo info = QHostInfo::fromName("qt-project.org");

QHostInfo通过IDNA和Punycode标准支持国际化的域名（IDN）。

要检索本地主机的名称，请使用静态的 QHostInfo::localHostName() 函数。

QHostInfo使用操作系统提供的机制来执行查找。根据RFC 6724，不能保证为某个域或主机注册的所有IP地址都会被返回。

注意：从Qt 4.6.1开始，QHostInfo使用多个线程进行DNS查询，而不是一个专门的DNS线程。这提高了性能，但也改变了使用lookupHost()时的信号发射顺序，与以前的Qt版本相比。

注意：从Qt 4.6.3开始，QHostInfo使用一个小型的内部60秒DNS缓存来提高性能。

See also QAbstractSocket, RFC 3492, and RFC 6724.

enum QHostInfo::HostInfoError#

这个枚举描述了在试图解析主机名时可能发生的各种错误。

Constant Value Description
QHostInfo::NoError 0 查询是成功的。
QHostInfo::HostNotFound 1 没有发现该主机的IP地址。
QHostInfo::UnknownError 2 发生了一个未知的错误。

See also error() and setError().

[static] QHostInfo::fromName(const &name)#

查询给定的主机名称的IP地址（es）。在查询过程中，该函数被阻塞，这意味着程序的执行被暂停，直到查询的结果准备好。在QHostInfo对象中返回查询的结果。

如果你向name传递一个字面的IP地址而不是主机名，QHostInfo将搜索该IP的域名（即，QHostInfo将执行一个反向的查询）。在成功的情况下，返回的QHostInfo将包含该主机名称的解析域名和IP地址。

See also lookupHost().

QHostInfo::hostName() const#

返回其IP地址被查询的主机名称。

See also setHostName() and localHostName().

[static] QHostInfo::localDomainName()#

返回本机的DNS域名。

注意： DNS域名与Windows网络中的域名不相关。

See also hostName().

[static] QHostInfo::localHostName()#

如果配置了主机名，则返回这台机器的主机名。注意，不保证主机名是全局唯一的，特别是如果它们是自动配置的。

这个函数不保证返回的主机名是一个完全合格的域名（FQDN）。为此，请使用fromName()将返回的名称解析为一个FQDN。

这个函数的返回值与QSysInfo::machineHostName()相同。

See also hostName() and localDomainName().

[static]int QHostInfo::lookupHost(const &name, *receiver, const char *member)#

查询与主机名name相关的IP地址，并返回查询的ID。当查询的结果准备好时，receiver中的槽或信号*成员被调用，参数为QHostInfo。然后可以检查QHostInfo对象以获得查找的结果。

例如，查找是由一个单一的函数调用来完成的:

QHostInfo::lookupHost("www.kde.org",
                      this, SLOT(lookedUp(QHostInfo)));

槽的实现会打印出查找返回的地址的基本信息，如果失败则报告一个错误:

void MyWidget::lookedUp(const QHostInfo &host)
{
    if (host.error() != QHostInfo::NoError) {
        qDebug() << "Lookup failed:" << host.errorString();
        return;
    }

    const auto addresses = host.addresses();
    for (const QHostAddress &address : addresses)
        qDebug() << "Found address:" << address.toString();
}

如果你向name传递一个字面的IP地址，而不是主机名，QHostInfo将搜索该IP的域名（即，QHostInfo将执行一个反向的查询）。在成功的情况下，产生的QHostInfo将包含主机名称的解析域名和IP地址。例子:

QHostInfo::lookupHost("4.2.2.1",
                      this, SLOT(lookedUp(QHostInfo)));

注意：如果你用lookupHost()启动多个请求，则不能保证信号的发出顺序。

See also abortHostLookup(), addresses(), error(), and fromName().

[static, since 5.9]template int QHostInfo::lookupHost(const QString &name, Functor functor)#

这是一个重载函数。

查询与主机名name相关的IP地址，并返回一个查询的ID。当查询的结果准备好后，functor被调用，参数为QHostInfo。然后可以检查QHostInfo对象以获得查询的结果。

这个functor将在调用lookupHost的线程中运行；该线程必须有一个正在运行的Qt事件循环。

注意：如果你用lookupHost()启动多个请求，则不能保证信号的发射顺序。

这个函数是在Qt 5.9中引入的。

See also abortHostLookup(), addresses(), error(), and fromName().

[static, since 5.9]template int QHostInfo::lookupHost(const QString &name, const QObject *context, Functor functor)#

这是一个重载函数。

查找与主机名name相关的IP地址，并返回一个用于查找的ID。当查询的结果准备好后，functor被调用，参数为QHostInfo。然后可以检查QHostInfo对象以获得查找的结果。

如果context在查找完成之前被销毁，functor将不会被调用。该functor将在context的线程中运行。上下文的线程必须有一个正在运行的Qt事件循环。

下面是该函数的另一个签名：

lookupHost(const QString &name, const QObject *receiver, PointerToMemberFunction function)

在这种情况下，当查找的结果准备好后，receiver'中的槽或信号function’被调用，参数为QHostInfo。然后可以检查QHostInfo对象以获得查找的结果。

注意：如果你用lookupHost()启动多个请求，则不能保证信号发出的顺序。

这个函数是在Qt 5.9中引入的。

See also abortHostLookup(), addresses(), error(), and fromName().
