详细描述#

当鼠标光标移入、移出或在小组件内，以及小组件有 Qt::WA_Hover 属性时，就会发生鼠标事件。

函数pos()给出了当前的光标位置，而oldPos()给出了旧的鼠标位置。

事件QEvent::HoverEnter和QEvent::HoverLeave，与事件QEvent::Enter和QEvent::Leave之间有一些相似之处。然而，它们略有不同，因为我们在HoverEnter和HoverLeave的事件处理程序中做了更新()。

QEvent::HoverMove也与QEvent::MouseMove略有不同。让我们考虑一个顶级窗口A包含一个子窗口B，而子窗口B又包含一个子窗口C（都启用了鼠标跟踪）：

现在，如果你在A的中间把光标从上面移到下面，你会得到以下QEvent::MouseMove事件。

  1. A::MouseMove
  2. B::MouseMove
  3. C::MouseMove

你将得到与QEvent::HoverMove相同的事件，只是无论事件是否被接受，该事件总是传播到顶层。它只有在Qt::WA_NoMousePropagation属性下才会停止传播。

在这种情况下，事件将以如下方式发生。

  1. A::HoverMove
  2. A::HoverMove, B::HoverMove
  3. A::HoverMove, B::HoverMove, C::HoverMove

成员函数文档#

QHoverEvent::QHoverEvent( type, const &pos, const &oldPos, modifiers = Qt::NoModifier, const *device = QPointingDevice::primaryPointingDevice())#

构建一个源自设备的悬停事件对象。

参数type必须是QEvent::HoverEnter, QEvent::HoverLeave, 或者QEvent::HoverMove。

pos是当前鼠标指针相对于接收部件的位置，而oldPos是它之前的这种位置。modifiers持有事件发生时所有键盘修改器的状态。

QHoverEvent::oldPos() const#

返回鼠标指针之前的位置，相对于接收事件的部件。如果没有先前的位置，oldPos()将返回与pos()相同的位置。

在QEvent::HoverEnter事件中，这个位置将始终是QPoint(-1, -1)。

See also pos().

QHoverEvent::oldPosF() const#

返回鼠标指针之前的位置，相对于接收事件的部件。如果没有先前的位置，oldPosF()将返回与posF()相同的位置。

在QEvent::HoverEnter事件中，这个位置将始终是QPointF(-1, -1)。

See also posF().
