详细描述#

QHttpMultiPart类似于RFC 2046中描述的MIME多部件消息，它将通过HTTP发送。一个多部分消息由任意数量的主体部分组成（见QHttpPart），这些部分由一个独特的边界分开。QHttpMultiPart的边界是用字符串 “boundary_.oOo._ “构建的，后面是随机字符，并提供足够的唯一性，以确保它不会出现在部件本身的内部。如果需要，边界仍然可以通过setBoundary()来设置。

作为一个例子，考虑下面的代码片段，它构建了一个包含文本部分和图像部分的多部分信息：

QHttpMultiPart *multiPart = new QHttpMultiPart(QHttpMultiPart::FormDataType);

QHttpPart textPart;
textPart.setHeader(QNetworkRequest::ContentDispositionHeader, QVariant("form-data; name=\"text\""));
textPart.setBody("my text");

QHttpPart imagePart;
imagePart.setHeader(QNetworkRequest::ContentTypeHeader, QVariant("image/jpeg"));
imagePart.setHeader(QNetworkRequest::ContentDispositionHeader, QVariant("form-data; name=\"image\""));
QFile *file = new QFile("image.jpg");
file->open(QIODevice::ReadOnly);
imagePart.setBodyDevice(file);
file->setParent(multiPart); // we cannot delete the file now, so delete it with the multiPart

multiPart->append(textPart);
multiPart->append(imagePart);

QUrl url("http://my.server.tld");
QNetworkRequest request(url);

QNetworkAccessManager manager;
QNetworkReply *reply = manager.post(request, multiPart);
multiPart->setParent(reply); // delete the multiPart with the reply
// here connect signals etc.

See also QHttpPart and QNetworkAccessManager::post().

成员类型文档#

enum QHttpMultiPart::ContentType#

RFC 2046和其他文件中描述的多部分子类型的已知内容类型列表。

Constant Value Description
QHttpMultiPart::MixedType 0 对应于 “multipart/mixed “子类型，意味着主体部分是相互独立的，如RFC 2046中所述。
QHttpMultiPart::RelatedType 1 对应于 “multipart/related “子类型，意味着主体部分是相互关联的，如RFC 2387中所述。
QHttpMultiPart::FormDataType 2 对应于 “multipart/form-data “子类型，意味着主体部分包含表单元素，如RFC 2388中所述。
QHttpMultiPart::AlternativeType 3 对应于 “multipart/alternative “子类型，意思是主体部分是相同信息的替代表示，如RFC 2046中所述。

See also setContentType().

void QHttpMultiPart::setBoundary(const &boundary)#

将边界设置为boundary。

通常情况下，你不需要自己生成一个边界；在构造时，边界是以字符串 “boundary_.oOo._ “启动的，后面是随机字符，并提供足够的唯一性以确保它不会出现在零件本身里面。

See also boundary().

void QHttpMultiPart::setContentType( contentType)#

设置内容类型为contentType。当通过QNetworkAccessManager::post()发送多部分信息时，该内容类型将被用于HTTP头部分。如果你想使用不包含在QHttpMultiPart::ContentType中的多部件子类型，你可以手工将 “Content-Type “头域添加到QNetworkRequest中，然后将此请求与多部件消息一起用于发布。

See also QHttpMultiPart::ContentType and QNetworkAccessManager::post().
