详细描述#

QHttp2Configuration控制HTTP/2的参数和设置，QNetworkAccessManager将用于在HTTP/2协议启用时发送请求和处理响应。

QHttp2Configuration目前支持的HTTP/2参数包括。

  • 用于连接级流量控制的会话窗口大小。将在需要时作为流上的’WINDOW_UPDATE’帧发送至远程对等体，标识符为0。
  • 用于流级流量控制的流接收窗口大小。作为’SETTINGS_INITIAL_WINDOW_SIZE’参数在初始SETTINGS帧中发送，当需要时，’WINDOW_UPDATE’帧将在QNetworkAccessManager打开的流上发送。
  • 最大的帧大小。这个参数限制了来自远程对等体的帧的最大有效载荷。由QNetworkAccessManager作为’SETTINGS_MAX_FRAME_SIZE’参数在初始’SETTINGS’帧中发送。
  • 服务器推送。允许启用或禁用服务器推送。作为 “SETTINGS_ENABLE_PUSH “参数在最初的 “SETTINGS “帧中发送。

QHttp2Configuration类还控制头压缩算法（HPACK）是否另外使用Huffman编码进行字符串压缩。

注意：配置必须在向给定的主机发送第一个请求（从而建立一个HTTP/2会话）之前设置。

**注意:**关于流量控制、服务器推送和 “SETTINGS “的细节可以在RFC 7540中找到。HPACK压缩算法的不同模式和参数在RFC 7541中描述。

See also QNetworkRequest::setHttp2Configuration(), QNetworkRequest::http2Configuration(), and QNetworkAccessManager.

QHttp2Configuration::QHttp2Configuration( &&other)#

复制-构造这个QHttp2Configuration。

QHttp2Configuration::QHttp2Configuration()#

默认构建一个QHttp2Configuration对象。

这样的配置有以下值。

  • 服务器推送被禁用
  • 哈夫曼字符串压缩被启用
  • 连接级流量控制的窗口大小为65535个字节
  • 流级流量控制的窗口大小是65535个八位字节
  • 帧大小为16384个八位字节

bool QHttp2Configuration::huffmanCompressionEnabled() const#

如果HPACK中的Huffman编码被启用，返回true。

See also setHuffmanCompressionEnabled.

unsigned int QHttp2Configuration::maxFrameSize() const#

返回HTTP/2帧可以有的最大有效载荷大小。默认（初始）值是16384个字节。

See also setMaxFrameSize().

bool QHttp2Configuration::serverPushEnabled() const#

如果服务器推送被启用，返回true。

注意：默认情况下，QNetworkAccessManager通过 “SETTINGS “框架禁用服务器推送。

See also setServerPushEnabled.

unsigned int QHttp2Configuration::sessionReceiveWindowSize() const#

返回连接级流量控制的窗口大小。QNetworkAccessManager将使用的默认值是2147483647字节。

See also setSessionReceiveWindowSize().

void QHttp2Configuration::setHuffmanCompressionEnabled(bool enable)#

如果enable为 “true”，HPACK压缩将额外使用Huffman编码压缩字符串。默认情况下是启用的。

注意：这个参数只影响QNetworkAccessManager正在发送的’HEADERS’帧。

See also huffmanCompressionEnabled.

bool QHttp2Configuration::setMaxFrameSize(unsigned int size)#

设置QNetworkAccessManager在发送其初始SETTINGS帧时将向服务器公布的最大帧尺寸。

注意：虽然这个尺寸被要求在16384和16777215之间的范围内，但携带有效载荷的帧中的实际有效载荷尺寸可能小于16384。

成功时返回 “true”，否则返回 “false”。

See also maxFrameSize().

void QHttp2Configuration::setServerPushEnabled(bool enable)#

如果enable为 “true”，远程服务器就有可能使用服务器推送来提前发送响应。

See also serverPushEnabled.

bool QHttp2Configuration::setSessionReceiveWindowSize(unsigned int size)#

设置连接级流量控制的窗口大小。size不能为0，不能超过2147483647个字节。

成功时返回 “true”，否则返回 “false”。

See also sessionReceiveWindowSize.

bool QHttp2Configuration::setStreamReceiveWindowSize(unsigned int size)#

设置流级流量控制的窗口大小。size不能为0，不能超过2147483647个字节。

成功时返回 “true”，否则返回 “false”。

See also streamReceiveWindowSize.

unsigned int QHttp2Configuration::streamReceiveWindowSize() const#

返回流级流量控制的窗口大小。QNetworkAccessManager将使用的默认值是65535个字节（见RFC 7540）。

See also setStreamReceiveWindowSize().

void QHttp2Configuration::swap( &other)#

将此配置与other配置进行交换。
