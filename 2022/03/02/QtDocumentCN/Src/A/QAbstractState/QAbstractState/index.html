详细说明#

QAbstractState 类是状态的抽象基类，是 QStateMachine 的一部分。 它定义了所有状态对象共有的接口。 QAbstractState 是 Qt 状态机框架 的一部分。

进入状态时会发出 entered() 信号。 退出状态时会发出 exited() 信号。

parentState() 函数返回状态的父状态。 machine() 函数返回状态所属的状态机。

Subclassing#

进入状态时调用onEntry()函数； 重新实现此函数以在进入状态时执行自定义处理。

退出状态时调用onExit()函数； 重新实现此函数以在状态退出时执行自定义处理。

属性文档#

[bindable read-only, since 5.4]active : bool#

注意： 此属性支持 QProperty 绑定。

此属性持有此状态的活动属性。 状态在 进入() 和 退出 之间处于活动状态 qabstractstate.html#exited)() 信号。

这个属性是在 Qt 5.4 中引入的。

[signal, since 5.4]void QAbstractState::activeChanged(bool active)#

当使用 active 作为参数更改 active 属性时会发出此信号。

这个函数是在 Qt 5.4 中引入的。

See also QAbstractState::active, entered(), and exited().

[private signal]void QAbstractState::entered()#

进入状态时会发出此信号（在调用 onEntry() 之后）。

注意：这是一个私人信号。 它可以用于信号连接，但不能由用户发出。

[private signal]void QAbstractState::exited()#

此信号在状态退出时发出（在调用 onExit() 之后）。

注意：这是一个私人信号。 它可以用于信号连接，但不能由用户发出。

破坏这个状态。

bool QAbstractState::active() const#

返回此状态是否处于活动状态。

注意： 属性活动的 Getter 函数。

[override virtual protected]bool QAbstractState::event( *e)#

*QAbstractState::machine() const#

返回此状态所属的状态机，如果状态不属于状态机，则返回“nullptr”。

[pure virtual protected]void QAbstractState::onEntry( *event)#

进入状态时调用此函数。 给定的 event 是导致进入状态的原因。 重新实现该函数以在进入状态时进行自定义处理。

[pure virtual protected]void QAbstractState::onExit( *event)#

退出状态时调用此函数。 给定的 event 是导致状态退出的原因。 重新实现此函数以在状态退出时执行自定义处理。

*QAbstractState::parentState() const#

返回此状态的父状态，如果该状态没有父状态，则返回“nullptr”。
