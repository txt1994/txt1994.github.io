详细说明#

在应用程序中，可以通过菜单、工具栏按钮和键盘快捷键调用许多常用命令。由于用户希望每个命令都以相同的方式执行，因此无论使用何种用户界面，将每个命令表示为 action 是很有用的。

操作可以添加到菜单和工具栏，并自动保持同步。例如，在文字处理器中，如果用户按下粗体工具栏按钮，将自动选中粗体菜单项。

QAction 可能包含图标、菜单文本、快捷方式、状态文本、“这是什么？”文本和工具提示。 其中大部分可以在构造函数中设置。它们也可以使用 setIcon(), setText(), setIconText(), setShortcut(), setStatusTip(), setWhatsThis()和 setToolTip()。对于菜单项，可以使用 setFont() 设置单个字体。

我们建议将动作创建为使用它们的窗口的子级。在大多数情况下，动作将是应用程序主窗口的子级。

QAction in widget applications#

创建 QAction 后，应将其添加到相关菜单和工具栏，然后连接到将执行操作的插槽。

例如：

const QIcon openIcon = QIcon::fromTheme("document-open", QIcon(":/images/open.png"));
QAction *openAct = new QAction(openIcon, tr("&Open..."), this);
openAct->setShortcuts(QKeySequence::Open);
openAct->setStatusTip(tr("Open an existing file"));
connect(openAct, &QAction::triggered, this, &MainWindow::open);
fileMenu->addAction(openAct);
fileToolBar->addAction(openAct);

使用 QWidget::addAction() 或 QGraphicsWidget::addAction 将操作添加到小部件.io/qt-6/qgraphicswidget.html#addAction)()。请注意，必须先将操作添加到小部件，然后才能使用它。当快捷方式应该是全局的（即 Qt::ApplicationShortcut 作为 Qt::ShortcutContext。

动作可以创建为独立的对象。但它们也可能在菜单构建过程中创建。 QMenu 类包含用于创建适合用作菜单项的操作的便利函数。

See also QMenu, QToolBar, and Qt Widgets - Application Example.

enum QAction::MenuRole#

此枚举描述了如何将操作移动到 macOS 上的应用程序菜单中。

Constant Value Description
QAction::NoRole 0 此操作不应放入应用程序菜单
QAction::TextHeuristicRole 1 此操作应根据 QMenuBar 文档中所述的操作文本放置在应用程序菜单中。
QAction::ApplicationSpecificRole 2 此操作应放在具有应用程序特定角色的应用程序菜单中
QAction::AboutQtRole 3 此操作处理“关于 Qt”菜单项。
QAction::AboutRole 4 此操作应放置在应用程序菜单中“关于”菜单项的位置。菜单项的文本将设置为“关于<应用程序名称>”。应用程序名称是从应用程序包中的 Info.plist 文件中获取的（参见 Qt for macOS - Deployment）。
QAction::PreferencesRole 5 此操作应放置在应用程序菜单中“首选项…”菜单项的位置。
QAction::QuitRole 6 此操作应放置在应用程序菜单中退出菜单项的位置。

设置此值仅对菜单栏的即时菜单中的项目有效，对这些菜单的子菜单无效。例如，如果您的菜单栏中有 File 菜单并且 File 菜单有一个子菜单，则为该子菜单中的操作设置 MenuRole 无效。他们永远不会被移动。

enum QAction::Priority#

此枚举定义用户界面中操作的优先级。

Constant Value Description
QAction::LowPriority 0 The action should not be prioritized in the user interface.
QAction::NormalPriority 128
QAction::HighPriority 256 The action should be prioritized in the user interface.

See also priority.

可检查的 : bool#

该属性保存动作是否为可检查动作

可检查动作是具有开/关状态的动作。 例如，在文字处理器中，粗体工具栏按钮可以打开或关闭。 不是切换动作的动作是命令动作； 简单地执行命令动作，例如 文件保存。 默认情况下，此属性为 false。

在某些情况下，一个切换操作的状态应该取决于其他切换操作的状态。 例如，“左对齐”、“居中”和“右对齐”切换操作是互斥的。 要实现独占切换，请将相关切换操作添加到 QActionGroup 并将 QActionGroup::exclusive 属性设置为 true。

enabled : bool#

该属性保存操作是否启用

用户不能选择禁用的操作。 它们不会从菜单或工具栏中消失，但它们的显示方式表明它们不可用。 例如，它们可能仅使用灰色阴影显示。

这是什么？ 对禁用操作的帮助仍然可用，前提是设置了 QAction::whatsThis 属性 .

当添加到的所有小部件（使用 QWidget::addAction()）被禁用或不可见时，该操作将被禁用 . 当一个动作被禁用时，不可能通过它的快捷方式来触发它。

默认情况下，此属性为 true（启用操作）。

iconVisibleInMenu : bool#

此属性保存操作是否应在菜单中显示图标

在某些应用程序中，在工具栏中使用带有图标的操作可能很有意义，但在菜单中却没有。 如果为真，则图标（如果有效）显示在菜单中，如果为假，则不显示。

默认遵循是否为应用程序设置了 Qt::AA_DontShowIconsInMenus 属性。 显式设置此属性会覆盖属性的存在（或不存在）。

例如：

QApplication app(argc, argv);
app.setAttribute(Qt::AA_DontShowIconsInMenus);  // Icons are *no longer shown* in menus
// ...
QAction *myAction = new QAction();
// ...
myAction->setIcon(SomeIcon);
myAction->setIconVisibleInMenu(true);   // Icon *will* be shown in menus for *this* action.

成员函数文档#

QAction::QAction(const &icon, const &text, *parent = nullptr)#

用 icon 和一些 text 和 parent 构造一个动作。 如果 parent 是一个动作组，该动作将自动插入到组中。

该操作使用 text 的剥离版本（例如，“&Menu Option…”变为“Menu Option”）作为工具按钮的描述性文本。 您可以通过使用 setText() 设置特定描述来覆盖它。 除非您使用 setToolTip() 指定不同的文本，否则工具提示将使用相同的文本。

QAction::QAction(const &text, *parent = nullptr)#

用一些 text 和 parent 构造一个动作。 如果 parent 是一个动作组，该动作将自动插入到组中。

[slot]void QAction::setDisabled(bool b)#

这是 enabled 属性的便利功能，对信号–插槽连接很有用。 如果 b 为真，则禁用该操作； 否则启用。

[slot]void QAction::toggle()#

这是 checked 属性的便利功能。连接到它以将选中状态更改为相反的状态。

[signal]void QAction::toggled(bool checked)#

每当可检查操作更改其 isChecked() 状态时，都会发出此信号。这可能是用户交互的结果，也可能是因为调用了 setChecked()。由于 setChecked() 更改了 [QAction](https://doc.qt.io/qt-6/qaction .html)，它发出 changed() 和 toggled()。

checked 如果动作被选中则为真，如果动作未被选中则为假。

Note: 通知信号 for property checked.

See also activate(), triggered(), and checked.

[slot]void QAction::trigger()#

这是一个调用activate(Trigger)的便利槽。

[signal]void QAction::triggered(bool checked = false)#

当用户激活操作时发出此信号；例如，当用户单击菜单选项、工具栏按钮或按下操作的快捷键组合时，或者当 trigger()被称为。值得注意的是，当 setChecked() 或 [toggle](https://doc.qt. io/qt-6/qaction.html#toggle)() 被调用。

如果动作是可检查的，如果动作被选中，checked 为真，如果动作未被选中，则为假。

See also activate(), toggled(), and checked.

销毁对象并释放分配的资源。

void QAction::setSeparator(bool b)#

如果 b 为真，则此操作将被视为分隔符。

分隔符的表示方式取决于插入它的小部件。在大多数情况下，分隔符操作将忽略文本、子菜单和图标。

根据 key 设置与平台相关的快捷键列表。调用此函数的结果将取决于当前运行的平台。请注意，此操作可以分配多个快捷方式。如果只需要主快捷方式，请改用 setShortcut。
