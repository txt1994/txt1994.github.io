详细描述#

QAbstractItemView 类是每个使用 QAbstractItemModel 的标准视图的基类。 QAbstractItemView 是一个抽象类，本身不能被实例化。 它提供了一个标准接口，用于通过信号和槽机制与模型进行互操作，使子类能够随着模型的更改而保持最新。 此类为键盘和鼠标导航、视口滚动、项目编辑和选择提供标准支持。 键盘导航实现了这个功能:

Keys Functionality
Arrow keys 更改当前项目并选择它。
Ctrl+Arrow keys 改变当前项目，但不选择它.
Shift+Arrow keys 改变当前项目并选择它。之前选择的项目不会被取消选择.
Ctrl+Space 切换对当前项目的选择.
Tab/Backtab 将当前项目改为下一个/上一个项目.
Home/End 选择模型中的第一个/最后一个项目。
Page up/Page down 按视图中可见行数向上/向下滚动显示的行。
Ctrl+A Selects all items in the model.

请注意，上表假设选择模式允许这些操作。例如，如果选择模式是QAbstractItemView::NoSelection，你就不能选择项目。

QAbstractItemView 类是 模型/视图类 之一，并且是 Qt 的 [模型/视图框架](../.. /M/Model_View_Programming/Model_View_Programming.md)。

继承QAbstractItemView的视图类只需要实现自己的视图特有的功能，比如绘制item、返回item的几何形状、查找item等。

QAbstractItemView 提供了常用的槽，例如 edit() 和 setCurrentIndex()。还提供了许多受保护的插槽，包括 dataChanged()、[rowsInserted](#virtual- protected-slotvoid-qabstractitemviewrowsinsertedconst-qmodelindex-parent-int-start-int-end)(), rowsAboutToBeRemoved(), selectionChanged()和currentChanged()。

根项由 rootIndex() 返回，当前项由 currentIndex() 返回。要确保项目可见，请使用 scrollTo()。

QAbstractItemView 的一些函数与滚动有关，例如 setHorizontalScrollMode() 和 setVerticalScrollMode()。要设置滚动条的范围，例如，您可以重新实现视图的 resizeEvent() 函数:

void MyView::resizeEvent(QResizeEvent *event) {
    horizontalScrollBar()->setRange(0, realWidth - width());
    ...
}

请注意，在显示小部件之前，范围不会更新。

其他几个功能与选择控制有关；例如 setSelectionMode() 和 setSelectionBehavior()。此类提供了一个默认选择模型来使用 (selectionModel())，但这可以通过使用 setSelectionModel() 的实例来替换QItemSelectionModel。

要完全控制项目的显示和编辑，您可以使用 setItemDelegate() 指定委托。

QAbstractItemView 提供了很多受保护的功能。一些与编辑有关，例如 edit() 和 commitData()，而另一些则是键盘和鼠标事件处理程序。

注意：如果继承 QAbstractItemView 并打算更新视口的内容，则应使用 viewport->update() 而不是 [update()](../../W /QWidget/QWidget.md#slotvoid-qwidgetupdate），因为所有的绘画操作都发生在视口上。

另请参阅 View Classes, Model/View Programming, QAbstractItemModel, and Chart Example.

enum QAbstractItemView::DragDropMode#

描述视图可以操作的各种拖放事件。 默认情况下，视图不支持拖放（NoDragDrop）。

Constant Value Description
QAbstractItemView::NoDragDrop 0 不支持拖放。
QAbstractItemView::DragOnly 1 视图支持拖动自己的项目
QAbstractItemView::DropOnly 2 视图接受 drop
QAbstractItemView::DragDrop 3 视图支持拖放
QAbstractItemView::InternalMove 4 视图仅接受来自其自身的移动（不是复制）操作。

请注意，使用的模型需要提供对拖放操作的支持。

这个枚举是在 Qt 4.2 中引入或修改的。

另请参阅 setDragDropMode() and Using drag and drop with item views.

enum QAbstractItemView::DropIndicatorPosition#

此枚举指示放置指示器相对于当前鼠标位置处的索引的位置:

Constant Value Description
QAbstractItemView::OnItem 0 该项目将被丢弃在索引上。
QAbstractItemView::AboveItem 1 该项目将被丢到索引之上。
QAbstractItemView::BelowItem 2 该项目将被丢到索引下面。
QAbstractItemView::OnViewport 3 该项目将被投放到视口中没有项目的区域。每个视图处理掉落到视口上的项目的方式取决于使用中的底层模型的行为。

enum QAbstractItemView::EditTrigger flags QAbstractItemView::EditTriggers#

这个枚举描述了将启动项目编辑的行动。

Constant Value Description
QAbstractItemView::NoEditTriggers 0 不可能进行编辑。
QAbstractItemView::CurrentChanged 1 只要当前项目发生变化，就开始编辑。
QAbstractItemView::DoubleClicked 2 当一个项目被双击时，编辑工作就开始了。
QAbstractItemView::SelectedClicked 4 当点击一个已经选定的项目时，编辑工作就开始了。
QAbstractItemView::EditKeyPressed 8 当在一个项目上按下平台编辑键时，编辑工作就开始了。
QAbstractItemView::AnyKeyPressed 16 当在一个项目上按下任何一个键时，编辑工作就开始了。
QAbstractItemView::AllEditTriggers 31 上述所有行动的编辑工作开始。

EditTriggers类型是QFlags的类型定义。它存储了EditTrigger值的OR组合。

enum QAbstractItemView::ScrollHint#

Constant Value Description
QAbstractItemView::EnsureVisible 0 滚动以确保该项目是可见的。
QAbstractItemView::PositionAtTop 1 滚动到视口顶部的项目位置。
QAbstractItemView::PositionAtBottom 2 Scroll to position the item at the bottom of the viewport.
QAbstractItemView::PositionAtCenter 3 Scroll to position the item at the center of the viewport.

enum QAbstractItemView::ScrollMode#

描述了滚动条应该如何表现。当设置滚动模式为ScrollPerPixel时，单步尺寸会自动调整，除非是用setSingleStep()明确设置。自动调整可以通过设置单步大小为-1来恢复。

Constant Value Description
QAbstractItemView::ScrollPerItem 0 视图将逐项滚动显示内容。
QAbstractItemView::ScrollPerPixel 1 视图将一次滚动一个像素的内容。

这个枚举是在 Qt 4.2 中引入或修改的。

enum QAbstractItemView::SelectionBehavior#

Constant Value Description
QAbstractItemView::SelectItems 0 选择单一项目。
QAbstractItemView::SelectRows 1 仅选择行。
QAbstractItemView::SelectColumns 2 仅选择列。

enum QAbstractItemView::SelectionMode#

This enum indicates how the view responds to user selections:

Constant Value Description
QAbstractItemView::SingleSelection 1 当用户选择一个项目时，任何已选择的项目都将变为未选择。用户可以在单击所选项目时通过按 Ctrl 键取消选择所选项目。
QAbstractItemView::ContiguousSelection 4 当用户以通常的方式选择一个项目时，选择被清除并选择新项目。但是，如果用户在单击项目时按下 Shift 键，则当前项目和单击项目之间的所有项目都被选中或取消选中，具体取决于单击项目的状态。
QAbstractItemView::ExtendedSelection 3 当用户以通常的方式选择一个项目时，选择被清除并选择新项目。但是，如果用户在单击某个项目时按下 Ctrl 键，则单击的项目将被切换，而所有其他项目都保持不变。如果用户在单击项目时按下 Shift 键，则当前项目和单击项目之间的所有项目都被选中或取消选中，具体取决于单击项目的状态。可以通过将鼠标拖动到多个项目上来选择它们。
QAbstractItemView::MultiSelection 2 当用户以通常的方式选择一个项目时，该项目的选择状态将被切换，而其他项目则保持不变。可以通过将鼠标拖动到多个项目上来切换它们。
QAbstractItemView::NoSelection 0 无法选择项目。

最常用的模式是SingleSelection和ExtendedSelection。

enum QAbstractItemView::State#

描述视图可以处于的不同状态。这通常仅在重新实现您自己的视图时才有意义。

Constant Value Description
QAbstractItemView::NoState 0 是默认状态。
QAbstractItemView::DraggingState 1 用户正在拖动项目。
QAbstractItemView::DragSelectingState 2 用户正在选择项目。
QAbstractItemView::EditingState 3 用户正在小部件编辑器中编辑项目。
QAbstractItemView::ExpandingState 4 用户正在打开项目的一个分支。
QAbstractItemView::CollapsingState 5 用户正在关闭项目分支。

void QAbstractItemView::setIndexWidget(const &index, *widget)#

在给定 index 的项目上设置给定的 widget，将小部件的所有权传递给视口。

如果 index 无效（例如，如果您传递根索引），此函数将不执行任何操作。

给定的 widget 的 autoFillBackground 属性必须设置为 true，否则小部件的背景将是透明的，同时显示模型数据和位于给定索引。

如果索引小部件 A 被索引小部件 B 替换，则索引小部件 A 将被删除。例如，在下面的代码片段中，QLineEdit 对象将被删除。

setIndexWidget(index, new QLineEdit);
...
setIndexWidget(index, new QTextEdit);

此功能仅应用于显示与数据项对应的可见区域内的静态内容。如果要显示自定义动态内容或实现自定义编辑器小部件，请改为子类 QStyledItemDelegate。

这个函数是在 Qt 4.1 中引入的。

另请参阅 indexWidget() and Delegate Classes.

void QAbstractItemView::setItemDelegate( *delegate)#

将此视图及其模型的项目委托设置为 delegate。如果您想完全控制项目的编辑和显示，这很有用。

任何现有的委托都将被删除，但不会被删除。 QAbstractItemView 不取得 delegate 的所有权。

警告： 你不应该在视图之间共享同一个委托实例。这样做可能会导致不正确或不直观的编辑行为，因为连接到给定委托的每个视图都可能收到 [closeEditor()](../../A/QAbstractItemDelegate/QAbstractItemDelegate.md#signalvoid-qabstractitemdelegatecloseeditorqwidget-editor-qabstractitemdelegateendedithint-hint- -nohint) 信号，并尝试访问、修改或关闭已关闭的编辑器。

另请参阅 itemDelegate().

void QAbstractItemView::setItemDelegateForColumn(int column, *delegate)#

设置此视图和模型使用的给定项目 delegate 用于给定 column。 column 上的所有项目都将由 delegate 绘制和管理，而不是使用默认委托（即 itemDelegate()）。

column 的任何现有列委托都将被删除，但不会被删除。 QAbstractItemView 不取得 delegate 的所有权。

注意： 如果委托已被分配给行和列，行委托将优先并管理相交的单元格索引。

警告： 你不应该在视图之间共享同一个委托实例。这样做可能会导致不正确或不直观的编辑行为，因为连接到给定委托的每个视图都可能收到 [closeEditor()](../../A/QAbstractItemDelegate/QAbstractItemDelegate.md#signalvoid-qabstractitemdelegatecloseeditorqwidget-editor-qabstractitemdelegateendedithint-hint- -nohint) 信号，并尝试访问、修改或关闭已关闭的编辑器。

void QAbstractItemView::setItemDelegateForRow(int row, *delegate)#

[virtual]void QAbstractItemView::setModel( *model)#

设置要显示的视图的 model。

此函数将创建并设置一个新的选择模型，替换之前使用 setSelectionModel() 设置的任何模型。但是，旧的选择模型不会被删除，因为它可能在多个视图之间共享。如果不再需要旧的选择模型，我们建议您删除它。这是通过以下代码完成的：

QItemSelectionModel *m = view->selectionModel();
view->setModel(new model);
delete m;

如果旧模型和旧选择模型都没有父对象，或者它们的父对象是长寿命对象，则最好调用它们的 [deleteLater](../../O/QObject/QObject.md# slotvoid-qobjectdeletelater)() 函数显式删除它们。

除非它是模型的父对象，否则视图不拥有模型的所有权，因为模型可以在许多不同的视图之间共享。

另请参阅 model(), selectionModel(), and setSelectionModel().
