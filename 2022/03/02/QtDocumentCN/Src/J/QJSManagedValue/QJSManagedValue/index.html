详细描述#

QJSManagedValue类允许与JavaScript值进行交互，其交互方式大多是你可以从JavaScript本身与它们进行交互。你可以获取和设置属性和原型，也可以访问数组。此外，你可以将值转换为JavaScript对象的Qt对应物。例如，一个Url对象可以转化为一个QUrl。

一个QJSManagedValue总是被绑定到一个特定的QJSEngine。你不能单独使用它。这意味着你不能让一个引擎的QJSManagedValue成为另一个引擎的QJSManagedValue的属性或proptotype。

与QJSValue相比，几乎所有由QJSManagedValue持有的值都存在于JavaScript堆中。没有内联或非托管存储。因此，你可以得到一个原始值的原型，你也可以得到一个字符串的lengthproperty。

只有默认构建的或从QJSManagedValues中移出的QJSManagedValues在JavaScript堆中不持有一个值。它们代表着undefined，它没有任何属性或原型。

与QJSValue相比，QJSManagedValue也不捕捉任何JavaScript异常。如果对QJSManagedValue的操作导致了一个错误，它通常会返回一个未定义的值和QJSEngine::hasError () 会返回true after。然后你可以使用QJSEngine::catchError ()来捕获异常，或者把它传到堆栈，由你自己决定。

注意：由于JavaScript堆上的值的引用必须在销毁时被释放，你不能将QJSManagedValue移动到不同的线程。销毁将在新的线程中进行，这将与原线程的垃圾收集器产生一个竞赛条件。这也意味着你不能持有一个QJSManagedValue超过其引擎的寿命。

处理QJSManagedValue的推荐方法是在堆栈中创建它，可能是通过移动一个QJSValue并添加一个引擎，然后对它执行必要的操作，最后将它移回QJSValue进行存储。在QJSManagedValue和QJSValue之间移动是很快的。

enum QJSManagedValue:: 类型#

这个枚举代表了JavaScript的本地类型，由ECMA-262规定。

常量 值 描述
QJSManagedValue::Undefined 0 undefined type
QJSManagedValue::Boolean 1 boolean type
QJSManagedValue::Number 2 number type
QJSManagedValue::String 3 string type
QJSManagedValue::Object 4 object type
QJSManagedValue::Symbol 5 symbol type
QJSManagedValue::Function 6 function type

注意， null值本身不是一种类型，而是一种特殊的对象。你可以使用isNull()方法查询一个QJSManagedValue的这个条件。此外，JavaScript没有整数类型，但它知道对数字的特殊处理，为只进行整数操作做准备。你可以使用isInteger()方法来查询QJSManagedValue，以了解它是否持有这种处理的结果。

QJSManagedValue:: QJSManagedValue ( QJSManagedValue && other )#

移动–从other构造一个QJSManagedValue。这使得other处于默认构建的状态，它代表未定义的，不属于任何引擎。

使用engine的堆从value创建一个QJSManagedValue。

QJSManagedValue:: QJSManagedValue ( value , * engine )#

从value创建一个QJSManagedValue，使用engine的堆。如果value本身是被管理的，而它所属的引擎不是engine，结果是一个undefinedvalue，并产生一个警告。

QJSManagedValue:: QJSManagedValue ()#

创建一个QJSManagedValue，代表JavaScript的undefinedvalue。这是唯一没有存储在JavaScript堆中的值。在一个默认构建的QJSManagedValue上调用engine()将返回nullptr。

QJSManagedValue &QJSManagedValue:: operator= ( QJSManagedValue && other )#

移动–从other分配一个QJSManagedValue。这使得other处于默认构造的状态，它代表未定义的，不属于任何引擎。

注意：这释放了这个QJSManagedValue在JavaScript堆中持有的内存槽。你不能把QJSManagedValue移动到与它所属的QJSEngine不同的线程中。

QJSManagedValue:: ~QJSManagedValue ()#

销毁 QJSManagedValue .

注意：这将释放它在JavaScript堆上的内存槽。你不能从不同的线程销毁一个QJSManagedValue，而不是它所属的QJSEngine。

QJSManagedValue:: call (const & arguments = {}) const#

若此 QJSManagedValue代表一个JavaScript函数对象，用给定的arguments 调用它，并返回结果。否则返回一个JavaScript的 “未定义的 “值。

参数必须是原始值，或者与这个QJSManagedValue属于同一个[QJSEngine]。否则，调用将不被执行，并返回一个JavaScriptundefined值。

QJSManagedValue:: callAsConstructor (const & arguments = {}) const#

返回 true 若此 QJSManagedValue has a property name ，否则返回 false . The properties of the prototype chain are not considered.

若此 QJSManagedValue表示一个JavaScript FunctionObject，用给定的arguments 调用它作为构造器，并返回结果。否则返回一个JavaScript的 “未定义 “值。

参数必须是原始值，或者与这个QJSManagedValue属于同一个QJSEngine。否则，调用不会被执行，并返回一个JavaScriptundefined值。

QJSManagedValue:: callWithInstance (const & instance , const & arguments = {}) const#

若此 QJSManagedValue 代表一个JavaScript FunctionObject，在instance上采用给定参数进行调用，并返回结果。否则会返回一个JavaScript的 “未定义的 “值。

参数和instance必须是原始值，或者与这个QJSManagedValue属于同一个[QJSEngine]。否则，该调用不会被执行，并返回一个JavaScript的 “未定义 “值。

bool QJSManagedValue:: toBoolean () const#

将被管理的值转换为布尔值。如果被管理的值是一个布尔值，那么这个值将被返回。否则，将按照JavaScript规则进行布尔值强制转换。

QJSManagedValue:: toDateTime () const#

若此QJSManagedValue持有一个JavaScript日期对象，则返回一个等价的[QDateTime]（qdatetime.html）。否则返回一个无效的。

int QJSManagedValue:: toInteger () const#

将管理的值转换为一个整数。首先按照toNumber()的规则将值转换为数字，然后按照给定的规则将JavaScript位移运算符的参数强制转换为32位整数，将其夹在整数范围内。

在内部，该值可能已经被存储为一个整数，在这种情况下，会采取快速的方法。

注意：将一个被管理的值转换为一个数字可能会抛出一个异常。特别是，符号不能被胁迫成数字，或者自定义valueOf()方法可能会抛出。在这种情况下，结果是0，引擎会在转换后携带一个错误。

注意：将数字胁迫成32位整数的JavaScript规则是不直观的。

QJSManagedValue:: toJSValue () const#

将这个QJSManagedValue复制到一个新的[QJSValue]（qjsvalue.html）。这比从一个QJSManagedValue移动构建一个QJSValue的效率要低，但可以保留QJSManagedValue。

double QJSManagedValue:: toNumber () const#

将被管理的值转换为一个数字。如果被管理的值持有一个数字，那么这个数字将被返回。否则，将按照JavaScript规则进行数字强制处理。

注意：将一个被管理的值转换为一个数字会产生一个异常。特别是，符号不能被强制转换成数字，或者一个自定义的valueOf()方法可能会抛出。在这种情况下，结果是0，并且引擎在转换后携带一个错误。

QJSManagedValue:: toPrimitive () const#

将被管理的值转换为一个QJSPrimitiveValue。如果被管理的值持有QJSPrimitiveValue所支持的类型，该值就被复制。否则，该值将被转换为一个字符串，并且该字符串被存储在QJSPrimitiveValue中。

注意：将一个被管理的值转换为一个字符串可能会抛出一个异常。特别是，符号不能被胁迫成字符串，或者自定义的toString()方法可能会抛出。在这种情况下，结果是未定义的值，引擎在转换后会携带一个错误。

const *QJSManagedValue:: toQMetaObject () const#

若此 QJSManagedValue持有一个QMetaObject指针，则返回该指针。否则返回nullptr。

*QJSManagedValue:: toQObject () const#

若此 QJSManagedValue持有一个QObject指针，则返回该指针。否则返回nullptr。

QJSManagedValue:: toRegularExpression () const#

若此 QJSManagedValue持有一个JavaScript正则表达式对象，返回一个等价的QRegularExpression。否则返回一个无效的。

QJSManagedValue:: toString () const#

将被管理的值转换为一个字符串。如果被管理的值是一个字符串，就会返回这个字符串。否则将按照JavaScript规则进行字符串强制处理。

注意：将一个被管理的值转换为一个字符串可能会产生一个异常。特别是，符号不能被强制转换成字符串，或者一个自定义的toString()方法可能会抛出。在这种情况下，结果是一个空字符串，引擎会在转换后携带一个错误。

QJSManagedValue:: toUrl () const#

若此 QJSManagedValue持有一个JavaScript Url对象，返回一个等价的QUrl。否则返回一个无效的。

QJSManagedValue:: toVariant () const#

将这个QJSManagedValue复制到一个新的QVariant。如果QJSManagedValue::isVariant()返回false，这也会创建一个有用的QVariant，QVariant可以容纳所有QJSManagedValue支持的类型。

QJSManagedValue:: type () const#

返回这个QJSManagedValue()的JavaScript类型。
