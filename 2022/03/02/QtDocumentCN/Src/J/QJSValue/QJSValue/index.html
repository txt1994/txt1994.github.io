详细描述#

QJSValue支持[ECMA-262]（http://www.ecma-international.org/publications/standards/Ecma-262.htm）标准中定义的类型。原始类型，即未定义、空、布尔、数字和字符串；以及对象和阵列类型。此外，还为Qt/C++类型提供了内置支持，如[QVariant]()和[QObject](qobject.html)。

对于基于对象的类型（包括日期和正则），使用QJSEngine中的newT()函数（例如，QJSEngine::newObject()）来创建所需类型的QJSValue。对于原始类型，使用QJSValue构造函数中的一个重载。对于其他类型，例如注册的小工具类型，如[QPoint]（qpoint.html），你可以使用[QJSEngine::toScriptValue]（qjsengine.html#toScriptValue）。

命名为isT()的方法（例如，isBool (), isUndefined ()）可以用来测试一个值是否属于某种类型。命名为toT()的方法（例如toBool(), toString()）可用于将一个QJSValue转换为另一种类型。你也可以使用通用的qjsvalue_cast()函数。

对象值有零个或多个属性，它们本身就是QJSValues。使用setProperty()来设置对象的一个属性，并调用property()来检索一个属性的值。

QJSEngine myEngine;
QJSValue myObject = myEngine.newObject();
QJSValue myOtherObject = myEngine.newObject();
myObject.setProperty("myChild", myOtherObject);
myObject.setProperty("name", "John Doe");

如果你想遍历一个脚本对象的属性，请使用QJSValueIterator类。

对象的值有一个内部的 “prototype “属性，可以用prototype()和setPrototype()访问它。

函数对象（isCallable()返回true的对象）可以通过调用call()来调用。） 构造函数可以通过调用callAsConstructor()来构造新对象。

使用 等于() 或 strictlyEquals() 来比较一个QJSValue和另一个QJSValue。

请注意，isObject()为真的QJSValue只携带一个实际对象的引用；复制QJSValue将只复制对象的引用，而不是对象本身。如果你想克隆一个对象（即把一个对象的属性复制到另一个对象），你可以在脚本代码中的 “for-in “语句或C++中的QJSValueIterator的帮助下这样做。

Working With Arrays#

要使用QJSValue创建一个数组。use QJSEngine::newArray ():

// Assumes that this class was declared in QML.
QJSValue jsArray = engine->newArray(3);

要设置数组中的单个元素，请使用setProperty(quint32 arrayIndex, const QJSValue &value)重载。例如，要用整数来填充上面的数组:

for (int i = 0; i < 3; ++i) {
    jsArray.setProperty(i, QRandomGenerator::global().generate());
}

要确定数组的长度，请访问"length" property。要访问数组元素，使用property(quint32 arrayIndex) 重载。下面的代码将我们上面创建的数组读回一个列表中:

QVector<int> integers;
const int length = jsArray.property("length").toInt();
for (int i = 0; i < length; ++i) {
    integers.append(jsArray.property(i).toInt());
}

另请参阅 QJSEngine and QJSValueIterator .

[since 5.12] enum QJSValue:: ErrorType#

使用这个枚举可以得到JavaScript语言特定类型的Error对象。

当在C++中模拟语言特性需要使用专门的异常类型时，它们可能是有用的。此外，它们可能有助于更清楚地传达某些典型情况，而不是抛出一个通用的JavaScript异常。例如，处理网络和资源定位器的代码可能会发现使用URIError类型来传播与畸形定位器有关的错误是非常有用的。

常量 值 描述
QJSValue::GenericError 1 一个通用的错误对象，但不属于特定的子类型。
QJSValue::RangeError 3 一个值不符合预期的集合或范围。
QJSValue::ReferenceError 4 引用了一个不存在的变量。
QJSValue::SyntaxError 5 遇到了一个无效的标记或标记序列，不符合语言的语法。
QJSValue::TypeError 6 一个操作数或参数与预期的类型不一致。
QJSValue::URIError 7 一个URI处理函数被错误地使用，或者提供的URI是畸形的。

该枚举在 Qt 5.12 引入或被修改。

enum QJSValue:: ObjectConversionBehavior#

这个枚举用于指定在转换为QVariant时，如何处理没有对应的本地Qt类型的JavaScript对象。

常量 值 描述
QJSValue::ConvertJSObjects 0 试图进行尽最大努力，可能是有损的转换。
QJSValue::RetainJSObjects 1 该值被保留为QJSValue包裹的QVariant。

常量 值 描述

QJSValue::NullValue 0 一个空值。

QJSValue:: call (const & args = QJSValueList()) const#

将这个QJSValue作为一个函数调用，将args作为参数传给函数，并使用globalObject()作为 “this “对象。返回从该函数返回的值。

若此 QJSValue 不可调用，call()不做任何事情，并返回一个未定义的 QJSValue 。

调用call()可能会导致脚本引擎发生异常；在这种情况下，call()会返回被抛出的值（通常是一个Error 对象）。你可以在返回值上调用isError()，以确定是否发生异常。

另请参阅 isCallable (), callWithInstance ()，和 callAsConstructor ().

QJSValue:: callAsConstructor (const & args = QJSValueList()) const#

创建新的 Object 并调用这个 QJSValue 作为构造函数，使用创建的对象作为`this’对象并传递args作为参数。如果构造函数调用的返回值是一个对象，那么该对象将被返回；否则将返回默认的构造对象。

若此 QJSValue 不是一个函数，callAsConstructor()不做任何事情，并返回一个未定义的 QJSValue 。

调用这个函数可能会导致脚本引擎发生异常；在这种情况下，被抛出的值（通常是一个`错误’对象）会被返回。你可以在返回值上调用isError()，以确定是否发生异常。

另请参阅 call () 和 QJSEngine::newObject ().

QJSValue:: callWithInstance (const & instance , const & args = QJSValueList()) const#

将此QJSValue作为一个函数调用，在函数调用中使用instance作为`this’对象，并将args作为参数传递给函数。返回从该函数返回的值。

若此QJSValue不是一个函数，call()不做任何事情，并返回一个未定义的QJSValue。

注意，若instance不是一个对象，全局对象（见QJSEngine::globalObject()）将被用作`this’对象。

调用call()可以导致脚本引擎发生异常；在这种情况下，call()返回被抛出的值（通常是一个Error对象）。你可以在返回值上调用isError()，以确定是否发生异常。

另请参阅 call ().

bool QJSValue:: deleteProperty (const & name )#

试图删除此对象的指定name 的属性。如果该属性被删除，则返回真，否则返回假。

这个函数的行为与JavaScript的删除操作符一致。特别是。

  • 非可配置的属性不能被删除。
  • 即使这个对象没有给定的name 的属性，这个函数也会返回true（也就是说，不存在的属性是 “琐碎的可删除”）。
  • 如果这个对象没有给定的name 的属性，但是prototype()链中的一个对象有，那么原型对象的属性不会被删除，并且这个函数返回true。

另请参阅 setProperty () 和 hasOwnProperty ().

bool QJSValue:: 等于 (const & other ) const#

返回true，若此 QJSValue等于 other ，否则返回false。比较遵循ECMA-262第11.9.3节 “抽象等价比较算法 “中描述的行为。

即使这个QJSValue的类型与other 值的类型不同，这个函数也可以返回真；也就是说，比较并不严格。例如，将数字9与字符串 “9 “进行比较会返回真；将未定义的值与空值进行比较会返回真；将原始值为6的数字对象与原始值为 “6 “的字符串对象进行比较会返回真；将数字1与布尔值true进行比较会返回真。如果你想进行没有这种隐含值转换的比较，请使用strictEquals()。

请注意，如果这个QJSValue或other 值是对象，调用这个函数对脚本引擎有副作用，因为引擎将调用对象的valueOf()函数（可能还有toString()），试图将对象转换为原始值（可能导致一个未捕获的脚本异常）。

另请参阅 strictlyEquals ().

QJSValue:: property (const & name ) const#

返回这个QJSValue的属性的值，并给出名称。如果不存在这样的属性，一个未定义的QJSValue被返回。

如果该属性是使用getter函数实现的（即设置了PropertyGetter标志），调用property()对脚本引擎有副作用，因为getter函数将被调用（可能导致未捕获的脚本异常）。如果发生了异常，property()会返回被抛出的值（通常是一个Error对象）。

要访问数组元素，请使用setProperty(quint32 arrayIndex, const QJSValue &value)重载来代替。

另请参阅 setProperty (), hasProperty ()，和 QJSValueIterator .

QJSValue:: property ( arrayIndex ) const#

这是重载函数。

返回这个QJSValue的属性的值，并给出名称。如果不存在这样的属性，一个未定义的QJSValue被返回。

如果该属性是使用getter函数实现的（即设置了PropertyGetter标志），调用prottReturns属性在给定的arrayIndex。

有可能以两种方式访问数组中的元素。第一种是使用数组索引作为属性名称：y()对脚本引擎有副作用，因为getter函数会被调用（可能会导致一个未捕获的脚本异常）。如果发生了异常，property()会返回被抛出的值（通常是一个Error对象）。

要访问数组元素，请使用setProperty(quint32 arrayIndex, const QJSValue &value)重载来代替：

qDebug() << jsValueArray.property(QLatin1String("4")).toString();

第二种是使用需要索引的重载。

qDebug() << jsValueArray.property(4).toString()。

这两种方法实现了相同的结果，只是后者。

  • 更容易使用（可以直接使用一个整数）
  • 更快（无需转换为整数）

如果此QJSValue不是一个数组对象，这个函数的行为就像property()被调用时使用arrayIndex的字符串表示。

设置这个QJSValue的属性的值，并给定name到value。

void QJSValue:: setProperty ( arrayIndex , const & value )#

这是重载函数。

设置给定arrayIndex到给定value的属性。

可以通过两种方式来修改数组中的元素。第一种是使用数组索引作为属性名称。

jsValueArray.setProperty(QLatin1String("4"), value)。

第二个是使用需要索引的重载。

jsValueArray.setProperty(4, value);

这两种方法实现了相同的结果，只是后者。

  • 更容易使用（可以直接使用一个整数）
  • 更快（无需转换为整数）。

如果此QJSValue不是一个数组对象，此函数的行为就像setProperty()被调用时使用arrayIndex的字符串表示。

另请参阅 property(quint32 arrayIndex) 和 Working With Arrays 。

void QJSValue:: setPrototype ( const & prototype )#

若此 QJSValue 是一个对象，将此对象的内部原型（__proto__ property）设置为prototype；如果 QJSValue 是空的，它将原型设置为空；否则不做任何事情。

内部原型不应该与名为 “prototype “的公共属性相混淆；公共原型通常只在作为构造器的函数上设置。

另请参阅 prototype() 和 isObject()。

bool QJSValue:: strictlyEquals (const & other ) const#

返回true，若此QJSValue等于other使用严格的比较（没有转换），否则返回false。 比较遵循ECMA-262第11.9.6节，”严格的平等比较算法 “中描述的行为。

如果这个QJSValue的类型与另一个值的类型不同，这个函数返回false。如果类型相等，结果取决于类型，如下表所示：

类型 结果
Undefined true
Null true
Bool 如果两个值都是真，则为真，否则为假
Number 如果其中一个值是NaN（非数字），则为假；如果数值相等，则为真，否则为假。
String 如果两个值都是完全相同的字符序列，则为真，否则为假。
Object 如果两个值指的是同一个对象，则为真，否则为假

QJSValue:: toVariant ( behavior ) const#

返回这个QJSValue的QVariant值，如果它可以被转换为QVariant；否则返回一个无效的QVariant。一些JavaScript类型和对象在Qt中具有本地表达式。这些被转换为它们的本地表达式。比如说:

Input Type 结果
Undefined 一个无效的QVariant。
Null 一个包含空指针的QVariant（[QMetaType::Nullptr]（#Type-enum））。
Bool 一个包含布尔值的QVariant。
Number 一个QVariant包含数字的值。
String 一个QVariant，包含字符串的值。
QVariant Object 结果是对象的QVariant值（没有转换）。
QObject Object 一个包含指向QObject的指针的QVariant。
Date Object 一个包含日期值的QVariant（toDateTime()）。
RegularExpression Object 一个包含正则表达式值的QVariant。

对于其他类型，behavior参数是相关的。如果给了ConvertJSObjects，就会尽力进行转换，但可能是有损的。通用的JavaScript对象被转换为QVariantMap。JavaScript数组被转换为QVariantList。每个属性或元素被转换为QVariant，递归；循环引用不被遵循。JavaScript函数对象被放弃。如果给了 “RetainJSObjects”，QJSValue被包装成一个QVariant凭借QVariant::fromValue()。由此产生的转换是无损的，但对象的内部结构不能立即访问。

另请参阅 isVariant ().
