详细描述#

QJsonDocument是一个包裹完整的JSON文档的类，它可以从UTF-8编码的文本表示法中读取该文档，并将其写入。

一个JSON文档可以通过QJsonDocument::fromJson ()从其基于文本的表示方式转换为QJsonDocument。toJson ()将其转换回文本。这个解析器非常快速和高效，可以将JSON转换为Qt使用的二进制表示。

解析后的文档的有效性可以用 ! isNull()

可以用isArray ()和isObject ()来查询一个文档是否包含一个数组或一个对象。文档中包含的数组或对象可以用array () 或 object ()来检索，然后进行读取或操作。

另请参阅 在 Qt 中支持 JSON and JSON 保存游戏范例 .

[since 5.1] enum QJsonDocument:: JsonFormat#

这个值定义了转换为QJsonDocument使用toJson()时产生的JSON字节数的格式。

常量 值 描述
QJsonDocument::Indented 0 定义人类可读的输出如下： { "Array" : [ true , 999 , "string" ] , "Key" : "Value" , "null" : null }
QJsonDocument::Compact 1 定义了一个紧凑的输出，如下所示: {"Array":[true,999,"string"],"Key":"Value","null":null}

该枚举在 Qt 5.1 引入或被修改。

[static] QJsonDocument:: fromJson (const & json , * error = nullptr)#

剖析 json 作为 UTF-8 编码 JSON 文档，并创建 QJsonDocument 从它。

返回有效 (非 null) QJsonDocument 若剖析成功。若它失败，返回文档将为 null，且可选 error 变量将包含有关错误的进一步细节。

另请参阅 toJson (), QJsonParseError ，和 isNull ().

[static] QJsonDocument:: fromVariant (const & variant )#

创建 QJsonDocument 从 QVariant variant .

若variant 包含任何其他类型，而不是 a QVariantMap, QVariantHash, QVariantList or QStringList ，返回的文档无效。

另请参阅 toVariant ().

bool QJsonDocument:: isEmpty () const#

bool QJsonDocument:: isNull () const#

返回 true 若此文档为 null。

null 是透过默认构造函数创建文档。

从UTF-8编码的文本或二进制格式创建的文件在解析过程中被验证。如果验证失败，返回的文件也将是空的。
