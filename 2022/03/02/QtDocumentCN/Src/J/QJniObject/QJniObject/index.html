详细描述#

QJniObject类包装了对一个Java对象的引用，确保它不被收集，并提供对大多数JNIEnv方法调用（成员，静态）和字段（setter，getter）的访问。它消除了许多通常需要的模板，直接访问JNI，为每个操作，包括异常处理。

注意：该API是为Android设计和测试的。它还没有在其他平台上进行测试。

一般注意事项#

  • 类名需要完全限定，例如："java/lang/String" 。
  • 方法签名写成"(ArgumentsTypes)ReturnType" ，见JNI类型。
  • 所有对象类型都以QJniObject的形式返回。

方法签名#

对于没有参数的函数，QJniObject提供了方便的函数，它将根据提供的模板类型使用正确的签名。比如说:

jint x = QJniObject::callMethod<jint>("getSize");
QJniObject::callMethod<void>("touch");

在其他情况下，你将需要自己提供签名，重要的是签名要与你要调用的函数相匹配。签名结构是"(ArgumentsTypes)ReturnType" 。签名中的数组类型必须有[前缀，完全限定的Object类型名称必须有L前缀和;后缀。

下面的例子演示了如何调用两个不同的静态函数。

// Java class
package org.qtproject.qt;
class TestClass
{
   static String fromNumber(int x) { ... }
   static String[] stringArray(String s1, String s2) { ... }
}

第一个函数的签名是"(I)Ljava/lang/String;" :

// C++ code
QJniObject stringNumber = QJniObject::callStaticObjectMethod("org/qtproject/qt/TestClass",
                                                             "fromNumber"
                                                             "(I)Ljava/lang/String;",
                                                             10);

而第二个函数的签名是"(Ljava/lang/String;Ljava/lang/String;) [Ljava/lang/String;":

// C++ code
QJniObject string1 = QJniObject::fromString("String1");
QJniObject string2 = QJniObject::fromString("String2");
QJniObject stringArray = QJniObject::callStaticObjectMethod("org/qtproject/qt/TestClass",
                                                            "stringArray"
                                                            "(Ljava/lang/String;Ljava/lang/String;)[Ljava/lang/String;"
                                                            string1.object<jstring>(),
                                                            string2.object<jstring>());

处理 Java 异常#

在调用可能抛出异常的Java函数后，在继续之前检查、处理和清除任何异常是很重要的。所有的QJniObject函数都通过报告和清除异常在内部处理异常，从而节省了客户端代码处理异常的需要。

注意：用户在直接使用 “JNIEnv “进行JNI调用时必须手动处理异常。在异常等待时进行其他JNI调用是不安全的。更多信息，请参阅 QJniEnvironment::checkAndClearExceptions ()。

Java 本机方法#

Java本地方法使得从Java中调用本地代码成为可能，这是通过在Java中创建一个函数声明并在其前面加上native关键字来实现的。在从Java中调用本地函数之前，你需要将Java本地函数映射到代码中的本地函数。映射函数可以通过调用QJniEnvironment::registerNativeMethods()完成。

下面的例子演示了如何做到这一点。

Java 实现：

class FooJavaClass
{
    public static void foo(int x)
    {
        if (x < 100)
            callNativeOne(x);
        else
            callNativeTwo(x);
    }
private static native void callNativeOne(int x);
private static native void callNativeTwo(int x);
}

C++ 实现：

static void fromJavaOne(JNIEnv *env, jobject thiz, jint x)
{
    Q_UNUSED(env);
    Q_UNUSED(thiz);
    qDebug() << x << "< 100";
}
static void fromJavaTwo(JNIEnv *env, jobject thiz, jint x)
{
    Q_UNUSED(env);
    Q_UNUSED(thiz);
    qDebug() << x << ">= 100";
}
void foo()
{
    // register the native methods first, ideally it better be done with the app start
    const JNINativeMethod methods[] =
                {{"callNativeOne", "(I)V", reinterpret_cast<void *>(fromJavaOne)},
                 {"callNativeTwo", "(I)V", reinterpret_cast<void *>(fromJavaTwo)}};
    QJniEnvironment env;
    env.registerNativeMethods("my/java/project/FooJavaClass", methods, 2);
    // Call the java method which will calls back to the C++ functions
    QJniObject::callStaticMethod<void>("my/java/project/FooJavaClass", "foo", "(I)V", 10);  // Output: 10 < 100
    QJniObject::callStaticMethod<void>("my/java/project/FooJavaClass", "foo", "(I)V", 100); // Output: 100 >= 100
}

Java 对象的生命周期#

大多数从Java收到的对象将是本地引用，并且只在你从本地方法返回之前保持有效。在那之后，该对象就有资格进行垃圾回收。如果你的代码在一个循环中创建了许多本地引用，你应该在每次迭代中手动删除它们，否则你可能会耗尽内存。更多信息，请参阅 JNI设计概述：全局和局部引用。在本地方法范围之外创建的本地引用必须手动删除，因为垃圾收集器不会自动释放它们，因为我们使用的是AttachCurrentThread。更多信息，见JNI提示：本地和全局引用。

如果你想保持一个Java对象的生命力，你需要为该对象创建一个新的全局引用，并在完成后释放它，或者构造一个新的QJniObject并让它管理该Java对象的生命周期。

注意：QJniObject只管理它自己的引用，如果你从一个全局或局部引用中构造一个QJniObject，该引用将不会被QJniObject释放。

QJniObject:: QJniObject ( jobject object )#

围绕Java对象object构建一个新的JNI对象。

注意： QJniObject将持有对Java对象object的引用，并在销毁时释放它。任何对QJniObject之外的Java对象object的引用都需要由调用者管理。在大多数情况下，你不应该用一个本地引用来调用这个函数，除非你打算自己管理这个本地引用。参见QJniObject::fromLocalRef()来转换一个本地引用到QJniObject。

[static] template T QJniObject:: callStaticMethod ( jclass clazz , jmethodID methodId , …)#

从clazz类中调用由methodId标识的静态方法，并带有任何后续参数。当clazz和methodId已经从以前的操作中被缓存时，该方法很有用。

QJniEnvironment env;
jclass javaMathClass = env.findClass("java/lang/Math");
jmethodID methodId = env.findStaticMethod(javaMathClass, "max", "(II)I");
if (methodId != 0) {
    jint a = 2;
    jint b = 4;
    jint max = QJniObject::callStaticMethod<jint>(javaMathClass, methodId, a, b);
}

[static] QJniObject:: fromLocalRef ( jobject localRef )#

创建 QJniObject 从本地JNI引用 localRef 。这个函数获取localRef的所有权，并在返回前释放它。

注意：只能用本地JNI引用来调用这个函数。例如，大多数原始的JNI调用，通过JNI环境，返回对一个java对象的本地引用。

jobject localRef = env->GetObjectArrayElement(array, index);
QJniObject element = QJniObject::fromLocalRef(localRef);

QJniObject qjniObject("could/not/find/Class") // ==> isValid() == false

jobject QJniObject:: object () const#

返回QJniObject所持有的对象，可以是jobject，也可以是T类型。T可以是[JNI对象类型]（#object-types）之一。

QJniObject string = QJniObject::fromString("Hello, JNI");
jstring jstring = string.object<jstring>();

注意：返回的对象仍然由这个QJniObject保持活力。为了在这个QJniObject的生命周期之外保持对象的活力，例如，为了记录它供以后使用，最简单的方法是将它存储在另一个具有适当生命周期的QJniObject中。或者，你可以为该对象创建一个新的全局引用并存储它，当你用完它时，注意释放它。

void functionScope()
{
    QString helloString("Hello");
    jstring myJString = 0;
    {
        QJniObject string = QJniObject::fromString(helloString);
        myJString = string.object<jstring>();
    }
   // Ops! myJString is no longer valid.
}

[since 6.2] jclass QJniObject:: objectClass () const#

返回由QJniObject作为jclass持有的类对象。

注意：返回的对象仍然由这个QJniObject保持活力。为了在这个QJniObject的生命周期之外保持对象的活力，例如，为了记录它供以后使用，最简单的方法是将它存储在另一个具有适当生命周期的QJniObject中。或者，你可以为该对象创建一个新的全局引用并存储它，当你用完它时，注意释放它。
