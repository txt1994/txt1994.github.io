详细描述#

当使用JNI时，JNIEnv类是一个指向函数表和每个JNI函数的成员函数的指针，这些函数通过该表间接进行。JNIEnv提供了大部分的JNI函数。每个C++本地函数都会收到一个JNIEnv 作为第一个参数。JNI环境不能在线程之间共享。

由于JNIEnv不做很多错误检查，如异常检查和清除，QJniEnvironment可以让你轻松做到这一点。

关于JNIEnv的更多信息，请参阅Java: Interface Function Table。

注意：本API是为Android设计和测试的。它还没有为其他平台进行测试。

enum class QJniEnvironment:: OutputMode#

常量 值 描述
QJniEnvironment::OutputMode::Silent 0 异常情况被无声地清理掉了

QJniEnvironment:: QJniEnvironment ()#

构建一个新的JNI环境对象，并将当前线程附加到Java VM上。

QJniEnvironment:: ~QJniEnvironment ()#

分离当前线程从Java VM和销毁QJniEnvironment对象。这将通过调用checkAndClearExceptions()来清除任何未决的异常。

bool QJniEnvironment:: checkAndClearExceptions ( outputMode = OutputMode::Verbose)#

根据outputMode，默默地清除任何悬而未决的异常，或者报告堆栈回溯。

与内部处理异常的QJniObject相比，如果你直接通过JNIEnv进行JNI调用，你需要在调用后使用此函数清除任何潜在的异常。关于可能抛出异常的JNIEnv调用的更多信息，请参阅 JNI Functions 。

返回 true 当一个未决的异常被清除时。

[static] bool QJniEnvironment:: checkAndClearExceptions ( JNIEnv * env , outputMode = OutputMode::Verbose)#

清除env的任何悬而未决的异常，根据outputMode，可以是静默的或报告堆栈回溯。当你已经有一个 “JNIEnv “指针时，这很有用，比如在一个本地函数的实现中。

与内部处理异常的QJniObject相比，如果你直接通过JNIEnv进行JNI调用，你需要在调用后使用此函数清除任何潜在的异常。关于可能抛出异常的JNIEnv调用的更多信息，请参阅 JNI Functions 。

返回 true 当一个未决的异常被清除时。

jclass QJniEnvironment:: findClass (const char * className )#

搜索 className 使用所有可用的类加载器。Qt on Android使用一个自定义的类加载器来加载所有的.jar文件，必须用它来寻找由该类加载器创建的任何类，因为这些类在使用默认类加载器时是不可见的。

如果className找不到，则返回类的指针或空。

这个函数的一个用例是搜索一个类来调用一个需要 “jclass “的JNI方法。当对同一个类对象进行多次JNI调用时，这可能比在每次调用中使用一个类名要快一点。此外，在进行JNI调用之前，这个调用会先寻找内部缓存的类，如果找到的话会返回这样的类。下面的代码片段创建了一个CustomClass'类的实例，然后调用printFromJava() `方法：

QJniEnvironment env;
jclass javaClass = env.findClass("org/qtproject/example/android/CustomClass");
QJniObject javaMessage = QJniObject::fromString("findClass example");
QJniObject::callStaticMethod<void>(javaClass, "printFromJava",
                                   "(Ljava/lang/String;)V", javaMessage.object<jstring>());

注意： 这个调用从内部缓存的类中返回一个对类对象的全局引用。

[since 6.2] jfieldID QJniEnvironment:: findField ( jclass clazz , const char * fieldName , const char * signature )#

搜索clazz类的一个成员字段。该字段由其fieldName和signature指定。

如果没有找到该字段，则返回该字段的ID或nullptr。

这个方法的一个用例是搜索类的字段并缓存它们的ID，这样它们以后就可以用来获取/设置字段。

[since 6.2] jmethodID QJniEnvironment:: findStaticMethod ( jclass clazz , const char * methodName , const char * signature )#

搜索类clazz的一个静态方法。该方法由其methodName和signature指定。

如果没有找到该方法，则返回该方法的ID或nullptr。

这个方法的一个用例是搜索类的方法并缓存它们的ID，以便以后可以用来调用这些方法。

QJniEnvironment env;
jclass javaClass = env.findClass("org/qtproject/example/android/CustomClass");
jmethodID methodId = env.findStaticMethod(javaClass,
                                          "staticJavaMethod",
                                          "(Ljava/lang/String;)V");
QJniObject javaMessage = QJniObject::fromString("findStaticMethod example");
QJniObject::callStaticMethod<void>(javaClass,
                                   methodId,
                                   javaMessage.object<jstring>());

该函数在 Qt 6.2 引入。

bool QJniEnvironment:: registerNativeMethods (const char * className , const JNINativeMethod [] methods , int size )#

注册数组methods中大小为size的Java方法，每个方法都可以调用类className中的本地C++函数。在调用这些方法之前，必须先注册这些方法。

如果注册成功，返回 “true”，否则返回 “false”。

方法数组中的每个元素由以下部分组成:

  • Java方法的名称
  • 方法签名
  • 将被执行的C++函数
const JNINativeMethod methods[] =
                        {{"callNativeOne", "(I)V", reinterpret_cast<void *>(fromJavaOne)},
                        {"callNativeTwo", "(I)V", reinterpret_cast<void *>(fromJavaTwo)}};
QJniEnvironment env;
env.registerNativeMethods("org/qtproject/android/TestJavaClass", methods, 2);

bool QJniEnvironment:: registerNativeMethods ( jclass clazz , const JNINativeMethod [] methods , int size )#

这是重载函数。

这个重载使用了之前缓存的jclass实例clazz。

JNINativeMethod methods[] {{"callNativeOne", "(I)V", reinterpret_cast<void *>(fromJavaOne)},
                           {"callNativeTwo", "(I)V", reinterpret_cast<void *>(fromJavaTwo)}};
QJniEnvironment env;
jclass clazz = env.findClass("org/qtproject/android/TestJavaClass");
env.registerNativeMethods(clazz, methods, 2);
