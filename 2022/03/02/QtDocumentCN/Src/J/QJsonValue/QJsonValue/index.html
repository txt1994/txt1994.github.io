详细描述#

JSON中的一个值可以是6种基本类型之一。

JSON 是结构化数据存储格式。它有 6 种基本数据类型：

  • bool QJsonValue::Bool
  • double QJsonValue::Double
  • string QJsonValue::String
  • array QJsonValue::Array
  • object QJsonValue::Object
  • null QJsonValue::Null

一个值可以代表上述任何一种数据类型。此外，QJsonValue有一个特殊的标志来表示未定义的值。这可以用isUndefined()来查询。

值的类型可以用type()或像isBool()、isString()等访问器来查询。同样，值可以通过toBool ()、toString ()等访问器转换为其存储的类型。

值在内部是严格类型化的，与QVariant 相反，不会尝试做任何隐式类型转换。这意味着转换到一个没有存储在值中的类型将返回一个默认构造的返回值。

QJsonValueRef#

QJsonValueRef是QJsonArray和QJsonObject的一个辅助类。当你得到一个QJsonValueRef类型的对象时，你可以像使用一个QJsonValue的引用一样使用它。如果你对它进行赋值，赋值将适用于QJsonArray或QJsonObject中的元素，你从该元素中得到了引用。

以下方法返回QJsonValueRef:

  • QJsonArray ::operator[](qsizetype i)
  • QJsonObject ::operator[](const QString & key) const

另请参阅 在 Qt 中支持 JSON and JSON 保存游戏范例 .

损失的信息和其他类型#

QVariant可以携带比在JSON中可表示的更多信息。如果QVariant不是上述类型之一，则不能保证转换，并可能在未来的Qt版本中发生变化，正如UUID那样。代码应努力不使用上述类型以外的任何其他类型。

若 QVariant::isNull () 返回真，则返回一个空的 QJsonValue ，或插入到列表或对象中，不管 QVariant 所带的类型如何。请注意，Qt 6.0中影响QVariant::isNull ()的行为变化也会影响这个函数。

一个浮点值如果是无穷大或NaN，将被转换为一个空的JSON值。自Qt 6.0以来，QJsonValue可以无损地存储任何64位有符号整数的全部精度，但在以前的版本中，超出±2^53范围的值可能会失去精度。大于或等于2^63的无符号64位值将失去精度或别名为负值，所以应该避免使用QMetaType::ULongLong。

对于上面没有列出的其他类型，将尝试转换为字符串，通常但不总是通过调用 QVariant::toString ()。如果转换失败，该值将被一个空的JSON值取代。请注意，QVariant::toString ()对大多数类型来说也是有损失的。例如，如果传递的QVariant代表原始字节数组数据，建议预先编码为Base64（或其他无损编码），否则将使用QString::fromUtf8()进行有损转换。

请注意，通过QVariant::toString ()进行的转换可能会在任何时候改变。QVariant和QJsonValue都可能在未来被扩展，以支持更多的类型，这将导致本函数执行转换的方式发生变化。

另请参阅 toVariant () 和 QCborValue::fromVariant ().
