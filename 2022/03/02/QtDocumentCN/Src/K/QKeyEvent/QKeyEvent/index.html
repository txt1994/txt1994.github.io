详细描述#

当按键被按下或释放时，按键事件被发送到具有键盘输入焦点的部件。

一个按键事件包含一个特殊的接受标志，表示接收器是否会处理这个按键事件。对于 QEvent::KeyPress 和 QEvent::KeyRelease ，这个标志是默认设置的，所以在对一个按键事件采取行动时不需要调用 accept() 。对于 QEvent::ShortcutOverride，接收者需要明确地接受该事件以触发重写。在一个按键事件上调用ignore()会将其传播到父部件。该事件在父部件链上传播，直到一个部件接受它或一个事件过滤器消耗它。

QWidget::setEnabled()函数可以被用来启用或禁用一个小组件的鼠标和键盘事件。

事件处理程序 QWidget::keyPressEvent(), QWidget::keyReleaseEvent(), QGraphicsItem::keyPressEvent () 和 QGraphicsItem::keyReleaseEvent () 接收键事件。

See also QFocusEvent and QWidget::grabKeyboard().

成员函数文档#

QKeyEvent::QKeyEvent( type, int key, modifiers, nativeScanCode, nativeVirtualKey, nativeModifiers, const &text = QString(), bool autorep = false, count = 1, const *device = QInputDevice::primaryKeyboard())#

构建一个按键事件对象。

参数type必须是QEvent::KeyPress, QEvent::KeyRelease, 或者QEvent::ShortcutOverride。

Int key是事件循环应该监听的Qt::Key的代码。如果key为0，事件就不是一个已知的键的结果；例如，它可能是一个合成序列或键盘宏的结果。modifiers持有键盘修改器，而给定的text是该键产生的Unicode文本。如果autorep为真，isAutoRepeat()将为真。count是事件中涉及的键的数量。

除了正常的按键事件数据外，还包含nativeScanCode、nativeVirtualKey和nativeModifiers。这些额外的数据被快捷键系统使用，以确定哪些快捷键需要触发。

QKeyEvent::QKeyEvent( type, int key, modifiers, const &text = QString(), bool autorep = false, count = 1)#

构建一个按键事件对象。

参数type必须是QEvent::KeyPress, QEvent::KeyRelease, 或者QEvent::ShortcutOverride。

Int key是事件循环应该监听的Qt::Key的代码。如果key为0，事件就不是一个已知的键的结果；例如，它可能是一个合成序列或键盘宏的结果。modifiers持有键盘修改器，而给定的text是该键产生的Unicode文本。如果autorep为真，isAutoRepeat()将为真。count是事件中涉及的键的数量。

int QKeyEvent::count() const#

返回该事件中涉及的键的数量。如果text()不是空的，这仅仅是字符串的长度。

See also Qt::WA_KeyCompression.

bool QKeyEvent::isAutoRepeat() const#

如果该事件来自于自动重复的按键，则返回 “true”；如果来自于初始按键，则返回 “false”。

请注意，如果该事件是一个部分由于自动重复的多键压缩事件，这个函数可以不确定地返回真或假。

int QKeyEvent::key() const#

返回被按下或释放的键的代码。

参见Qt::Key中的键盘代码列表。这些代码与底层窗口系统无关。注意，这个函数不区分大写字母和非大写字母，为此请使用text()函数（返回该键生成的Unicode文本）。

值为0或Qt::Key_unknown意味着该事件不是一个已知键的结果；例如，它可能是一个合成序列、一个键盘宏的结果，或由于键事件压缩。

See also Qt::WA_KeyCompression.

[since 6.0] QKeyEvent::keyCombination() const#

返回一个QKeyCombination对象，其中包含该事件携带的key()和modifiers()。

这个函数是在Qt 6.0中引入的。

bool QKeyEvent::matches( key) const#

如果关键事件与给定的标准key相匹配，返回 “true”；否则返回 “false”。

QKeyEvent::modifiers() const#

返回事件发生后立即存在的键盘修改器标志。

QKeyEvent::nativeModifiers() const#

返回一个按键事件的本地修改器。如果键事件不包含这些数据，则返回0。

注意：本地修改器可能是0，即使键事件包含扩展信息。

QKeyEvent::nativeScanCode() const#

返回按键事件的本地扫描代码。如果按键事件不包含这些数据，则返回0。

注意：本地扫描代码可能是0，即使键事件包含扩展信息。

QKeyEvent::nativeVirtualKey() const#

返回本地虚拟键，或键事件的键符号。如果键事件不包含这些数据，则返回0。

注意：本地虚拟键可能是0，即使键事件包含扩展信息。

QKeyEvent::text() const#

返回该键产生的Unicode文本。

当按下Shift、Control、Alt和Meta等修饰键时的返回值在不同平台上有所不同，可能会返回一个空字符串。

注意： key()将总是返回一个有效的值，与修改键无关。

See also Qt::WA_KeyCompression.
