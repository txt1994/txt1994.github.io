详细描述#

在其最常见的形式中，按键序列描述了必须一起使用的按键组合，以执行某些动作。键序与QAction对象一起使用，以指定哪些键盘快捷键可以用来触发动作。

按键序列可以通过三种不同的方式被构建为键盘快捷键使用。

  • 对于标准快捷键，可以使用标准键来请求与每个快捷键相关的特定平台的键序。
  • 对于自定义快捷键，可以使用人类可读的字符串，如 “Ctrl+X”，这些字符串可以被翻译成适合不同语言用户的快捷键。翻译是在”QShortcut “环境下进行的。
  • 对于硬编码的快捷方式，可以用Qt::Key和Qt::Modifier枚举值定义的数值组合来指定整数键代码。每个键代码由单个 Qt::Key 值和零个或多个修饰符组成，例如 Qt::SHIFT、Qt::CTRL、Qt::ALT 和 Qt::META 。

例如，Ctrl P可能是一个序列，用作打印文档的快捷方式，可以用以下任何一种方式指定:

QKeySequence(QKeySequence::Print);
QKeySequence(tr("Ctrl+P"));
QKeySequence(tr("Ctrl+p"));
QKeySequence(Qt::CTRL | Qt::Key_P);
QKeySequence(Qt::CTRL + Qt::Key_P); // deprecated

注意，对于字母来说，规范字符串中使用的大小写并不重要。在上面的例子中，用户不需要按住Shift键来激活用 “Ctrl+P “指定的快捷方式。然而，对于其他键来说，使用Shift作为一个未指定的额外修改键可能会导致应用程序的用户感到困惑，因为他们的键盘布局与开发者使用的键盘布局不同。参见下面的键盘布局问题部分以了解更多细节。

最好是尽可能地使用标准的快捷键。当为非标准的快捷方式创建按键序列时，你应该使用人类可读的字符串，而不是硬编码的整数值。

QKeySequence对象可以被转换为QString，以获得序列的人类可读翻译版本。类似地，toString()函数可以产生人类可读的字符串，用于菜单中。在macOS上，适当的符号被用来描述使用Macintosh键盘上特殊键的键盘快捷方式。

另一种指定硬编码键码的方法是使用字符的Unicode码位；例如，’A’给出了与[Qt::Key_A]（qt.html#Key-enum）相同的键序。

注意：在macOS上，对 “Ctrl”、Qt::CTRL、Qt::Key_Control 和 [Qt::ControlModifier](qt. html#KeyboardModifier-enum)对应于Macintosh键盘上的命令键，而对 “Meta “的引用、Qt::META、Qt::Key_Meta和Qt::MetaModifier对应于Control**键。macOS上的开发者可以在所有平台上使用相同的快捷键描述，他们的应用程序将自动在macOS上按照预期工作。

Keyboard Layout Issues#

许多键序规格是由开发人员根据某些类型的键盘布局选择的，而不是选择代表动作名称的第一个字母的键，如Ctrl S（”Ctrl+S”）或Ctrl C（”Ctrl+C”）。此外，由于某些符号只能在某些键盘布局上借助于修改键来输入，用于一个键盘布局的键序可能会映射到不同的键，或者根本没有映射到任何键，或者需要一个额外的修改键才能在不同的键盘布局上使用。

例如，快捷键Ctrl plus和Ctrl minus经常被用作图形应用程序中缩放操作的快捷键，它们可以分别被指定为 “Ctrl++”和 “Ctrl+-“。然而，这些快捷键的指定和解释方式取决于键盘布局。挪威键盘的用户会注意到，**+和-键在键盘上并不相邻，但仍能激活这两个快捷键而不需要按Shift键。然而，使用英式键盘的用户将需要按住Shift键来输入+**符号，这使得该快捷方式实际上与 “Ctrl+Shift+=”相同。

尽管一些开发者可能会采用完全指定他们在键盘上使用的所有修饰符来激活一个快捷键，但这也会导致不同键盘布局的用户产生意想不到的行为。

例如，一个使用英式键盘的开发者可能会决定指定 “Ctrl+Shift+=”作为键序，以便创建一个与Ctrl plus相同的快捷方式，这很巧。然而，**=键需要使用挪威键盘上的Shift键来访问，这使得所需的快捷键实际上是Ctrl Shift Shift =**（一个不可能的组合键）。

因此，在指定可用于各种不同键盘布局的按键序列时，人可读的字符串和硬编码的按键代码在使用上都会有问题。只有使用标准快捷键才能保证用户能够使用开发者想要的快捷键。

尽管这样，我们可以通过确保使用人类可读的字符串来解决这个问题，使得为不同语言的用户翻译按键序列成为可能。对于那些键盘具有他们所使用语言的最典型布局的用户来说，这种方法将是成功的。

GNU Emacs Style Key Sequences#

类似于GNU Emacs中使用的密钥序列，最多允许四个密钥代码，可以通过使用多参数构造器来创建，也可以通过传递逗号分隔的密钥序列的人类可读字符串。

例如，Ctrl X后面是Ctrl C，可以用以下任何一种方式指定按键序列：

QKeySequence(tr("Ctrl+X, Ctrl+C"));
QKeySequence(Qt::CTRL | Qt::Key_X, Qt::CTRL | Qt::Key_C);
QKeySequence(Qt::CTRL + Qt::Key_X, Qt::CTRL + Qt::Key_C); // deprecated

警告：在创建QKeySequence之前，必须已经构建了一个QApplication实例；否则，你的应用程序可能会崩溃。

See also QShortcut.

QKeySequence::NativeText 0 按键序列是一个特定平台的字符串。这意味着它将被翻译显示，在Mac上，它将类似于菜单栏中的一个键序。这个枚举最好在你想向用户显示该字符串时使用。

QKeySequence::QKeySequence( key)#

为给定的key构建一个QKeySequence对象。其结果将取决于当前运行的平台。

结果对象将基于key的键绑定列表中的第一个元素。

QKeySequence::QKeySequence(const &keysequence)#

复制构造函数。制作一个keysequence的副本。

QKeySequence::QKeySequence( k1, k2 = QKeyCombination::fromCombined(0), k3 = QKeyCombination::fromCombined(0), k4 = QKeyCombination::fromCombined(0))#

用最多4个键k1、k2、k3和k4构建一个按键序列。

See also QKeyCombination.

QKeySequence::QKeySequence(int k1, int k2 = 0, int k3 = 0, int k4 = 0)#

用最多4个键k1、k2、k3和k4构建一个按键序列。

键码在 Qt::Key 中列出，并且可以与修改器（见 Qt::Modifier）结合，如 [Qt::SHIFT](qt. html#Modifier-enum），Qt::CTRL，Qt::ALT，或 Qt::META。

QKeySequence::QKeySequence(const &key, format = NativeText)#

根据格式，从键字符串创建一个键序。

例如，”Ctrl+O “给出CTRL+’O’。字符串 “Ctrl”、”Shift”、”Alt “和 “Meta “被识别，以及它们在”QShortcut “上下文中的翻译对应物（使用 QObject::tr()）。

最多可以输入四个按键代码，用逗号隔开，例如：”Alt+X,Ctrl+S,Q”。

这个构造函数通常与tr()一起使用，这样就可以在翻译中替换快捷键：

QMenu *file = new QMenu(this);
file->addAction(tr("&Open..."), this, SLOT(open()),
                  QKeySequence(tr("Ctrl+O", "File|Open")));

请注意 “文件|打开 “的译者注释。这绝不是必要的，但它为人类译者提供了一些背景。

QKeySequence::matches(const &seq) const#

将序列与seq相匹配。如果匹配成功，返回ExactMatch；如果seq不完全匹配，返回PartialMatch；如果两个序列没有共同点，返回NoMatch。如果seq比较短，则返回NoMatch。

[static] QKeySequence::mnemonic(const &text)#

返回text中记忆体的快捷键序列，如果没有找到记忆体，则返回一个空键序列。

例如，mnemonic(“E&xit”)返回Qt::ALT+Qt::Key_X，mnemonic(“&Quit”)返回ALT+Key_Q，mnemonic(“Quit”)返回一个空QKeySequence。

我们提供了一个英文的常用助记符列表。在撰写本文时，微软和开放集团似乎没有为其他语言发布相应的建议。

QKeySequence::toString( format = PortableText) const#

基于format，返回按键序列的字符串表示。

例如，Qt::CTRL+Qt::Key_O的值会产生 “Ctrl+O”。如果按键序列有多个按键代码，在返回的字符串中每个都用逗号隔开，例如 “Alt+X, Ctrl+Y, Z”。字符串 “Ctrl”、”Shift “等在”QShortcut “上下文中使用QObject::tr()翻译。

如果按键序列没有按键，将返回一个空字符串。

在macOS上，如果format是QKeySequence::NativeText，返回的字符串类似于在菜单栏中显示的序列；否则，字符串使用 “便携式 “格式，适合写入文件。

See also fromString().

bool QKeySequence::operator<(const &other) const#

提供此密钥序列和other密钥序列的任意比较。所能保证的是，如果两个键序列相等，操作者返回false；如果键序列不相等，则返回(ks1 < ks2) == !( ks2 < ks1)。

这个函数在某些情况下是有用的，例如，如果你想使用QKeySequence对象作为QMap中的键。

See also operator==(), operator!=(), operator>(), operator<=(), and operator>=().

void qt_set_sequence_auto_mnemonic(bool b)#

指定是否应该尊重菜单项、标签等的助记符。在Windows和X11上，这个功能默认是打开的；在MacOS上，它是关闭的。当这个功能关闭时（也就是说，当b为假时），QKeySequence::mnemonic()总是返回一个空字符串。

注意：这个函数没有在Qt的任何头文件中声明。要在你的应用程序中使用它，请在调用它之前声明该函数的原型。
