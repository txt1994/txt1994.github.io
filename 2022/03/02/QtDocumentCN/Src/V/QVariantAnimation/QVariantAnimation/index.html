详细描述#

这个类是动画框架的一部分。它用作属性和项目动画的基类，具有共享功能的功能。

该类在QVariant上执行插值，但将使用插值的值留给其子类。目前，Qt 提供了QPropertyAnimation，它为 Qt属性设置动画。如果您希望为此类属性设置动画，请参阅QPropertyAnimation类描述。

然后可以通过调用setStartValue ()和setEndValue ()设置属性的开始值和结束值，最后调用start ()开始动画。QVariantAnimation 将插入目标对象的属性并发出valueChanged ()。要对当前值的变化做出反应，您必须重新实现updateCurrentValue () 虚函数或连接到所述信号。

还可以在位于开始值和结束值之间的指定步长处设置值。然后，插值将以指定的步骤接触这些点。请注意，开始值和结束值定义为 0.0 和 1.0 处的键值。

有两种方法可以影响 QVariantAnimation 插入值的方式。您可以通过调用setEasingCurve ()设置缓动曲线，并通过调用setDuration () 配置持续时间。您可以通过创建 QVariantAnimation 的子类并重新实现 virtual interpolated () 函数来更改QVariant的插值方式。

如果您不想将QVariant声明为 Qt 属性，则子类化 QVariantAnimation 可能是一种替代方法。但是请注意，在大多数情况下，您最好将QVariant声明为属性。

并非所有QVariant类型都受支持。以下是当前支持的QVariant类型的列表：

  • Int
  • UInt
  • Double
  • Float
  • QLine
  • QLineF
  • QPoint
  • QPointF
  • QSize
  • QSizeF
  • QRect
  • QRectF
  • QColor

如果您需要对其他变体类型（包括自定义类型）进行插值，则必须自己为这些类型实现插值。为此，您可以为给定类型注册一个插值器函数。该函数接受 3 个参数：开始值、结束值和当前进度。

例子：

QVariant myColorInterpolator(const QColor &start, const QColor &end, qreal progress)
{
    ...
        return QColor(...);
}
...
    qRegisterAnimationInterpolator<QColor>(myColorInterpolator);

另一种选择是重新实现interpolated ()，它返回被插值的插值。

另请参阅QPropertyAnimation、QAbstractAnimation和动画框架。

属性文档#

[read-only]currentValue : const #

此属性保存动画的当前值。

该属性描述当前值；开始值和结束值之间的插值，使用当前时间进行进度。值本身是从interpolated () 中获取的，在动画运行时会重复调用。

QVariantAnimation在当前值改变时调用虚拟updateCurrentValue () 函数。这对于需要跟踪更新的子类特别有用。例如，QPropertyAnimation使用此函数为 Qt属性设置动画。

访问功能：

QVariant currentValue() const

通知信号：

void valueChanged(const QVariant &value)

See also startValue and endValue.

[bindable]duration : int#

注意：此属性支持QProperty绑定。

该属性保存动画的持续时间

此属性描述动画的持续时间（以毫秒为单位）。默认持续时间为 250 毫秒。

另见QAbstractAnimation::duration ()。

[bindable]easingCurve : #

注意：此属性支持QProperty绑定。

该属性保存动画的缓动曲线

此属性定义动画的缓动曲线。默认情况下，使用线性缓动曲线，从而产生线性插值。提供了其他曲线，例如QEasingCurve::InCirc，它提供了一个圆形入口曲线。另一个例子是QEasingCurve::InOutElastic，它对插值变量的值提供弹性效果。

QVariantAnimation会使用QEasingCurve::valueForProgress () 将动画的“归一化进度”（currentTime / totalDuration）转化为动画实际使用的有效进度。调用interpolated ()时，正是这个有效的进度。此外，keyValues中的步骤是指这个有效的进展。

缓动曲线与插值器、interpolated () 虚函数和动画的持续时间一起使用，以控制当前值如何随着动画的进行而变化。

endValue : #

该属性保存动画的结束值

此属性描述动画的结束值。

访问功能：

QVariant endValue() const
void setEndValue(const QVariant &value)

See also startValue.

startValue : #

该属性保存动画的可选开始值

此属性描述动画的可选开始值。如果省略，或者如果指定了一个空QVariant作为开始值，则动画将在动画开始时使用结束的当前位置。

访问功能：

QVariant startValue() const
void setStartValue(const QVariant &value)

See also endValue.

[virtual protected] QVariantAnimation::interpolated(const &from, const &to, progress) const#

这个虚函数返回变量from和to之间的线性插值，在progress中，通常是一个介于 0 和 1 之间的值。您可以在QVariantAnimation的子类中重新实现此函数以提供您自己的插值算法。

请注意，为了使插值与返回值小于 0 或大于 1的QEasingCurve一起工作（例如QEasingCurve::InBack），您应该确保它可以外推。如果数据类型的语义不允许外推，则此函数应妥善处理。

如果您希望您的类处理 Qt 已经支持的类型，您应该调用此函数的QVariantAnimation实现（有关支持的类型列表，请参见类QVariantAnimation描述）。

另请参见QEasingCurve。

QVariantAnimation::keyValueAt( step) const#

返回给定step的关键帧值。给定的step必须在 0 到 1 的范围内。如果**step没有KeyValue，则返回无效的QVariant。

另见keyValues () 和setKeyValueAt ()。

QVariantAnimation::keyValues() const#

返回此动画的关键帧。

另请参见keyValueAt () 和setKeyValues ()。

void QVariantAnimation::setKeyValueAt( step, const &value)#

在给定的step中使用给定的value创建一个关键帧。给定的步长必须在 0 到 1 的范围内。

另请参见setKeyValues () 和keyValueAt ()。

void QVariantAnimation::setKeyValues(const &keyValues)#

用给定的keyValues替换当前的一组关键帧。关键帧的步长必须在 0 到 1 的范围内。

另请参见keyValues () 和keyValueAt ()。

[override virtual protected]void QVariantAnimation::updateCurrentTime(int)#

重新实现: QAbstractAnimation::updateCurrentTime(int currentTime).

[virtual protected]void QVariantAnimation::updateCurrentValue(const &value)#

每次动画的当前值改变时都会调用这个虚函数。value参数是新的当前值。

基类实现什么也不做。

See also currentValue.

[override virtual protected]void QVariantAnimation::updateState( newState, oldState)#

重新实现: QAbstractAnimation::updateState(QAbstractAnimation::State newState, QAbstractAnimation::State oldState).

相关非成员#

template void qRegisterAnimationInterpolator(QVariant (*)(const T &, const T &, qreal) func)#

为模板类型注册一个自定义插值函数T。必须在构造动画之前注册插值器。要取消注册（并使用默认插值器） ，请将func设置为nullptr.

注意：这个函数是线程安全的。
