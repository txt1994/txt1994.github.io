详细描述#

QByteArrayView 引用它不拥有的原始字节的连续部分。它充当各种类字节数组数据的接口类型，无需先构造 QByteArray。

字节数组数据可以表示为数组（或数组兼容的数据结构，例如 QByteArray、std::basic_string 等.) 的 char、signed char、unsigned char 或 std::byte。

QByteArrayView 被设计为接口类型；它的主要用例是作为函数参数类型。当 QByteArrayViews 用作自动变量或数据成员时，必须注意确保引用的数据（例如，由 [QByteArray] 拥有（QByteArray.md） ) 在所有代码路径上都比 QByteArrayView 的寿命更长，以免字节数组视图最终引用已删除的数据。

当用作接口类型时，QByteArrayView 允许单个函数接受各种类似字节数组的数据源。因此，一个接受 QByteArrayView 的函数同时替换了几个函数重载（例如，QByteArray、const char * 等）允许将更多字节数组数据源传递给函数。

QByteArrayView 应该按值传递，而不是通过引用到常量：

void myfun1(QByteArrayView bv);        // preferred
void myfun2(const QByteArrayView &bv); // compiles and works, but slower

如果你想让你的用户在他们可以传递给你的函数的数据类型方面有最大的自由度，请伴随 QByteArrayView 重载和重载

  • char

    : 这个重载可以委托给 QByteArrayView 版本:

    void fun(QByteArrayView bv);
void fun(char ch) { fun(QByteArrayView(&ch, 1)); }

    即使出于技术原因，QByteArrayView 本身也无法提供 char 构造函数。

  • QByteArray：如果您存储字节数组的未修改副本，因此想利用 QByteArray 的隐式共享。

QByteArrayView 也可以用作函数的返回值。 如果您调用返回 QByteArrayView 的函数，请特别注意不要将 QByteArrayView 保留的时间长于函数承诺保持引用数据活动的时间。 如果有疑问，通过调用 toByteArray() 将 QByteArrayView 转换为 [QByteArray](https ://doc.qt.io/qt-6/qbytearray.html)。

qsizetype QByteArrayView::lastIndexOf( bv, qsizetype from) const#

qsizetype QByteArrayView::lastIndexOf(char ch, qsizetype from = -1) const#

在此字节数组视图中，分别返回 bv 查看的字节序列的最后一次出现的开始或字节 ch 的最后一次出现的开始的索引位置，从索引位置 from 向后搜索。 如果 from 为 -1，则从最后一个字符开始搜索； 如果 from 为 -2，则在倒数第二个字符处，依此类推。 如果未找到匹配项，则返回 -1。

注意： 当搜索长度为 0 的 bv 时，数据末尾的匹配会被否定的 from 排除在搜索之外，即使通常认为 -1 是从搜索开始 视图的结尾：最后的匹配是在最后一个字符之后，所以它被排除在外。 要包含这样一个最终的空匹配，要么给 from 一个正值，要么完全省略 from 参数。

也可以看看 indexOf() and contains().

template QByteArrayView::QByteArrayView(const char (&)[Size] data = Size)#

在 char 数组 data 上构造一个字节数组视图。 视图覆盖数组，直到遇到第一个 '\0' 或 Size，以先到者为准。 如果您需要完整的数组，请改用 fromArray()。

data 必须在此字节数组视图对象的生命周期内保持有效。

注意： 此构造函数仅适用于 char 数组文字。 其背后的原因是为了与预定义“足够大”数组但仅使用一些预分配空间的 C 库兼容。 为了在隐式构造函数重载中以直观的方式支持这一点，我们需要在第一个 char(0) 处停止。 这对于 char 数组是合乎逻辑的，但对于 std::byte 数组则不然。

也可以看看 fromArray.

template QByteArrayView::QByteArrayView(const Byte *data)#

在 data 上构造一个字节数组视图。 长度是通过扫描第一个 Byte(0) 来确定的。

data 必须在此字节数组视图对象的生命周期内保持有效。

将 nullptr 作为 data 传递是安全的，并且会生成空字节数组视图。

如果 data 不是数组并且 Byte 是兼容的字节类型，则此构造函数仅参与重载决议。

也可以看看 Compatible Byte Types.

template <typename Byte, if_compatible_byte> QByteArrayView::QByteArrayView(const Byte *first, const Byte *last)#

在 first 上构造一个长度为 (last - first) 的字节数组视图。

范围 [first,last) 必须在此 QByteArrayView 的生命周期内保持有效。

如果 last 也是 nullptr，则将 \nullptr 作为 first 传递是安全的，并且会导致空字节数组视图。

如果 last 在 first 之前，或者 first 是 nullptr 而 last 不是，则行为未定义。

如果 Byte 是兼容的字节类型，则此构造函数仅参与重载决议。

也可以看看 Compatible Byte Types.
