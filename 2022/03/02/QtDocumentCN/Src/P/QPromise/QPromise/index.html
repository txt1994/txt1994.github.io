详细描述#

QPromise提供了一种简单的方法，以异步的方式将用户定义的计算的进度和结果传达给QFuture。为了使通信发挥作用，QFuture必须由QPromise构建。

当需要细粒度的控制或者需要高水平的通信基元来配合QFuture时，你可以使用基于QPromise的工作负载作为Qt Concurrent框架的一个替代。

promise和future协作的最简单的情况是一个单一的结果通信:

QPromise<int> promise;
QFuture<int> future = promise.future();

QScopedPointer<QThread> thread(QThread::create([] (QPromise<int> promise) {
    promise.start();   // notifies QFuture that the computation is started
    promise.addResult(42);
    promise.finish();  // notifies QFuture that the computation is finished
}, std::move(promise)));
thread->start();

future.waitForFinished();  // blocks until QPromise::finish is called
future.result();  // returns 42

根据设计，QPromise是一个只移动的对象。这种行为有助于确保每当承诺被销毁时，相关的未来对象会得到通知，而不会永远等待结果的出现。然而，如果想用同一个承诺来报告不同线程的结果，这就不太方便了。目前还没有具体的方法来做到这一点，但已知的机制是存在的，例如使用智能指针或原始指针/引用。如果你想复制你的承诺并在多个地方同时使用它，QSharedPointer是一个不错的默认选择。从某种意义上说，原始指针或引用更容易，而且可能表现得更好（因为不需要做资源管理），但可能导致悬空。

下面是一个关于如何在多个线程中使用承诺的例子:

QSharedPointer<QPromise<int>> sharedPromise(new QPromise<int>());
QFuture<int> future = sharedPromise->future();

// ...

// here, QPromise is shared between threads via a smart pointer
QScopedPointer<QThread> threads[] = {
    QScopedPointer<QThread>(QThread::create([] (auto sharedPromise) {
        sharedPromise->addResult(0, 0);  // adds value 0 by index 0
    }, sharedPromise)),
    QScopedPointer<QThread>(QThread::create([] (auto sharedPromise) {
        sharedPromise->addResult(-1, 1);  // adds value -1 by index 1
    }, sharedPromise)),
    QScopedPointer<QThread>(QThread::create([] (auto sharedPromise) {
        sharedPromise->addResult(-2, 2);  // adds value -2 by index 2
    }, sharedPromise)),
    // ...
};
// start all threads
for (auto& t : threads)
    t->start();

// ...

future.resultAt(0);  // waits until result at index 0 becomes available. returns value  0
future.resultAt(1);  // waits until result at index 1 becomes available. returns value -1
future.resultAt(2);  // waits until result at index 2 becomes available. returns value -2

See also QFuture.

成员函数文档#

bool QPromise::addResult(T &&result, int index = -1)#

bool QPromise::addResult(const T &result, int index = -1)#

将result添加到内部结果集合的index位置。如果没有指定索引，result将被添加到集合的末端。

当result被添加到集合中时返回true。

当这个承诺处于取消或完成状态时，或者当result被拒绝时，返回false。addResult() 如果在集合中已经有另一个结果存储在相同的索引中，则拒绝result。

你可以通过调用QFuture::resultAt()来获得一个特定索引的结果。

注意：可以指定一个任意的索引并请求该索引的结果。然而，一些QFuture方法是用连续的结果来操作的。例如，使用QFuture::resultCount()或QFuture::const_iterator 的迭代方法。为了获得所有可用的结果，而不考虑是否存在索引间隙，使用QFuture::results()。

QPromise::QPromise( &&other)#

从other移动构造一个新的QPromise。

See also operator=().

QPromise::QPromise()#

构建一个具有默认状态的QPromise。

&QPromise::operator=(QPromise &&other)#

Move将other分配给这个承诺，并返回对这个承诺的引用。

QPromise::~QPromise()#

销毁该承诺。

注意：承诺在销毁时隐含地过渡到取消状态，除非用户事先调用finish()。

void QPromise::finish()#

报告计算完成。 完成后，调用 addResult() 时不会添加新结果。 此方法伴随 start()。

See also QFuture::isFinished(), QFuture::waitForFinished(), and start().

QPromise::future() const#

返回与此承诺关联的未来。

bool QPromise::isCanceled() const#

返回计算是否已经被QFuture::cancel()函数取消。返回值 “true “表示计算应该结束并调用finish()。

注意：取消后，当前可用的结果仍然可以被future访问，但在调用addResult()时，新的结果将不会被添加。

void QPromise::setException(const &e)#

设置异常e为计算的结果。

注：在整个计算执行过程中，你最多可以设置一个异常。

注意：这个方法不能在QFuture::cancel()或finish()之后使用。

See also isCanceled().

void QPromise::setException(std::exception_ptr e)#

这是一个重载函数。

void QPromise::setProgressRange(int minimum, int maximum)#

设置计算的进度范围为minimum和maximum之间。

如果maximum小于minimum，则minimum成为唯一合法的值。

进度值被重置为minimum。

进度范围的使用可以通过setProgressRange(0, 0)禁用。在这种情况下，进度值也会被重置为0。

See also QFuture::progressMinimum(), QFuture::progressMaximum(), and QFuture::progressValue().

void QPromise::setProgressValue(int progressValue)#

将计算的进度值设置为progressValue。有可能只递增进度值。这是一个调用setProgressValueAndText(progressValue, QString())的方便方法。

如果progressValue跌出了进度范围，这个方法就没有效果。

See also QFuture::progressValue() and setProgressRange().

void QPromise::setProgressValueAndText(int progressValue, const &progressText)#

将计算的进度值和进度文本分别设置为progressValue和progressText。可以只增加进度值。

注意：如果承诺处于取消或完成状态，该函数没有效果。

See also QFuture::progressValue(), QFuture::progressText(), QFuture::cancel(), and finish().

void QPromise::start()#

报告计算已经开始。调用这个方法对于说明计算的开始是很重要的，因为QFuture方法依赖于这个信息。

注意：当start()被从一个新创建的线程调用时，需要特别注意。在这种情况下，由于线程调度的实现细节，该调用可能会自然地被延迟。

See also QFuture::isStarted(), QFuture::waitForFinished(), and finish().

void QPromise::suspendIfRequested()#

有条件地暂停当前的执行线程并等待，直到被QFuture的相应方法恢复或取消。除非计算被QFuture::suspend()或其他相关方法请求暂停，否则该方法不会阻塞。如果你想检查执行是否已经被暂停，请使用QFuture::isSuspended()。

注意：当在多个线程中使用同一个承诺时，只要至少有一个使用该承诺的线程暂停，QFuture::isSuspended()就会变成true。

下面的代码片断显示了暂停机制的用法:

// Create promise and future
QPromise<int> promise;
QFuture<int> future = promise.future();

promise.start();
// Start a computation thread that supports suspension and cancellation
QScopedPointer<QThread> thread(QThread::create([] (QPromise<int> promise) {
    for (int i = 0; i < 100; ++i) {
        promise.addResult(i);
        promise.suspendIfRequested();   // support suspension
        if (promise.isCanceled())       // support cancellation
            break;
    }
    promise.finish();
}, std::move(promise)));
thread->start();

QFuture::suspend()请求暂停相关的承诺:

future.suspend();

在QFuture::isSuspended()变成true之后，你可以得到中间结果。

future.resultCount();  // returns some number between 0 and 100
for (int i = 0; i < future.resultCount(); ++i) {
    // process results available before suspension
}

暂停时，您可以恢复或取消等待的计算：

future.resume();  // resumes computation, this call will unblock the promise
// alternatively, call future.cancel() to stop the computation

future.waitForFinished();
future.results();  // returns all computation results - array of values from 0 to 99

See also QFuture::resume(), QFuture::cancel(), QFuture::setSuspended(), and QFuture::toggleSuspended().

void QPromise::swap( &other)#

用这个承诺交换other承诺。这个操作非常快，而且从不失败。
