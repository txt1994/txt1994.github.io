详细描述#

QProperty 是实现 [Qt Bindable Properties] (bindableproperties.html) 的类之一。你可以给它赋值，也可以通过value()函数或T转换操作符读取它。你也可以将该属性与一个动态计算值的表达式绑定，即绑定表达式。它被表示为一个C++的lambda，可以用来表达你的应用程序中不同属性之间的关系。

QProperty::~QProperty()#

销毁该属性。

template QPropertyNotifier QProperty::addNotifier(Functor f)#

订阅给定的functor f作为回调，每当属性的值发生变化时就会调用。

回调f被期望是一个具有普通调用操作符（）的类型，没有任何参数。这意味着你可以提供一个C++ lambda表达式，一个std::函数，甚至一个带有调用操作符的自定义结构。

返回的属性变化处理程序对象会保持对订阅的跟踪。当它超出范围时，回调会被取消订阅。

这个方法在某些情况下比onValueChanged()更容易使用，因为返回的对象不是一个模板。因此，它可以更容易地被存储，例如，作为一个类中的成员。

See also onValueChanged() and subscribe().

QPropertyBinding QProperty::binding() const#

返回与此属性相关的绑定表达式。如果不存在这样的关联，将返回一个默认构造的QPropertyBinding。

See also setBinding().

template QPropertyChangeHandler QProperty::onValueChanged(Functor f)#

注册给定的漏斗f作为回调，每当属性的值发生变化时就会被调用。在每个值的变化中，处理程序要么被立即调用，要么被推迟，这取决于上下文。

回调f被期望是一个具有普通调用操作符（）的类型，没有任何参数。这意味着你可以提供一个C++ lambda表达式，一个std::函数，甚至一个带有调用操作符的自定义结构。

返回的属性变更处理程序对象会保持对注册的跟踪。当它超出范围时，回调被取消注册。

QPropertyBinding QProperty::setBinding(const QPropertyBinding &newBinding)#

将此属性的值与提供的newBinding表达式联系起来，并返回先前关联的绑定。第一次读取属性值时，绑定被评估。每当绑定的依赖性改变时，绑定将在下次读取此属性的值时被重新评估。

See also binding().

bool QProperty::setBinding(const QUntypedPropertyBinding &newBinding)#

这是一个重载函数。

将此属性的值与提供的newBinding表达式联系起来。第一次读取属性值时，绑定被评估。每当绑定的依赖性改变时，绑定将在下次读取此属性的值时被重新评估。

如果该属性的类型与绑定函数返回的类型相同，则返回真；否则返回假。

template QPropertyBinding QProperty::setBinding(Functor f)#

这是一个重载函数。

将此属性的值与所提供的漏斗f相关联，并返回先前关联的绑定。在第一次读取属性值时，通过调用f的调用操作符（）来评估绑定。每当绑定的依赖性改变时，绑定将在下次读取该属性值时被重新评估。

See also Formulating a Property Binding.

template QPropertyChangeHandler QProperty::subscribe(Functor f)#

订阅给定的函数f作为一个回调，该回调会被立即调用，并且在未来该属性的值发生变化时被调用。在每个值的变化中，处理程序要么被立即调用，要么被推迟，这取决于上下文。

回调f应该是一个可以被复制的类型，并且有一个没有任何参数的普通调用操作符（）。这意味着你可以提供一个C++ lambda表达式，一个std::函数，甚至一个带有调用操作符的自定义结构。

返回的属性变化处理程序对象会保持对订阅的跟踪。当它超出范围时，回调被取消订阅。

QPropertyBinding QProperty::takeBinding()#

将绑定表达式与该属性解除关联并返回。调用这个函数后，只有当你给它分配了一个新的值，或者设置了一个新的绑定时，该属性的值才会改变。

QProperty::parameter_type QProperty::value() const#

返回该属性的值。在返回值之前，这可能会评估一个与此属性绑定的绑定表达式。

See also setValue().

非相关成员#

[since 6.2]void beginPropertyUpdateGroup()#

标志着一个属性更新组的开始。在这个组内，改变一个属性既不会立即更新任何附属属性，也不会触发改变通知。这些都被推迟，直到调用endPropertyUpdateGroup来结束这个组。

组可以被嵌套。在这种情况下，只有在最外层的组被结束后，延迟才会结束。

注意：只有在所有受该组影响的属性值被更新为新值后，才会发送变更通知。这允许在多个属性需要更新时重新建立一个类的不变性，防止任何外部观察者注意到一个不一致的状态。

这个函数是在 Qt 6.2 中引入的。

See also Qt::endPropertyUpdateGroup.

[since 6.2]void endPropertyUpdateGroup()#

结束一个属性更新组。如果最外层的组已经被结束，并且现在发生了延迟的绑定评估和通知。

警告：调用endPropertyUpdateGroup而不在前面调用beginPropertyUpdateGroup会导致未定义行为。

这个函数是在 Qt 6.2 中引入的。

See also Qt::beginPropertyUpdateGroup.
