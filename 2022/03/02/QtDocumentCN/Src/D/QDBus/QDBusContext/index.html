详细描述#

当一个槽由于信号传递或由于远程方法调用而在一个对象中被调用时，有时需要知道发生的背景。特别是，如果槽确定它想在稍后的机会发送回复，或者它想用一个错误来回复，就需要上下文了。

QDBusContext类是访问上下文的一个替代方案，它不需要修改由Qt D-Bus XML 编译器(qdbusxml2cpp)生成的代码。

QDBusContext是通过使用QDBusConnection::registerObject()从被导出的对象中子类化出来而使用。下面的例子说明了这种用法:

class MyObject: public QObject,
                protected QDBusContext
{
    Q_OBJECT
    QDBusConnection conn;
    QDBusMessage msg;
    //...
protected slots:
    void process();
public slots:
    void methodWithError();
    QString methodWithDelayedReply();
};
void MyObject::methodWithError()
{
    sendErrorReply(QDBusError::NotSupported,
                   "The method call 'methodWithError()' is not supported");
}
QString MyObject::methodWithDelayedReply()
{
    conn = connection();
    msg = message();
    setDelayedReply(true);
    QMetaObject::invokeMethod(this, "process", Qt::QueuedConnection);
    return QString();
}

这个例子说明了两种典型的用法，即发送错误的回复和延迟回复的用法。

注意：不要同时子类化QDBusContext和[QDBusAbstractAdaptor]（qdbusabstractadaptor.html）。QDBusContext应该出现在真正的对象中，而不是适配器中。如果有必要从适配器代码中确定上下文，请使用公共继承，并通过 QObject::parent ()访问这些函数。

bool QDBusContext:: calledFromDBus () const#

返回 true 如果我们正在处理一个D-Bus调用。如果这个函数返回 “true”，这个类中的其他函数是可用的。

当这个函数返回 “false “时，访问这些函数是未定义的，可能导致崩溃。

void QDBusContext:: sendErrorReply (const & name , const & msg = QString()) const#

向调用者发送一个错误名称作为回复。可选的msg参数是一个人类可读的文本，解释失败的原因。

如果发送了一个错误，Qt D-Bus将忽略被调用槽的返回值和任何输出参数。

void QDBusContext:: sendErrorReply ( type , const & msg = QString()) const#

这是重载函数。

向调用者发送一个错误type作为回复。可选的msg参数是一个人可读的文本，解释失败的原因。

如果发送了一个错误，Qt D-Bus将忽略被调用槽的返回值和任何输出参数。

void QDBusContext:: setDelayedReply ( bool enable ) const#

设置这个调用是否会有一个延迟回复。

若enable为false，Qt D-Bus将在被调用槽返回后，自动生成一个回复给调用者，如果需要的话。

若enable为true，Qt D-Bus将不会产生自动回复。它也会忽略槽的返回值和任何输出参数。相反，被调用的对象负责存储传入的消息，并在稍后的时间发送一个回复或错误。

未能发送回复将导致D-Bus产生一个自动超时的错误。

另请参阅 isDelayedReply ().
