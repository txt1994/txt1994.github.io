QDBusAbstractInterface 类#

QDBusAbstractInterface 类是 Qt D-Bus 绑定中所有 D-Bus 接口的基类 (允许访问远程接口)。 更多…

Header: #include
CMake: find_package(Qt6 COMPONENTS Dbus REQUIRED) target_link_libraries(mytarget PRIVATE Qt6::Dbus)
qmake: QT += dbus
继承： QObject
继承者： QDBusConnectionInterface and QDBusInterface

详细描述#

生成的代码类也派生自QDBusAbstractInterface，这里描述的所有方法也对生成的代码类有效。除了这里描述的方法，生成的代码类还为远程方法提供了成员函数，这使得编译时可以检查正确的参数和返回值，以及属性类型匹配和信号参数匹配。

template QDBusPendingCall QDBusAbstractInterface:: asyncCall (const QString & method , Args &&… args )#

调用此接口上的方法method，并将args传给该方法。所有的args必须可以转换为[QVariant]（qvariant.html）。

呼叫 “的参数通过D-Bus作为输入参数传递给远程函数。返回的QDBusPendingCall对象可以用来找出关于回复的信息。

它可以按以下方式使用：

QString value = retrieveValue();
QDBusPendingCall pcall = interface->asyncCall(QLatin1String("Process"), value);
QDBusPendingCallWatcher *watcher = new QDBusPendingCallWatcher(pcall);
QObject::connect(watcher, SIGNAL(finished(QDBusPendingCallWatcher*)),
                this, SLOT(callFinishedSlot(QDBusPendingCallWatcher*)));

这个例子说明了带0、1和2个参数的函数调用，并说明了每个参数传递的不同类型（第一次调用"ProcessWorkUnicode"将包含一个Unicode字符串，第二次调用"ProcessWork"将包含一个字符串和一个字节阵列）。

注意：在Qt 5.14之前，这个函数最多只接受八（8）个参数。

另请参阅 asyncCallWithArgumentList ().

QDBusAbstractInterface:: asyncCallWithArgumentList (const & method , const < > & args )#

在此接口上放置一个对method指定的远程方法的调用，使用args作为参数。这个函数返回一个QDBusPendingCall对象，可以用来跟踪回复的状态，并在回复到达时访问其内容。

通常情况下，你应该使用asyncCall()来放置调用。

注意： 此函数是 thread-safe .

template QDBusMessage QDBusAbstractInterface:: call (const QString & method , Args &&… args )#

调用此接口上的方法method，并将args传给该方法。所有的args必须可以转换为[QVariant]（qvariant.html）。

呼叫 “的参数通过D-Bus传递给远程函数作为输入参数。输出参数在QDBusMessage回复中返回。如果回复是一个错误回复，lastError()也将被设置为错误信息的内容。

它可以按以下方式使用:

QString value = retrieveValue();
QDBusMessage reply;
QDBusReply<int> api = interface->call(QLatin1String("GetAPIVersion"));
if (api >= 14)
  reply = interface->call(QLatin1String("ProcessWorkUnicode"), value);
else
  reply = interface->call(QLatin1String("ProcessWork"), QLatin1String("UTF-8"), value.toUtf8());

这个例子说明了带0、1和2个参数的函数调用，并说明了每个参数传递的不同类型（第一次调用"ProcessWorkUnicode"将包含一个Unicode字符串，第二次调用"ProcessWork"将包含一个字符串和一个字节阵列）。

注意：在Qt 5.14之前，这个函数最多只接受八（8）个参数。

另请参阅 callWithArgumentList ().

template QDBusMessage QDBusAbstractInterface:: call ( QDBus::CallMode mode , const QString & method , Args &&… args )#

这是重载函数。

调用此接口上的方法method，并将args传给该方法。所有的args必须可以转换为[QVariant]（qvariant.html）。

如果mode是 “NoWaitForReply”，那么这个函数在发出调用后将立即返回，而不等待远程方法的回复。否则，mode表示该函数是否应该在等待回复时激活Qt Event Loop。

如果这个函数为了等待回复而重新进入Qt事件循环，它将排除用户输入。在等待过程中，它可能向你的应用程序传递信号和其他方法调用。因此，每当用call()进行调用时，它必须准备好处理重入的问题。

注意： 在Qt 5.14之前，这个函数最多只接受八（8）个参数。

另请参阅 callWithArgumentList ().

QDBusAbstractInterface:: callWithArgumentList ( mode , const & method , const < > & args )#

在这个接口上对method指定的远程方法进行调用，使用args作为参数。这个函数返回收到的回复信息，可以是普通的QDBusMessage::ReplyMessage （表示成功）或者QDBusMessage::ErrorMessage （如果调用失败）。参数mode指定了这个调用应该如何放置。

如果调用成功，lastError()将被清除；否则，它将包含这个调用产生的错误。

通常，你应该使用call()来放置调用。

警告： 若使用 UseEventLoop ，你的代码必须准备好处理任何重入：在这个函数返回之前，其他方法调用和信号可能被传递，以及其他Qt队列信号和事件。

注意： 此函数是 thread-safe .

bool QDBusAbstractInterface:: callWithCallback (const & method , const < > & args , * receiver , const char * returnMethod , const char * errorMethod )#

在这个接口上对method指定的远程方法进行调用，使用args作为参数。这个函数在排队调用后立即返回。来自远程函数的回复被传递给对象receiver的returnMethod。如果发生错误，会调用对象receiver上的errorMethod。

此函数返回 true 如果排队成功。它并不表示执行的调用成功了。如果失败，则调用errorMethod。如果排队失败，此函数返回 false ，没有槽会被调用。

returnMethod必须有函数调用所返回的类型作为其参数。可以选择，它可以有一个QDBusMessage参数作为它的最后或唯一的参数。errorMethod必须有一个QDBusError作为它的唯一参数。

另请参阅 QDBusError and QDBusMessage .

bool QDBusAbstractInterface:: callWithCallback (const & method , const < > & args , * receiver , const char * slot )#

这是重载函数。

这个函数已被废弃。请使用重载版本。

使用args作为参数，在此接口上放置对method指定的远程方法的调用。该函数在排队调用后立即返回。来自远程函数的回复或其发出的任何错误都会被传递到对象receiver的slot槽中。

此函数返回 true 如果排队成功：这并不表示调用成功。如果失败，槽将被调用，并有一个错误信息。在这种情况下，lastError()将不会被设置。

另请参阅 QDBusError and QDBusMessage .

bool QDBusAbstractInterface:: isValid () const#

返回 true 如果这是对一个远程对象的有效引用。如果在创建这个接口的过程中出现了错误（例如，如果远程应用程序不存在），它将返回`alse。

注意：当处理远程对象时，在创建QDBusInterface时不一定能确定它是否存在。

void QDBusAbstractInterface:: setTimeout ( int timeout )#

将未来所有DBus调用的超时时间以毫秒为单位设置为timeout。-1表示默认的DBus超时（通常是25秒）。

另请参阅 timeout ().

返回超时的当前值，单位是毫秒。-1表示默认的DBus超时（通常是25秒）。
