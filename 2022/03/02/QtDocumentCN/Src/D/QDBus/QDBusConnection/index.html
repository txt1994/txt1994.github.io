详细描述#

这个类是一个D-Bus会话的初始点。使用它，你可以获得对远程对象、接口的访问；将远程信号连接到你的对象的槽中；注册对象等等。

D-Bus连接是通过connectToBus()函数创建的，它打开一个与服务器守护进程的连接，并进行初始握手，将该连接与一个名字联系起来。进一步尝试使用相同的名字进行连接将返回相同的连接。

然后使用disconnectFromBus()函数关闭连接。

一旦断开连接，调用connectToBus ()将不会重新建立连接，你必须创建一个新的QDBusConnection实例。

为了方便两种最常见的连接类型，sessionBus()和systemBus()函数分别返回会话服务器守护进程和系统服务器守护进程的开放连接。这些连接在第一次使用时被打开，在运行QCoreApplication析构器时被关闭。

D-Bus还支持点对点的连接，不需要总线服务器守护进程。使用这种设施，两个应用程序可以互相交谈并交换信息。这可以通过向connectToBus()函数传递一个地址来实现，该函数是由另一个D-Bus应用程序使用QDBusServer打开的。

enum QDBusConnection:: BusType#

指定总线连接的类型。有效的总线类型是。

常量 值 描述
QDBusConnection::SessionBus 0 会话总线，与正在运行的桌面会话相关联
QDBusConnection::SystemBus 1 系统总线，用于与全系统的进程进行通信
QDBusConnection::ActivationBus 2 激活总线，启动服务的总线的 “别名”。

在会话总线上，人们可以找到同一用户的其他应用程序，它们共享同一个桌面会话（因此得名）。然而，在系统总线上，通常会发现整个系统共享的进程。

取消注册一个对象路径的模式。

常量 值 描述

QDBusConnection:: asyncCall (const & message , int timeout = -1) const#

通过该连接发送message并立即返回。这个函数只适用于方法调用。它返回一个类型为QDBusPendingCall的对象，可以用来跟踪回复的状态。

如果在timeout毫秒内没有收到回复，那么将传递一个自动的错误，表明调用的到期。默认的timeout是-1，它将被替换为一个适合进程间通信的实现定义的值（一般是25秒）。这个超时也是QDBusPendingCall::waitForFinished()中等待的上限。

见 QDBusInterface::asyncCall ()函数，以获得更友好的放置调用的方式。

QDBusConnection:: baseService () const#

如果这个QDBusConnection对象已经连接，则返回这个连接的唯一连接名，或者一个空的QString否则。

唯一的连接名是一个字符串，形式为”:x.xxx” (其中x是十进制数字)，在连接时由D-Bus服务器守护程序分配。它在总线中唯一地标识了这个客户。

对于点对点的连接，这个函数返回一个空的QString。

QDBusConnection:: call (const & message , mode = QDBus::Block, int timeout = -1) const#

在这个连接上发送消息，并阻塞，等待回复，最多为timeout毫秒。这个函数只适用于方法调用。它的返回值是回复信息，其类型为QDBusMessage::ReplyMessage 或者QDBusMessage::ErrorMessage 。

如果在timeout毫秒内没有收到回复，那么将自动发送一个错误，表明调用已经到期。默认的timeout是-1，它将被替换为一个适合进程间通信的实现定义的值（通常是25秒）。

见QDBusInterface::call()函数，以更友好的方式放置调用。

警告： 若mode为QDBus::BlockWithGui ，该函数将重新进入Qt事件循环，以等待回复。在等待过程中，它可能向你的应用程序传递信号和其他方法调用。因此，每当用call()进行调用时，它必须准备好处理一个重入。

bool QDBusConnection:: callWithCallback (const & message , * receiver , const char * returnMethod , const char * errorMethod , int timeout = -1) const#

通过这个连接发送message并立即返回。当收到回复时，在receiver对象中调用returnMethod方法。如果发生错误，将调用errorMethod方法。

如果在timeout毫秒内没有收到回复，就会有一个自动的错误被传递，表明调用已经到期。默认的timeout是-1，它将被替换为一个适合进程间通信的实现定义的值（一般是25秒）。

这个函数只适用于方法调用。只要参数类型匹配，并且没有错误发生，就可以保证槽被精确地调用一次，并给出回复。

返回 true 如果消息被发送，或false如果消息不能被发送。

bool QDBusConnection:: connect (const & service , const & path , const & interface , const & name , * receiver , const char * slot )#

将service、path、interface和name参数指定的信号连接到插槽slot在对象receiver。参数service和path可以是空的，表示连接到任何远程应用程序的( interface , name )对的任何信号。

返回 true 若连接成功。

警告：信号只有在参数匹配的情况下才会被传递给槽。这个验证只有在收到信号时才能进行，而不是在连接时。

bool QDBusConnection:: connect (const & service , const & path , const & interface , const & name , const & signature , * receiver , const char * slot )#

这是重载函数。

将信号连接到槽slot在对象receiver。与之前的connect()重载不同，这个函数允许人们使用signature变量指定要连接的参数签名。然后，该函数将验证该签名是否可以被传递到slot指定的槽中，否则返回false。

返回 true 若连接成功。

注意：该函数验证信号签名是否与槽的参数相匹配，但它不验证实际的信号是否在远程服务中以给定的签名存在。

bool QDBusConnection:: connect (const & service , const & path , const & interface , const & name , const & argumentMatch , const & signature , * receiver , const char * slot )#

这是重载函数。

将信号连接到槽slot在对象receiver。与之前的connect()重载不同，这个函数允许人们使用signature变量指定要连接的参数签名。然后，该函数将验证该签名是否可以被传递到slot指定的插槽中，否则返回false。

argumentMatch参数列出要匹配的字符串参数，按顺序排列。注意，为了匹配一个空字符串，你需要传递一个QString，它是空的但不是空的（即QString(“”)）。一个空的 QString 会跳过该位置的匹配。

返回 true 若连接成功。

注意：此函数验证信号的签名是否与槽的参数相匹配，但它并不验证实际的信号是否存在于远程服务中的给定签名。

[static] QDBusConnection:: localMachineId ()#

返回D-Bus系统已知的本地机器ID。每个运行D-Bus的节点或主机都有一个唯一的标识符，如果它们共享像文件系统这样的资源，可以用来将其与其他主机区分开。

请注意，本地机器的ID并不保证在系统启动时是持久的，所以这个标识符不应该存储在持久性存储中（像文件系统）。它只保证在这个启动会话的有效期内保持不变。

QDBusConnection:: name () const#

返回此连接的连接名称，如connectToBus()的名称参数。

连接名称可以用来唯一地识别与总线的实际底层连接。从一个连接中产生的副本将总是隐含地共享底层连接，因此将有相同的连接名称。

反之，两个具有不同连接名称的连接将总是连接到不同的总线上，或者在该总线上具有不同的唯一名称（由baseService()返回）。

另请参阅 connectToBus () 和 disconnectFromBus ().

*QDBusConnection:: objectRegisteredAt (const & path ) const#

返回用registerObject()在path给出的对象路径上注册的对象。

bool QDBusConnection:: registerObject (const & path , * object , options = ExportAdaptors)#

在路径path处注册对象object并返回 true 如果注册成功。参数options指定了对象object的多少将通过D-Bus暴露。

这个函数不会替换现有的对象：如果已经有一个对象在路径path注册，这个函数将返回false。首先使用unregisterObject()来取消注册。

ExportChildObjects标志根据注册对象的路径和子对象的QObject::objectName导出D-Bus上的子对象。因此，子对象有一个对象名称是很重要的。

你不能把一个对象注册为一个用ExportChildObjects注册的对象的子对象。

[since 5.5] bool QDBusConnection:: registerObject (const & path , const & interface , * object , options = ExportAdaptors)#

这是重载函数。

在路径path处注册对象object，接口名称interface并返回 true 如果注册成功。参数options指定了对象object的多少将通过D-Bus暴露。

这个函数不会替换现有的对象：如果在路径path已经有一个对象被注册，这个函数将返回false。首先使用unregisterObject()来取消注册。

ExportChildObjects标志根据注册对象的路径和子对象的QObject::objectName导出D-Bus上的子对象。因此，子对象有一个对象名称是很重要的。

你不能把一个对象注册为一个用ExportChildObjects注册的对象的子对象。

该函数在 Qt 5.5 引入。

另请参阅 unregisterService () 和 QDBusConnectionInterface::registerService ().

返回 true 如果消息被成功排队，否则为false。

[static] QDBusConnection:: systemBus ()#

返回 QDBusConnection 在系统总线上打开的对象。这个函数返回的对象引用是有效的，直到QCoreApplication的析构器被运行，这时连接将被关闭，对象被删除。

void QDBusConnection:: unregisterObject (const & path , mode = UnregisterNode)#

解除一个对象的注册，该对象是通过registerObject()在path给出的对象路径上注册的，如果mode是QDBusConnection::UnregisterTree，它的所有子对象也是如此。

注意，你不能取消那些没有被registerObject()注册的对象。
