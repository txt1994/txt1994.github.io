详细描述#

该类用于通过D-Bus向远程应用程序发送参数并接收它们。D-Bus提供了一个可扩展的类型系统，基于一些原始类型和它们的关联。关于类型系统的更多信息，请参见Qt D-Bus类型系统页面。

QDBusArgument是Qt D-Bus类型系统的中心类，它提供了集合和解集原始类型的函数。然后通过在数组、字典或结构中关联一个或多个原始类型来创建复合类型。

下面的例子说明了如何使用[Qt D-Bus类型系统]（qdbustypesystem.html）构造一个包含整数和字符串的结构:

struct MyStructure
{
    int count;
    QString name;
    // ...
};
Q_DECLARE_METATYPE(MyStructure)
// 将MyStructure的数据导入一个D-Bus参数中
QDBusArgument &operator<<(QDBusArgument &argument, const MyStructure &myStruct)
{
    argument.beginStructure();
    argument << myStruct.count << myStruct.name;
    argument.endStructure();
    return argument;
}
// 从D-Bus参数中获取MyStructure数据。
const QDBusArgument &operator>>(const QDBusArgument &argument, MyStructure &myStruct)
{
    argument.beginStructure();
    argument >> myStruct.count >> myStruct.name;
    argument.endStructure();
    return argument;
}

在使用QDBusArgument之前，该类型必须通过qDBusRegisterMetaType()注册。因此，在你程序的某个地方，你应该添加以下代码:

qDBusRegisterMetaType<MyStructure>();

一旦注册，一个类型就可以用于对外的方法调用（用QDBusAbstractInterface::call()放置），来自注册对象的信号发射或来自远程应用的传入调用。

需要注意的是，operator<< 和operator>> 流函数在结构的情况下必须总是产生相同数量的条目，无论是在读还是在写（marshalling和demarshalling），否则调用和信号可能开始默默地失败。

下面的例子说明了在一个可能包含无效数据的类的背景下的这种错误用法:

//bad code
    // Wrongly marshall the MyTime data into a D-Bus argument
    QDBusArgument &operator<<(QDBusArgument &argument, const MyTime &mytime)
    {
        argument.beginStructure();
        if (mytime.isValid)
            argument << true << mytime.hour
                     << mytime.minute << mytime.second;
        else
            argument << false;
        argument.endStructure();
        return argument;
    }

在这个例子中，两个operator<< 和 operator>> 函数都可能产生不同的读/写次数。这可能会混淆Qt D-Bus类型系统，应该避免。

另请参阅 QDBusAbstractInterface , The Qt D-Bus type system , Using Adaptors ，和 qdbus_cast ().

enum QDBusArgument:: ElementType#

这个枚举描述了参数所持有的元素的类型。

常量 值 描述
QDBusArgument::BasicType 0 一个基本元素，它被QVariant所理解。以下类型被认为是基本的：bool, byte, short, ushort, int, uint, qint64, quint64, double, QString , QByteArray , QDBusObjectPath , QDBusSignature
QDBusArgument::VariantType 1 变体元素（QDBusVariant)
QDBusArgument::ArrayType 2 一个数组元素，通常用QList 表示。注意：QByteArray 和关联映射不被视为数组，即使D-Bus协议将它们作为数组来传输。
QDBusArgument::StructureType 3 一个由结构代表的自定义类型，如QDateTime、QPoint等。
QDBusArgument::MapType 4 一个关联的容器，如QMap<键，值>或QHash<键，值>。
QDBusArgument::MapEntryType 5 一个关联容器中的一个条目：键和值都构成一个地图条目类型。
QDBusArgument::UnknownType -1 类型不明，或者我们已经到了列表的最后。

QDBusArgument:: asVariant () const#

以QVariant的形式返回当前参数。基本类型将被解码并返回QVariant中，但对于复杂类型，这个函数将返回QDBusArgument对象中的QVariant。解码该参数是调用者的责任（例如，通过调用其中的asVariant()）。

例如，如果当前参数是一个INT32，这个函数将返回一个QVariant，其参数类型为QMetaType::Int。对于一个INT32的数组，它将返回一个QVariant包含QDBusArgument。

如果发生错误或者没有更多的参数可以解码（即，我们在参数列表的末端），这个函数将返回一个无效的QVariant。

void QDBusArgument:: beginArray ( id )#

打开一个新的D-Bus数组，适合追加元类型id的元素。

这个函数通常用在operator<< streaming operators中，如下面的例子:

// Append an array of MyElement types
QDBusArgument &operator<<(QDBusArgument &argument, const MyArray &myArray)
{
    argument.beginArray(qMetaTypeId<MyElement>());
    for (const auto &element : myArray)
        argument << element;
    argument.endArray();
    return argument;
}

如果你想调用的类型是QList或任何Qt的容器类，它只需要一个模板参数，你不需要为它声明一个operator<<函数，因为Qt D-Bus提供了通用模板来完成调用数据的工作。这同样适用于STL的序列容器，如std::list，std::vector，等。

另请参阅 endArray (), beginStructure ()，和 beginMap ().

void QDBusArgument:: beginArray () const#

递归到D-Bus数组中，允许提取数组元素。

这个函数通常用在 operator>> 流运算器 “中，如下面的例子:

// 提取一个MyArray数组中的MyElement元素
const QDBusArgument &operator>>(const QDBusArgument &argument, MyArray &myArray)
{
    argument.beginArray();
    myArray.clear();
    while (!argument.atEnd()) {
        MyElement element;
        argument >> element;
        myArray.append(element);
    }
    argument.endArray();
    return argument;
}

如果你想解封的类型是QList或任何Qt的容器类，它只需要一个模板参数，你不需要为它声明一个operator>>函数，因为Qt D-Bus提供通用模板来完成解封数据的工作。这同样适用于STL的序列容器，如std::list，std::vector，等。

另请参阅 atEnd (), beginStructure ()，和 beginMap ().

void QDBusArgument:: beginMap ( keyMetaType , valueMetaType )#

打开一个适合追加元素的新D-Bus地图。地图是将一个条目（键）与另一个条目（值）联系起来的容器，例如Qt的QMap或QHash。地图的键和值元类型的id必须在keyMetaType和valueMetaType中传递。

这个函数通常用在operator<< streaming operators中，如下面的例子:

// Append a dictionary that associates ints to MyValue types
QDBusArgument &operator<<(QDBusArgument &argument, const MyDictionary &myDict)
{
    argument.beginMap(QMetaType::fromType<int>(), QMetaType::fromType<MyValue>());
    MyDictionary::const_iterator i;
    for (i = myDict.cbegin(); i != myDict.cend(); ++i) {
        argument.beginMapEntry();
        argument << i.key() << i.value();
        argument.endMapEntry();
    }
    argument.endMap();
    return argument;
}

你通常不需要为关联容器（如QHash或std::map）提供一个operator<< 或operator>>函数，因为Qt D-Bus提供了通用模板来完成数据编排的工作。

另请参阅 endMap (), beginStructure (), beginArray ()，和 beginMapEntry ().

void QDBusArgument:: beginMap () const#

递归到D-Bus地图中，允许提取地图的元素。

这个函数通常用于operator>> streaming运算符中，如下面的例子。

// Extract a MyDictionary map that associates integers to MyElement items
const QDBusArgument &operator>>(const QDBusArgument &argument, MyDictionary &myDict)
{
    argument.beginMap();
    myDict.clear();
    while (!argument.atEnd()) {
        int key;
        MyElement value;
        argument.beginMapEntry();
        argument >> key >> value;
        argument.endMapEntry();
        myDict.insert(key, value);
    }
    argument.endMap();
    return argument;
}

如果你想解封的类型是QMap或QHash，你不需要为它声明一个operator>>函数，因为Qt D-Bus提供了通用模板来完成解封数据的工作。

另请参阅 endMap (), beginStructure (), beginArray ()，和 beginMapEntry ().

void QDBusArgument:: beginStructure ()#

打开一个新的D-Bus结构，适合添加新的参数。

这个函数通常用在operator<< streaming operators中，如下面的例子:

QDBusArgument &operator<<(QDBusArgument &argument, const MyStructure &myStruct)
{
    argument.beginStructure();
    argument << myStruct.member1 << myStruct.member2;
    argument.endStructure();
    return argument;
}

结构可以包含其他结构，所以下面的代码也是有效的:

QDBusArgument &operator<<(QDBusArgument &argument, const MyStructure &myStruct)
{
    argument.beginStructure();
    argument << myStruct.member1 << myStruct.member2;
    argument.beginStructure();
    argument << myStruct.member3.subMember1 << myStruct.member3.subMember2;
    argument.endStructure();
    argument << myStruct.member4;
    argument.endStructure();
    return argument;
}

另请参阅 endStructure (), beginArray ()，和 beginMap ().

template QMetaType qDBusRegisterMetaType ()#

注册 T 采用 Qt D-Bus Type System 和Qt meta-type system ，如果它还没有注册。

要注册一个类型，必须用Q_DECLARE_METATYPE()宏将其声明为一个元类型，然后像下面的例子那样注册:

#include <QDBusMetaType>
qDBusRegisterMetaType<MyClass>();

若T不是Qt的容器类之一，operator<< 和operator>>在T和QDBusArgument之间的流操作必须已经声明。参见Qt D-Bus Type System页面，了解更多关于如何声明此类类型的信息。

这个函数返回该类型的Qt元类型ID（与qRegisterMetaType()返回的值相同）。

注意： 继承可流式类型（包括容器 QList , QHash 或 QMap ）的 T 可以被流化而无需提供自定义的 operator<< 和 operator>> ，这一特性从Qt 5.7开始被废弃，因为它忽略了T中除了基类之外的一切。没有诊断方法。你应该始终为你希望流化的所有类型提供这些操作符，而不是依赖Qt为基类提供的流操作符。

注意： 此函数是 thread-safe .

另请参阅 Qt D-Bus Type System , qRegisterMetaType ()，和 QMetaType .

template T qdbus_cast (const QDBusArgument & arg )#

试图将arg的内容拆成T类型。例如：

MyType item = qdbus_cast<Type>(argument);

请注意，它等同于以下内容:

MyType item;
argument >> item;
