详细描述#

QDBusAbstractAdaptor 类是所有打算使用 D-Bus 向外部世界提供接口的对象的起点。这是通过将一个或多个从 QDBusAbstractAdaptor 派生的类附加到一个普通的 QObject 然后使用 QDBusConnection::registerObject 注册该 QObject 来完成的. QDBusAbstractAdaptor 对象旨在成为轻量级包装器，主要只是将调用中继到真实对象（其父对象）和来自它的信号。

每个 QDBusAbstractAdaptor 派生类都应该使用类定义中的 Q_CLASSINFO 宏定义它正在实现的 D-Bus 接口。请注意，这种方式只能公开一个接口。

QDBusAbstractAdaptor 使用信号、槽和属性的标准 QObject 机制来确定要导出到总线的信号、方法和属性。 QDBusAbstractAdaptor 派生类发出的任何信号都将通过对象注册的任何 D-Bus 连接自动中继。

从 QDBusAbstractAdaptor 派生的类必须使用 new 运算符在堆上创建，并且不能被用户删除（当它们连接的对象也被删除时，它们将被自动删除）。

另请参阅 Using adaptors and QDBusConnection .

[protected] void QDBusAbstractAdaptor:: setAutoRelaySignals ( bool enable )#

切换真实对象的自动信号中继（见object()）。

自动信号中继包括在两个类中具有完全相同方法签名的父类上的信号之间的连接。

若enable设置为真，则连接信号；若设置为假，则断开所有信号。

另请参阅 autoRelaySignals ().

宏文档编制#

Q_NOREPLY#

Q_NOREPLY宏可以用来标记一个被调用的方法，在从QDBusInterface::call()返回之前不等待它完成处理。被调用的方法不能返回任何输出参数，如果它返回，任何这样的参数将被丢弃。

你可以在你自己的适配器中使用这个宏，在类的声明中把它放在方法的返回值（必须是 “void”）之前，如例子所示。

Q_NOREPLY void myMethod();

它在方法实现中的存在（在类声明之外）是可选的。

另请参阅 使用 Qt D-Bus 适配器 .
