详细描述#

当处理D-Bus总线服务或通过D-Bus处理远程应用程序时，会发生一些错误情况。这种错误状况有时会通过一个返回的错误值或QDBusError来表示。

C++和Java的异常是D-Bus错误的有效类比：远程应用程序和总线可能决定抛出一个错误条件，而不是正常返回一个返回值。然而，Qt D-Bus的实现并没有使用C++的异常抛出机制，所以你会在返回的回复中收到QDBusError（参见QDBusReply::error()）。

QDBusError对象被用来检查从总线和远程应用程序中收到的错误名称和消息。当从D-Bus调用时，你不应该自己创建这样的对象来发出错误的信号：相反，使用QDBusMessage::createError () 和 QDBusConnection::send ()。

另请参阅 QDBusConnection::send (), QDBusMessage ，和 QDBusReply .

enum QDBusError:: ErrorType#

为了方便验证由D-Bus实现和总线守护进程本身产生的最常见的D-Bus错误，QDBusError可以与一组预先定义的值进行比较:

常量 值 描述
QDBusError::NoError 0 QDBusError is invalid (i.e., the call succeeded)
QDBusError::Other 1 QDBusError包含一个不属于众所周知的错误。
QDBusError::Failed 2 调用失败（org.freedesktop.DBus.Error.Failed ）。
QDBusError::NoMemory 3 内存不足（org.freedesktop.DBus.Error.NoMemory ）。
QDBusError::ServiceUnknown 4 被调用的服务不知道（org.freedesktop.DBus.Error.ServiceUnknown ）。
QDBusError::NoReply 5 被调用的方法没有在指定的超时时间内回复（org.freedesktop.DBus.Error.NoReply）。
QDBusError::BadAddress 6 给出的地址无效（ org.freedesktop.DBus.Error.BadAddress ）。
QDBusError::NotSupported 7 不支持该调用/操作（org.freedesktop.DBus.Error.NotSupported ）。
QDBusError::LimitsExceeded 8 分配给这个进程/调用/连接的限制超过了预先定义的值（org.freedesktop.DBus.Error.LimitsExceeded）。
QDBusError::AccessDenied 9 该调用/操作试图访问一个不允许的资源（org.freedesktop.DBus.Error.AccessDenied）。
QDBusError::NoServer 10 文档中没有说这是为了什么 ( org.freedesktop.DBus.Error.NoServer )
QDBusError::Timeout 11 文档中没有说这是为了什么 ( org.freedesktop.DBus.Error.Timeout )
QDBusError::NoNetwork 12 Documentation doesn’t say what this is for ( org.freedesktop.DBus.Error.NoNetwork )
QDBusError::AddressInUse 13 QDBusServer试图绑定一个已经被使用的地址（org.freedesktop.DBus.Error.AddressInUse）。
QDBusError::Disconnected 14 在QDBusConnection断开连接（org.freedesktop.DBus.Error.Disconnected）后，调用/进程/消息被发送。
QDBusError::InvalidArgs 15 传递给这个调用/操作的参数无效（org.freedesktop.DBus.Error.InvalidArgs）。
QDBusError::UnknownMethod 16 在这个对象/接口中没有找到所调用的方法和给定的参数（org.freedesktop.DBus.Error.UnknownMethod ）。
QDBusError::TimedOut 17 Documentation doesn’t say… ( org.freedesktop.DBus.Error.TimedOut )
QDBusError::InvalidSignature 18 类型签名无效或不兼容（org.freedesktop.DBus.Error.InvalidSignature）。
QDBusError::UnknownInterface 19 这个对象的接口不知道（org.freedesktop.DBus.Error.UnknownInterface）。
QDBusError::UnknownObject 20 对象路径指向一个不存在的对象 ( org.freedesktop.DBus.Error.UnknownObject )
QDBusError::UnknownProperty 21 该属性在此接口中不存在（org.freedesktop.DBus.Error.UnknownProperty）。
QDBusError::PropertyReadOnly 22 属性设置失败，因为该属性是只读的（org.freedesktop.DBus.Error.PropertyReadOnly ）。
QDBusError::InternalError 23 发生了一个内部错误
QDBusError::InvalidObjectPath 25 提供的对象路径是无效的。
QDBusError::InvalidService 24 所请求的服务是无效的。
QDBusError::InvalidMember 27 该成员是无效的。
QDBusError::InvalidInterface 26 接口是无效的。

[static] QDBusError:: errorString ( error )#

返回与错误条件error相关的错误名称。

bool QDBusError:: isValid () const#

返回 true 如果这是一个有效的错误条件（即，如果有一个错误），否则就是false。

QDBusError:: message () const#

返回被呼叫者与此错误相关的消息。错误信息是实现定义的，通常包含一个人类可读的错误代码，尽管这并不意味着它适合你的最终用户。

QDBusError:: name () const#

返回这个错误的名称。错误名称类似于D-Bus接口名称，如org.freedesktop.DBus.InvalidArgs。

另请参阅 type ().
