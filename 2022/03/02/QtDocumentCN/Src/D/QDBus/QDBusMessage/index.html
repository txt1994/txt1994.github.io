详细描述#

这个对象可以代表总线上可能发生的四种不同类型的消息（MessageType）中的任何一种。

  • 方法调用
  • 方法的返回值
  • 信号发射
  • 错误代码

这种类型的对象是通过静态的createError()、createMethodCall()和createSignal()函数创建的。使用QDBusConnection::send()函数来发送消息。

enum QDBusMessage:: MessageType#

可能的信息类型:

常量 值 描述
QDBusMessage::MethodCallMessage 1 代表一个传出或传入的方法调用的信息
QDBusMessage::SignalMessage 4 代表发出或收到的信号发射的信息
QDBusMessage::ReplyMessage 2 代表方法调用的返回值的信息
QDBusMessage::ErrorMessage 3 代表响应方法调用的错误条件的信息
QDBusMessage::InvalidMessage 0 无效信息：在从D-Bus收到的信息中永远不会设置这一点。

[static] QDBusMessage:: createMethodCall (const & service , const & path , const & interface , const & method )#

构建一个新的DBus消息，代表一个方法调用。一个方法调用总是通知它的目标地址（service , path , interface 和 method）。

如果方法名称是唯一的，DBus总线允许在一个给定的远程对象上调用一个方法而不指定目标接口。然而，如果远程对象上的两个接口输出了相同的方法名，结果是未定义的（这两个接口中的一个可能被调用或返回错误）。

当在点对点的环境中使用DBus时（即不是在总线上），service参数是可选的。

QDBusInterface类为同步方法的调用提供了一个更简单的抽象。

这个函数返回一个QDBusMessage对象，可以用QDBusConnection::call()发送。

[static] QDBusMessage:: createSignal (const & path , const & interface , const & name )#

构造一个新的DBus消息，它具有给定的path、interface和name，代表一个信号发射。

一个DBus信号从一个应用程序发出，并被所有从该接口监听该信号的应用程序收到。

返回的QDBusMessage对象可以使用QDBusConnection::send()函数发送。

[static, since 5.6] QDBusMessage:: createTargetedSignal (const & service , const & path , const & interface , const & name )#

用给定的path、interface和name构造一个新的DBus消息，代表向特定目的地发射信号。

一个DBus信号是从一个应用程序发出的，并且只被拥有目标服务名称的应用程序接收。

返回的QDBusMessage对象可以使用QDBusConnection::send()函数发送。

该函数在 Qt 5.6 引入。

void QDBusMessage:: setAutoStartService ( bool enable )#

将自动启动标志设置为enable。这个标志只对方法调用信息有意义，它告诉D-Bus服务器要么自动启动负责该服务名称的服务，要么不自动启动它。

默认情况下，这个标志为真，也就是说，一个服务是自动启动的。这意味着。

当这个方法调用所指向的服务已经在运行时，这个方法调用就会被发送给它。如果该服务还没有运行，D-Bus守护进程被请求自动启动分配给该服务名称的服务。这是由放置在D-Bus服务器已知目录中的.service文件处理的。这些文件都包含一个服务名称和一个程序的路径，当这个服务名称被请求时，这个程序就会被执行。

另请参阅 autoStartService ().

void QDBusMessage:: setDelayedReply ( bool enable ) const#

设置消息是否会在以后被回复（如果enable为真），或者是否应该由Qt D-Bus生成一个自动回复（如果enable为假）。

在D-Bus中，所有的方法调用都必须生成对调用者的回复，除非调用者明确表示不这样做（见isReplyRequired()）。QtDBus为任何被调用的槽自动生成这样的回复，但它也允许槽表明它们是否将在稍后的时间，在函数完成处理后负责发送回复。

另请参阅 isDelayedReply () 和 Delayed Replies .

[since 5.12] void QDBusMessage:: setInteractiveAuthorizationAllowed ( bool enable )#

将交互式授权标志设置为enable。这个标志只对方法调用信息有意义，它告诉D-Bus服务器，方法的调用者准备在实际方法被处理之前等待交互式授权的发生（例如通过Polkit）。

默认情况下，这个标志是假的，另一端应该及时地做出任何非交互式的授权决定。

org.freedesktop.DBus.Error.InteractiveAuthorizationRequired error表示授权失败，但是如果设置了这个标志就可以成功了。

该函数在 Qt 5.12 引入。

另请参阅 isInteractiveAuthorizationAllowed ().
