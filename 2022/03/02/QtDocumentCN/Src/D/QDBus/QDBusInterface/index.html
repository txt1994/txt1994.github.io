详细描述#

QDBusInterface是一个通用的访问器类，用来放置对远程对象的调用，连接到远程对象导出的信号，以及获取/设置远程属性的值。这个类对于动态访问远程对象非常有用：也就是说，当你没有生成代表远程接口的代码时。

调用通常是通过使用call()函数来进行的，该函数构造消息，在总线上发送消息，等待回复并对回复进行解码。信号是通过使用普通的QObject::connect ()函数连接的。最后，属性是通过 QObject::property () 和 QObject::setProperty () 函数访问的。

下面的代码片段演示了如何在一个名为 “com.example.Calculator “的远程应用程序中执行 “2+2 “的数学运算，该应用程序通过会话总线访问。

QDBusInterface remoteApp( "com.example.Calculator", "/Calculator/Operations",
                          "org.mathematics.RPNCalculator" );
remoteApp.call( "PushOperand", 2 );
remoteApp.call( "PushOperand", 2 );
remoteApp.call( "ExecuteOperation", "+" );
QDBusReply<int> reply = remoteApp.call( "PopOperand" );
if ( reply.isValid() )
    printf( "%d", reply.value() );          // prints 4

另请参阅 Qt D-Bus XML 编译器 (qdbusxml2cpp) .

成员函数文档编制#

QDBusInterface:: QDBusInterface (const & service , const & path , const & interface = QString(), const & connection = QDBusConnection::sessionBus(), * parent = nullptr)#

创建一个动态的QDBusInterface对象，与对象上的接口interface相关，在路径path上的服务service上，使用给定的connection。若interface是一个空字符串，创建的对象将参考通过反省该对象找到的所有接口的合并。否则，如果interface不是空的，QDBusInterface对象将被缓存起来，以加速同一接口的进一步创建。

parent被传递给基类构造函数。

如果远程服务service不存在，或者在试图获取远程接口interface的描述时发生错误，那么创建的对象将是无效的（参见isValid()）。

[virtual] QDBusInterface:: ~QDBusInterface ()#

销毁对象接口，并释放所使用的任何资源。
