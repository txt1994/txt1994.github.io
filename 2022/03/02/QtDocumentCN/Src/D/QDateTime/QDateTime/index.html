详细描述#

QDateTime 对象编码日历日期和时钟时间（“日期时间”）。它结合了 QDate 和 QTime 类的特性。它可以从系统时钟中读取当前日期时间。它提供了比较日期时间和通过添加秒数、天数、月数或年数来操作日期时间的函数。

QDateTime 可以描述日期时间相对于 local time ，到 UTC ，到指定的 [offset from UTC ](qt.html#TimeSpec- enum) 或指定的 time zone ，与 QTimeZone 类一起使用。例如，“欧洲/柏林”时区将应用德国使用的夏令时规则。相比之下，与 UTC +3600 秒的偏移比 UTC 早一小时（通常以 ISO 标准表示法写为“UTC+01:00”），没有夏令时偏移或更改。使用本地时间或指定时区时，会考虑时区转换，例如夏令时（DST；但见下文）的开始和结束。用于表示日期时间的系统选择被描述为它的“时间规范”。

QDateTime 对象通常通过在构造函数中显式给出日期和时间来创建，或者通过使用诸如 currentDateTime () 或 fromMSecsSinceEpoch ( ）。可以使用 setDate () 和 setTime () 更改日期和时间。还可以使用 setMSecsSinceEpoch () 函数设置日期时间，该函数从 1970 年 1 月 1 日 00:00:00 开始计算时间，以毫秒为单位。 [fromString ](qdatetime.html# fromString)() 函数返回一个 QDateTime，给定一个字符串和用于解释字符串中日期的日期格式。

QDateTime::currentDateTime () 返回一个 QDateTime，它表示相对于本地时间的当前时间。 QDateTime::currentDateTimeUtc () 返回一个 QDateTime，它表示相对于 UTC 的当前时间。

date () 和 time () 函数提供对日期时间的日期和时间部分的访问。 toString () 函数以文本格式提供相同的信息。

QDateTime 提供了一套完整的运算符来比较两个 QDateTime 对象，其中较小的表示较早，较大的表示较晚。

您可以使用 addMSecs () 将日期时间增加（或减少）给定的毫秒数，使用 addSecs () 的秒数，或使用 [addDays ] 的天数(qdatetime.html#addDays)()。同样，可以使用 addMonths () 和 addYears ()。 daysTo () 函数返回之间的天数两个日期时间， secsTo () 返回两个日期时间之间的秒数， msecsTo () 返回两个日期时间之间的毫秒数。这些操作在适用的情况下了解夏令时 (DST) 和其他时区转换。

使用 toTimeSpec () 以本地时间或 UTC 表示日期时间， toOffsetFromUtc () 以与 UTC 的偏移量表示，或 toTimeZone )() 来表示它相对于一般时区。您可以使用 timeSpec () 来找出 QDateTime 对象存储其时间的相对时间规范。当它是 Qt::TimeZone 时，您可以使用 timeZone () 找出它正在使用的区域。

注意： QDateTime 不考虑闰秒。

备注#

注意： 所有字符串格式的转换都是使用 C 语言环境完成的。有关本地化转换，请参阅 QLocale 。

注意： 公历中没有 0 年。那一年的日期被认为是无效的。 -1 年是“基督之前的 1 年”或“普通时代之前的 1 年”。公元 1 月 1 日的前一天是公元前 1 年 12 月 31 日。

有效日期范围#

QDateTime 可以表示的值范围取决于内部存储实现。 QDateTime 当前存储在 qint64 中，作为对日期和时间进行编码的串行毫秒值。这将日期范围限制在大约 +/- 2.92 亿年，而 QDate 的范围是 +/- 20 亿年。创建具有极值的 QDateTime 时必须小心，以免溢出存储。支持值的确切范围取决于 Qt::TimeSpec 和时区。

使用时区#

QDateTime 使用系统的时区信息来确定当前的本地时区及其与 UTC 的偏移量。如果系统配置不正确或不是最新的，QDateTime 将给出错误的结果。

QDateTime 同样使用系统提供的信息来确定其他时区与 UTC 的偏移量。如果此信息不完整或过时，QDateTime 将给出错误结果。参见QTimeZone文档编制了解更多细节。

在现代 Unix 系统上，这意味着 QDateTime 通常尽可能地拥有有关历史转换（包括 DST，见下文）的准确信息。在系统不支持历史时区数据的 Windows 上，不会保持关于时区转换的历史准确性，特别是包括 DST。

夏令时 (DST)#

QDateTime 考虑了标准时间和夏令时之间的转换。例如，如果转换是在凌晨 2 点并且时钟前进到凌晨 3 点，则从 02:00:00 到 02:59:59.999 有一个“缺失”小时，QDateTime 认为该小时无效。执行的任何日期算术都将考虑这个缺失的时间并返回有效结果。例如，将一分钟添加到 01:59:59 将得到 03:00:00。

对于系统time_t可以表示的日期时间（在具有32位time_t的系统上从1901-12-14到2038-01-18；如果类型是64位，QDateTime可以表示全范围） ，标准系统 API 用于确定本地时间与 UTC 的偏移量。对于这些系统 API 未处理的日期时间，使用 QTimeZone::systemTimeZone ()。在任何一种情况下，所使用的偏移信息都取决于系统，并且可能不完整，或者对于过去的历史而言，是不准确的。无论如何，对于未来的日期，当地时区的偏移量和 DST 规则可能会在该日期到来之前发生变化。

UTC 偏移量#

与 UTC 的偏移量没有明确的大小限制，但在使用 toString () 和 fromString () 方法时有一个隐式限制 [+|-]hh:mm 格式，有效地将范围限制为 +/- 99 小时 59 分钟和整分钟。 请注意，目前没有时区的偏移量超出 ±14 小时的范围，并且所有已知的偏移量都是五分钟的倍数。

另请参阅 QDate , QTime , QDateTimeEdit ，和 QTimeZone .

成员函数文档编制#

QDateTime:: toString ( format , cal = QCalendar()) const#

QDateTime:: toString (const & format , cal = QCalendar()) const#

以字符串形式返回日期时间。 format 参数确定结果字符串的格式。如果提供了 cal，则它确定用于表示日期的日历；它默认为公历。请参阅 QTime::toString () 和 QDate::toString () 分别了解支持的时间和日期说明符。

任何用单引号括起来的字符序列都将逐字包含在输出字符串中（去掉引号），即使它包含格式化字符。在输出中，两个连续的单引号 (“‘’”) 被替换为单引号。格式字符串中的所有其他字符都逐字包含在输出字符串中。

支持不带分隔符的格式（例如“ddMM”），但必须小心使用，因为结果字符串并不总是可靠地可读（例如，如果“dM”产生“212”，它可能意味着 12 月 2 日或 21 日二月）。

示例格式字符串（假设 QDateTime 是 2001 年 5 月 21 日 14:13:09.120）：

格式 结果
dd.MM.yyyy 21.05.2001
ddd MMMM d yy Tue May 21 01
hh:mm:ss.zzz 14:13:09.120
hh:mm:ss.z 14:13:09.12
hⓂs ap 2:13:9 pm

如果日期时间无效，将返回一个空字符串。

注意： 日期和月份名称以及 AM/PM 指示以英语（C 语言环境）给出。 如果使用本地化的月份和日期名称以及 AM/PM 的本地化形式，请使用 QLocale::system().toDateTime()。

另请参阅 fromString (), QDate::toString (), QTime::toString ()，和 QLocale::toString ().

QDateTime:: QDateTime ( date , time , spec = Qt::LocalTime, int offsetSeconds = 0)#

使用由 spec 和 offsetSeconds 秒定义的时间规范，使用给定的 date 和 time 构造一个日期时间。

若 date 有效和 time 不是，时间将设置为午夜。

若 spec 不是 Qt::OffsetFromUTC then offsetSeconds 将被覆盖。

若 spec is Qt::OffsetFromUTC and offsetSeconds is 0 then the timeSpec () 将设置为 [Qt::UTC] (qt.html#TimeSpec-enum)，即0秒的偏移量。

若 spec 为 Qt::TimeZone ，则规范将设置为 Qt::LocalTime ，即当前系统时区。 要创建 Qt::TimeZone datetime，请使用正确的构造函数。

QDateTime:: addDays ( ndays ) const#

返回 QDateTime 对象，包含比该对象的日期时间晚 ndays 天的日期时间（如果 ndays 为负数，则更早）。

若 timeSpec () 是 Qt::LocalTime 或 Qt::TimeZone 以及结果日期和 时间落在标准时间到夏令时转换小时，则结果将相应调整，即如果转换是在凌晨 2 点，时钟向前到凌晨 3 点，结果落在凌晨 2 点和凌晨 3 点之间，那么结果将被调整为 凌晨 3 点后跌倒。

另请参阅 daysTo (), addMonths (), addYears ()，和 addSecs ().

QDateTime:: addMonths ( int nmonths ) const#

返回 QDateTime 对象，包含一个比该对象的日期时间晚nmonths个月的日期时间（如果nmonths是负数，则更早）。

若 timeSpec () 是 Qt::LocalTime 或 Qt::TimeZone ，而结果的日期和时间是在标准时间到夏令时的过渡小时，那么结果将被相应调整，即。 例如，如果过渡时间是凌晨2点，而时钟前进到凌晨3点，结果落在凌晨2点和3点之间，那么结果将被调整为落在凌晨3点之后。

返回 QDateTime 对象，该对象包含一个比该对象的日期时间晚nyears的日期时间（如果nyears是负数，则更早）。

QDateTime:: daysTo (const & other ) const#

返回从这个日期时间到另一个日期时间的天数。天数的计算方法是，从这个日期时间到另一个日期时间之间的午夜次数。这意味着，从23:55到第二天的0:05，相差10分钟也算一天。

如果*另一个日期时间比这个日期时间早，返回的值是负数。

范例：

QDateTime startDate(QDate(2012, 7, 6), QTime(8, 30, 0));
QDateTime endDate(QDate(2012, 7, 7), QTime(16, 30, 0));
qDebug() << "Days from startDate to endDate: " << startDate.daysTo(endDate);
startDate = QDateTime(QDate(2012, 7, 6), QTime(23, 55, 0));
endDate = QDateTime(QDate(2012, 7, 7), QTime(0, 5, 0));
qDebug() << "Days from startDate to endDate: " << startDate.daysTo(endDate);
qSwap(startDate, endDate); // Make endDate before startDate.
qDebug() << "Days from startDate to endDate: " << startDate.daysTo(endDate);

另请参阅 addDays (), secsTo ()，和 msecsTo ().

[static] QDateTime:: fromString (const & string , const & format , cal = QCalendar())#

返回 QDateTime，由字符串表示，使用format，如果字符串不能被解析，则为无效的日期时间。

如果提供日历，则使用cal，否则使用Gregorian。

除了通过QDate::fromString ()和QTime::fromString ()识别格式字符串中的表达式来表示日期和时间的一部分外，该方法还支持:

Expression Output
t the timezone (for example “CEST”)

如果没有’t’格式指定符，则使用系统的本地时区。所有其他字段的默认值，见QDate::fromString () 和 QTime::fromString ()。

例如：

QDateTime dateTime = QDateTime::fromString("1.30.1", "M.d.s");
// dateTime is January 30 in 1900 at 00:00:01.
dateTime = QDateTime::fromString("12", "yy");
// dateTime is January 1 in 1912 at 00:00:00.

所有其他输入字符将被视为文本。任何用单引号括起来的非空字符序列也将被当作文本处理（去掉引号），而不会被解释为表达式。

QTime time1 = QTime::fromString("131", "HHh");
// time1 is 13:00:00
QTime time1 = QTime::fromString("1apA", "1amAM");
// time1 is 01:00:00
QDateTime dateTime2 = QDateTime::fromString("M1d1y9800:01:02",
                                            "'M'M'd'd'y'yyhh:mm:ss");
// dateTime is 1 January 1998 00:01:02

如果不满足格式，将返回一个无效的QDateTime。如果格式得到满足，但string代表一个无效的日期时间（例如，在时区转换所跳过的空隙中），则返回一个无效的QDateTime，其toMSecsSinceEpoch（）代表一个接近的有效日期时间。将其传递给 fromMSecsSinceEpoch ()会产生一个有效的日期时间，但它并不忠实于被解析的字符串。

没有前导零的表达式（d, M, h, m, s, z）将是贪婪的。这意味着它们将使用两位数（或者三位，对于z），即使这将使它们超出范围和/或为其他部分留下太少的数字。

QDateTime dateTime = QDateTime::fromString("130", "Mm"); // invalid

这可能意味着 1 月 1 日 00:30.00，但 M 将获得两位数。

string 的错误指定字段将导致返回无效的 QDateTime 。 例如，考虑以下代码，其中两位数年份 12 被读取为 1912（请参阅下表了解所有字段默认值）； 结果日期时间无效，因为 1912 年 4 月 23 日是星期二，而不是星期一：

QString string = "Monday, 23 April 12 22:51:41";
QString format = "dddd, d MMMM yy hh:mm:ss";
QDateTime invalid = QDateTime::fromString(string, format);

正确的代码是：

QString string = "Tuesday, 23 April 12 22:51:41";
QString format = "dddd, d MMMM yy hh:mm:ss";
QDateTime valid = QDateTime::fromString(string, format);

注意：日期和月份名称以及 AM/PM 指示必须用英语（C 语言环境）给出。 如果使用本地化的月份和日期名称以及 AM/PM 的本地化形式，请使用 QLocale::system().toDateTime()。

另请参阅 toString (), QDate::fromString (), QTime::fromString ()，和 QLocale::toDateTime ().

[since 5.2] int QDateTime:: offsetFromUtc () const#

以秒为单位返回此日期时间与 UTC 的偏移量。

结果取决于 timeSpec ()：

  • Qt::UTC偏移量为 0。
  • Qt::OffsetFromUTC偏移量是最初设置的值。
  • Qt::LocalTime 返回本地时间与 UTC 的偏移量。
  • Qt::TimeZone返回时区使用的偏移量。

对于最后两个，将返回该日期和时间的偏移量，同时考虑夏令时偏移量。 偏移量是给定时区的本地时间或时间与 UTC 时间的差值； 它在 UTC（本初子午线以东）之前的时区为正，对于 UTC（本初子午线以西）之后的时区为负。

QDateTime:: toString ( format = Qt::TextDate) const#

这是重载函数。

以格式给定的字符串形式返回日期时间。

若 format 为 Qt::TextDate ，字符串以默认方式格式化。日期和月份名称将使用英文。这种格式的一个例子是“Wed May 20 03:40:13 1998”。对于本地化格式，请参阅 QLocale::toString ()。

若 format 为 Qt::ISODate ，字符串格式对应于日期和时间表示的 ISO 8601 扩展规范，格式为 yyyy-MM-ddTHH:mm:ss [Z|[+|-]HH:mm]，取决于QDateTime的timeSpec()。若timeSpec () 是 Qt::UTC , Z 将附加到字符串中；如果 timeSpec () 是 Qt::OffsetFromUTC ，则从 UTC 开始的小时和分钟偏移量将附加到字符串中。要在 ISO 8601 日期中包含毫秒，请使用 format Qt::ISODateWithMs ，它对应于 yyyy-MM-ddTHH:mm:ss.zzz[Z|[+| -]HH:mm]。

若 format 为 Qt::RFC2822Date ，字符串按照 RFC 2822 格式化。

如果日期时间无效，将返回一个空字符串。

警告： Qt::ISODate 格式仅对 0 到 9999 范围内的年份有效。

另请参阅 fromString (), QDate::toString (), QTime::toString ()，和 QLocale::toString ().
