详细描述#

对话框窗口是顶级窗口，主要用于短期任务和与用户简短交流。QDialogs 可以是模态 (或非模态) 的。QDialogs 可以提供 返回值 ，且它们可以有 默认按钮 。QDialogs 也可拥有 QSizeGrip 在其右下角，使用 setSizeGripEnabled ().

注意：QDialog (及任何其它 Widget 有类型 Qt::Dialog ) 使用父级 Widget 与其它 Qt 类稍有不同。对话框始终是顶层 Widget，但若它有父级，其默认位置是居中父级的顶层 Widget 顶部 (若它本身不在顶层)。它还会共享父级的任务栏条目。

使用重载 QWidget::setParent () 函数去改变 QDialog 小部件的所有权。此函数允许明确设置要重设父级的 Widget 的窗口标志；使用重载函数将清零指定 Widget 的窗口系统特性窗口标志 (尤其，它将重置 Qt::Dialog 标志)。

注意： 对话框的父级关系 not 隐含对话框将始终被堆叠在父级窗口顶部。要确保对话框始终在顶部，使对话框模态。这也适用于对话框本身的子级窗口。要确保对话框的子级窗口停留在对话框顶部，也使子级窗口模态。

模态对话框#

A modal 对话框是在同一应用程序中阻塞其它可见窗口输入的对话框。用于从用户请求文件名 (或用于设置应用程序首选项) 的通常是模态对话框。对话框可以为 应用程序模态 (默认) 或 窗口模态 .

当一个应用程序模态对话框被打开时，用户必须完成与该对话框的交互并关闭它，然后才能访问该应用程序中的任何其他窗口。窗口模态对话框只阻止对与该对话框相关的窗口的访问，允许用户继续使用应用程序中的其他窗口。

显示模态对话框最常见的方式是调用其exec()函数。当用户关闭对话框时，exec ()将提供一个有用的返回值。为了关闭对话框并返回适当的值，你必须连接一个默认的按钮，例如一个OK按钮到accept()槽，一个Cancel按钮到reject() 槽。另外，你可以用 “接受 “或 “拒绝 “来调用完成()槽。

另一种方法是调用 setModal (true) 或 setWindowModality ()，然后调用 show ()。与exec()不同，show()立即将控制权返回给调用者。调用 setModal (true) 对于进度对话框特别有用，在这种情况下，用户必须有能力与对话框进行交互，例如取消一个长期运行的操作。如果你使用show ()和setModal (true)一起执行一个长时间的操作，你必须在处理过程中定期调用QCoreApplication::processEvents ()以便让用户与对话框交互。(参见QProgressDialog。

非模态对话框#

A modeless 对话框是独立于同一应用程序中其它窗口运转的对话框。文字处理程序中的查找 替换对话框经常是非模态的，以允许用户与应用程序主窗口和对话框进行交互。

非模态对话框被显示使用 show ()，它把控制立即返回给调用者。

若援引 show() 函数在隐藏对话框之后，对话框将显示在其原始位置。这是因为窗口管理器决定程序员未明确放置的窗口位置。要保留用户已移动对话框的位置，把其位置保存在 closeEvent() 处理程序，然后把对话框移动到该位置，在再次展示它之前。

默认按钮#

对话框的默认按钮是用户按回车键（Return）时按下的按钮。这个按钮用来表示用户接受对话框的设置并希望关闭对话框。使用 QPushButton::setDefault (), QPushButton::isDefault () 和 QPushButton::autoDefault () 来设置和控制对话框的默认按钮。

Esc 键#

若用户在对话框中按下 Esc 键， QDialog::reject () 会被调用。这将导致窗口被关闭： 关闭事件 不可以是 ignored .

可扩展性#

可扩展性是指以两种方式显示对话框的能力：显示最常用选项的部分对话框，以及显示所有选项的完整对话框。通常，一个可扩展的对话框最初会以部分对话框的形式出现，但有一个更多的切换按钮。如果用户按下更多按钮，对话框就会展开。扩展范例](qtwidgets-dialogs-extension-example.html)展示了如何使用Qt实现可扩展对话框。

返回值 (模态对话框)#

模态对话框通常用于需要返回值的情况，例如，指示用户是按了OK还是Cancel。对话框可以通过调用 accept () 或 reject () 插槽来关闭，而 exec () 会返回 接受或拒绝的适当位置。exec ()调用返回对话的结果。如果对话没有被销毁，result ()也可以得到结果。

为了修改你的对话框的关闭行为，你可以重新实现accept()、reject()或done()。closeEvent()函数应该只在保留对话框的位置或覆盖标准关闭或拒绝行为时重新实施。

代码范例#

模态对话框：

void EditorWindow::countWords()
{
    WordCountDialog dialog(this);
    dialog.setWordCount(document().wordCount());
    dialog.exec();
}

非模态对话框：

void EditorWindow::find()
{
    if (!findDialog) {
        findDialog = new FindDialog(this);
        connect(findDialog, &FindDialog::findNext,
                this, &EditorWindow::findNext);
    }
    findDialog->show();
    findDialog->raise();
    findDialog->activateWindow();
}

特性文档编制#

modal : bool#

此特性保持是否 show ()应以模态或无模态方式弹出对话框。

默认情况下此特性为 false ，show ()会以无模式方式弹出对话框。将此属性设置为true，相当于将QWidget::windowModality设置为Qt::ApplicationModal 。

exec()忽略了这个属性的值，总是以模态的方式弹出对话框。

访问函数：

bool isModal () const
void setModal (bool modal )

另请参阅 QWidget::windowModality , show ()，和 exec ().

sizeGripEnabled : bool#

此特性保持是否启用大小握把

A QSizeGrip 被放置在对话框右下角，当此属性被启用时。默认情况下，大小握把是禁用的。

访问函数：

bool isSizeGripEnabled () const
void setSizeGripEnabled (bool)

成员函数文档编制#

QDialog:: QDialog ( * parent = nullptr, f = Qt::WindowFlags())#

构造对话框采用父级 parent .

对话框总是一个顶层小组件，但如果它有一个父对象，它的默认位置是在父对象的顶部居中。它也会共享父对象的任务栏条目。

小组件的标志f被传递给QWidget构造函数。例如，如果你不希望在对话框的标题栏里有一个What’s This按钮，可以在f中传递Qt::WindowTitleHint | Qt::WindowSystemMenuHint 。

另请参阅 QWidget::setWindowFlags ().

[virtual slot] void QDialog:: accept ()#

隐藏模态对话框并将结果代码设为 接受 .

另请参阅 reject () 和 done ().

[signal] void QDialog:: accepted ()#

当对话框被用户接受或通过调用 accept () 或 done () 采用 QDialog::Accepted 自变量，该信号会被发射出来。

请注意，当用 hide () 或 setVisible (false) 隐藏对话框时，这个信号不会*被发出来。这包括在对话框可见时删除它。

另请参阅 finished () 和 rejected ().

[virtual slot] void QDialog:: done ( int r )#

QDialog::Accepted 或QDialog::Rejected，accepted()或rejected()信号也将分别被发出。

如果这个对话框是用 exec ()显示的，那么 done() 也会导致本地事件循环结束，并且 exec () 会返回 r 。

与QWidget::close ()一样，如果Qt::WA_DeleteOnClose 标志被设置，done()将删除该对话框。如果对话框是应用程序的主部件，应用程序就会终止。如果对话框是最后一个关闭的窗口，[QGuiApplication::lastWindowClosed ] (qguiapplication.html#lastWindowClosed)() 信号被发射。

另请参阅 accept (), reject (), QApplication::activeWindow ()，和 QCoreApplication::quit ().

[virtual slot] int QDialog:: exec ()#

把对话框展示成 模态对话框 ，阻塞直到用户关闭它。函数返回 DialogCode 结果。

如果对话框是 应用程序模态 ，用户在关闭对话框之前不能与同一个应用程序中的任何其他窗口进行交互。如果对话框是窗口模态，在对话框打开时，只有与父窗口的交互被阻止。默认情况下，对话框是应用模态的。

注意：避免使用这个函数；而是使用 open() 。与exec()不同，open()是异步的，并且不会旋转一个额外的事件循环。这可以防止一系列危险的bug发生（例如，在通过exec()打开对话框时删除对话框的父级）。当使用open()时，你可以连接到QDialog的finished信号，以便在对话框关闭时获得通知。

另请参阅 open (), show (), result ()，和 setWindowModality ().

[signal] void QDialog:: finished ( int result )#

当对话框的*结果代码被设置时，这个信号就会被发射出来，无论是用户还是通过调用 done (), accept ()，或 reject ()。

请注意，当用hide ()或setVisible (false)隐藏对话框时，这个信号不会*发出来。这包括在对话框可见时删除它。

另请参阅 accepted () 和 rejected ().
