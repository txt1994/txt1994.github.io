成员类型文档#

enum QEventPoint::State flags QEventPoint::States#

指定该事件点的状态。

Constant Value Description
QEventPoint::Unknown Qt::TouchPointUnknownState 状态不明。
QEventPoint::Stationary Qt::TouchPointStationary 事件点并没有移动。
QEventPoint::Pressed Qt::TouchPointPressed 触摸点或按钮被按下。
QEventPoint::Updated Qt::TouchPointMoved 事件点已经更新。
QEventPoint::Released Qt::TouchPointReleased 触摸点或按钮被释放。

States类型是QFlags的一个类型定义。它存储了State值的OR组合。

特性文档#

accepted : bool#

这个属性持有事件点的接受状态。

在基于widget的应用程序中，这个属性不被使用，因为一个widget接受或拒绝一个完整的QInputEvent才有意义。

然而，在Qt Quick中，项目或事件处理程序通常只接受QTouchEvent中实际参与手势的单个点，而其他点可以传递给其他项目或处理程序。为了保持一致性，这适用于任何QPointerEvent；并且只有当QPointerEvent中的所有点都被接受时，才会进行传递。

访问函数:

void setAccepted(bool accepted = true)

See also QEvent::accepted.

[read-only]ellipseDiameters : const #

此属性持有触摸点的边界椭圆的宽度和高度。

返回值的单位是逻辑像素。大多数触摸屏不会检测到接触点的形状，也没有鼠标或平板设备可以检测到它，所以空大小是最常见的值。在某些触摸屏上，直径可能不为零，而且总是相等（椭圆被近似为圆）。

访问函数:

这个属性持有这个点的角度方向。

返回值的单位是度，其中零（默认）表示手指、令牌或触控笔指向上方，负的角度表示它向左旋转，正的角度表示它向右旋转。大多数触摸屏不检测旋转，所以零是最常见的值。

[read-only]uniqueId : const #

这个属性持有这个点或标记的唯一ID，如果有的话。

它通常是无效的（见isValid()），因为触摸屏不能唯一地识别手指。

当它来自QTabletEvent时，它标识了正在使用的触控笔的序列号。

当TUIO驱动与支持它们的触摸屏一起使用时，它可以识别一个特定的标记（靶标对象）。

访问函数:

QPointingDeviceUniqueId uniqueId() const

[read-only]velocity : const #

这个属性持有一个速度向量，单位是每秒像素，在屏幕或桌面的坐标系中。

注意：如果设备的能力包括QInputDevice::Velocity，这意味着速度来自于操作系统（也许是触摸硬件或驱动提供的）。但通常 “速度 “功能没有被设置，表示速度是由Qt计算的，使用一个简单的卡尔曼滤波器来提供一个平滑的平均速度，而不是一个瞬时值。有效地，它告诉了用户在过去几个事件中拖动这个点的速度和方向，其中最近的事件影响最大。

访问函数:

QVector2D velocity() const

See also QInputDevice::capabilities() and QInputEvent::device().
