详细描述#

Qt的主事件循环（QCoreApplication::exec()）从事件队列中获取本地窗口系统事件，将其翻译成QEvents，并将翻译好的事件发送到QObjects。

一般来说，事件来自底层窗口系统（自发()返回true），但也可以使用QCoreApplication::sendEvent()和QCoreApplication::postEvent()手动发送事件（自发() 返回false）。

QObjects通过调用其QObject::event()函数来接收事件。该函数可以在子类中被重新实现，以定制事件处理并添加额外的事件类型；QWidget::event()是一个明显的例子。默认情况下，事件被派发到事件处理程序中，比如 QObject::timerEvent() 和 QWidget::mouseMoveEvent()。QObject::installEventFilter()允许一个对象拦截为另一个对象准备的事件。

基本的QEvent只包含一个事件类型参数和一个 “接受 “标志。接受标志通过accept()设置，并通过ignore()清除。它是默认设置的，但不要依赖它，因为子类可以选择在它们的构造函数中清除它。

QEvent的子类包含描述特定事件的额外参数。

See also QObject::event(), QObject::installEventFilter(), QCoreApplication::sendEvent(), QCoreApplication::postEvent(), and QCoreApplication::processEvents().

User events should have values between User and MaxUser:

Constant Value Description
QEvent::User 1000 用户定义的事件。
QEvent::MaxUser 65535 最后的用户事件ID。

为了方便起见，你可以使用registerEventType()函数来为你的应用程序注册和保留一个自定义事件类型。这样做将允许你避免意外地重复使用已经在你的应用程序中其他地方使用的自定义事件类型。

Property Documentation#

accepted : bool#

此属性持有事件对象的接受标志。

设置接受参数表示事件接收器想要这个事件。不需要的事件可能会被传播到父部件。默认情况下，isAccepted()被设置为真，但不要依赖这个，因为子类可以选择在他们的构造函数中清除它。

为了方便，接受标志也可以用accept()来设置，用ignore()来清除。

注意：接受一个QPointerEvent隐含地接受了该事件携带的所有点。

void QEvent::accept()#

设置事件对象的接受标志，相当于调用setAccepted(true)。

设置接受参数表明事件接收者想要这个事件。不需要的事件可能会被传播到父部件。

See also ignore().

创建并返回此事件的一个相同的副本。

void QEvent::ignore()#

清除事件对象的接受标志参数，相当于调用setAccepted(false)。

清除接受参数表示事件接收器不想要这个事件。不需要的事件可能会被传播到父部件。

See also accept().

[static]int QEvent::registerEventType(int hint = -1)#

注册并返回一个自定义的事件类型。如果所提供的hint可用，它将被使用，否则它将返回一个介于QEvent::User和QEvent::MaxUser之间、尚未被注册的值。如果提示的值不在QEvent::User和QEvent::MaxUser之间，则被忽略。

如果所有可用的值已经被占用或程序正在关闭，则返回-1。

Note: 这个函数是 thread-safe.

bool QEvent::spontan eous() const#

如果事件发生在应用程序之外（系统事件），返回 “true”；否则返回 “false”。
