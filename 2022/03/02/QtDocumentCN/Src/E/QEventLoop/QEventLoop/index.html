详细描述#

在任何时候，你都可以创建一个QEventLoop对象，并对其调用exec()来启动一个本地事件循环。在事件循环中，调用exit()将迫使exec() 返回。

See also QAbstractEventDispatcher.

成员类型文档#

enum QEventLoop::ProcessEventsFlag flags QEventLoop::ProcessEventsFlags#

这个枚举控制processEvents()函数处理的事件类型。

Constant Value Description
QEventLoop::AllEvents 0x00 所有事件。请注意，DeferredDelete事件被特别处理。参见 QObject::deleteLater() 了解更多细节。
QEventLoop::ExcludeUserInputEvents 0x01 不处理用户输入事件，如ButtonPress和KeyPress。请注意，这些事件不会被丢弃；它们将在下次调用processEvents()时被传递，但不包括ExcludeUserInputEvents标志。
QEventLoop::ExcludeSocketNotifiers 0x02 不处理套接字通知器事件。请注意，这些事件不会被丢弃；它们将在下次调用processEvents()时被传递，但不包括ExcludeSocketNotifiers标志。
QEventLoop::WaitForMoreEvents 0x04 如果没有待处理的事件，则等待事件。

ProcessEventsFlags类型是QFlags的一个类型定义。它存储了ProcessEventsFlag值的一个OR组合。

See also processEvents().

[slot]void QEventLoop::exit(int returnCode = 0)#

告诉事件循环以一个返回代码退出。

在这个函数被调用后，事件循环从调用exec()中返回。exec()函数返回returnCode。

根据惯例，returnCode为0表示成功，任何非零值表示错误。

请注意，与同名的C库函数不同，这个函数*会返回给调用者–停止的是事件处理。

See also QCoreApplication::quit(), quit(), and exec().

int QEventLoop::exec( flags = AllEvents)#

进入主事件循环并等待，直到exit()被调用。返回传递给exit()的值。

如果指定了flags，只有flags所允许的类型的事件才会被处理。

有必要调用这个函数来启动事件处理。主事件循环接收来自窗口系统的事件，并将这些事件分派给应用程序的小部件。

一般来说，在调用exec()之前不能进行任何用户交互。作为一个特例，像QMessageBox这样的模态小部件可以在调用exec()之前使用，因为模态小部件使用它们自己的本地事件循环。

为了使你的应用程序执行空闲处理（即只要没有待处理的事件就执行一个特殊的函数），使用一个超时为0的QTimer。更复杂的空闲处理方案可以使用processEvents()实现。

See also QCoreApplication::quit(), exit(), and processEvents().

bool QEventLoop::isRunning() const#

如果事件循环正在运行，返回 “true”；否则返回 “false”。从调用exec()到调用exit()时，事件循环被认为是在运行。

See also exec() and exit().

bool QEventLoop::processEvents( flags = AllEvents)#

处理一些符合flags的未决事件。如果待处理的事件被处理，则返回 “true”；否则返回 “false”。

如果你有一个长期运行的操作，并想在不允许用户输入的情况下显示其进度，即通过使用ExcludeUserInputEvents标志，这个函数就特别有用。

这个函数只是QAbstractEventDispatcher::processEvents()的一个包装器。详情见该函数的文档。

void QEventLoop::processEvents( flags, int maxTime)#

处理符合flags的待处理事件，最长时间为maxTime毫秒，或者直到没有更多的事件需要处理，以较短的时间为准。如果你有一个长期运行的操作，并想在不允许用户输入的情况下显示其进度，即通过使用ExcludeUserInputEvents标志，这个功能就特别有用。

注意：

  • 该函数不连续处理事件；它在所有可用的事件处理完后返回。
  • 指定WaitForMoreEvents标志没有意义，将被忽略。

void QEventLoop::wakeUp()#

唤醒事件循环。

See also QAbstractEventDispatcher::wakeUp().
