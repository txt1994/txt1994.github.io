详细描述#

QLabel用于显示文本或图像。没有提供用户交互功能。标签的视觉外观可以通过各种方式进行配置，它可以用来为另一个部件指定一个焦点记忆键。

一个QLabel可以包含以下任何内容类型：

Content Setting
Plain text 传递一个QString给setText()。
Rich text 将一个包含丰富文本的QString传递给setText()。
A pixmap 传递一个QPixmap给setPixmap()。
A movie 传递一个QMovie给setMovie()。
A number 传递一个int或double给setNum()，它将数字转换为纯文本。
Nothing 与空的纯文本相同。这是默认的。由clear()设置。

警告：在向构造函数传递QString或调用setText()时，请确保对输入进行消毒，因为QLabel试图猜测它是以纯文本还是以富文本（HTML 4标记的一个子集）显示文本。您可能希望明确调用setTextFormat()，例如，如果您希望文本为纯文本格式，但无法控制文本来源（例如，显示从网络加载的数据时）。

当使用这些函数改变内容时，之前的任何内容都会被清除掉。

默认情况下，标签显示左对齐，垂直居中的文本和图像，其中要显示的文本中的任何标签都是自动展开。然而，QLabel的外观可以通过几种方式进行调整和微调。

内容在QLabel部件区域内的定位可以通过setAlignment()和setIndent()进行调整。文本内容也可以用setWordWrap()沿着字的边界换行。例如，这段代码在右下角设置了一个带有两行文字的下沉式面板（两行文字都与标签的右侧平齐）：

QLabel *label = new QLabel(this);
label->setFrameStyle(QFrame::Panel | QFrame::Sunken);
label->setText("first line\nsecond line");
label->setAlignment(Qt::AlignBottom | Qt::AlignRight);

QLabel从QFrame继承的属性和功能也可以用来指定任何给定的标签所要使用的部件框架。

一个QLabel经常被用作一个交互式部件的标签。对于这种用途，QLabel提供了一个有用的机制来添加一个助记符（见QKeySequence），该助记符将把键盘焦点设置到其他部件（称为QLabel的 “伙伴”）。比如说:

QLineEdit *phoneEdit = new QLineEdit(this);
QLabel *phoneLabel = new QLabel("&Phone:", this);
phoneLabel->setBuddy(phoneEdit);

在这个例子中，当用户按下Alt+P时，键盘焦点被转移到标签的伙伴（QLineEdit）。如果这个伙伴是一个按钮（继承自QAbstractButton），触发助记符将模拟一个按钮点击。

See also QLineEdit, QTextEdit, QPixmap, QMovie, and GUI Design Handbook: Label.

特性文档#

alignment : #

该属性持有标签内容的对齐方式

默认情况下，标签的内容是左对齐和垂直居中的。

访问功能:

Qt::Alignment alignment() const
void setAlignment(Qt::Alignment)

See also text.

[read-only]hasSelectedText : const bool#

这个属性表示是否有任何文本被选中

hasSelectedText()如果部分或全部文本被用户选中，返回 “true”；否则返回 “false”。

默认情况下，该属性为 “false”。

注意：标签上设置的textInteractionFlags需要包括TextSelectableByMouse或TextSelectableByKeyboard。

访问功能:

bool hasSelectedText() const

See also selectedText().

indent : int#

这个属性保持标签的文本缩进，单位是像素。

如果标签显示文本，如果alignment()是Qt::AlignLeft，则缩进适用于左边缘；如果alignment()是[Qt::AlignRight](qt. 如果alignment()是Qt::AlignTop，则到右边缘；如果alignment()是Qt::AlignBottom则到底部边缘。）

如果缩进是负的，或者没有设置缩进，标签会按如下方式计算有效缩进。如果frameWidth()为0，则有效缩进为0。如果frameWidth()大于0，则有效缩进为该部件当前font()中 “x “字符的一半宽度。

默认情况下，缩进为-1，意味着有效缩进是按照上述方式计算的。

访问功能:

int indent() const
void setIndent(int)

See also alignment, margin, frameWidth(), and font().

margin : int#

此属性保存了边距的宽度

margin是指框架最里面的像素和内容最外面的像素之间的距离。

默认边距为0。

访问功能:

int margin() const
void setMargin(int)

See also indent.

openExternalLinks : bool#

指定QLabel是否应该使用QDesktopServices::openUrl()自动打开链接，而不是发出linkActivated() 信号。

注意：标签上设置的textInteractionFlags需要包括LinksAccessibleByMouse或LinksAccessibleByKeyboard。

默认值为false。

访问功能:

void setOpenExternalLinks(bool open)

See also textInteractionFlags().

启用后，如果标签显示一个像素图，它将缩放像素图以填充可用空间。

[read-only]selectedText : const #

此属性用于保存所选文本

如果没有选定的文本，这个属性的值是一个空字符串。

默认情况下，这个属性包含一个空字符串。

注意：标签上设置的textInteractionFlags需要包括TextSelectableByMouse或TextSelectableByKeyboard。

访问功能:

QString selectedText() const

See also hasSelectedText().

text : #

此属性持有标签的文本

如果没有设置文本，这将返回一个空字符串。设置文本将清除之前的任何内容。

文本将被解释为纯文本或富文本，这取决于文本格式的设置；见setTextFormat()。默认设置是Qt::AutoText；也就是说，QLabel将尝试自动检测文本集的格式。关于富文本的定义，请参见支持的HTML子集。

如果已经设置了一个好友，好友的助记符将从新的文本中更新。

请注意，QLabel非常适合于显示小型富文本文档，比如小型文档从标签的调色板和字体属性中获得它们的文档特定设置（字体、文本颜色、链接颜色）。对于大型文档，请在只读模式下使用QTextEdit代替。QTextEdit在必要时还可以提供一个滚动条。

注意：如果文本包含丰富的文本，该功能可以实现鼠标跟踪。

访问功能:

QString text() const
void setText(const QString &)

See also setTextFormat(), setBuddy(), and alignment.

textFormat : #

该属性持有标签的文本格式

請看 Qt::TextFormat 枚举对可能的选项的解释。

默认格式是Qt::AutoText。

访问功能:

Qt::TextFormat textFormat() const
void setTextFormat(Qt::TextFormat)

See also text().

textInteractionFlags : #

指定标签在显示文本时应如何与用户输入互动。

如果旗標包含 Qt::LinksAccessibleByKeyboard ，焦點策略也會自動設定為 Qt::StrongFocus 。如果 Qt::TextSelectableByKeyboard 被設定，那麼焦點策略會被設定為 Qt::ClickFocus 。

默认值是 Qt::LinksAccessibleByMouse 。

访问功能:

Qt::TextInteractionFlags textInteractionFlags() const
void setTextInteractionFlags(Qt::TextInteractionFlags flags)

wordWrap : bool#

该属性持有标签的文字包装策略。

如果这个属性是true，那么标签文本在必要的时候会在断字处被包起来；否则就完全不被包起来。

默认情况下，文字包装是禁用的。

访问功能:

bool wordWrap() const
void setWordWrap(bool on)

See also text.

void QLabel::setBuddy( *buddy)#

将此标签的好友设置为buddy。

当用户按下这个标签所指示的快捷键时，键盘焦点会被转移到这个标签的好友小部件上。

伙伴机制只适用于包含有一个以安培符为前缀的文本的QLabels，即’&’。这个字符被设置为快捷键。详情见QKeySequence::mnemonic()文档（要显示一个真正的安培号，请使用’&&’）。

在一个对话框中，你可能会创建两个数据输入部件和一个标签，并设置几何布局，这样每个标签就在其数据输入部件的左边（它的 “伙伴”），例如：

QLineEdit *nameEdit  = new QLineEdit(this);
QLabel    *nameLabel = new QLabel("&Name:", this);
nameLabel->setBuddy(nameEdit);
QLineEdit *phoneEdit  = new QLineEdit(this);
QLabel    *phoneLabel = new QLabel("&Phone:", this);
phoneLabel->setBuddy(phoneEdit);
// (layout setup not shown)

通过上面的代码，当用户按Alt+N时，焦点会跳到姓名栏，当用户按Alt+P时，焦点会跳到电话栏。

要取消先前设置的好友，请在buddy设置为nullptr时调用此函数。

See also buddy(), setText(), QShortcut, and setAlignment().
