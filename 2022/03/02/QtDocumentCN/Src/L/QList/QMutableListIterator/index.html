详细描述#

QList同时具有Java-style iterators和STL-style iterators。STL风格的迭代器更有效，应该是首选。

使用迭代器的一个替代方法是使用索引位置。大多数QList的成员函数将索引作为它们的第一个参数，这使得访问、插入和删除项目都可以不使用迭代器。

QMutableListIterator允许你在QList（或QQueue）上迭代并修改列表。如果你不想修改列表（或者有一个const QList），使用稍快的QListIterator代替。

QMutableListIterator的构造函数需要一个QList作为参数。在构造之后，迭代器位于列表的最开始（第一个项目之前）。下面是如何依次迭代所有元素的方法：

QList<float> list;
...
QMutableListIterator<float> i(list);
while (i.hasNext())
    qDebug() << i.next();

next()函数返回列表中的下一个项目，并使迭代器前进。与STL风格的迭代器不同，Java风格的迭代器指向中间的项目，而不是直接指向项目。第一次调用next()将迭代器推进到第一个和第二个项目之间的位置，并返回第一个项目；第二次调用next()将迭代器推进到第二个和第三个项目之间的位置，返回第二个项目；以此类推。

下面是如何以相反的顺序迭代这些元素的方法：

QMutableListIterator<float> i(list);
i.toBack();
while (i.hasPrevious())
    qDebug() << i.previous();

如果你想找到一个特定值的所有出现，在一个循环中使用 findNext() 或 findPrevious() 。

如果你想在遍历列表时删除项目，使用remove()。如果你想修改一个项目的值，使用setValue()。如果你想在列表中插入一个新的项目，使用 insert()。

例子：

QMutableListIterator<int> i(list);
while (i.hasNext()) {
    int val = i.next();
    if (val < 0) {
        i.setValue(-val);
    } else if (val == 0) {
        i.remove();
    }
}

这个例子遍历了一个列表，用负数的绝对值替换了负数，并消除了零。

在任何时候，一个给定的列表上只能有一个可变迭代器处于活动状态。此外，当迭代器处于活动状态时，不应该直接对列表进行修改（而不是通过迭代器），因为这可能会使迭代器失效，导致未定义的行为。

See also QListIterator and QList::iterator.

成员函数文档#

bool QMutableListIterator::findPrevious(const T &value)#

从当前迭代器位置开始向后搜索value。如果找到value，返回true；否则返回false。

调用后，如果找到了value，迭代器被定位在匹配项之前；否则，迭代器被定位在容器的前面。

See also findNext().

bool QMutableListIterator::findNext(const T &value)#

从当前迭代器位置开始向前搜索value。如果找到value，返回 “true”；否则返回 “false”。

调用后，如果找到了value，迭代器被定位在匹配项之后；否则，迭代器被定位在容器的后面。

See also findPrevious().

bool QMutableListIterator::hasPrevious() const#

如果迭代器后面至少有一个项目，即迭代器不在容器的前面，则返回 “true”；否则返回 “false”。

See also hasNext() and previous().

bool QMutableListIterator::hasNext() const#

如果在迭代器前面至少有一个项目，即迭代器没有在容器的后面，则返回 “true”；否则返回 “false”。

See also hasPrevious() and next().

void QMutableListIterator::toBack()#

将迭代器移到容器的后面（最后一个项目之后）。

See also toFront() and previous().

void QMutableListIterator::toFront()#

将迭代器移到容器的前面（在第一个项目之前）。

See also toBack() and next().

&QMutableListIterator::operator=(QList &container)#

使得迭代器在list上操作。迭代器被设置为在列表的前面（第一个项目之前）。

See also toFront() and toBack().

QMutableListIterator::QMutableListIterator( &list)#

构建一个用于遍历list的迭代器。迭代器被设置为在列表的前面（第一个项目之前）。

See also operator=().

void QMutableListIterator::insert(const T &value)#

在当前迭代器的位置插入value。调用后，迭代器的位置就在插入的项目之后。

See also remove() and setValue().

T &QMutableListIterator::next()#

返回一个对下一个项目的引用，并使迭代器前进一个位置。

在位于容器后面的迭代器上调用这个函数会导致未定义的结果。

See also hasNext(), peekNext(), and previous().

T &QMutableListIterator::peekNext() const#

返回对下一个项目的引用，而不移动迭代器。

在位于容器后面的迭代器上调用这个函数会导致未定义的结果。

See also hasNext(), next(), and peekPrevious().

T &QMutableListIterator::peekPrevious() const#

返回对前一个项目的引用，而不移动迭代器。

在位于容器前端的迭代器上调用这个函数会导致未定义的结果。

See also hasPrevious(), previous(), and peekNext().

T &QMutableListIterator::previous()#

返回对前一个项目的引用，并将迭代器向后移动一个位置。

在位于容器前端的迭代器上调用这个函数会导致未定义的结果。

See also hasPrevious(), peekPrevious(), and next().

void QMutableListIterator::remove()#

移除使用其中一个遍历函数（next(), previous()）跳过的最后一个项目。findNext(), findPrevious()).

例子:

QMutableListIterator<int> i(list);
while (i.hasNext()) {
    int val = i.next();
    if (val < -32768 || val > 32767)
        i.remove();
}

See also insert() and setValue().

void QMutableListIterator::setValue(const T &value) const#

用value替换使用其中一个遍历函数跳过的最后一个项目的值。

遍历函数是next(), previous(), findNext(), 和findPrevious()。

例子：

QMutableListIterator<double> i(list);
while (i.hasNext()) {
    double val = i.next();
    i.setValue(std::sqrt(val));
}

See also value(), remove(), and insert().

const T &QMutableListIterator::value() const#

返回使用其中一个遍历函数（next(), previous(), findNext(), findPrevious()）跳过的最后一个项目的值。

在调用next()或findNext()后，value()等同于peekPrevious()。在调用previous()或findPrevious()后，value()等同于peekNext()。

See also setValue().

T &QMutableListIterator::value()#

这是一个重载函数。

返回一个非静态的引用，指向使用其中一个遍历函数跳过的最后一个项目的值。
