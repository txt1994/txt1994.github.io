详细描述#

QUtf8StringView 引用它不拥有的 UTF-8 字符串的连续部分。它充当各种 UTF-8 字符串的接口类型，无需先构造QString或QByteArray。

UTF-8 字符串可以表示为 、 或 的数组（或数组兼容的数据结构，例如 std:: char8_tbasic_string等char）。signed char``unsigned char

QUtf8StringView 被设计为接口类型；它的主要用例是作为函数参数类型。当 QUtf8StringViews 用作自动变量或数据成员时，必须注意确保引用的字符串数据（例如，由 std::u8string 拥有）在所有代码路径上都比 QUtf8StringView 寿命长，以免字符串视图最终引用已删除数据。

当用作接口类型时，QUtf8StringView 允许单个函数接受多种 UTF-8 字符串数据源。一个接受 QUtf8StringView 的函数因此替换了几个函数重载（例如QByteArray），同时允许将更多的字符串数据源传递给函数，例如u8"Hello World"( char8_tC++20) 或char(C++17)字符串字面量。char8_t使用 QUtf8StringView 时，C++17 和 C++20 之间的不兼容性消失了。

像所有视图一样，QUtf8StringViews 应该按值传递，而不是通过引用到常量：

void myfun1(QUtf8StringView sv);        // preferred
void myfun2(const QUtf8StringView &sv); // compiles and works, but slower

如果您想让您的用户在可以将哪些字符串传递给您的函数方面给予最大的自由，请考虑改用QAnyStringView。

QUtf8StringView 也可以用作函数的返回值。如果您调用返回 QUtf8StringView 的函数，请特别注意不要使 QUtf8StringView 保持的时间长于函数承诺保持引用的字符串数据处于活动状态的时间。如果有疑问，通过调用toString () 将 QUtf8StringView 转换为QString来获得对数据的强引用。

QUtf8StringView 是文字类型。

兼容的字符类型#

  • QUtf8StringView 接受各种字符类型的字符串：
    • char（签名和未签名）
    • char8_t（仅限 C++20）

大小和子字符串#

QUtf8StringView 函数中的所有大小和位置都以 UTF-8 代码点表示（即 UTF-8 多字节序列根据其长度计为两个、三个或四个）。QUtf8StringView 不会尝试通过 UTF-8 多字节序列来检测或阻止切片。这类似于QStringView和代理对的情况。

C++20、char8_t 和 QUtf8StringView#

在 C++20 中，u8""字符串文字将其类型从 更改const char[]为const char8_t[]. 如果 Qt 6 可能依赖于 C++20，QUtf8StringView 将char8_t本地存储，并且以下函数和别名将使用 (pointers to) char8_t：

  • storage_type, value_type, etc
  • begin(), end(), data(), etc
  • front(), back(), at(), operator

这就是 Qtf8StringView 在 Qt 7 中的预期外观，但对于 Qt 6，这是不可能的。Qt 没有在未来十年将用户锁定在 C++17 时代的界面中，而是在不同的（内联）命名空间中提供了两个 QUtf8StringView 类。第一个，在 namespace 中q_no_char8_t，具有value_type并且是const char普遍可用的。第二个，在 namespace 中q_has_char8_t，具有value_typeconst char8_t并且仅在以 C++20 模式编译时可用。

q_no_char8_t是一个内联命名空间，与 C++ 版本无关，以避免意外的二进制不兼容。要使用该char8_t版本，您需要使用q_has_char8_t::QUtf8StringView.

在内部，两者都是相同模板类 QBasicUtf8StringView 的实例化。请不要在源代码中使用模板类的名称。

另请参见QAnyStringView、QUtf8StringView和QString。
