详细说明#

它用于解析在 URL 中找到的查询字符串，如下所示：

像上面这样的查询字符串用于在 URL 中传输选项，通常被解码为多个键值对。上面的一个将在其列表中包含两个条目，键为“type”和“color”。QUrlQuery 还可用于从查询的各个组件创建适合在QUrl::setQuery () 中使用的查询字符串。

解析查询字符串的最常见方法是在构造函数中通过将查询字符串传递给它来对其进行初始化。否则，可以使用setQuery () 方法设置要解析的查询。在使用setQueryDelimiters () 函数设置非标准分隔符后，该方法还可用于解析带有非标准分隔符的查询。

可以使用query () 再次获取编码后的查询字符串。这将获取所有内部存储的项目并使用分隔符对字符串进行编码。

编码#

QUrlQuery 中的所有 getter 方法都支持QUrl::ComponentFormattingOptions类型的可选参数，包括query ()，它指示如何对相关数据进行编码。除了QUrl::FullyDecoded 之外，返回的值仍然必须被视为百分比编码的字符串，因为某些值不能以解码形式表示（如控制字符、无法解码为 UTF-8 的字节序列）。因此，百分比字符始终由字符串“%25”表示。

处理空格和加号（“+”）#

Web 浏览器通常将 HTML FORM 元素中的空格编码为加号 (“+”)，并将加号编码为其百分比编码形式 (%2B)。但是，管理 URL 的 Internet 规范不考虑空格和加号等价字符。

因此，QUrlQuery 永远不会将空格字符编码为“+”，也永远不会将“+”解码为空格字符。相反，空格字符将以编码形式呈现为“%20”。

为了支持 HTML 表单的编码，QUrlQuery 也从不将“%2B”序列解码为加号，也不编码加号。事实上，在键、值或查询字符串中找到的任何“%2B”或“+”序列都与写入的完全一样（“%2b”到“%2B”的大写除外）。

全解码#

使用QUrl::FullyDecoded格式，所有百分比编码的序列都将被完全解码，并且 ‘%’ 字符用于表示自身。QUrl::FullyDecoded应该小心使用，因为它可能会导致数据丢失。有关可能丢失哪些数据的信息，请参阅QUrl::FullyDecoded的文档。

仅当在不需要百分比编码的上下文中处理呈现给用户的文本时，才应使用此格式化模式。请注意，QUrlQuery 设置器和查询方法不支持对应的QUrl::DecodedMode解析，因此使用QUrl::FullyDecoded获取键列表可能会导致在对象中找不到键。

非标准分隔符#

默认情况下，QUrlQuery 使用等号 (“=”) 将键与其值分开，并使用与号 (“&”) 将键值对彼此分开。可以通过调用setQueryDelimiters ()更改 QUrlQuery 用于解析和重构查询的分隔符。

非标准分隔符应从 RFC 3986 所称的“子分隔符”中选择。他们是：

sub-delims    = "!" / "$" / "&" / "'" / "(" / ")"
              / "*" / "+" / "," / ";" / "="

不支持使用其他字符，可能会导致意外行为。QUrlQuery 不会验证您是否传递了有效的分隔符。

See also QUrl.

QUrlQuery:: QUrlQuery (const & queryString )#

QUrlQuery:: QUrlQuery (const & url )#

构造一个 QUrlQuery 对象并使用默认查询分隔符解析在url URL 中找到的查询字符串。要使用其他分隔符解析查询字符串，应首先使用 setQueryDelimiters() 设置它们，然后使用setQuery ( ) 设置查询。

另见QUrl::query ()。

销毁此QUrlQuery对象。

void QUrlQuery::addQueryItem(const &key, const &value)#

将对key = value附加到 URL 的查询字符串的末尾。此方法不会覆盖可能使用相同键存在的现有项目。

注意：此方法不像 HTML 表单那样将空格 (ASCII 0x20) 和加号 (“+”) 视为相同。如果您需要将空格表示为加号，请使用实际的加号。

另请参见hasQueryItem () 和queryItemValue ()。

QUrlQuery::allQueryItemValues(const &key, encoding = QUrl::PrettyDecoded) const#

返回查询字符串值的列表，其键等于URL 中的键，使用 encoding 中指定的选项对返回值进行编码。如果未找到key ，则此函数返回一个空列表。

另请参见queryItemValue () 和addQueryItem ()。

void QUrlQuery::clear()#

通过删除当前存储的所有键值对来清除此QUrlQuery对象。如果查询分隔符已更改，此函数将保留更改后的值。

另请参见isEmpty () 和setQueryDelimiters ()。

[static]char16_t QUrlQuery::defaultQueryPairDelimiter()#

返回用于分隔键值对的默认字符，一个 & 符号 (“&”)。

注意：在 Qt 6 之前，此函数返回QChar。

另请参见setQueryDelimiters ()、queryPairDelimiter () 和defaultQueryValueDelimiter ()。

[static]char16_t QUrlQuery::defaultQueryValueDelimiter()#

返回查询中用于分隔键和值的默认字符，即等号 (“=”)。

注意：在 Qt 6 之前，此函数返回QChar。

另请参见setQueryDelimiters ()、queryValueDelimiter () 和defaultQueryPairDelimiter ()。

bool QUrlQuery::hasQueryItem(const &key) const#

如果有一个查询字符串对，其键等于URL 中的键，则true。

另请参见addQueryItem () 和queryItemValue ()。

bool QUrlQuery::isEmpty() const#

返回true此QUrlQuery对象是否不包含键值对，例如在默认构造之后或在解析空查询字符串之后。

另请参见setQuery () 和clear ()。

QUrlQuery::query( encoding = QUrl::PrettyDecoded) const#

返回重建的查询字符串，由当前存储在此QUrlQuery对象中的键值对组成，并由为此对象选择的查询分隔符分隔。使用encoding参数给出的选项对键和值进行编码。

对于此函数，唯一不明确的分隔符是哈希（“#”），因为在 URL 中，它用于将查询字符串与可能跟随的片段分开。

返回字符串中键值对的顺序与原始查询完全相同。

另请参见setQuery ()、QUrl::setQuery ()、QUrl::fragment () 和Encoding。

QUrlQuery::queryItemValue(const &key, encoding = QUrl::PrettyDecoded) const#

从 URL 返回与键key关联的查询值，使用encoding中指定的选项对返回值进行编码。如果没有找到 key，这个函数返回一个空字符串。如果你需要区分一个空值和一个不存在的键，你应该首先使用hasQueryItem () 检查键的存在。

如果键key被多次定义，此函数将返回找到的第一个，按照它们在查询字符串中出现的顺序或使用addQueryItem () 添加的顺序。

另请参见addQueryItem ()、allQueryItemValues () 和Encoding。

<QPair<, > > QUrlQuery::queryItems( encoding = QUrl::PrettyDecoded) const#

返回 URL 的查询字符串，作为键和值的映射，使用encoding中指定的选项对项目进行编码。元素的顺序与在查询字符串中找到的或用setQueryItems () 设置的顺序相同。

另请参见setQueryItems () 和Encoding。

在query () 中重构查询字符串或在setQuery () 中解析时，返回用于分隔键值对的字符。

另请参见setQueryDelimiters () 和queryValueDelimiter ()。

QUrlQuery::queryValueDelimiter() const#

在query () 中重构查询字符串或在setQuery () 中解析时，返回用于分隔键和值的字符。

另请参见setQueryDelimiters () 和queryPairDelimiter ()。

void QUrlQuery::removeAllQueryItems(const &key)#

从 URL 中删除所有 key 等于key的查询字符串对。

另请参见removeQueryItem ()。

void QUrlQuery::removeQueryItem(const &key)#

从 URL 中删除其 key 等于key的查询字符串对。如果有多个项目的键等于key，它会按照它们在查询字符串中出现的顺序删除第一个项目或使用addQueryItem () 添加。

另请参见removeAllQueryItems ()。

void QUrlQuery::setQuery(const &queryString)#

解析queryString中的查询字符串并将内部项目设置为在那里找到的值。如果使用setQueryDelimiters ()指定了任何分隔符，此函数将使用它们而不是默认分隔符来解析字符串。

See also query().

void QUrlQuery::setQueryDelimiters( valueDelimiter, pairDelimiter)#

设置用于分隔键和值之间以及 URL 查询字符串中的键值对之间的字符。默认值分隔符为“=”，默认对分隔符为“&”。

valueDelimiter将用于分隔键和值，而pairDelimiter将用于分隔键值对。在查询字符串的键和值的编码表示中出现的任何这些定界字符在查询() 中返回时都会进行百分比编码。

如果valueDelimiter设置为 ‘(‘ 并且pairDelimiter为 ‘)’，则上述查询字符串将改为如下表示：

http://www.example.com/cgi-bin/drawgraph.cgi?type(pie)color(green)

注意：非标准分隔符应从 RFC 3986 所称的“子分隔符”中选择。他们是： 

sub-delims    = "!" / "$" / "&" / "'" / "(" / ")"
              / "*" / "+" / "," / ";" / "="

不支持使用其他字符，可能会导致意外行为。此方法不会验证您是否传递了有效的分隔符。

另请参见queryValueDelimiter () 和queryPairDelimiter ()。

void QUrlQuery::setQueryItems(const <QPair<, > > &query)#

将此QUrlQuery对象中的项目设置为query。query中元素的顺序被保留。

注意：此方法不像 HTML 表单那样将空格 (ASCII 0x20) 和加号 (“+”) 视为相同。如果您需要将空格表示为加号，请使用实际的加号。

另请参见queryItems () 和isEmpty ()。

void QUrlQuery::swap( &other)#

将此 URL 查询实例与other. 此功能非常快且永不失败。

QUrlQuery::toString( encoding = QUrl::PrettyDecoded) const#

将此QUrlQuery作为QString返回。encoding可用于指定返回值的 URL 字符串编码。

bool QUrlQuery::operator!=(const &other) const#

如果other不等于此QUrlQuery则返回true。否则，返回false。

See also operator==().

bool QUrlQuery::operator==(const &other) const#

如果此对象和另一个对象包含相同的内容、以相同的顺序并使用相同的查询分隔符，则返回true。
