详细描述#

QNetworkDatagram可以与QUdpSocket类一起使用，以表示一个UDP（用户数据报协议）数据报中包含的全部信息。QNetworkDatagram封装了一个数据报的下列信息：

  • 有效载荷数据。
  • 发送方地址和端口号。
  • 目的地地址和端口号。
  • 剩余的跳数限制（在IPv4中，这个字段通常被称为 “生存时间”–TTL）。
  • 收到数据报的网络接口索引或要发送的网络接口索引。

QUdpSocket将尽可能地在所有操作系统上匹配常见的行为，但在某些操作系统中，并非上述所有元数据都能获得。在用QUdpSocket::writeDatagram()发送时无法在数据报上设置的元数据将被默默地丢弃。

在接收时，senderAddress()和senderPort()属性包含发送数据报的对等体的地址和端口，而destinationAddress()和destinationPort()包含数据报中包含的目标。这通常是当前机器的本地地址，但它也可以是一个IPv4广播地址（如 “255.255.255.255”）或一个IPv4或IPv6组播地址。应用程序可能会发现，确定数据报是否通过单播寻址专门发送给这台机器，或者是否发送给了多个目的地是很有用的。

发送时，senderAddress()和senderPort()应包含发送时要使用的本地地址。发件人地址必须是分配给本机的地址，可以通过QNetworkInterface获得，而端口号必须是套接字所绑定的端口号。任何一个字段都可以不设置，将由操作系统用默认值填入。destinationAddress()和destinationPort()字段可以设置为一个不同于UDP套接字当前关联的目标地址。

通常，当发送一个数据报以回复之前收到的数据报时，人们会将destinationAddress()设置为传入数据报的senderAddress()，端口号也是如此。为了促进这一共同过程，QNetworkDatagram提供了makeReply()函数。

hopCount()函数对于收到的数据报来说，包含了该数据包的剩余跳数限制。在发送时，它包含要设置的跳数限制。大多数协议会把这个值设置为默认值，让操作系统决定使用的最佳值。IPv4上的组播经常使用这个字段来表示组播组的范围（链路本地、组织本地或全球）。

interfaceIndex()函数包含接收数据包的操作系统接口的索引。这个值与可以在QHostAddress::scopeId()属性上设置的值相同，并且与QNetworkInterface::index()属性匹配。当向全局地址发送数据包时，没有必要设置接口索引，因为操作系统会使用系统的路由表选择正确的接口。当向链接本地目的地发送数据报时，无论是单播还是多播，这个属性都很重要。

特点支持#

QNetworkDatagram的某些功能不是在所有操作系统中都支持的。在所有系统中，只有远程主机的地址和端口（接收数据包的发送方和发送数据包的目的地）被支持。在大多数操作系统中，其他功能只支持IPv6。软件应在运行时检查其余部分是否可以确定为IPv4地址。

目前的功能支持情况如下:

成员函数文档#

QNetworkDatagram::makeReply(const &payload) &&#

QNetworkDatagram::makeReply(const &payload) const &#

创建一个新的QNetworkDatagram，代表对这个传入数据报的回复，并将有效载荷数据设置为payload。这个函数是一个非常方便的方法，可以将数据报回复给原发件人。

例子:

void Server::readPendingDatagrams()
{
    while (udpSocket->hasPendingDatagrams()) {
        QNetworkDatagram datagram = udpSocket->receiveDatagram();
        QByteArray replyData = processThePayload(datagram.data());
        udpSocket->writeDatagram(datagram.makeReply(replyData));
    }
}

这个函数特别方便，因为它将自动从这个数据报中复制参数到新的数据报中。

  • 这个数据报的发送者地址和端口被复制到新数据报的目的地址和端口。
  • 这个数据报的接口索引（如果有的话）被复制到新数据报的接口索引。
  • 只有当地址是IPv6全局（非多播）地址时，该数据报的目的地址和端口才会被复制到新数据报的发送者地址和端口。
  • 新数据报的跳数限制被重置为默认值（-1）。

如果QNetworkDatagram在未来的Qt版本中被修改为携带更多的元数据，这个函数将适当地复制这些元数据。

如果这个数据报的目标地址是一个IPv4地址，则不会被复制，因为如果不对分配给本机的所有地址进行详尽的搜索，就不可能将IPv4广播地址与普通IPv4地址区分开来。试图发送一个发件人地址等于广播地址的数据报可能会失败。然而，这不应该影响通信，因为具有多个IPv4地址的网络接口并不常见，所以操作系统将选择的地址很可能是对等体能够理解的地址。

注意：这个函数带有rvalue-和lvalue-引用限定符重载，所以在调用makeReply之前，如果可能的话，最好确保这个对象是rvalue，这样才能更好地利用移动语义。为了实现这一点，上面的例子将使用:

udpSocket->writeDatagram(std::move(datagram).makeReply(replyData));

QNetworkDatagram::QNetworkDatagram(const &other)#

创建一个other数据报的副本，包括有效载荷和元数据。

要创建一个适合在回复中发送的数据报，请使用QNetworkDatagram::makeReply()。

QNetworkDatagram::QNetworkDatagram(const &data, const &destinationAddress = QHostAddress(), port = 0)#

创建一个QNetworkDatagram对象，并将data作为有效载荷数据，以及destinationAddress和port作为数据报的目的地址。

QNetworkDatagram::QNetworkDatagram()#

创建一个没有有效载荷数据和未定义目标地址的QNetworkDatagram对象。

有效载荷可以通过使用setData()来修改，目的地址可以通过setDestination()来设置。

如果目标地址未被定义，QUdpSocket::writeDatagram()将试图通过使用QUdpSocket::connectToHost()将数据报发送至最后关联的地址。

&QNetworkDatagram::operator=(const &other)#

复制other数据报，包括有效载荷和元数据。

要创建一个适合在回复中发送的数据报，请使用QNetworkDatagram::makeReply()。

void QNetworkDatagram::clear()#

清除这个QNetworkDatagram对象中的有效载荷数据和元数据，将它们重置为默认值。

QNetworkDatagram::data() const#

返回这个数据报的数据有效载荷。对于从网络上收到的数据报，它包含数据报的有效载荷。对于一个发出的数据报，它是要发送的数据报。

注意，数据报可以在没有数据的情况下被发送，所以返回的QByteArray可能是空的。

See also setData().

QNetworkDatagram::destinationAddress() const#

返回与此数据报相关的目标地址。对于从网络上收到的数据报，它是对等节点发送数据报的地址，它可以是本机的本地地址，也可以是多播或广播地址。对于传出的数据报，它是数据报应该发送的地址。

如果在这个数据报上没有设置目标地址，返回的对象将向QHostAddress::isNull()报告真。

See also senderAddress(), destinationPort(), and setDestination().

int QNetworkDatagram::destinationPort() const#

返回与此数据报相关的目的地的端口号。对于从网络上收到的数据报，它是对等节点发送数据报的本地端口号。对于传出的数据报，它是该数据报应该被发送到的对等端口。

如果没有目标地址与该数据报相关联，该函数返回-1。

See also destinationAddress(), senderPort(), and setDestination().

int QNetworkDatagram::hopLimit() const#

返回与此数据报相关的跳数限制。跳数限制是指在数据报过期并向数据报发送方发送错误之前允许转发IP数据包的节点数量。在IPv4中，这个值通常被称为 “生存时间”（TTL）。

如果这个数据报是从网络上收到的，这就是数据报接收后的剩余跳数，并由转发数据包的每个节点减去1。值为-1表示没有获得跳数限制。

如果这是一个外发数据报，这是发送时要在IP头中设置的值。值为-1表示操作系统应选择该值。

See also setHopLimit().

QNetworkDatagram::interfaceIndex() const#

返回这个数据报所关联的接口索引。接口索引是一个正数，在操作系统中唯一标识了网络接口。这个数字与QNetworkInterface::index()为该接口返回的值一致。

如果这个数据报是从网络上接收的，这是该数据包从哪个接口接收的索引。如果这是一个外发数据报，这是数据报应该被发送的接口的索引。

值为0表示接口索引未知。

See also setInterfaceIndex().

bool QNetworkDatagram::isNull() const#

如果这个QNetworkDatagram对象为空，则返回真。这个函数与isValid()相反。

bool QNetworkDatagram::isValid() const#

如果这个QNetworkDatagram对象是有效的，返回true。一个有效的QNetworkDatagram对象至少包含一个发送者或接收者地址。有效的数据报可以包含空的有效载荷。

QNetworkDatagram::senderAddress() const#

返回与此数据报相关的发送者地址。对于从网络上收到的数据报，它是发送该数据报的对等节点的地址。对于发出的数据报，它是发送时要使用的本地地址。

如果在这个数据报上没有设置发件人地址，返回的对象将向QHostAddress::isNull()报告真。

See also destinationAddress(), senderPort(), and setSender().

int QNetworkDatagram::senderPort() const#

返回与此数据报相关的发送者的端口号。对于从网络上收到的数据报，它是对等节点发送数据报的端口号。对于传出的数据报，它是该数据报应从的本地端口。

如果没有发件人地址与此数据报相关联，此函数返回-1。

See also senderAddress(), destinationPort(), and setSender().

void QNetworkDatagram::setData(const &data)#

将此数据报的数据有效载荷设置为data。通常没有必要在收到的数据报上调用这个函数。对于传出的数据报，这个函数设置将在网络上发送的数据。

由于数据报可以为空，一个空的QByteArray是data的一个有效值。

See also data().

void QNetworkDatagram::setDestination(const &address, port)#

将与该数据报相关的目标地址设置为地址address和端口号port。目的地址和端口号通常在接收时由QUdpSocket设置，因此不需要在接收的数据报上调用这个函数。

对于传出的数据报，该函数可用于设置数据报应被发送的地址。它可以是用于与对等体通信的单播地址，也可以是用于向一组设备发送的广播或多播地址。

See also QUdpSocket::writeDatagram(), destinationAddress(), destinationPort(), and setSender().

void QNetworkDatagram::setHopLimit(int count)#

设置与此数据报相关的跳数限制为count。跳数限制是指在IP数据包过期并向数据包发送者发送错误之前允许转发的节点数量。在IPv4中，这个值通常被称为 “生存时间”（TTL）。

通常没有必要在从网络上收到的数据报上调用这个函数。

如果这是一个发出的数据包，这是发送时要在IP头中设置的值。该值的有效范围是1到255。这个函数也接受一个-1的值，表示由操作系统来选择这个值。

See also hopLimit().

void QNetworkDatagram::setInterfaceIndex( index)#

将该数据报所关联的接口索引设置为index。接口索引是一个正数，在操作系统中唯一标识了网络接口。这个数字与QNetworkInterface::index()为该接口返回的值一致。

通常没有必要对从网络上收到的数据包调用这个函数。

如果这是一个外发数据包，这是数据报应该被发送的接口的索引。值为0表示操作系统应该根据其他因素来选择接口。

注意，接口索引也可以用QHostAddress::setScopeId()来设置IPv6目标地址，然后用setDestination()来设置。如果在目的地址中设置的范围ID和index不一样，并且都不是0，那么操作系统就无法确定哪个接口将发送数据报。

See also interfaceIndex() and setInterfaceIndex().

void QNetworkDatagram::setSender(const &address, port = 0)#

将与该数据报相关的发送方地址设置为地址address和端口号port。发件人地址和端口号通常在接收时由QUdpSocket设置，因此没有必要在接收的数据报上调用这个函数。

对于发出的数据报，该函数可用于设置数据报应携带的地址。地址address通常必须是分配给本机的本地地址之一，这些地址可以通过QNetworkInterface获得。如果不设置，操作系统会根据有关目的地选择最合适的地址来使用。

端口号port必须是与套接字相关的端口号，如果有的话。可以用0的值来表示操作系统应该选择端口号。

See also QUdpSocket::writeDatagram(), senderAddress(), senderPort(), and setDestination().
