详细描述#

Cookie是一些小的信息，像HTTP这样的无状态协议用来在不同的请求中保持一些持久的信息。

一个cookie是由远程服务器在回复一个请求时设置的，当进一步发送请求时，它希望同样的cookie被送回来。

QNetworkCookie持有一个从网络上收到的这样的cookie。一个cookie有一个名字和一个值，但这些对应用程序来说是不透明的（也就是说，存储在其中的信息对应用程序没有意义）。一个cookie有一个相关的路径名称和域，这表明该cookie何时应再次被发送到服务器。

一个cookie还可以有一个过期日期，表明其有效性。如果过期日期不存在，这个cookie就被认为是一个 “会话cookie”，当应用程序退出时（或当它的会话概念结束时）应该被丢弃。

QNetworkCookie提供了一种使用QNetworkCookie::parseCookies()函数从HTTP头格式解析cookie的方法。然而，当在QNetworkReply中收到时，cookie就已经被解析了。

这个类实现了Netscape最初的cookie规范所描述的cookie，它有点类似于RFC 2109规范，加上“HttpOnly “扩展。较新的RFC 2965规范（使用Set-Cookie2头）不被支持。

See also QNetworkCookieJar, QNetworkRequest, and QNetworkReply.

成员类型文档#

enum QNetworkCookie::RawForm#

这个枚举与toRawForm()函数一起使用，以声明应返回cookie的哪种形式。

Constant Value Description
QNetworkCookie::NameAndValueOnly 0 使toRawForm()只返回cookie的 “NAME=VALUE “部分，适合在客户端请求的 “Cookie: “头中发回给服务器。多个cookie在 “Cookie: “头域中用分号隔开。
QNetworkCookie::Full 1 使toRawForm()返回完整的cookie内容，适合在服务器的 “Set-Cookie: “头中发送给客户端。

请注意，只有Cookie的完整形式可以被解析回其原始内容。

See also toRawForm() and parseCookies().

[since 6.1]enum class QNetworkCookie::SameSite#

Constant Value Description
QNetworkCookie::SameSite::Default 0 SameSite未被设置。可以被浏览器解释为无或不严。
QNetworkCookie::SameSite::None 1 Cookie可以在所有情况下发送。这曾经是默认的，但最近的浏览器使Lax成为默认的，现在会要求cookie既要安全又要设置SameSite=None。
QNetworkCookie::SameSite::Lax 2 Cookies是在第一方请求和第三方网站发起的GET请求中发送的。这是现代浏览器的默认做法（自2020年中期起）。
QNetworkCookie::SameSite::Strict 3 Cookies只在第一方背景下发送。

这个枚举是在Qt 6.1中引入或修改的。

See also setSameSitePolicy() and sameSitePolicy().

成员函数文档#

QNetworkCookie::QNetworkCookie(const &other)#

通过复制other的内容创建一个新的QNetworkCookie对象。

QNetworkCookie::QNetworkCookie(const &name = QByteArray(), const &value = QByteArray())#

创建一个新的QNetworkCookie对象，将cookie的名称初始化为name，其值为value。

一个cookie只有在它有一个名字的情况下才是有效的。然而，该值对应用程序来说是不透明的，而且是空的可能对远程服务器有意义。

&QNetworkCookie::operator=(const &other)#

将QNetworkCookie对象的内容other复制到这个对象。

QNetworkCookie::~QNetworkCookie()#

销毁这个QNetworkCookie对象。

QNetworkCookie::domain() const#

返回这个cookie所关联的域。这与cookie字符串的 “域 “字段相对应。

注意，这里的域名可能以点开始，这不是一个有效的主机名。然而，它意味着这个cookie匹配所有以该域名结尾的主机名。

See also setDomain().

QNetworkCookie::expirationDate() const#

返回这个cookie的过期日期。如果这个cookie是一个会话cookie，返回的QDateTime将是无效的。如果日期是过去的，这个cookie已经过期，不应该再被送回远程服务器。

过期日期与cookie字符串中的 “expires “条目的参数相对应。

See also isSessionCookie() and setExpirationDate().

bool QNetworkCookie::hasSameIdentifier(const &other) const#

如果这个cookie与*其他的cookie具有相同的标识符元组，则返回`true’。标识符元组由名称、域和路径组成。

See also operator==().

bool QNetworkCookie::isHttpOnly() const#

如果这个cookie的 “HttpOnly “标志被启用，则返回true。

一个 “HttpOnly “的cookie只能由网络请求和回复来设置和检索；也就是说，HTTP协议。它不能被运行在浏览器上的脚本访问。

See also isSecure().

bool QNetworkCookie::isSecure() const#

如果在cookie字符串中指定了 “安全 “选项，则返回true，否则返回false。

安全cookie可能包含私人信息，不应通过未加密的连接重新发送。

See also setSecure().

bool QNetworkCookie::isSessionCookie() const#

如果这个cookie是一个会话cookie，返回 “true”。会话cookie是一种没有过期日期的cookie，这意味着当应用程序的会话概念结束时（通常是应用程序退出时），它应该被丢弃。

See also expirationDate() and setExpirationDate().

QNetworkCookie::name() const#

返回这个cookie的名称。一个cookie的唯一强制性字段是它的名字，没有它就不被认为有效。

See also setName() and value().

[since 5.0]void QNetworkCookie::normalize(const &url)#

如果cookie的路径和域以前是空的，这个函数将它们规范化。参数url被用来确定正确的域和路径。

这个函数从Qt 5.0 引入。

[static]<> QNetworkCookie::parseCookies(const &cookieString)#

解析从服务器响应的 “Set-Cookie: “头中收到的cookie字符串cookieString。如果有一个解析错误，这个函数会返回一个空的列表。

由于HTTP头可以同时设置一个以上的cookie，这个函数返回一个QList<QNetworkCookie>，每个被解析的cookie都有一个。

See also toRawForm().

QNetworkCookie::path() const#

返回与这个cookie相关的路径。这与cookie字符串的 “path “字段相对应。

See also setPath().

[since 6.1] QNetworkCookie::sameSitePolicy() const#

如果在cookie字符串中指定了”SameSite “选项，则返回 “SameSite::Default”，如果不存在的话。

这个函数从Qt 6.1 引入。

See also setSameSitePolicy().

void QNetworkCookie::setDomain(const &domain)#

将与此cookie相关的域设置为domain。

See also domain().

void QNetworkCookie::setExpirationDate(const &date)#

将这个cookie的过期日期设置为date。给这个cookie设置一个无效的过期日期将意味着它是一个会话cookie。

See also isSessionCookie() and expirationDate().

void QNetworkCookie::setHttpOnly(bool enable)#

将这个cookie的 “HttpOnly “标志设置为enable。

See also isHttpOnly().

void QNetworkCookie::setName(const &cookieName)#

将这个cookie的名字设置为cookieName。注意，将cooki名称设置为空的QByteArray将使这个cookie无效。

See also name() and value().

void QNetworkCookie::setPath(const &path)#

将与此cookie相关的路径设置为path。

See also path().

[since 6.1]void QNetworkCookie::setSameSitePolicy( sameSite)#

将这个cookie的”SameSite “选项设置为sameSite。

这个函数从Qt 6.1 引入。

See also sameSitePolicy().

void QNetworkCookie::setSecure(bool enable)#

将这个cookie的安全标志设置为enable。

安全cookie可能包含私人信息，不应通过未加密的连接重新发送。

See also isSecure().

void QNetworkCookie::setValue(const &value)#

将这个cookie的值设置为value。

See also value() and name().

[since 5.0]void QNetworkCookie::swap( &other)#

将这个cookie与othercookie交换。这个功能非常快，而且从不失败。

这个函数从Qt 5.0 引入。

QNetworkCookie::toRawForm( form = Full) const#

返回这个QNetworkCookie的原始形式。这个函数返回的QByteArray适合作为HTTP头，在服务器响应（Set-Cookie头）或客户端请求（Cookie头）中。你可以从两种格式中选择一种，使用form。

See also parseCookies().

QNetworkCookie::value() const#

返回这个cookie的值，如cookie字符串中指定的。请注意，如果一个cookie的值是空的，它仍然是有效的。

Cookie的名-值对被认为是对应用程序不透明的：也就是说，它们的值没有任何意义。

See also setValue() and name().

bool QNetworkCookie::operator!=(const &other) const#

如果这个cookie不等于other，则返回true。

See also operator==().

[since 5.0]bool QNetworkCookie::operator==(const &other) const#

如果这个cookie等于other，则返回true。这个函数只在cookie的所有字段都相同时返回true。

然而，在某些情况下，两个同名的cookie可以被认为是相等的。

这个函数从Qt 5.0 引入。

See also operator!=() and hasSameIdentifier().
