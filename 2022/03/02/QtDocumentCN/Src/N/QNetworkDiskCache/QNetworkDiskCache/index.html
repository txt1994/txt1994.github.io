详细描述#

QNetworkDiskCache使用QDataStream将每个url存储在cacheDirectory中的自己的文件里。使用qCompress对文本MimeType的文件进行压缩。数据只在insert()和updateMetaData()中写到磁盘上。

目前，你不能与一个以上的磁盘缓存共享相同的缓存文件。

QNetworkDiskCache默认将缓存在系统中使用的空间限制为50MB。

注意你必须设置缓存目录才能工作。

网络磁盘缓存可以通过以下方式启用：

QNetworkAccessManager *manager = new QNetworkAccessManager(this);
QNetworkDiskCache *diskCache = new QNetworkDiskCache(this);
diskCache->setCacheDirectory("cacheDir");
manager->setCache(diskCache);

在发送请求时，为了控制何时使用缓存和何时使用网络的偏好，请考虑以下几点：

// do a normal request (preferred from network, as this is the default)
QNetworkRequest request(QUrl(QString("http://qt-project.org")));
manager->get(request);

// do a request preferred from cache
QNetworkRequest request2(QUrl(QString("http://qt-project.org")));
request2.setAttribute(QNetworkRequest::CacheLoadControlAttribute, QNetworkRequest::PreferCache);
manager->get(request2);

要检查响应是来自缓存还是来自网络，可以采用以下方法:

void replyFinished(QNetworkReply *reply) {
    QVariant fromCache = reply->attribute(QNetworkRequest::SourceIsFromCacheAttribute);
    qDebug() << "page from cache?" << fromCache.toBool();
}

[virtual protected] QNetworkDiskCache::expire()#

清理缓存，使其大小低于最大缓存大小。返回缓存的当前大小。

当缓存的当前大小大于maximumCacheSize()时，较老的缓存文件被移除，直到总大小小于maximumCacheSize()的90%，先从最老的开始，使用文件创建日期来确定一个缓存文件的年龄。

子类可以重新实现这个函数来改变缓存文件的删除顺序，同时考虑到应用程序所知道的QNetworkDiskCache所不知道的信息，例如，缓存被访问的次数。

注意： cacheSize()的调用在当前缓存大小未知时过期。

See also maximumCacheSize() and fileMetaData().

void QNetworkDiskCache::setCacheDirectory(const &cacheDir)#

设置存储缓存文件的目录为cacheDir。

QNetworkDiskCache将创建这个目录，如果它不存在。

准备好的缓存项目在插入时将被存储在新的缓存目录中。

See also cacheDirectory() and QStandardPaths::CacheLocation.

void QNetworkDiskCache::setMaximumCacheSize( size)#

设置磁盘缓存的最大尺寸为size。

如果新的大小小于当前缓存的大小，那么缓存将调用expire()。

See also maximumCacheSize().
