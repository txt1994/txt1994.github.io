详细描述#

QNetworkInformation通过插件提供了一个跨平台的网络相关信息的接口。

各种插件可以有不同的功能支持，因此你可以根据需要的功能来load()插件。

QNetworkInformation是一个单例，从第一个成功的load()开始就保持活动，直到QCoreApplication对象被销毁。如果你销毁并重新创建QCoreApplication对象，你必须再次调用load()。

See also QNetworkInformation::Feature.

成员类型文档#

enum class QNetworkInformation::Feature flags QNetworkInformation::Features#

列出一个插件目前可能支持的所有功能。这可以在QNetworkInformation::load()中使用。

Constant Value Description
QNetworkInformation::Feature::Reachability 0x1 如果该插件支持这个功能，那么reachability属性将提供有用的结果。否则它将总是返回Reachability::Unknown。参见[QNetworkInformation::Reachability]（#Reachability-enum）。
QNetworkInformation::Feature::CaptivePortal 0x2 如果该插件支持这个功能，那么isBehindCaptivePortal属性将提供有用的结果。否则它将总是返回 “false”。

Features类型是QFlags的一个类型化定义。它存储了一个OR组合的特征值。

enum class QNetworkInformation::Reachability#

Constant Value Description
QNetworkInformation::Reachability::Unknown 0 如果返回这个值，那么我们可能已经连接上了，但操作系统仍未确认完全连接，或者这个功能不被支持。
QNetworkInformation::Reachability::Disconnected 1 表示系统可能根本没有连接。
QNetworkInformation::Reachability::Local 2 表示系统已连接到网络，但可能只能访问本地网络的设备。
QNetworkInformation::Reachability::Site 3 表示系统已连接到网络，但可能只能访问本地子网或内网的设备。
QNetworkInformation::Reachability::Online 4 表示系统已连接到网络，能够访问互联网。

See also QNetworkInformation::reachability.

特性文档#

[read-only, since 6.2]isBehindCaptivePortal : const bool#

让你知道用户的设备是否在圈养门户后面。

这个属性表明用户的设备目前是否已知是在一个限制性门户网站后面。这个功能依赖于操作系统对限制性门户网站的检测，在不报告这个的系统上不被支持。在不支持该功能的系统上，该属性将始终返回 “false”。

这个属性是在Qt 6.2中引入的。

访问功能:

bool isBehindCaptivePortal() const

通知者信号：

void isBehindCaptivePortalChanged(bool state)

[read-only]reachability : const #

此属性持有系统的网络连接的当前状态。

表示可以预期的连接水平。请注意，这只是基于插件/操作系统的报告。在某些情况下，这被认为是错误的。例如，在Windows上，默认情况下，”在线 “检查是由Windows连接到微软拥有的服务器进行的。如果这个服务器由于任何原因被阻止，那么它将假定它没有在线可及性。正因为如此，你不应该在试图进行连接之前将其作为预检查。

访问功能:

QNetworkInformation::Reachability reachability() const

通知者信号:

void reachabilityChanged(QNetworkInformation::Reachability newReachability)

[static]QNetworkInformation *QNetworkInformation::instance()#

返回一个指向QNetworkInformation实例的指针，如果有的话。

[static]bool QNetworkInformation::load( backend)#

试图加载一个名称与backend匹配的后端（不考虑大小写）。

如果它成功加载了请求的后端，或者它已经被加载，则返回true。否则返回 “false”。

See also instance.

[static]bool QNetworkInformation::load( features)#

加载一个支持features的后端。

如果它成功加载了请求的后端，或者它已经被加载，则返回true。否则返回 “false”。

See also instance.

bool QNetworkInformation::supports( features) const#

如果当前加载的后端支持features，则返回 “true”。
