QNetworkAddressEntry Class#

QNetworkAddressEntry类存储了一个网络接口所支持的一个IP地址，以及其相关的网络掩码和广播地址。 More…

详细描述#

每个网络接口可以包含零个或多个IP地址，而这些地址又可以与一个网络掩码和/或一个广播地址相关联（取决于操作系统的支持）。

该类代表一个这样的组。

成员类型文档#

[since 5.11]enum QNetworkAddressEntry::DnsEligibilityStatus#

这个枚举表明一个给定的主机地址是否有资格被公布在域名系统（DNS）或其他类似的名称解析机制中。一般来说，如果一个地址是可以在不确定的时间内到达这台机器的地址，那么这个地址就适合公布，尽管它不一定是永久性的。例如，通过DHCP获得的地址通常符合条件，但通过加密生成的临时IPv6地址则不符合条件。

Constant Value Description
QNetworkAddressEntry::DnsEligibilityUnknown -1 Qt和操作系统无法确定这个地址是否应该被发布。如果找不到任何符合条件的地址，应用程序可能需要进一步应用启发式方法。
QNetworkAddressEntry::DnsEligible 1 This address is eligible for publication in DNS.
QNetworkAddressEntry::DnsIneligible 0 这个地址不应该在DNS中公布，也不应该传送给其他各方，除非是作为一个外发数据包的源地址。

这个枚举是在Qt 5.11中引入或修改的。

See also dnsEligibility() and setDnsEligibility().

QNetworkAddressEntry::broadcast() const#

返回与IPv4地址和网络掩码相关的广播地址。它通常可以通过将净掩码包含0的IP地址位设置为1而得到。（换句话说，通过将IP地址与净掩码的倒数进行位运算）

对于IPv6地址来说，这个成员总是空的，因为在该系统中，广播的概念已经被放弃，转而使用组播。特别是，对应于本地网络中所有节点的主机组可以通过 “所有节点 “的特殊组播组（地址FF02::1）到达。

See also setBroadcast().

[since 5.11]void QNetworkAddressEntry::clearAddressLifetime()#

重置该地址的首选寿命和有效寿命。在此调用后，isLifetimeKnown()将返回false。

这个函数从Qt 5.11引入。

See also preferredLifetime(), validityLifetime(), isLifetimeKnown(), and setAddressLifetime().

[since 5.11] QNetworkAddressEntry::dnsEligibility() const#

返回这个地址是否有资格在域名系统（DNS）或类似的名称解析机制中公布。

一般来说，如果一个地址是一个可以在不确定的时间内到达本机的地址，那么它就适合公布，尽管它不一定是永久性的。例如，通过DHCP获得的地址通常是合格的，但通过加密生成的临时IPv6地址则不合格。

在某些系统中，QNetworkInterface需要启发式地确定哪些地址是合格的。

这个函数从Qt 5.11引入。

See also isLifetimeKnown(), isPermanent(), and setDnsEligibility().

QNetworkAddressEntry::ip() const#

该函数返回在一个网络接口中发现的一个IPv4或IPv6地址。

See also setIp().

[since 5.11]bool QNetworkAddressEntry::isLifetimeKnown() const#

如果地址寿命是已知的，返回 “true”，如果不是，返回 “false”。如果不知道，preferredLifetime()和validityLifetime()都将返回QDeadlineTimer::Forever。

这个函数从Qt 5.11引入。

See also preferredLifetime(), validityLifetime(), setAddressLifetime(), and clearAddressLifetime().

[since 5.11]bool QNetworkAddressEntry::isPermanent() const#

如果这个地址在这个接口上是永久的，返回 “true”，如果是临时的，返回 “false”。永久地址是没有过期时间的，通常是静态的（手动配置的）。

如果不能确定这一信息，该函数返回`true’。

注意：取决于操作系统和网络配置工具，如果工具没有将细节正确地告知操作系统，临时地址有可能被解释为永久地址。

这个函数从Qt 5.11引入。

See also isLifetimeKnown(), validityLifetime(), and isTemporary().

[since 5.11]bool QNetworkAddressEntry::isTemporary() const#

如果这个地址在这个接口上是临时的，返回true，如果是永久的，返回false。

这个函数从Qt 5.11引入。

See also isLifetimeKnown(), validityLifetime(), and isPermanent().

QNetworkAddressEntry::netmask() const#

返回与IP地址相关的网络掩码。网络掩码以IP地址的形式表示，如255.255.0.0。

对于IPv6地址，前缀长度被转换为一个地址，其中设置为1的比特数等于前缀长度。对于前缀长度为64位（最常见的值），网络掩码将被表示为一个QHostAddress，持有地址FFF:FFF:FFF:FFF:。

See also setNetmask() and prefixLength().

[since 5.11] QNetworkAddressEntry::preferredLifetime() const#

如果知道的话，返回这个地址变得废弃（不再是首选）的最后期限。如果地址的寿命不知道（见isLifetimeKnown()），这个函数总是返回[QDeadlineTimer::Forever]（qdeadlinetimer.html#ForeverConstant-enum）。

当一个地址被优先使用时，它可能被操作系统作为新的、流出的数据包的源地址。在它被废弃后，它将对传入的数据包再保持一段时间的有效性，直到最后被删除（见validityLifetime()）。

这个函数从Qt 5.11引入。

See also validityLifetime(), isLifetimeKnown(), setAddressLifetime(), and clearAddressLifetime().

int QNetworkAddressEntry::prefixLength() const#

返回这个IP地址的前缀长度。前缀长度与网络掩码中设置为1的位数一致（见netmask()）。对于IPv4地址，该值在0到32之间。对于IPv6地址，它包含在0到128之间，是表示地址的首选形式。

如果不能确定前缀长度，该函数返回-1（即netmask()返回一个空的QHostAddress()）。

See also setPrefixLength() and netmask().

[since 5.11]void QNetworkAddressEntry::setAddressLifetime( preferred, validity)#

将此地址的首选寿命和有效寿命分别设置为preferred和validity的期限。在这次调用之后，isLifetimeKnown()将返回true，即使两个参数都是QDeadlineTimer::Forever。

这个函数从Qt 5.11引入。

See also preferredLifetime(), validityLifetime(), isLifetimeKnown(), and clearAddressLifetime().

void QNetworkAddressEntry::setBroadcast(const &newBroadcast)#

将此QNetworkAddressEntry对象的广播IP地址设置为newBroadcast。

See also broadcast().

[since 5.11]void QNetworkAddressEntry::setDnsEligibility( status)#

将该地址的DNS资格标志设置为status。

这个函数从Qt 5.11引入。

See also dnsEligibility().

void QNetworkAddressEntry::setIp(const &newIp)#

将QNetworkAddressEntry对象包含的IP地址设置为newIp。

See also ip().

void QNetworkAddressEntry::setNetmask(const &newNetmask)#

将此QNetworkAddressEntry对象包含的网络掩码设置为newNetmask。设置网络掩码的同时也设置了前缀的长度，以匹配新的网络掩码。

See also netmask() and setPrefixLength().

void QNetworkAddressEntry::setPrefixLength(int length)#

设置此IP地址的前缀长度为length。对于这种类型的IP地址，length的值必须是有效的：对于IPv4地址在0到32之间，对于IPv6地址在0到128之间。设置为任何无效的值相当于设置为-1，这意味着 “没有前缀长度”。

设置前缀长度也会设置净掩码（见netmask()）。

See also prefixLength() and setNetmask().

[since 5.0]void QNetworkAddressEntry::swap( &other)#

将这个网络地址条目实例与other交换。这个功能非常快，而且从不失败。

这个函数从Qt 5.0引入。

[since 5.11] QNetworkAddressEntry::validityLifetime() const#

如果知道的话，返回这个地址变得无效并将从网络堆栈中删除的期限。如果地址寿命不知道（见isLifetimeKnown()），这个函数总是返回QDeadlineTimer::Forever 。

当一个地址是有效的，它将被操作系统接受为本机的有效目标地址。它是否被用作新的、传出的数据包的源地址，除其他规则外，由首选寿命控制（见preferredLifetime()）。

这个函数从Qt 5.11引入。

See also preferredLifetime(), isLifetimeKnown(), setAddressLifetime(), and clearAddressLifetime().

bool QNetworkAddressEntry::operator!=(const &other) const#

如果该网络地址条目与other条目不同，则返回true。

bool QNetworkAddressEntry::operator==(const &other) const#

如果该网络地址条目与other条目相同，则返回true。
