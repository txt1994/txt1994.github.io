详细描述#

QGlobalStatic类是使用Q_GLOBAL_STATIC()时导出的前端API。关于何时使用它以及它的要求的讨论，请参见该宏的文档。

通常情况下，你不会直接使用这个类，而是使用Q_GLOBAL_STATIC()或Q_GLOBAL_STATIC_WITH_ARGS()宏，如下所示:

Q_GLOBAL_STATIC(MyType, staticType)

上面的例子创建了一个名为 “staticType “的QGlobalStatic类型的对象。在上述声明之后，staticType对象可以像指针一样被使用，保证被精确初始化一次。除了作为指针使用外，该对象还提供了两个方法来确定全局的当前状态：existence() 和 isDestroyed()。

See also Q_GLOBAL_STATIC() and Q_GLOBAL_STATIC_WITH_ARGS().

成员类型文档#

QGlobalStatic::Type#

这种类型相当于传递给Q_GLOBAL_STATIC()或Q_GLOBAL_STATIC_WITH_ARGS()宏的Type参数。它被用于一些函数的返回类型中。

成员函数文档#

bool QGlobalStatic::exists() const#

如果全局静态对象已经完成初始化（也就是说，如果该类型的构造函数已经返回），该函数返回true。具体来说，如果初始化仍在进行中，该函数会返回false。

一旦这个函数返回了一次true，它就不会再返回false，直到程序被重新启动或者包含全局静态的插件或库被卸载和重新加载。

这个函数在程序执行的任何时候调用都是安全的：它不会失败，也不会导致死锁。此外，如果内容还没有被创建，它不会导致内容被创建。

如果可以确定全局静态对象的初始条件，并希望避免可能昂贵的构造操作，那么这个函数就很有用。

例如，在下面的代码示例中，这个函数被用来短路创建名为globalState的全局静态，并返回一个默认值：

Q_GLOBAL_STATIC(MyType, globalState)
QString someState()
{
    if (globalState.exists())
        return globalState->someState;
    return QString();
}

线程安全通知：这个函数是线程安全的，因为它可以在任何时候从任何线程中被调用，并总是返回一个有效的回复。但是由于构造的非原子性，这个函数可能在构造完成后的短时间内返回错误。

内存排序通知：此函数不施加任何内存排序保证。这反而是由返回内容的指针或引用的访问函数提供的。如果你绕过访问器函数并试图访问由构造函数设置的一些全局状态，请确保使用QAtomicInt或QAtomicPointer提供的正确内存排序语义。

See also isDestroyed().

bool QGlobalStatic::isDestroyed() const#

如果全局静态对象已经完成销毁（也就是说，如果该类型的销毁器已经返回），该函数返回true。具体来说，如果销毁工作仍在进行中，该函数会返回false。

一旦这个函数返回了一次true，它就不会再返回false，直到程序被重新启动或者包含全局静态的插件或库被卸载和重新加载。

这个函数在程序执行的任何时候调用都是安全的：它不会失败，也不会导致死锁。此外，如果内容还没有被创建，它不会导致内容被创建。

这个函数在程序关闭时可能被执行的代码中很有用，可以确定内容是否仍然可以被访问。

See also exists().

*QGlobalStatic::operator QGlobalStatic::Type *()#

这个函数返回这个全局静态内容的地址。如果内容还没有被创建，它们将被这个函数以线程安全的方式创建。如果内容已经被销毁，这个函数将返回一个空指针。

例如，这个函数可以用来将指向全局静态内容的指针存储在一个局部变量中，从而避免对该函数的多次调用。Q_GLOBAL_STATIC()的实现已经相当高效了，但在性能关键的部分，帮助编译器一点可能是有用的。比如说。

Q_GLOBAL_STATIC(MyType, globalState)
QString someState()
{
    MyType *state = globalState;
    if (!state) {
        // we're in a post-destruction state
        return QString();
    }
    if (state->condition)
        return state->value1;
    else
        return state->value2;
}

See also operator->() and operator*().

&QGlobalStatic::operator*()#

这个函数返回对这个全局静态内容的一个引用。如果内容还没有被创建，它们将被这个函数以线程安全的方式创建。

这个函数不检查内容是否已经被销毁。如果这个函数在对象被销毁后被调用，它将返回一个无效的引用，不得使用。

*QGlobalStatic::operator->()#

这个函数返回这个全局静态内容的地址。如果内容还没有被创建，它们将被这个函数以线程安全的方式创建。

这个函数不检查内容是否已经被销毁，也不会返回null。如果这个函数在对象被销毁后被调用，它将返回一个不应该被解除引用的悬空指针。

宏文档#

[since 5.1]Q_GLOBAL_STATIC(Type, VariableName)#

创建一个类型为QGlobalStatic的全局静态对象，其名称为VariableName，其行为是指向Type的指针。由Q_GLOBAL_STATIC创建的对象在第一次使用时就会初始化，这意味着它不会增加应用程序或库的加载时间。此外，该对象在所有平台上都是以线程安全的方式初始化的。

这个宏的典型用法如下，在全局范围内（也就是在任何函数体之外）：

Q_GLOBAL_STATIC(MyType, staticType)

这个宏的目的是取代非POD（Plain Old Data，或者用C++11的术语说，不是由琐碎的类型构成的）的全局静态对象，因此得名。例如，下面的C++代码创建了一个全局静态：

static MyType staticType;

与Q_GLOBAL_STATIC相比，假设MyType是一个具有构造函数、析构函数或其他非POD的类或结构，上述方法有以下缺点。

  • 它需要在加载时初始化MyType（也就是说，在加载库或应用程序时调用MyType的默认构造函数）。
  • 即使该类型从未被使用，也会被初始化。
  • 在不同的翻译单元中，初始化和销毁的顺序是不确定的，导致可能在初始化前或销毁后使用。
  • 如果它在一个函数中被发现（也就是说，不是全局的），它将在第一次使用时被初始化，但目前许多编译器（截至2013年）并不保证初始化是线程安全的。

Q_GLOBAL_STATIC宏解决了上述所有问题，它保证了首次使用时的线程安全初始化，并允许用户查询该类型是否已经被销毁，以避免销毁后的使用问题（参见QGlobalStatic::isDestroyed()）。

构造函数和破坏函数#

对于Q_GLOBAL_STATIC，类型Type必须是可公开默认构造和可公开破坏的。对于Q_GLOBAL_STATIC_WITH_ARGS()，必须有一个与所传参数相匹配的公共构造函数。

不可能将Q_GLOBAL_STATIC用于具有受保护或私有默认构造函数或析构函数的类型（对于Q_GLOBAL_STATIC_WITH_ARGS()，必须有一个与参数匹配的受保护或私有构造函数）。如果相关类型的成员是受保护的，那么可以通过从该类型派生并创建公共的构造函数和析构函数来解决这个问题。如果该类型的成员是私有的，那么在派生之前就必须有一个友好声明。

例如，以下内容足以在先前定义的 “MyOtherType “的基础上创建 “MyType”，该类型有一个受保护的默认构造函数和/或受保护的析构函数（或者将它们作为私有的，但将 “MyType “定义为友方）。

class MyType : public MyOtherType { };
Q_GLOBAL_STATIC(MyType, staticType)

MyType不需要主体，因为析构器是一个隐含的成员，如果没有定义其他的构造函数，那么默认的构造函数也是如此。然而，为了与Q_GLOBAL_STATIC_WITH_ARGS()一起使用，一个合适的构造函数体是必要的：

class MyType : public MyOtherType
{
public:
    MyType(int i) : MyOtherType(i) {}
};
Q_GLOBAL_STATIC_WITH_ARGS(MyType, staticType, (42))

另外，如果编译器支持C++11的继承构造函数，我们可以写：

class MyType : public MyOtherType
{
public:
    using MyOtherType::MyOtherType;
};
Q_GLOBAL_STATIC_WITH_ARGS(MyType, staticType, (42))

放置#

Q_GLOBAL_STATIC宏在全局范围内创建一个必须是静态的类型。不可能将Q_GLOBAL_STATIC宏放在一个函数中（这样做会导致编译错误）。

更重要的是，这个宏应该放在源文件中，而不是放在头文件中。由于产生的对象具有静态链接性，如果该宏被放在头文件中，并被多个源文件所包含，那么该对象将被多次定义，不会导致链接错误。相反，每个翻译单元将引用一个不同的对象，这可能导致微妙的、难以追踪的错误。

不推荐的用途#

请注意，该宏不建议用于POD类型或具有C++11 constexpr构造函数的类型（trivially constructible and destructible）。对于这些类型，我们仍然建议使用常规的静态，无论是全局的还是函数局部的。

这个宏可以工作，但它会增加不必要的开销。

构建时的重入性、线程安全、死锁和异常安全问题#

Q_GLOBAL_STATIC宏创建了一个对象，该对象在第一次使用时以线程安全的方式初始化自己：如果多个线程试图同时初始化该对象，那么只有一个线程会继续初始化，而所有其他线程则等待完成。

如果初始化过程抛出一个异常，初始化将被视为未完成，并将在控制权达到对象的任何使用时再次尝试。如果有任何线程在等待初始化，其中一个线程将被唤醒以尝试初始化。

该宏不保证来自同一线程的重入性。如果全局静态对象被直接或间接地从构造函数内部访问，肯定会发生死锁。

此外，如果两个Q_GLOBAL_STATIC对象在两个不同的线程上被初始化，并且每个线程的初始化序列都访问了另一个，可能会发生死锁。出于这个原因，我们建议保持全局静态构造器的简单性，如果做不到这一点，就确保在构造过程中没有对全局静态的交叉依赖。

销毁#

如果对象在程序的生命周期内从未被使用，除了QGlobalStatic::exists()和QGlobalStatic::isDestroyed()函数外，类型Type的内容将不会被创建，也不会有任何退出时间操作。

如果该对象被创建，它将在退出时被销毁，类似于C语言的atexit函数。事实上，在大多数系统中，如果库或插件在退出前从内存中卸载，也会调用析构器。

由于销毁是在程序退出时发生的，所以没有提供线程安全。这包括插件或库卸载的情况。此外，由于析构器不应该抛出异常，所以也没有提供异常安全。

然而，重入是允许的：在销毁过程中，可以访问全局静态对象，返回的指针将与销毁开始前的指针相同。在销毁完成后，访问全局静态对象是不允许的，除非在QGlobalStatic API中指出。

这个函数是在Qt 5.1中引入的。

See also Q_GLOBAL_STATIC_WITH_ARGS() and QGlobalStatic.

[since 5.1]Q_GLOBAL_STATIC_WITH_ARGS(Type, VariableName, Arguments)#

创建一个类型为QGlobalStatic的全局和静态对象，名称为VariableName，由参数Arguments初始化，表现为指向Type的指针。由Q_GLOBAL_STATIC_WITH_ARGS创建的对象在第一次使用时就会初始化，这意味着它不会增加应用程序或库的加载时间。此外，该对象在所有平台上都是以线程安全的方式初始化的。

这个宏的典型用法如下，在全局范围内（也就是在任何函数体之外）：

Q_GLOBAL_STATIC_WITH_ARGS(MyType, staticType, (42, "Hello", "World"))

Arguments宏的参数必须总是包括小括号，或者，如果C++11的统一初始化被允许，则包括大括号。

除了用提供的参数对内容进行实际初始化外，这个宏的行为与Q_GLOBAL_STATIC()相同。请参阅该宏的文档以了解更多信息。

这个函数是在Qt 5.1中引入的。

See also Q_GLOBAL_STATIC() and QGlobalStatic.
