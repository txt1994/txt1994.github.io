详细描述#

颜色通常以 RGB（红色、绿色和蓝色）分量来指定，但也可以根据 HSV（色调、饱和度和值）和 CMYK（青色、品红色、黄色和黑色）来指定颜色 成分。 此外，可以使用颜色名称指定颜色。 颜色名称可以是任何 SVG 1.0 颜色名称。

RGB HSV CMYK

QColor 构造函数根据 RGB 值创建颜色。要基于 HSV 或 CMYK 值创建 QColor，请分别使用 toHsv () 和 toCmyk () 函数。这些函数使用所需格式返回颜色的副本。此外，静态 fromRgb ()、fromHsv () 和 fromCmyk () 函数根据指定值创建颜色。或者，可以使用 convertTo () 函数（以所需格式返回颜色的副本）或 setRgb ( ), setHsv () 和 setCmyk () 函数改变 this 颜色的格式。 spec () 函数说明颜色是如何指定的。

可以通过将 RGB 字符串（例如“#112233”）或 ARGB 字符串（例如“#ff112233”）或颜色名称（例如“blue”）传递给 [setNamedColor](# setNamedColor)() 函数。颜色名称取自 SVG 1.0 颜色名称。 name () 函数以“#RRGGBB”格式返回颜色名称。也可以使用 setRgb ()、setHsv () 和 setCmyk () 设置颜色。要获得更浅或更深的颜色，请使用 lighter () 和 darker () 函数区分。

isValid () 函数指示 QColor 是否完全合法。例如，RGB 值超出范围的 RGB 颜色是非法的。出于性能原因，QColor 大多忽略非法颜色，因此，使用无效颜色的结果是未定义的。

颜色分量可以单独检索，例如使用 red ()、hue () 和 cyan ()。也可以使用 getRgb ()、getHsv () 和 getCmyk () 函数一次性检索颜色分量的值。使用 RGB 颜色模型，还可以使用 rgb () 访问颜色分量。

有几个相关的非成员： QRgb 是代表 RGB 值三元组 (r, g, b) 的无符号整数的 typdef。请注意，它还可以保存 alpha 通道的值（有关更多信息，请参阅融合 Alpha 的绘画部分）。 qRed ()、qBlue () 和 qGreen () 函数返回给定 QRgb 值的相应分量，而qRgb () 和 qRgba () 函数根据给定的组件值创建并返回 QRgb 三元组。最后，qAlpha () 函数返回提供的 QRgb 的 alpha 分量，和 qGray () 函数根据给定值。

QColor 独立于平台和设备。 QColormap 类把颜色映射到硬件。

有关一般绘画的更多信息，请参阅 描绘系统 文档编制。

整数 vs 浮点数的精度#

QColor 支持浮点精度并提供所有颜色分量函数的浮点版本，例如 getRgbF()、hueF() 和 fromCmykF()。 请注意，由于组件是使用 16 位整数存储的，因此使用 setRgbF () 设置的值与 getRgbF ( ) 函数由于四舍五入。

虽然基于整数的函数采用 0-255 范围内的值（hue () 必须具有 0-359 范围内的值），但浮点函数接受 0.0 - 1.0 范围内的值。

融合 Alpha 的绘制#

QColor 还支持 alpha 混合轮廓和填充。 颜色的 alpha 通道指定透明效果，0 表示完全透明的颜色，而 255 表示完全不透明的颜色。 例如：

// Specify semi-transparent red
painter.setBrush(QColor(255, 0, 0, 127));
painter.drawRect(0, 0, width() / 2, height());
// Specify semi-transparent blue
painter.setBrush(QColor(0, 0, 255, 127));
painter.drawRect(0, 0, width(), height() / 2);

预定义的颜色#

在 QColorConstants 命名空间中有 20 个预定义的 QColor 对象，包括黑色、白色、原色和次要颜色、这些颜色的较暗版本以及三种灰度。 此外，QColorConstants::Svg命名空间为标准SVG颜色关键字名称定义了QColor对象。

QColorConstants::Color0 、QColorConstants::Color1 和 QColorConstants::Transparent 颜色用于特殊目的。

QColorConstants::Color0 （零像素值）和 QColorConstants::Color1 （非零像素值）是用于在 QBitmaps 中绘制的特殊颜色。 使用 QColorConstants::Color0 绘制将位图位设置为 0（透明；即背景），使用 c{QColorConstants::Color1} 绘制将位设置为 1（不透明；即前景）。

QColorConstants::Transparent用于表示透明像素。 使用此值进行绘制时，将使用适合所使用的底层像素格式的像素值。

由于历史原因，在 Qt::GlobalColor 枚举中也提供了 20 种预定义颜色。

最后，QColor 识别各种颜色名称（作为字符串）； QColor 知道的静态 colorNames () 函数返回 QStringList 颜色名称。

扩展 RGB 颜色模型#

扩展的 RGB 颜色模型，也称为 scRGB 颜色空间，与 RGB 颜色模型相同，只是它允许值小于 0.0 和大于 1.0。 这使得可以表示在 RGB 颜色空间范围之外的颜色，但仍对 RGB 颜色空间内的颜色使用相同的值。

HSV 颜色模型#

RGB 模型是面向硬件的。 它的表现与大多数显示器显示的相近。 相比之下，HSV 以更适合人类对颜色的感知的方式表示颜色。 例如，“强于”、“暗于”和“相反”的关系在 HSV 中很容易表达，但在 RGB 中更难表达。

HSV 与 RGB 一样，具有三个组成部分：

  • H，对于色调，如果颜色是彩色（不是灰色），则在 0 到 359 的范围内，如果是灰色，则无意义。 它代表大多数人熟悉的色轮上的度数。 红色为 0（度），绿色为 120，蓝色为 240。

  • S，饱和度，在0到255的范围内，越大，颜色越强。 浅灰色的饱和度接近 0； 非常强烈的颜色饱和度接近 255。

  • V，代表值，在 0 到 255 的范围内，代表颜色的亮度或亮度。 0 为黑色； 255 尽可能远离黑色。

下面是一些例子：纯红色是H=0，S=255，V=255； 深红色，略微向洋红色移动，可能是 H=350（相当于 -10），S=255，V=180； 浅灰色的浅红色的 H 约为 0（例如 350-359 或 0-10），S 约为 50-100，S=255。

Qt 为非彩色返回 -1 的色调值。 如果你传递一个太大的色调值，Qt 会强制它进入范围。 色调 360 或 720 被视为 0； 色调 540 被视为 180。

除了标准的 HSV 模型之外，Qt 还提供了一个 alpha 通道来显示 alpha-blended drawing。

HSL 颜色模型#

HSL 与 HSV 类似，但是 HSL 指定的不是 Value 参数，而是一个 Lightness 参数，该参数映射到颜色的亮度略有不同。

同样，HSL 饱和度值通常与相同颜色的 HSV 饱和度值不同。 hslSaturation () 提供颜色的 HSL 饱和度值，而 saturation () 和 hsvSaturation () 提供 HSV 饱和度值。

色调值在 HSL 和 HSV 中定义为相同。

CMYK 颜色模型#

虽然 RGB 和 HSV 颜色模型用于在计算机显示器上显示，但 CMYK 模型用于印刷机和一些硬拷贝设备的四色印刷过程。

CMYK 有四种成分，均在 0-255 范围内：青色 (C)、品红色 (M)、黄色 (Y) 和黑色 (K)。青色、品红色和黄色称为减色； CMYK 颜色模型从白色表面开始创建颜色，然后通过应用适当的组件减去颜色。虽然结合青色、品红色和黄色会产生黑色，但减去一个或多个会产生任何其他颜色。当以不同的百分比组合时，这三种颜色可以创建整个颜色光谱。

混合 100% 的青色、品红色和黄色确实会产生黑色，但结果并不令人满意，因为它会浪费墨水、增加干燥时间并在打印时产生浑浊的颜色。出于这个原因，在专业印刷中添加了黑色以提供纯黑色调；因此称为“四色工艺”。

除了标准的 CMYK 模型，Qt 还提供了一个 alpha 通道来显示 alpha-blended drawing。

另请参阅 QPalette , QBrush ，和 QColorConstants .

相关非成员#

QRgb#

格式为 #AARRGGBB 的 ARGB 四元组，相当于无符号整数。

该类型还保存 alpha 通道的值。 默认的 alpha 通道是 ff，即不透明。 欲了解更多信息，请参阅融合Alpha的绘画章节。

以下是如何创建 QRgb 值的一些示例：

const QRgb rgb1 = 0x88112233;
const QRgb rgb2 = QColor("red").rgb();
const QRgb rgb3 = qRgb(qRed(rgb1), qGreen(rgb2), qBlue(rgb2));
const QRgb rgb4 = qRgba(qRed(rgb1), qGreen(rgb2), qBlue(rgb2), qAlpha(rgb1));

另请参阅 qRgb (), qRgba (), QColor::rgb ()，和 QColor::rgba ().
