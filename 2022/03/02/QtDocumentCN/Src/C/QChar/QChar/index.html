详细描述#

在 Qt 中，Unicode 字符是 16 位实体，没有任何标记或结构。 这个类代表这样一个实体。 它重量轻，因此可以在任何地方使用。 大多数编译器将其视为 unsigned short 。

QChar 提供了完整的测试/分类功能，与其他格式相互转换，从组合 Unicode 转换为分解 Unicode，并在您要求时尝试比较和大小写转换。

分类函数包括标准 C++ 头文件 （以前称为 <ctype.h>）中的函数，但对所有 Unicode 字符进行操作，而不仅仅是 ASCII 范围。如果字符是某种类型的字符，它们都返回 true；否则他们返回假。这些分类函数是 isNull ()（如果字符是 ‘\0’，则返回 true），isPrint()（如果字符是任何类型的可打印字符，则返回 true，包括whitespace), isPunct () (任何类型的标点符号), isMark () (Unicode 标记), isLetter () (一个字母), isNumber () (任何类型的数字字符，不只是 0-9)，isLetterOrNumber ()，和 isDigit () （十进制数字）。所有这些都是 category () 的包装器，它们返回每个字符的 Unicode 定义的类别。其中一些还计算派生属性（例如 isSpace () 返回 true ，如果字符属于 Separator_* 类别或来自 Other_Control category 的异常代码点）。

QChar 还提供了direction ()，表示这个字符的“自然”书写方向。 joiningType () 函数指示角色如何与其邻居连接（主要用于阿拉伯语或叙利亚语），最后是 hasMirrored ()，它指示角色在连接时是否需要镜像 以“不自然”的书写方向打印。

使用 decomposition () 可以将组合的 Unicode 字符（如 ring ）转换为分解的 Unicode（“a”后跟“ring above”）。

在 Unicode 中，比较不一定是可能的，而且大小写转换充其量是非常困难的。 Unicode，覆盖了“整个”世界，也包括了世界上大部分的大小写和排序问题。 operator==() 和朋友会纯粹根据字符的数字 Unicode 值（码位）进行比较，而 toUpper () 和 toLower () 会在该字符具有明确定义的大写/小写等效项。对于依赖于区域设置的比较，请使用 QString::localeAwareCompare ()。

转换函数包括 unicode ()（转为标量）、toLatin1 ()（转为标量，但将所有非Latin-1 字符转为0）、[row ](# row)() (给出 Unicode 行), cell () (给出 Unicode 单元), digitValue () (给出任意数字字符的整数值),和许多构造函数。

QChar 提供了构造函数和强制转换运算符，可以轻松地与传统的 8 位 char s 相互转换。如果您定义了 QT_NO_CAST_FROM_ASCII 和 QT_NO_CAST_TO_ASCII ，如 QString 文档中所述，您将需要显式调用 fromLatin1 ()，或使用 QLatin1Char ，要从 8 位 char 构造 QChar，您需要调用 toLatin1 () 来取回 8 位值。

从 Qt 6.0 开始，大多数 QChar 构造函数都是 explicit。 这样做是为了避免意外混合整数类型和字符串时的危险错误。 您可以通过定义宏“QT_IMPLICIT_QCHAR_CONSTRUCTION”来选择退出（并使这些构造函数隐式）。

更多信息见 “About the Unicode Character Database” .

另请参阅 Unicode , QString ，和 QLatin1Char .

[static, since 6.0] R QChar:: fromUcs4 ( char32_t c )#

返回一个匿名结构，其中

  • 包含一个char16_t chars[2] array。
  • 可以隐含地转换为一个QStringView，和
  • 用C++11范围内的for循环进行迭代。

如果c需要代理，chars[0]包含高位代理，chars[1]包含低位代理，并且QStringView大小为2。否则，chars[0] 包含c，chars[1] 是null，和QStringView有大小1。

这样可以方便地使用结果：

QString s;
s += QChar::fromUcs4(ch);
							
for (char16_t c16 : QChar::fromUcs4(ch))
    use(c16);

该函数在 Qt 6.0 引入。

另请参阅 fromUcs2 () 和 requiresSurrogates ().

宏文档编制#

[since 6.0] QT_IMPLICIT_QCHAR_CONSTRUCTION#

定义这个宏使得某些QChar构造函数成为隐式而非显式。这样做是为了执行安全转换：

QString str = getString();
if (str == 123) {
    // Oops, meant str == "123". By default does not compile,
    // *unless* this macro is defined, in which case, it's interpreted
    // as `if (str == QChar(123))`, that is, `if (str == '{')`.
    // Likely, not what we meant.
}

提供这个宏是为了保持现有代码的正常运行；建议使用显式转换和/或QLatin1Char来代替。

比如说：

QChar c1 =  'x'; // OK，除非定义了QT_NO_CAST_FROM_ASCII。
QChar c2 = u'x'; // always OK, recommended
QChar c3 = QLatin1Char('x'); // always OK, recommended
// from int to 1 UTF-16 code unit: must guarantee that the input is <= 0xFFFF
QChar c4 = 120;        // compile error, unless QT_IMPLICIT_QCHAR_CONSTRUCTION is defined
QChar c5(120);         // OK (direct initialization)
auto  c6 = QChar(120); // ditto
// from int/char32_t to 1/2 UTF-16 code units:
// 𝄞 'MUSICAL SYMBOL G CLEF' (U+1D11E)
auto c7 = QChar(0x1D11E);           // compiles, but undefined behavior at runtime
auto c8 = QChar::fromUcs4(0x1D11E);       // always OK
auto c9 = QChar::fromUcs4(U'\U0001D11E'); // always OK
// => use c8/c9 as QStringView objects

该函数在 Qt 6.0 引入。

另请参阅 QLatin1Char , QChar::fromUcs4 ，和 QT_NO_CAST_FROM_ASCII .
