详细描述#

关闭事件被发送给用户想要关闭的部件，通常是通过从窗口菜单中选择 “关闭”，或者点击***标题栏按钮。当你调用QWidget::close()以编程方式关闭一个部件时，它们也会被发送。

关闭事件包含一个标志，表明接收者是否想让小部件被关闭。當一個 widget 接受 close 事件時，它會被隱藏起來（如果它是用 Qt::WA_DeleteOnClose 的旗號來創建的話，則會被銷毀）。如果它拒绝接受关闭事件，则不会发生任何事情。(在X11下，窗口管理器有可能强行关闭窗口；但在写这篇文章的时候，我们还不知道有任何窗口管理器会这样做。)

事件处理程序 QWidget::closeEvent () 接收关闭事件。这个事件处理程序的默认实现是接受关闭事件。如果你不希望你的小部件被隐藏，或者想要一些特殊的处理方式，你应该重新实现这个事件处理程序，并且ignore()该事件。

应用实例中的closeEvent()显示了一个关闭事件处理程序，询问是否在关闭前保存文档。

如果你想让部件在关闭时被删除，请用Qt::WA_DeleteOnClose标记来创建它。这对于多窗口应用程序中的独立顶层窗口非常有用。

当QObject被删除时，它们会发出destroy()信号。

如果最后一个顶层窗口被关闭，QGuiApplication::lastWindowClosed () 信号被发射。

isAccepted () 函数返回 true 如果事件的接收者同意关闭窗口部件；调用 accept () 同意关闭窗口部件，如果此事件的接收者不希望窗口部件被关闭，则调用 ignore ()。
