详细描述#

正则表达式，或regexps，是处理字符串和文本的一个非常强大的工具。这在许多情况下都很有用，例如：

审定 一个regexp可以测试一个子串是否符合某些标准，例如，是一个整数或不包含空格。
搜索 regexp提供了比简单子串匹配更强大的模式匹配，例如，匹配mail、letter或correspondence中的一个词，但不匹配email、mailman、mailer、letterbox等词。
搜索和替换 一个重合表达式可以用不同的子串替换所有出现的子串，例如，用*&替换所有出现的&，除非&后面已经有一个amp;*。
字符串拆分 一个regexp可以用来识别一个字符串应该被分割的地方，例如分割以tab为分隔符的字符串。

本文绝不是关于使用正则表达式进行模式匹配的完整参考资料，以下部分需要读者具备一些关于Perl类正则表达式及其模式语法的基本知识。

关于正则表达式的好的参考资料包括。

  • Mastering Regular Expressions（第三版），Jeffrey E. F. Friedl著，ISBN 0-596-52812-4。
  • pcrepattern(3) man page，描述了 PCRE（与 Perl 兼容的正则表达式的参考实现）支持的模式语法。
  • Perl的正则表达式文档 和 Perl的正则表达式教程。

简介#

QRegularExpression实现了与Perl兼容的正则表达式。它完全支持 Unicode。关于 QRegularExpression 所支持的正则表达式语法的概述，请参考前面提到的 pcrepattern(3) man page。一个正则表达式由两部分组成：一个模式字符串和一组改变模式字符串含义的**模式选项。

你可以通过向QRegularExpression构造函数传递一个字符串来设置模式字符串。

QRegularExpression re("a pattern");

这将模式字符串设置为`一个模式’。你也可以使用setPattern()函数在一个现有的QRegularExpression对象上设置一个模式:

QRegularExpression re;
re.setPattern("another pattern");

注意，由于C++的字面字符串规则，你必须用另一个反斜线来转义模式字符串内的所有反斜线。

// matches two digits followed by a space and a word
QRegularExpression re("\\d\\d \\w+");

// matches a backslash
QRegularExpression re2("\\\\");

或者，你可以使用原始字符串字面，在这种情况下，你不需要在模式中转义反斜线，`R”(…) “之间的所有字符都被视为原始字符。正如你在下面的例子中看到的，这简化了模式的编写:

// matches two digits followed by a space and a word
QRegularExpression re(R"(\d\d \w+)");

pattern()函数返回当前为一个QRegularExpression对象设置的模式：

QRegularExpression re("a third pattern");
QString pattern = re.pattern(); // pattern == "a third pattern"

模式选择#

模式字符串的含义可以通过设置一个或多个模式选项来修改。例如，可以通过设置QRegularExpression::CaseInsensitiveOption来设置模式不敏感地匹配大小写。

你可以通过将这些选项传递给QRegularExpression构造函数来设置这些选项，如：

// matches "Qt rocks", but also "QT rocks", "QT ROCKS", "qT rOcKs", etc.
QRegularExpression re("Qt rocks", QRegularExpression::CaseInsensitiveOption);

或者，你可以在一个现有的QRegularExpressionObject上使用setPatternOptions()函数:

QRegularExpression re("^\\d+$");
re.setPatternOptions(QRegularExpression::MultilineOption);
// re matches any line in the subject string that contains only digits (but at least one)

通过使用patternOptions()函数，可以获得当前设置在QRegularExpression对象上的模式选项:

QRegularExpression re = QRegularExpression("^two.*words$", QRegularExpression::MultilineOption
                                                           | QRegularExpression::DotMatchesEverythingOption);

QRegularExpression::PatternOptions options = re.patternOptions();
// options == QRegularExpression::MultilineOption | QRegularExpression::DotMatchesEverythingOption

请参考QRegularExpression::PatternOption枚举文档，以了解每个模式选项的更多信息。

匹配类型和匹配选项#

match()和globalMatch()函数的最后两个参数设置匹配类型和匹配选项。匹配类型是QRegularExpression::MatchType枚举的值；通过使用NormalMatch匹配类型（默认）来选择 “传统 “匹配算法。也可以启用正则表达式对主题字符串的部分匹配：详见partial matching 部分。

匹配选项是一组由一个或多个 QRegularExpression::MatchOption 值组成的。它们改变了正则表达式与主题字符串进行特定匹配的方式。请参考 QRegularExpression::MatchOption 枚举文档以了解更多细节。

正常匹配#

为了进行匹配，你可以简单地调用match()函数，传递一个要匹配的字符串。我们把这个字符串称为主题字符串。match()函数的结果是一个QRegularExpressionMatch对象，可用于检查匹配的结果。比如说：

// match two digits followed by a space and a word
QRegularExpression re("\\d\\d \\w+");
QRegularExpressionMatch match = re.match("abc123 def");
bool hasMatch = match.hasMatch(); // true

如果匹配成功，（隐含的）捕获组号0可以用来检索整个模式所匹配的子串（也请参见关于提取捕获的子串的部分）:

QRegularExpression re("\\d\\d \\w+");
QRegularExpressionMatch match = re.match("abc123 def");
if (match.hasMatch()) {
    QString matched = match.captured(0); // matched == "23 def"
    // ...
}

也可以通过将偏移量作为match()函数的参数，在主题字符串内的任意偏移量处开始匹配。在下面的例子中，"12 abc"没有被匹配，因为匹配是从偏移量1开始的:

QRegularExpression re("\\d\\d \\w+");
QRegularExpressionMatch match = re.match("12 abc 45 def", 1);
if (match.hasMatch()) {
    QString matched = match.captured(0); // matched == "45 def"
    // ...
}

提取捕获的子字符串#

QRegularExpressionMatch对象还包含模式字符串中被捕获组捕获的子字符串的信息。captured()函数将返回由第n个捕获组捕获的字符串:

QRegularExpression re("^(\\d\\d)/(\\d\\d)/(\\d\\d\\d\\d)$");
QRegularExpressionMatch match = re.match("08/12/1985");
if (match.hasMatch()) {
    QString day = match.captured(1); // day == "08"
    QString month = match.captured(2); // month == "12"
    QString year = match.captured(3); // year == "1985"
    // ...
}

模式中的捕获组从1开始编号，隐含的捕获组0被用来捕获与整个模式相匹配的子串。

通过使用captureStart()和capturedEnd()函数，也可以检索每个捕获的子串的起点和终点（在主题字符串内）:

QRegularExpression re("abc(\\d+)def");
QRegularExpressionMatch match = re.match("XYZabc123defXYZ");
if (match.hasMatch()) {
    int startOffset = match.capturedStart(1); // startOffset == 6
    int endOffset = match.capturedEnd(1); // endOffset == 9
    // ...
}

所有这些函数都有一个重载，以QString为参数，以提取named捕获的子字符串。比如说:

QRegularExpression re("^(?<date>\\d\\d)/(?<month>\\d\\d)/(?<year>\\d\\d\\d\\d)$");
QRegularExpressionMatch match = re.match("08/12/1985");
if (match.hasMatch()) {
    QString date = match.captured("date"); // date == "08"
    QString month = match.captured("month"); // month == "12"
    QString year = match.captured("year"); // year == 1985
}

全范围内的匹配#

全局匹配*对于在一个主题字符串中找到一个给定的正则表达式的所有出现是非常有用的。假设我们想从一个给定的字符串中提取所有的词，其中一个词是与模式\w+相匹配的子串。

QRegularExpression::globalMatch返回一个QRegularExpressionMatchIterator，它是一个类似Java的前向迭代器，可以用来迭代结果。比如说：

QRegularExpression re("(\\w+)");
QRegularExpressionMatchIterator i = re.globalMatch("the quick fox");

因为它是一个类似于Java的迭代器，QRegularExpressionMatchIterator将紧紧指向第一个结果之前。每个结果都会以QRegularExpressionMatch对象的形式返回。如果至少还有一个结果，hasNext()函数将返回true，而next()将返回下一个结果并推进迭代器。继续前面的例子：

QStringList words;
while (i.hasNext()) {
    QRegularExpressionMatch match = i.next();
    QString word = match.captured(1);
    words << word;
}
// words contains "the", "quick", "fox"

你也可以使用peekNext()来获得下一个结果，而不用推进迭代器。

也可以简单地在基于范围的for循环中使用QRegularExpression::globalMatch的结果，例如像这样:

// using a raw string literal, R"(raw_characters)", to be able to use "\w"
// without having to escape the backslash as "\\w"
QRegularExpression re(R"(\w+)");
QString subject("the quick fox");
for (const QRegularExpressionMatch &match : re.globalMatch(subject)) {
    // ...
}

可以向globalMatch()函数传递一个起始偏移量和一个或多个匹配选项，就像用match()进行正常匹配一样。

局部匹配#

当到达主题字符串的末尾，但需要更多的字符来成功完成匹配时，就会得到*部分匹配。请注意，部分匹配通常比正常匹配的效率要低得多，因为不能采用许多优化的匹配算法。

在调用QRegularExpression::match或QRegularExpression::globalMatch时，必须通过指定PartialPreferCompleteMatch或[PartialPreferFirstMatch]（#MatchType-enum）的匹配类型明确请求部分匹配。如果发现部分匹配，那么对match()返回的QRegularExpressionMatch对象调用hasMatch函数将返回 “false”，但hasPartialMatch()将返回 “true”。

当发现部分匹配时，不会返回捕获的子串，与整个匹配相对应的（隐式）捕获组0会捕获主题字符串的部分匹配子串。

请注意，如果找到了部分匹配，要求部分匹配仍然可以导致完全匹配；在这种情况下，hasMatch()将返回true，而hasPartialMatch()则返回false。一个QRegularExpressionMatch同时报告部分匹配和完全匹配的情况从未发生。

部分匹配主要在两种情况下有用：实时验证用户输入和增量/多段式匹配。

验证用户输入#

假设我们希望用户以特定的格式输入一个日期，例如 “MMM dd, yyyy”。我们可以用一个模式来检查输入的有效性，比如：

^(Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec) \d\d?, \d\d\d\d$

(这个模式并不捕捉无效的日子，但为了例子的目的，我们还是保留它）。

我们希望在用户输入时*用这个正则表达式来验证输入，这样我们就可以在用户输入时立即报告错误（例如，用户打错了键）。为了做到这一点，我们必须区分三种情况。

  • 输入不可能与正则表达式匹配。
  • 输入与正则表达式匹配。
  • 输入的内容现在与正则表达式不匹配，但如果有更多的字符被添加到其中，就会匹配。

请注意，这三种情况正好代表了 QValidator 的可能状态（参见 QValidator::State 枚举）。

特别是，在最后一种情况下，我们希望正则表达式引擎报告一个部分匹配：我们成功地将模式与主题字符串匹配，但由于遇到了主题的结尾，匹配无法继续。然而，请注意，匹配算法应该继续并尝试所有的可能性，如果发现了一个完整的（非部分）匹配，那么应该报告这个匹配，并接受输入字符串为完全有效。

这种行为是由PartialPreferCompleteMatch匹配类型实现的。比如说:

QString pattern("^(Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec) \\d\\d?, \\d\\d\\d\\d$");
QRegularExpression re(pattern);

QString input("Jan 21,");
QRegularExpressionMatch match = re.match(input, 0, QRegularExpression::PartialPreferCompleteMatch);
bool hasMatch = match.hasMatch(); // false
bool hasPartialMatch = match.hasPartialMatch(); // true

如果与主题字符串匹配相同的正则表达式导致完全匹配，则会照常报告:

QString input("Dec 8, 1985");
QRegularExpressionMatch match = re.match(input, 0, QRegularExpression::PartialPreferCompleteMatch);
bool hasMatch = match.hasMatch(); // true
bool hasPartialMatch = match.hasPartialMatch(); // false

另一个具有不同模式的例子，显示了倾向于完全匹配而不是部分匹配的行为:

QRegularExpression re("abc\\w+X|def");
QRegularExpressionMatch match = re.match("abcdef", 0, QRegularExpression::PartialPreferCompleteMatch);
bool hasMatch = match.hasMatch(); // true
bool hasPartialMatch = match.hasPartialMatch(); // false
QString captured = match.captured(0); // captured == "def"

在这种情况下，子模式abc\\w+X与主题字符串部分匹配；但是，子模式def与主题字符串完全匹配，因此报告了一个完整的匹配。

如果在匹配时发现多个部分匹配（但没有完全匹配），那么QRegularExpressionMatch对象将报告发现的第一个匹配。比如说:

QRegularExpression re("abc\\w+X|defY");
QRegularExpressionMatch match = re.match("abcdef", 0, QRegularExpression::PartialPreferCompleteMatch);
bool hasMatch = match.hasMatch(); // false
bool hasPartialMatch = match.hasPartialMatch(); // true
QString captured = match.captured(0); // captured == "abcdef"

递增/多段式匹配#

增量匹配是部分匹配的另一个用例。假设我们想在一个大文本中找到一个正则表达式的出现（也就是与正则表达式相匹配的子串）。为了做到这一点，我们想把大文本分成小块 “喂 “给正则表达式引擎。显而易见的问题是，如果与正则表达式相匹配的子串跨越了两个或更多的小块，会发生什么情况。

在这种情况下，正则表达式引擎应该报告一个部分匹配，这样我们就可以再次添加新的数据进行匹配，（最终）得到一个完整的匹配。这意味着正则表达式引擎可能会假定在主题字符串的末尾*之外还有其他字符。这不是从字面上理解的–引擎永远不会尝试访问主题中最后一个字符之后的任何字符。

QRegularExpression在使用PartialPreferFirstMatch匹配类型时实现了这种行为。这种匹配类型一经发现就会报告部分匹配，而其他的匹配选项不会被尝试（即使它们可能导致完全匹配）。比如说：

QRegularExpression re("abc|ab");
QRegularExpressionMatch match = re.match("ab", 0, QRegularExpression::PartialPreferFirstMatch);
bool hasMatch = match.hasMatch(); // false
bool hasPartialMatch = match.hasPartialMatch(); // true

发生这种情况是因为在匹配交替运算符的第一个分支时，发现了部分匹配，因此匹配停止，没有尝试第二个分支。另一个例子：

QRegularExpression re("abc(def)?");
QRegularExpressionMatch match = re.match("abc", 0, QRegularExpression::PartialPreferFirstMatch);
bool hasMatch = match.hasMatch(); // false
bool hasPartialMatch = match.hasPartialMatch(); // true

这显示了量词看似反直觉的行为：由于?是贪婪的，那么引擎在匹配了"abc"之后，首先尝试继续匹配；但后来匹配到了主题字符串的末端，因此报告了部分匹配。这在下面的例子中更加令人惊讶：

QRegularExpression re("(abc)*");
QRegularExpressionMatch match = re.match("abc", 0, QRegularExpression::PartialPreferFirstMatch);
bool hasMatch = match.hasMatch(); // false
bool hasPartialMatch = match.hasPartialMatch(); // true

如果我们记住引擎期望主题字符串只是我们要寻找的整个文本的一个子串，就很容易理解这种行为（也就是我们之前说的，引擎假定在主题字符串的末尾还有其他字符）。

由于*量词是贪婪的，那么报告一个完整的匹配可能是一个错误，因为在当前主题"abc"之后，可能还有其他"abc"的出现。例如，完整的文本可能是 “abcabcX”，因此要报告的*正确的匹配（在完整的文本中）应该是"abcabc"；通过只与前面的"abc"匹配，我们得到的是部分匹配。

错误处理#

由于模式字符串中的语法错误，QRegularExpression对象有可能是无效的。如果正则表达式有效，isValid()函数将返回真，否则返回假：

QRegularExpression invalidRe("(unmatched|parenthesis");
bool isValid = invalidRe.isValid(); // false

你可以通过调用errorString()函数获得更多关于具体错误的信息；此外，patternErrorOffset()函数将返回模式字符串中的偏移量。

QRegularExpression invalidRe("(unmatched|parenthesis");
if (!invalidRe.isValid()) {
    QString errorString = invalidRe.errorString(); // errorString == "missing )"
    int errorOffset = invalidRe.patternErrorOffset(); // errorOffset == 22
    // ...
}

如果尝试用无效的QRegularExpression进行匹配，那么返回的QRegularExpressionMatch对象也将是无效的（也就是说，其isValid()函数将返回false）。这同样适用于试图进行全局匹配。

不支持Perl兼容的正则表达式特性#

QRegularExpression并不支持与Perl兼容的正则表达式中的所有功能。最值得注意的是，不支持捕获组的重复名称，使用它们会导致未定义的行为。

这可能会在未来的Qt版本中改变。

调试使用QRegularExpression的代码#

QRegularExpression内部使用及时编译器（JIT）来优化匹配算法的执行。JIT大量使用自修改代码，这可能会导致调试工具（如Valgrind）崩溃。如果你想使用QRegularExpression调试程序，你必须启用所有对自修改代码的检查（例如，Valgrind的--smc-check命令行选项）。启用这种检查的缺点是，你的程序运行速度会大大降低。

为了避免这一点，如果你在调试模式下编译Qt，JIT默认是禁用的。我们可以通过设置QT_ENABLE_REGEXP_JIT环境变量为非零或零，来覆盖默认值，并启用或禁用JIT的使用（在调试或发布模式下）。

See also QRegularExpressionMatch and QRegularExpressionMatchIterator.

enum QRegularExpression::MatchType#

MatchType枚举定义了应该针对主题字符串尝试的匹配类型。

Constant Value Description
QRegularExpression::NormalMatch 0 正常匹配完成。
QRegularExpression::PartialPreferCompleteMatch 1 模式字符串与主题字符串进行部分匹配。如果发现部分匹配，则记录下来，并像往常一样尝试其他匹配方式。如果找到一个完整的匹配，那么它将优先于部分匹配；在这种情况下，只有完整的匹配被报告。如果没有找到完整的匹配（而只有部分匹配），那么就报告部分匹配。
QRegularExpression::PartialPreferFirstMatch 2 模式字符串与主题字符串进行部分匹配。如果发现部分匹配，则停止匹配并报告部分匹配结果。在这种情况下，其他的匹配方式（可能会导致完全匹配）不会被尝试。此外，这种匹配类型假定主题字符串只是一个较大文本的子串，并且（在这个文本中）在主题字符串的末尾还有其他字符。这可能会导致令人惊讶的结果；更多细节见部分匹配部分的讨论。
QRegularExpression::NoMatch 3 不做任何匹配。这个值被默认构建的 QRegularExpressionMatch 或 QRegularExpressionMatchIterator 作为匹配类型返回。使用这种匹配类型对用户来说不是很有用，因为从来没有发生过匹配。这个枚举值已经在Qt 5.1中被引入。

enum QRegularExpression::PatternOption flags QRegularExpression::PatternOptions#

PatternOption枚举定义了对模式字符串的解释方式的修改，因此也定义了模式与主题字符串的匹配方式。

Constant Value Description
QRegularExpression::NoPatternOption 0x0000 没有设置模式选项。
QRegularExpression::CaseInsensitiveOption 0x0001 该模式应该以不区分大小写的方式与主题字符串匹配。这个选项对应于Perl正则表达式中的/i修改器。
QRegularExpression::DotMatchesEverythingOption 0x0002 模式字符串中的点元字符（.）允许匹配主题字符串中的任何字符，包括新行（通常，点不匹配新行）。这个选项对应于Perl正则表达式中的/s修饰符。
QRegularExpression::MultilineOption 0x0004 模式字符串中的密语(^)和美元($)元字符允许分别在主题字符串的任何换行之后和之前，以及主题字符串的最开始和最后进行匹配。这个选项对应于Perl正则表达式中的/m修饰符。
QRegularExpression::ExtendedPatternSyntaxOption 0x0008 模式字符串中任何未被转义且在一个字符类之外的空白都会被忽略。此外，一个字符类之外的未转义的锐利字符（**#**）会导致所有后续字符被忽略，直到第一个换行（包括）。这可以用来增加模式字符串的可读性，也可以在正则表达式中加入注释；如果模式字符串是从文件中加载的，或者是由用户编写的，这就特别有用，因为在C++代码中，总是可以使用字符串字面的规则来在模式字符串之外加入注释。这个选项对应于Perl正则表达式中的/x修改器。
QRegularExpression::InvertedGreedinessOption 0x0010 量词的贪婪性被颠倒了。*, +, ?, {m,n}等变成懒惰，而它们的懒惰版本（*?, +?, ?, {m,n}?等）变成贪婪。在Perl正则表达式中，没有与该选项相当的选项。
QRegularExpression::DontCaptureOption 0x0020 非命名的捕获组不捕获子字符串；命名的捕获组仍按原计划工作，以及对应于整个匹配的隐式捕获组编号0。在Perl正则表达式中，没有与该选项相当的选项。
QRegularExpression::UseUnicodePropertiesOption 0x0040 w'、d’等字符类的含义，以及它们的对应物（W'、D’等）的含义，从只匹配ASCII字符变为匹配具有相应Unicode属性的任何字符。例如，d'被改为匹配任何具有Unicode Nd（十进制数字）属性的字符；w’被改为匹配任何具有Unicode L（字母）或N（数字）属性的字符，加上下划线，以此类推。这个选项对应于Perl正则表达式中的/u修改器。

[static, since 5.15] QRegularExpression::escape( str)#

转义str的所有字符，使其在作为正则表达式模式字符串使用时不再具有任何特殊含义，并返回转义后的字符串。例如：

QString escaped = QRegularExpression::escape("a(x) = f(x) + g(x)");
// escaped == "a\\(x\\)\\ \\=\\ f\\(x\\)\\ \\+\\ g\\(x\\)"

这对于从任意字符串建立模式是非常方便的：

QString pattern = "(" + QRegularExpression::escape(name) +
    "|" + QRegularExpression::escape(nickname) + ")";
QRegularExpression re(pattern);

注意：这个函数实现了Perl的quotemeta算法，用反斜杠转义了str中的所有字符，除了[A-Z]、[a-z]和[0-9]范围内的字符，以及下划线（_）字符。与Perl的唯一区别是，str内的字面NUL会用序列"\\0"（反斜杠+'0'）来转义，而不是"\\\0"（反斜杠+NUL）。

这个函数是在Qt 5.15中引入的。

[since 5.1] QRegularExpression::namedCaptureGroups() const#

返回一个captureCount()+1元素的列表，包含模式字符串中命名的捕获组名称。列表的排序是这样的：如果有名字的话，列表中位于i位置的元素是第i个捕获组的名字，如果捕获组没有名字的话，则是空字符串。

例如，给定一个正则表达式

(?<day>\d\d)-(?<month>\d\d)-(?<year>\d\d\d\d) (\w+) (?<name>\w+)

namedCaptureGroups()将返回以下列表。

("", "day", "month", "year", "", "name")

这相当于0号捕获组（对应于整个匹配）没有名字，1号捕获组有名字 “日”，2号捕获组有名字 “月”，等等。

如果正则表达式无效，则返回一个空列表。

这个函数是在Qt 5.1中引入的。

参见 isValid(), QRegularExpressionMatch::captured(), 和 QString::isEmpty() 。

[static, since 5.15] QRegularExpression::wildcardToRegularExpression( pattern, options = DefaultWildcardConversion)#

返回给定glob pattern的正则表达式。这个转换是针对文件路径球化的，这意味着特别是路径分隔符得到了特殊的处理。这意味着它不仅仅是一个从 ““ to “.“的基本转换。

QString wildcard = QRegularExpression::wildcardToRegularExpression("*.jpeg")。
// 将会匹配名称为以下的文件。
// foo.jpeg
// f_o_o.jpeg
// föö.jpeg

默认情况下，返回的正则表达式是完全锚定的。换句话说，不需要对结果再次调用 anchoredPattern() 。要得到一个不被锚定的正则表达式，请传递 UnanchoredWildcardConversion 作为转换options 。

这个实现紧跟glob模式的通配符的定义。

c 除了下面提到的那些，任何字符都代表自己。因此，c匹配字符c。
? 匹配任何单个字符。它与完整的重构公式中的**.**相同。
***** 匹配零或更多的任何字符。它与完整的重构公式中的**.***相同。
[abc] 匹配括号内的一个字符。
[a-c] 匹配括号内给出的一个字符的范围。
[!abc] 匹配括号中没有给出的一个字符。它与完整的重构公式中的**[^abc]**相同。
[!a-c] 匹配一个不在括号中给出的范围内的字符。这与完整的regexp中的**[^a-c]**相同。

注意：反斜杠()字符在这里不是转义字符。为了匹配其中的一个特殊字符，请将其放在方括号中（例如，[?]）。

关于实现的更多信息可以在:

  • 维基百科的Glob文章
  • `man 7 glob’.

这个函数是在Qt 5.15中引入的。

See also escape().
