详细描述#

一个消息框显示一个主要的文本来提醒用户注意某个情况，一个信息文本来进一步解释这个提醒或向用户提问，以及一个可选的详细文本来提供更多的数据，如果用户要求的话。消息框也可以显示一个图标和标准按钮来接受用户的回应。

提供了两个使用QMessageBox的API，即基于属性的API，和静态函数。调用一个静态函数是比较简单的方法，但它不如使用基于属性的API灵活，而且结果的信息量较小。建议使用基于属性的API。

基于属性的API#

要使用基于属性的API，构建一个QMessageBox的实例，设置所需的属性，并调用exec()来显示消息。最简单的配置是只设置消息文本属性。

QMessageBox msgBox;
msgBox.setText("The document has been modified.");
msgBox.exec();

用户必须点击OK按钮来解除消息框。GUI的其他部分会被封锁，直到消息框被取消。

比起仅仅提醒用户一个事件，一个更好的方法是同时询问用户该如何处理。将问题存储在informative text属性中，并将standard buttons属性设置为你想作为用户响应的按钮集。这些按钮是由StandardButtons的值通过使用位数OR操作符组合而成的。按钮的显示顺序是与平台有关的。例如，在Windows上，保存显示在取消的左边，而在Mac OS上，这个顺序是相反的。

将你的一个标准按钮标记为你的默认按钮。

QMessageBox msgBox;
msgBox.setText("The document has been modified.");
msgBox.setInformativeText("Do you want to save your changes?");
msgBox.setStandardButtons(QMessageBox::Save | QMessageBox::Discard | QMessageBox::Cancel);
msgBox.setDefaultButton(QMessageBox::Save);
int ret = msgBox.exec();

这是macOS指南中推荐的方法。类似的指南适用于其他平台，但要注意不同平台对信息文本的不同处理方式。

exec()槽返回被点击的按钮的StandardButtons 值。

switch (ret) {
    case QMessageBox::Save:
        // Save was clicked
        break;
    case QMessageBox::Discard:
        // Don't Save was clicked
        break;
    case QMessageBox::Cancel:
        // Cancel was clicked
        break;
    default:
        // should never be reached
        break;
}

为了给用户提供更多的信息以帮助他回答问题，可以设置详细文本属性。如果设置了详细文本属性，将显示**显示细节…**按钮。

点击**显示细节……**按钮显示详细的文本。

富文本和文本格式属性#

详细文本属性总是被解释为纯文本。主要文本和信息文本属性可以是纯文本或富文本。这些字符串会根据text format属性的设置进行解释。默认设置是auto-text。

请注意，对于一些包含XML元字符的纯文本字符串，自动文本富文本检测测试可能会失败，导致你的纯文本字符串被错误地解释为富文本。在这些罕见的情况下，使用Qt::convertFromPlainText()将你的纯文本字符串转换为视觉上等同的富文本字符串，或者使用setTextFormat()明确设置text format 属性。

严重程度以及图标和Pixmap属性#

QMessageBox支持四种预定义的消息严重程度，或者说消息类型，它们实际上只在各自显示的预定义图标上有所不同。通过将图标属性设置为预定义图标之一来指定四个预定义消息类型之一。以下规则是指导方针:

Question 因为在正常操作过程中提出了一个问题。
Information 有关正常操作的报告信息。
Warning 用于报告非关键性的错误。
Critical 用于报告关键错误。

预定义图标不是由QMessageBox定义的，而是由样式提供的。默认值是No Icon。其他情况下，消息框都是一样的。当使用标准图标时，请使用表格中推荐的图标，或者使用你的平台的样式指南推荐的图标。如果没有一个标准图标适合你的消息框，你可以通过设置icon pixmap属性而不是设置icon属性来使用一个自定义图标。

总之，要设置一个图标，可以使用**setIcon()来设置一个标准图标，或者**setIconPixmap()来设置一个自定义图标。

静态函数API#

使用静态函数API构建消息框，虽然很方便，但不如使用基于属性的API灵活，因为静态函数签名缺乏设置信息文本和详细文本 属性的参数。一个解决方法是使用title参数作为消息框的主文本，text参数作为消息框的信息文本。因为这有一个明显的缺点，就是使消息框的可读性降低，平台指南不推荐这样做。微软视窗用户界面指南*推荐使用应用程序名称作为窗口的标题，这意味着如果你在主文本之外有一个信息文本，你必须将其连接到text参数。

注意，静态函数签名在其按钮参数方面有所改变，现在用于设置标准按钮和默认按钮。

静态函数可用于创建信息()、问题()、警告()和关键()消息框。

int ret = QMessageBox::warning(this, tr("My Application"),
                               tr("The document has been modified.\n"
                                  "Do you want to save your changes?"),
                               QMessageBox::Save | QMessageBox::Discard
                               | QMessageBox::Cancel,
                               QMessageBox::Save);

标准对话框](qtwidgets-dialogs-standarddialogs-example.html) 示例展示了如何使用QMessageBox和其他内置的Qt对话框。

高级用法#

如果标准按钮对你的消息框来说不够灵活，你可以使用addButton()重载，它接受一个文本和一个ButtonRole来添加自定义按钮。ButtonRole被QMessageBox用来决定屏幕上的按钮的顺序（根据平台的不同而不同）。你可以在调用exec()后测试clickedButton的值。比如说：

QMessageBox msgBox;
QPushButton *connectButton = msgBox.addButton(tr("Connect"), QMessageBox::ActionRole);
QPushButton *abortButton = msgBox.addButton(QMessageBox::Abort);

msgBox.exec();

if (msgBox.clickedButton() == connectButton) {
    // connect
} else if (msgBox.clickedButton() == abortButton) {
    // abort
}

默认键和逃逸键#

默认按钮（即在按下Enter时激活的按钮）可以用setDefaultButton()指定。如果没有指定默认按钮，QMessageBox会尝试根据消息框中使用的按钮的button roles来找到一个。

可以使用setEscapeButton()来指定逃生按钮（当Esc被按下时激活的按钮）。如果没有指定逃生按钮，QMessageBox会尝试用这些规则找到一个。

  1. 如果只有一个按钮，那么它就是Esc被按下时激活的按钮。
  2. 2.如果有一个Cancel按钮，它就是在按下Esc时激活的按钮。
    1. 如果正好有一个按钮具有拒绝角色或否角色，它就是在按下Esc时激活的按钮。

当使用这些规则无法确定逃生按钮时，按下Esc没有任何效果。

See also QDialogButtonBox, GUI Design Handbook: Message Box, Standard Dialogs Example, and Qt Widgets - Application Example.

成员类型文档#

enum QMessageBox::ButtonRole#

这个枚举描述了可以用来描述按钮盒中的按钮的角色。这些角色的组合就像旗帜一样，用来描述其行为的不同方面。

Constant Value Description
QMessageBox::InvalidRole -1 该按钮是无效的。
QMessageBox::AcceptRole 0 点击该按钮将导致接受该对话框（如确定）。
QMessageBox::RejectRole 1 点击该按钮会导致对话框被拒绝（如取消）。
QMessageBox::DestructiveRole 2 点击该按钮会导致破坏性的改变（例如，放弃改变），并关闭对话框。
QMessageBox::ActionRole 3 点击按钮会使对话框内的元素发生变化。
QMessageBox::HelpRole 4 可以点击该按钮来请求帮助。
QMessageBox::YesRole 5 该按钮是一个类似 “是 “的按钮。
QMessageBox::NoRole 6 该按钮是一个类似 “不 “的按钮。
QMessageBox::ApplyRole 8 该按钮应用当前的变化。
QMessageBox::ResetRole 7 该按钮将对话框的字段重置为默认值。

See also StandardButton.

enum QMessageBox::Icon#

这个枚举有以下值:

Constant Value Description
QMessageBox::NoIcon 0 信息框没有任何图标。
QMessageBox::Question 4 图标，表示该信息是在问问题。
QMessageBox::Information 1 一个图标，表明该信息没有什么不正常的地方。
QMessageBox::Warning 2 一个图标，表示该信息是一个警告，但可以处理。
QMessageBox::Critical 3 图标，表明该信息代表一个关键问题。

这些枚举描述了标准按钮的标志。每个按钮都有一个定义的ButtonRole。

Constant Value Description
QMessageBox::Ok 0x00000400 一个用AcceptRole定义的 “确定 “按钮。
QMessageBox::Open 0x00002000 一个用AcceptRole定义的 “打开 “按钮。
QMessageBox::Save 0x00000800 一个用AcceptRole定义的 “保存 “按钮。
QMessageBox::Cancel 0x00400000 一个用RejectRole定义的 “取消 “按钮。
QMessageBox::Close 0x00200000 一个用RejectRole定义的 “关闭 “按钮。
QMessageBox::Discard 0x00800000 一个 “丢弃 “或 “不保存 “的按钮，取决于平台，用DestructiveRole定义。
QMessageBox::Apply 0x02000000 一个用ApplyRole定义的 “应用 “按钮。
QMessageBox::Reset 0x04000000 一个用ResetRole定义的 “复位 “按钮。
QMessageBox::RestoreDefaults 0x08000000 一个用ResetRole定义的 “恢复默认值 “按钮。
QMessageBox::Help 0x01000000 一个用HelpRole定义的 “帮助 “按钮。
QMessageBox::SaveAll 0x00001000 一个用AcceptRole定义的 “全部保存 “按钮。
QMessageBox::Yes 0x00004000 一个用YesRole定义的 “是 “按钮。
QMessageBox::YesToAll 0x00008000 一个用YesRole定义的 “是全部 “按钮。
QMessageBox::No 0x00010000 一个用NoRole定义的 “No “按钮。
QMessageBox::NoToAll 0x00020000 一个用NoRole定义的 “拒绝所有 “按钮。
QMessageBox::Abort 0x00040000 一个用RejectRole定义的 “Abort “按钮。
QMessageBox::Retry 0x00080000 一个用AcceptRole定义的 “重试 “按钮。
QMessageBox::Ignore 0x00100000 一个用AcceptRole定义的 “忽略 “按钮。
QMessageBox::NoButton 0x00000000 一个无效的按钮。

The following values are obsolete:

Constant Value Description
QMessageBox::YesAll YesToAll 用YesToAll代替。
QMessageBox::NoAll NoToAll 使用NoToAll代替。
QMessageBox::Default 0x00000100 使用information(), warning()等的defaultButton参数来代替，或者调用setDefaultButton()。
QMessageBox::Escape 0x00000200 调用setEscapeButton()代替。
QMessageBox::FlagMask 0x00000300
QMessageBox::ButtonMask ~FlagMask

StandardButtons类型是QFlags的一个类型定义。它存储了StandardButton值的一个OR组合。

See also ButtonRole and standardButtons.

特性文档#

detailedText : #

此属性持有要在详情区显示的文本。

该文本将被解释为纯文本。

默认情况下，此属性包含一个空字符串。

访问功能:

QString detailedText() const
void setDetailedText(const QString &text)

See also QMessageBox::text and QMessageBox::informativeText.

icon : #

此属性持有消息框的图标

消息框的图标可以用其中一个值来指定:

  • QMessageBox::NoIcon
  • QMessageBox::Question
  • QMessageBox::Information
  • QMessageBox::Warning
  • QMessageBox::Critical

默认是QMessageBox::NoIcon。

用来显示实际图标的像素图取决于当前的GUI风格。你也可以通过设置icon pixmap属性为图标设置一个自定义的像素图。

访问功能:

QMessageBox::Icon icon() const
void setIcon(QMessageBox::Icon)

See also iconPixmap.

iconPixmap : #

此属性持有当前图标

消息框当前使用的图标。请注意，通常很难绘制一个在所有GUI风格中看起来都合适的像素图；你可能想为每个平台提供一个不同的像素图。

默认情况下，这个属性是未定义的。

访问功能:

QPixmap iconPixmap() const
void setIconPixmap(const QPixmap &pixmap)

See also icon.

informativeText : #

此属性持有信息性文本，为信息提供更全面的描述。

信息性文本可用于扩展text()，以向用户提供更多信息。在Mac上，这个文本以小的系统字体出现在text()下面。在其他平台上，它只是被附加到现有的文本中。

默认情况下，这个属性包含一个空字符串。

访问功能:

QString informativeText() const
void setInformativeText(const QString &text)

See also QMessageBox::text and QMessageBox::detailedText.

standardButtons : #

消息框中标准按钮的集合

此属性控制了哪些标准按钮被消息框使用。

默认情况下，此属性不包含任何标准按钮。

访问功能:

QMessageBox::StandardButtons standardButtons() const
void setStandardButtons(QMessageBox::StandardButtons buttons)

See also addButton().

text : #

这个属性持有要显示的消息框文本。

该文本将被解释为纯文本或富文本，这取决于文本格式设置（QMessageBox::textFormat）。默认设置是Qt::AutoText，也就是说，消息框将尝试自动检测文本的格式。

这个属性的默认值是一个空字符串。

访问功能:

QString text() const
void setText(const QString &text)

See also textFormat, QMessageBox::informativeText, and QMessageBox::detailedText.

textFormat : #

此属性持有消息框所显示的文本的格式

消息框当前使用的文本格式。请参阅 Qt::TextFormat 枚举，了解可能的选项的解释。

默认格式是Qt::AutoText。

访问功能:

Qt::TextFormat textFormat() const
void setTextFormat(Qt::TextFormat format)

See also setText().

[since 5.1]textInteractionFlags : #

指定消息框的标签应如何与用户输入交互。

默认值取决于样式。

这个函数是在Qt 5.1中引入的。

访问功能:

Qt::TextInteractionFlags textInteractionFlags() const
void setTextInteractionFlags(Qt::TextInteractionFlags flags)

See also QStyle::SH_MessageBox_TextInteractionFlags.

成员函数文档#

QMessageBox::QMessageBox( icon, const &title, const &text, buttons = NoButton, *parent = nullptr, f = Qt::Dialog | Qt::MSWindowsFixedSizeDialogHint)#

用给定的icon、title、text和标准的buttons构建一个消息框。可以在任何时候使用addButton()添加标准或自定义按钮。parent和f参数被传递给QDialog构造函数。

消息框是一个application modal对话框。

在macOS上，如果parent不是nullptr，并且你希望你的消息框作为该父级的Qt::Sheet出现，请将消息框的window modality设置为Qt::WindowModal（默认）。否则，该消息框将是一个标准的对话框。

See also setWindowTitle(), setText(), setIcon(), and setStandardButtons().

[override virtual slot]int QMessageBox::exec()#

重新实现: QDialog::exec()。

将消息框显示为一个modal dialog，阻塞直到用户关闭它。

当使用带有标准按钮的QMessageBox时，这个函数返回一个StandardButton值，表示被点击的标准按钮。当使用带有自定义按钮的QMessageBox时，该函数返回一个不透明的值；使用clickedButton()来确定哪个按钮被点击了。

注意： result()函数还返回StandardButton值，而不是QDialog::DialogCode。

用户不能与同一应用程序中的任何其他窗口进行交互，直到他们通过点击按钮或使用窗口系统提供的机制来关闭该对话框。

See also show() and result().

[static]void QMessageBox::about( *parent, const &title, const &text)#

显示一个简单的关于框，标题为title，文本为text。about框的父级是parent。

about()在四个地方寻找一个合适的图标。

  1. 如果存在的话，它更倾向于parent->icon()。
  2. 2.如果没有，它将尝试包含parent的顶级widget。
  3. 如果失败了，它会尝试活动窗口。
  4. 作为最后的手段，它使用信息图标。

关于框有一个标有 “OK “的按钮。在macOS上，关于框是作为一个无模式窗口弹出的；在其他平台上，它目前是应用模式。

See also QWidget::windowIcon() and QApplication::activeWindow().

这对于包含在应用程序的帮助菜单中很有用，如Menus例子所示。

*QMessageBox::clickedButton() const#

返回被用户点击的按钮，如果用户点击了Esc键而没有设置逃生按钮，则返回`nullptr’。

QMessageBox messageBox(this);
QAbstractButton *disconnectButton =
    messageBox.addButton(tr("Disconnect"), QMessageBox::ActionRole);
...
    messageBox.exec();
if (messageBox.clickedButton() == disconnectButton) {
    ...
}

See also standardButton() and button().

[static] QMessageBox::critical( *parent, const &title, const &text, buttons = Ok, defaultButton = NoButton)#

在指定的parent widget前面打开一个带有给定的title和text的关键消息框。

标准的按钮被添加到消息框中。defaultButton指定了在按下*Enter时使用的按钮。defaultButton必须是指在buttons中给出的一个按钮。如果defaultButton是QMessageBox::NoButton，QMessageBox会自动选择一个合适的默认值。

返回被点击的标准按钮的身份。如果Esc被按下，则返回逃生按钮。

消息框是一个应用模式对话框。

警告：在执行对话框的过程中不要删除parent。如果你想这样做，你应该使用QMessageBox构造函数之一自己创建对话框。

See also question(), warning(), and information().

See also setDefaultButton(), addButton(), and QPushButton::setDefault().

*QMessageBox::escapeButton() const#

返回当escape被按下时被激活的按钮。

默认情况下，QMessageBox会尝试自动检测一个转义按钮，如下所示。

  1. 如果只有一个按钮，它将被作为逃生按钮。
  2. 如果有一个Cancel按钮，就把它作为逃生按钮。
  3. 仅在macOS上，如果正好有一个具有QMessageBox::RejectRole角色的按钮，它就会成为逃生按钮。

当逃生按钮不能被自动检测到时，按Esc没有效果。

See also setEscapeButton() and addButton().
